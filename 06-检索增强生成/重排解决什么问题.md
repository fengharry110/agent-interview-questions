# 重排解决什么问题？

## 面试问题

重排解决什么问题？

## 考察重点

- 是否理解重排在检索系统中的定位
- 是否掌握重排解决的典型问题
- 是否理解重排和初排的分工
- 是否有重排策略的设计能力

## 核心结论

重排（Re-ranking）解决的是初排结果"相关性排序不够精准"的问题。初排追求召回速度，使用轻量级模型（如向量检索）快速筛选；重排追求排序精度，使用重型模型（如Cross-Encoder）精细排序。

## 参考回答

搜索"Python异步编程"：

**初排（向量检索）**：
- 快速从千万文档中找到相关的100个
- 但这100个的排序可能不准：可能文档A和B都相关，但B应该排第一
- 原因：向量模型追求的是语义相似，不是精确相关性

**重排（Cross-Encoder）**：
- 对这100个文档精细排序
- 判断文档A和查询的相关性到底有多高
- 输出精确的相关性分数，重新排序

打个比方：
- 初排像海选：快速从10000人中选出100人
- 重排像决赛：仔细比较这100人的实力，排出一二三

```python
class WhyReRanking:
    """
    初排的局限性
    """
    
    # 局限性1：向量模型的双向性问题
    vector_model_limitation = """
    向量检索本质是：query和document在语义空间中的距离

    问题：向量模型在编码时是双向的，但检索时是单向的

    示例：
    query = "Python异步编程"
    doc = "本文介绍Python的asyncio库"

    向量模型看到的是完整的query和doc
    但检索时，query和doc可能在语义空间很接近
    然而，如果doc是"Python同步编程详解"
    加上"asyncio"关键词的doc也可能被召回（因为embedding相似）
    """
    
    # 局限性2：多词查询的部分匹配问题
    multi_query_limitation = """
    query = "Python异步编程 性能优化"

    理想：同时包含"Python异步编程"和"性能优化"的文档排第一

    向量检索：可能召回只匹配一部分的文档
    - doc1: "Python异步编程详解" (匹配了第一部分)
    - doc2: "Python性能优化技巧" (匹配了第二部分)
    - doc3: "异步编程性能对比" (匹配了"异步"和"性能")

    向量距离可能无法精确反映这种多维度匹配
    """
    
    # 局限性3：领域特定语义
    domain_specific_limitation = """
    query = "Python中的装饰器"

    通用向量模型：可能找不到专讲装饰器的文档
    因为"装饰器"在通用语义中不是高频词

    重排阶段：可以引入领域知识判断
```

### 两阶段检索架构

```python
class TwoStageRetrieval:
    """
    两阶段检索：初排 + 重排
    """
    
    def __init__(self):
        self.first_stage = BiEncoderRetriever()
        self.second_stage = CrossEncoderReranker()
    
    def retrieve_and_rerank(
        self,
        query: str,
        top_k_initial: int = 100,
        top_k_final: int = 10
    ) -> List[Document]:
        """
        两阶段检索流程
        """
        # 第一阶段：快速向量检索，召回Top-100
        initial_results = self.first_stage.search(
            query=query,
            top_k=top_k_initial
        )
        
        # 第二阶段：精细排序，输出Top-10
        reranked_results = self.second_stage.rerank(
            query=query,
            documents=initial_results,
            top_k=top_k_final
        )
        
        return reranked_results
    
    def get_pipeline_latency(self) -> Dict[str, float]:
        """估算各阶段延迟"""
        return {
            "first_stage_latency_ms": 10,   # 向量检索：10ms
            "second_stage_latency_ms": 500,  # Cross-Encoder：500ms（但只处理100个）
            "total_latency_ms": 510
        }
```

### Cross-Encoder重排

```python
class CrossEncoderReranker:
    """
    Cross-Encoder重排模型
    """
    
    def __init__(self, model_name: str = "cross-encoder/ms-marco"):
        self.model = AutoModelForSequenceClassification.from_pretrained(model_name)
        self.tokenizer = AutoTokenizer.from_pretrained(model_name)
    
    def rerank(
        self,
        query: str,
        documents: List[Document],
        top_k: int = 10
    ) -> List[RerankedDocument]:
        """
        Cross-Encoder对query-document对进行相关性打分
        """
        # 1. 准备输入
        pairs = [(query, doc.content) for doc in documents]
        
        # 2. 批量编码
        inputs = self.tokenizer(
            pairs,
            padding=True,
            truncation=True,
            return_tensors="pt",
            max_length=512
        )
        
        # 3. 前向传播
        with torch.no_grad():
            outputs = self.model(**inputs)
            scores = outputs.logits.squeeze(-1).numpy()
        
        # 4. 按分数排序
        doc_scores = list(zip(documents, scores))
        doc_scores.sort(key=lambda x: x[1], reverse=True)
        
        # 5. 返回Top-K
        return [
            RerankedDocument(doc=doc, score=float(score))
            for doc, score in doc_scores[:top_k]
        ]
```

### 轻量级重排策略

```python
class LightweightReranker:
    """
    轻量级重排策略（不依赖大型模型）
    """
    
    def __init__(self):
        self.bm25 = BM25Okapi()
        self.exact_matcher = ExactMatcher()
    
    def rerank(
        self,
        query: str,
        documents: List[Document],
        top_k: int = 10
    ) -> List[RerankedDocument]:
        """
        结合多信号的轻量级重排
        """
        signals = []
        
        for doc in documents:
            # 1. BM25分数
            bm25_score = self._calculate_bm25(query, doc.content)
            
            # 2. 精确匹配信号
            exact_matches = self._count_exact_matches(query, doc.content)
            
            # 3. 位置信号
            position_score = self._calculate_position_score(doc)
            
            # 4. 质量信号
            quality_score = doc.metadata.get("quality_score", 0.5)
            
            # 5. 综合分数
            final_score = (
                0.4 * self._normalize(bm25_score) +
                0.3 * exact_matches / max(len(query.split()), 1) +
                0.1 * position_score +
                0.2 * quality_score
            )
            
            signals.append(final_score)
        
        # 排序
        doc_signals = list(zip(documents, signals))
        doc_signals.sort(key=lambda x: x[1], reverse=True)
        
        return [
            RerankedDocument(doc=doc, score=float(score))
            for doc, score in doc_signals[:top_k]
        ]
    
    def _normalize(self, score: float) -> float:
        """归一化到0-1"""
        return 1 / (1 + np.exp(-score))
```

### LambdaMART重排

```python
class LambdaMARReranker:
    """
    LambdaMART学习to rank重排
    """
    
    def __init__(self):
        self.model = None
        self.feature_extractor = RankingFeatures()
    
    def train(self, training_data: List[RankingSample]):
        """
        训练LambdaMART模型
        """
        # 1. 提取特征
        X = [self.feature_extractor.extract(q, doc) for q, doc, _ in training_data]
        y = [label for _, _, label in training_data]
        
        # 2. 训练模型
        self.model = lgb.LGBMRanker(
            objective="lambdarank",
            metric="ndcg",
            n_estimators=100
        )
        self.model.fit(X, y)
    
    def rerank(
        self,
        query: str,
        documents: List[Document],
        top_k: int = 10
    ) -> List[RerankedDocument]:
        """
        使用LambdaMART重排
        """
        # 1. 提取特征
        features = [self.feature_extractor.extract(query, doc) for doc in documents]
        
        # 2. 预测分数
        scores = self.model.predict(features)
        
        # 3. 排序
        doc_scores = list(zip(documents, scores))
        doc_scores.sort(key=lambda x: x[1], reverse=True)
        
        return [
            RerankedDocument(doc=doc, score=float(score))
            for doc, score in doc_scores[:top_k]
        ]


class RankingFeatureExtractor:
    """
    排序特征提取器
    """
    
    def extract(self, query: str, doc: Document) -> List[float]:
        """
        提取用于排序的特征
        """
        features = []
        
        # BM25相关
        features.append(self.bm25_score(query, doc.content))
        features.append(self.bm25_score(query, doc.title))
        
        # 向量相似度
        features.append(self.vector_similarity(query, doc.content))
        
        # 精确匹配
        features.append(self.exact_match_ratio(query, doc.content))
        features.append(self.query_length_match(query, doc))
        
        # 文档质量
        features.append(doc.metadata.get("quality_score", 0.5))
        features.append(doc.metadata.get("authority_score", 0.5))
        
        # 新鲜度
        features.append(self.recency_score(doc.published_date))
        
        # 多样性
        features.append(self.topic_diversity(doc))
        
        return features
```

### BERT重排模型

```python
class BERTReRanker:
    """
    基于BERT的交互式重排
    """
    
    def __init__(self, model_name: str = "bert-base-uncased"):
        self.model = BertForNextSentencePrediction.from_pretrained(model_name)
        self.tokenizer = BertTokenizer.from_pretrained(model_name)
    
    def rerank(
        self,
        query: str,
        documents: List[Document],
        top_k: int = 10
    ) -> List[RerankedDocument]:
        """
        使用BERT进行深度语义匹配重排
        """
        results = []
        
        for doc in documents:
            # 构造[CLS] query [SEP] document [SEP]
            encoded = self.tokenizer(
                query,
                doc.content,
                padding=True,
                truncation=True,
                max_length=512,
                return_tensors="pt"
            )
            
            with torch.no_grad():
                outputs = self.model(**encoded)
                # 使用pooler output或logits作为分数
                score = outputs.pooler_output.mean().item()
            
            results.append(RerankedDocument(doc=doc, score=score))
        
        # 排序
        results.sort(key=lambda x: x.score, reverse=True)
        return results[:top_k]
```

### 多样性重排

```python
class DiversityReranker:
    """
    考虑多样性的重排
    """
    
    def __init__(self, base_reranker):
        self.base_reranker = base_reranker
        self.lambda_diversity = 0.3
    
    def rerank(
        self,
        query: str,
        documents: List[Document],
        top_k: int = 10
    ) -> List[RerankedDocument]:
        """
        结合相关性和多样性的重排
        使用MMR-like策略
        """
        # 1. 获取基础相关性分数
        base_results = self.base_reranker.rerank(query, documents, top_k=len(documents))
        
        # 2. MMR选择
        selected = []
        remaining = base_results.copy()
        
        while len(selected) < top_k and remaining:
            best_idx = None
            best_score = float('-inf')
            
            for i, candidate in enumerate(remaining):
                # 相关性分数
                relevance = candidate.score
                
                # 与已选文档的最小相似度（越大说明越需要包含）
                if selected:
                    min_similarity = min(
                        self._cosine_sim(candidate.doc.embedding, s.doc.embedding)
                        for s in selected
                    )
                else:
                    min_similarity = 0
                
                # MMR分数
                mmr = (
                    (1 - self.lambda_diversity) * relevance +
                    self.lambda_diversity * (1 - min_similarity)
                )
                
                if mmr > best_score:
                    best_score = mmr
                    best_idx = i
            
            if best_idx is not None:
                selected.append(remaining[best_idx])
                remaining.pop(best_idx)
        
        return selected
```

## 工程实践

### 重排模型选择决策树

```
文档规模 & 延迟要求
         │
         ├── 小规模（<10万）+ 低延迟（<50ms） ──→ BM25 + 轻量特征
         │
         ├── 中规模（10万-100万）+ 中延迟（50-200ms） ──→ 向量+LambdaMART
         │
         └── 大规模（>100万）+ 可接受延迟（200-500ms） ──→ 向量+Cross-Encoder

查询复杂度
         │
         ├── 简单查询（短） ──→ BM25 + 向量混合
         │
         └── 复杂查询（长） ──→ Cross-Encoder或BERT交互
```

```python
class RerankingMetrics:
    """
    重排效果评估
    """
    
    @staticmethod
    def evaluate(
        original_ranking: List[str],
        reranked_ranking: List[str],
        relevance_labels: Dict[str, int]
    ) -> Dict[str, float]:
        """
        评估重排前后的效果
        """
        # NDCG@K
        original_ndcg = calculate_ndcg(original_ranking, relevance_labels, k=10)
        reranked_ndcg = calculate_ndcg(reranked_ranking, relevance_labels, k=10)
        
        # MRR
        original_mrr = calculate_mrr(original_ranking, relevance_labels)
        reranked_mrr = calculate_mrr(reranked_ranking, relevance_labels)
        
        # 提升
        ndcg_improvement = (reranked_ndcg - original_ndcg) / original_ndcg * 100
        mrr_improvement = (reranked_mrr - original_mrr) / original_mrr * 100
        
        return {
            "original_ndcg": original_ndcg,
            "reranked_ndcg": reranked_ndcg,
            "ndcg_improvement_pct": ndcg_improvement,
            "original_mrr": original_mrr,
            "reranked_mrr": reranked_mrr,
            "mrr_improvement_pct": mrr_improvement
        }
```

## Claude Code的重排实现

```python
class ClaudeCodeReranker:
    """
    Claude Code使用多种重排策略
    """
    
    def __init__(self):
        self.stage1 = BiEncoderRetriever()
        self.stage2 = CrossEncoderReranker()  # 对代码片段重排
        self.stage3 = DiversityReranker()      # 保证结果多样性
    
    def rerank_code_snippets(
        self,
        query: str,
        snippets: List[CodeSnippet],
        top_k: int = 5
    ) -> List[RerankedSnippet]:
        """
        代码片段重排流程
        """
        # 1. 阶段1：向量检索初排
        initial = self.stage1.search(query, snippets, top_k=50)
        
        # 2. 阶段2：Cross-Encoder精细排序
        # 代码片段可能很短，用Cross-Encoder判断更精准
        cross_reranked = self.stage2.rerank(query, initial, top_k=20)
        
        # 3. 阶段3：多样性重排
        # 保证返回的代码片段来自不同文件/不同模式
        final = self.stage3.rerank(query, cross_reranked, top_k=top_k)
        
        return final
```

## 常见误区

- **重排代替初排**：重排只处理初排结果，不适合直接从海量文档重排
- **忽视延迟**：重排模型通常很重，需要控制重排数量
- **只用单一模型**：组合多个信号往往比单一重排模型好
- **忽视多样性**：只优化相关性可能导致结果重复

建议对比实验：同一批检索结果，分别用向量检索排序和重排排序，观察NDCG、MRR等指标的变化。重点理解两阶段架构的设计初衷：初排用轻量模型保证召回速度，重排用重量模型保证排序精度。
