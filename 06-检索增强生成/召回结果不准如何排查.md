# 召回结果不准如何排查？

## 面试问题

召回结果不准如何排查？

## 考察重点

- 是否理解检索系统的召回、排序、质量评估全链路
- 能否说清楚不同类型检索问题的定位方法
- 是否能结合具体场景给出排查方案
- 是否理解embedding模型与检索效果的关联

## 核心结论

召回结果不准的问题需要分层排查：先定位是Embedding问题还是检索策略问题，再定位是文档库问题还是查询理解问题，最后定位到具体环节。

## 排查框架

### 分层定位法

```
召回问题排查链路：

用户查询
    ↓
[Query层]  query理解是否正确？
    ↓ Yes → [Embedding层]  embedding模型是否适合？
    ↓ No  → 修复query理解
    ↓
[检索层]  检索策略是否正确？
    ↓ Yes → [文档库层]  文档是否覆盖？
    ↓ No  → 修复检索策略
    ↓
[排序层]  排序是否合理？
    ↓ Yes → [后处理层]  结果处理是否有问题？
    ↓ No  → 修复排序/后处理
    ↓
输出结果
```

class QueryUnderstandingValidator:
    def validate(self, query: str, expected_intent: str) -> ValidationResult:
        # 1. 检测query中的关键实体
        entities = self.extract_entities(query)
        
        # 2. 验证实体是否被正确识别
        missing_entities = self.check_missing_entities(entities, expected_intent)
        
        # 3. 检测query类型
        query_type = self.classify_query_type(query)
        
        # 4. 检测语言
        query_lang = self.detect_language(query)
        
        return ValidationResult(
            entities_found=entities,
            missing_entities=missing_entities,
            query_type=query_type,
            language=query_lang,
            issues=self.identify_issues(entities, missing_entities, query_type)
        )
    
    def check_missing_entities(self, entities: List[str], expected: str) -> List[str]:
        expected_entities = self.extract_entities(expected)
        return [e for e in expected_entities if e not in entities]
```

class EmbeddingQualityChecker:
    def __init__(self, embedding_model):
        self.model = embedding_model
        self.test_cases = self.load_test_cases()
    
    def check_embedding_quality(self) -> QualityReport:
        # 1. 语义相似度测试
        semantic_scores = self.test_semantic_similarity()
        
        # 2. 召回率测试
        recall_scores = self.test_recall()
        
        # 3. 多语言测试（如果适用）
        multilingual_scores = self.test_multilingual()
        
        return QualityReport(
            semantic=semantic_scores,
            recall=recall_scores,
            multilingual=multilingual_scores,
            overall=self.calculate_overall_score(semantic_scores, recall_scores)
        )
    
    def test_semantic_similarity(self) -> Dict[str, float]:
        test_pairs = [
            ("king", "queen", 0.8),  # expected similarity
            ("car", "automobile", 0.9),
            ("dog", "cat", 0.6),
        ]
        scores = {}
        for w1, w2, expected in test_pairs:
            v1 = self.model.encode(w1)
            v2 = self.model.encode(w2)
            similarity = cosine_similarity(v1, v2)
            scores[f"{w1}-{w2}"] = similarity
        return scores
```

### Embedding问题定位

```python
class EmbeddingProblemLocator:
    def locate_problems(self, query: str, retrieved_docs: List[Document]) -> List[Problem]:
        problems = []
        
        # 问题1：相似度高但语义不相关
        for doc in retrieved_docs:
            if doc.score > 0.8 and not self.is_semantically_related(query, doc):
                problems.append(Problem(
                    type="false_positive",
                    doc_id=doc.id,
                    score=doc.score,
                    reason="High cosine similarity but low semantic relevance"
                ))
        
        # 问题2：语义相关但相似度低
        if self.is_semantically_related(query, retrieved_docs[0]) and retrieved_docs[0].score < 0.5:
            problems.append(Problem(
                type="false_negative",
                doc_id=retrieved_docs[0].id,
                score=retrieved_docs[0].score,
                reason="Semantic relevance but low cosine similarity"
            ))
        
        return problems
```

## 检索层排查

### 检索参数检查

```python
class RetrievalParameterChecker:
    def check_parameters(self, retrieval_config: dict, index_stats: dict) -> List[Issue]:
        issues = []
        
        # 1. 检查top_k是否合理
        if retrieval_config['top_k'] > index_stats['total_docs'] * 0.1:
            issues.append(Issue(
                severity="warning",
                message=f"top_k={retrieval_config['top_k']} may be too large for {index_stats['total_docs']} docs"
            ))
        
        # 2. 检查相似度阈值
        if retrieval_config.get('min_score', 0) > 0.9:
            issues.append(Issue(
                severity="error",
                message="min_score too high may cause false negatives"
            ))
        
        # 3. 检查filter条件
        if retrieval_config.get('filters'):
            issues.extend(self.check_filter_effectiveness(retrieval_config['filters']))
        
        return issues
```

### 检索策略问题

```python
class RetrievalStrategyAnalyzer:
    def analyze(self, query: str, results: List[Document], expected: List[str]) -> Analysis:
        # 1. 计算召回率
        recall = self.calculate_recall(results, expected)
        
        # 2. 计算精确率
        precision = self.calculate_precision(results, expected)
        
        # 3. MRR (Mean Reciprocal Rank)
        mrr = self.calculate_mrr(results, expected)
        
        # 4. 分析错误类型
        errors = self.categorize_errors(results, expected)
        
        return Analysis(
            recall=recall,
            precision=precision,
            mrr=mrr,
            errors=errors,
            diagnosis=self.diagnose(recall, precision, errors)
        )
    
    def diagnose(self, recall, precision, errors) -> str:
        if recall < 0.5 and precision < 0.5:
            return "检索策略问题：召回和精确率都低，建议检查索引和query"
        elif recall < 0.5:
            return "召回不足：可能索引有问题或query表达不准确"
        elif precision < 0.5:
            return "精确率不足：可能检索策略过于宽松或query理解有误"
        else:
            return "结果基本正常，检查排序策略"
```

## 文档库层排查

### 文档覆盖度检查

```python
class DocumentCoverageChecker:
    def check_coverage(self, query: str, index: VectorIndex) -> CoverageReport:
        # 1. 提取query关键词
        keywords = self.extract_keywords(query)
        
        # 2. 检查每个关键词的文档覆盖
        coverage = {}
        for keyword in keywords:
            doc_count = index.count_docs_containing(keyword)
            coverage[keyword] = {
                'count': doc_count,
                'percentage': doc_count / index.total_docs
            }
        
        # 3. 识别覆盖不足的关键词
        low_coverage = [k for k, v in coverage.items() if v['percentage'] < 0.01]
        
        return CoverageReport(
            keyword_coverage=coverage,
            low_coverage_keywords=low_coverage,
            is_covered=len(low_coverage) == 0
        )
```

### 文档 Chunk 问题

```python
class ChunkQualityAnalyzer:
    def analyze_chunks(self, doc_id: str, chunks: List[Chunk]) -> ChunkAnalysis:
        issues = []
        
        for i, chunk in enumerate(chunks):
            # 检查chunk是否太小
            if chunk.token_count < 50:
                issues.append(ChunkIssue(
                    chunk_index=i,
                    type="too_small",
                    message=f"Chunk too short ({chunk.token_count} tokens)"
                ))
            
            # 检查chunk是否太大
            if chunk.token_count > 1024:
                issues.append(ChunkIssue(
                    chunk_index=i,
                    type="too_large",
                    message=f"Chunk too large ({chunk.token_count} tokens)"
                ))
            
            # 检查chunk是否截断了完整语义单元
            if chunk.is_truncated:
                issues.append(ChunkIssue(
                    chunk_index=i,
                    type="truncated",
                    message="Chunk was truncated, may miss complete context"
                ))
            
            # 检查chunk是否有重复
            if self.has_significant_overlap(chunk, chunks[:i]):
                issues.append(ChunkIssue(
                    chunk_index=i,
                    type="overlap",
                    message="Chunk significantly overlaps with previous chunks"
                ))
        
        return ChunkAnalysis(doc_id=doc_id, issues=issues)
```

## 实战排查流程

### 快速定位检查清单

```
1. [Query层]
   □ query能否被正确解析？
   □ 关键实体是否被识别？
   □ query类型判断是否正确？

2. [Embedding层]
   □ test set召回率是否达标？
   □ 语义相似度测试是否通过？
   □ embedding是否过期/需更新？

3. [检索层]
   □ 检索参数是否合理？
   □ filter条件是否过于严格？
   □ top_k设置是否合适？

4. [文档库层]
   □ 相关文档是否在库中？
   □ chunk切分是否合理？
   □ 是否有最新的文档更新？

5. [排序层]
   □ 排序分数分布是否正常？
   □ 高分结果是否真正相关？
   □ 排序是否稳定？
```

## 常见问题诊断

| 症状 | 可能原因 | 排查方法 |
|------|---------|---------|
| 召回结果完全不相关 | Embedding模型问题/索引损坏 | 测试semantic similarity/重建索引 |
| 只能召回部分相关 | query理解问题/文档覆盖不足 | 检查query改写/补充文档 |
| 召回多但不准确 | 排序策略问题/阈值过低 | 调整排序权重/提高阈值 |
| 结果时好时坏 | 索引不一致/分片问题 | 检查索引状态/合并分片 |
