# 如何评估检索质量？

## 面试问题

如何评估检索质量？

## 考察重点

- 是否理解检索系统各层的评估指标
- 是否能区分离线评估和在线评估的适用场景
- 是否掌握多维度评估体系的设计方法
- 是否有基于业务目标进行评估优先级排序的能力

## 核心结论

检索质量评估需要分层进行：召回层评估"找没找到"，精排层评估"找得准不准"，业务层评估"找得有用没用"。回答这个问题时，要建立从底层指标到业务指标的完整链路，并说明各层指标的计算方法和业务含义。

## 参考回答

检索质量评估不是单一指标能完成的。以一个代码智能助手为例，用户输入"如何实现快速排序"，系统可能召回100个文档，但：
1. 前50个都是排序算法理论文档，没有代码实现
2. 第51-80个是Python实现，但用户用的是TypeScript
3. 第81-100个是TypeScript快速排序，但实现有bug

这个例子说明：召回率高不等于检索质量高，需要分层评估。

## 深度展开

### 检索评估框架

检索系统分为四层，每层有对应的评估指标：

```
┌─────────────────────────────────────────────────────────────┐
│                        业务层评估                            │
│         任务完成率、用户满意度、转化率、留存率                 │
├─────────────────────────────────────────────────────────────┤
│                        排序层评估                            │
│            NDCG、MAP、MRR、Precision@K、MRR@K               │
├─────────────────────────────────────────────────────────────┤
│                        精排层评估                            │
│              点击率、通过率、收藏率、分享率                   │
├─────────────────────────────────────────────────────────────┤
│                        召回层评估                            │
│                  Recall@K、HitRate、Coverage                 │
└─────────────────────────────────────────────────────────────┘
```

### 召回层指标

召回层解决"找没找到"的问题，核心指标是Recall@K和HitRate。

Recall@K = (相关文档出现在前K个结果中的数量) / (所有相关文档的数量)

```python
def recall_at_k(retrieved_docs: List[str], relevant_docs: Set[str], k: int) -> float:
    """
    计算Recall@K
    retrieved_docs: 检索返回的文档ID列表（已按相关性排序）
    relevant_docs: 真正的相关文档集合
    k: 取前K个结果计算
    """
    retrieved_at_k = set(retrieved_docs[:k])
    recall = len(retrieved_at_k & relevant_docs) / len(relevant_docs) if relevant_docs else 0.0
    return recall

def hit_rate_at_k(retrieved_docs: List[str], relevant_docs: Set[str], k: int) -> float:
    """
    计算HitRate@K - 是否在Top-K中至少命中一个
    """
    retrieved_at_k = set(retrieved_docs[:k])
    return 1.0 if len(retrieved_at_k & relevant_docs) > 0 else 0.0

# 示例
retrieved = ["doc1", "doc2", "doc3", "doc4", "doc5"]
relevant = {"doc2", "doc7", "doc9"}

print(f"Recall@5: {recall_at_k(retrieved, relevant, 5)}")  # 1/3 ≈ 0.333
print(f"HitRate@3: {hit_rate_at_k(retrieved, relevant, 3)}")  # 1.0 (命中doc2)
```

**Coverage覆盖率**也很重要，衡量检索系统能覆盖多少比例的查询：

```python
def coverage_rate(queries: List[str], retrieval_func: callable, corpus: List[Document]) -> float:
    """
    计算查询覆盖率 - 有多少查询能召回至少一个结果
    """
    covered = 0
    for query in queries:
        results = retrieval_func(query, corpus)
        if len(results) > 0:
            covered += 1
    return covered / len(queries) if queries else 0.0
```

### 精排层指标

精排层解决"排得准不准"的问题，核心指标是MRR、MAP和NDCG。

**MRR (Mean Reciprocal Rank)** - 第一个相关结果的位置倒数：

```python
def mean_reciprocal_rank(
    queries: List[str],
    retrieval_func: callable,
    relevance_func: callable
) -> float:
    """
    MRR = 平均(1/第一个相关文档的位置)
    只关心第一个相关结果的位置
    """
    reciprocal_ranks = []
    for query in queries:
        results = retrieval_func(query)  # 返回排序后的文档列表
        for rank, doc in enumerate(results, start=1):
            if relevance_func(query, doc):  # 判断是否相关
                reciprocal_ranks.append(1.0 / rank)
                break
        else:
            reciprocal_ranks.append(0.0)  # 没有找到相关文档
    return sum(reciprocal_ranks) / len(reciprocal_ranks)

# 示例：查询"快速排序实现"
# 返回: [doc_py_quicksort, doc_java_quicksort, doc_theory_sorting]
# 如果只有doc_java_quicksort是相关的，位置是2，RR = 1/2 = 0.5
```

**MAP (Mean Average Precision)** - 所有相关文档的平均精确率的均值：

```python
def average_precision(
    retrieved_docs: List[str],
    relevant_docs: Set[str]
) -> float:
    """
    AP = Σ(每个相关文档的Precision@K) / 相关文档总数
    衡量所有相关文档在结果中的分布质量
    """
    num_relevant = len(relevant_docs)
    if num_relevant == 0:
        return 0.0
    
    precisions = []
    relevant_found = 0
    for i, doc in enumerate(retrieved_docs, start=1):
        if doc in relevant_docs:
            relevant_found += 1
            precisions.append(relevant_found / i)
    
    return sum(precisions) / num_relevant

def mean_average_precision(
    queries: List[str],
    retrieval_func: callable,
    relevance_func: callable
) -> float:
    aps = []
    for query in queries:
        results = retrieval_func(query)
        relevant = {doc for doc in results if relevance_func(query, doc)}
        aps.append(average_precision(results, relevant))
    return sum(aps) / len(aps)
```

**NDCG (Normalized Discounted Cumulative Gain)** - 考虑相关度等级的排序质量：

```python
def dcg_at_k(relevances: List[float], k: int) -> float:
    """
    DCG@K = Σ(REL_i / log2(i+1)), i=1 to k
    相关度越高的文档排在前面，DCG值越大
    """
    dcg = 0.0
    for i, rel in enumerate(relevances[:k], start=1):
        dcg += rel / np.log2(i + 1)
    return dcg

def ndcg_at_k(
    retrieved_relevances: List[float],
    ideal_relevances: List[float],
    k: int
) -> float:
    """
    NDCG@K = DCG@K / IDCG@K
    归一化到[0,1]，1表示完美排序
    """
    dcg = dcg_at_k(retrieved_relevances, k)
    idcg = dcg_at_k(sorted(ideal_relevances, reverse=True), k)
    return dcg / idcg if idcg > 0 else 0.0

# 示例：相关度等级 [3, 2, 0, 1] vs [3, 2, 1, 0]
# 计算两者NDCG，前者应该更高
```

### 业务层指标

业务层指标直接衡量检索对业务目标的影响：

```python
@dataclass
class BusinessMetrics:
    task_completion_rate: float      # 任务完成率
    user_satisfaction_score: float   # 用户满意度
    click_through_rate: float        # 点击率
    conversion_rate: float            # 转化率
    retention_rate: float            # 留存率
    abandonment_rate: float           # 放弃率
    
    def weighted_score(self, weights: Dict[str, float]) -> float:
        """根据权重计算综合业务得分"""
        score = 0.0
        for metric, weight in weights.items():
            value = getattr(self, metric, 0.0)
            score += value * weight
        return score
```

### Claude Code的评估实践

Claude Code使用多层次评估体系验证检索质量：

```python
class ClaudeCodeRetrievalEvaluator:
    def __init__(self):
        self.offline_metrics = OfflineMetrics()
        self.online_metrics = OnlineMetrics()
    
    def evaluate_recall_quality(
        self,
        test_queries: List[RetrievalQuery],
        expected_docs: Dict[str, List[str]]
    ) -> Dict[str, float]:
        """离线评估召回质量"""
        results = {}
        for query in test_queries:
            retrieved = self.retrieve(query)
            expected = expected_docs.get(query.id, [])
            
            recall = self.offline_metrics.recall_at_k(
                retrieved, set(expected), k=10
            )
            mrr = self.offline_metrics.mean_reciprocal_rank(
                retrieved, set(expected)
            )
            results[query.id] = {"recall": recall, "mrr": mrr}
        
        return results
    
    def evaluate_ranking_quality(
        self,
        query: str,
        retrieved_docs: List[Document],
        relevance_labels: List[int]
    ) -> float:
        """评估排序质量（NDCG）"""
        relevances = [rel for _, rel in sorted(
            zip(retrieved_docs, relevance_labels),
            key=lambda x: x[0].score,
            reverse=True
        )]
        ideal_relevances = sorted(relevance_labels, reverse=True)
        
        return self.offline_metrics.ndcg_at_k(
            relevances, ideal_relevances, k=len(relevances)
        )
```

### 评估数据集构建

高质量的评估数据集是准确评估的前提：

```python
class RetrievalDatasetBuilder:
    def __init__(self, seed: int = 42):
        self.rng = random.Random(seed)
        self.queries = []
        self.qrels = {}  # query_id -> {doc_id: relevance_score}
    
    def add_query_with_relevance(
        self,
        query: str,
        doc_relevances: Dict[str, int],
        source: str = "human"
    ):
        """
        添加查询及其相关文档标注
        relevance_score: 0(不相关), 1(部分相关), 2(相关), 3(高度相关)
        """
        query_id = self._generate_query_id(query)
        self.queries.append({"id": query_id, "text": query, "source": source})
        self.qrels[query_id] = doc_relevances
    
    def split(self, train_ratio: float = 0.8) -> tuple:
        """划分训练集和测试集"""
        self.rng.shuffle(self.queries)
        split_idx = int(len(self.queries) * train_ratio)
        train_queries = self.queries[:split_idx]
        test_queries = self.queries[split_idx:]
        return train_queries, test_queries
    
    def export_bm25_format(self) -> str:
        """导出为BM25评估格式"""
        lines = []
        for query in self.queries:
            qid = query["id"]
            relevant_docs = [
                f"{doc_id}:{score}"
                for doc_id, score in self.qrels[qid].items()
                if score > 0
            ]
            lines.append(f"{qid}\t{' '.join(relevant_docs)}")
        return "\n".join(lines)
```

### 在线评估体系

```python
class OnlineRetrievalMonitor:
    def __init__(self, time_window_hours: int = 24):
        self.time_window = time_window_hours * 3600
        self.events = deque(maxlen=10000)
    
    def record_retrieval_event(
        self,
        query_id: str,
        user_id: str,
        retrieved_doc_ids: List[str],
        clicked_doc_id: str = None,
        session_duration: float = None
    ):
        """记录用户检索行为事件"""
        self.events.append({
            "timestamp": time.time(),
            "query_id": query_id,
            "user_id": user_id,
            "retrieved": retrieved_doc_ids,
            "clicked": clicked_doc_id,
            "session_duration": session_duration,
            "position_clicked": retrieved_doc_ids.index(clicked_doc_id) 
                                if clicked_doc_id in retrieved_doc_ids 
                                else None
        })
    
    def calculate_ctr_by_position(self) -> Dict[int, float]:
        """计算各位置的点击率"""
        position_clicks = defaultdict(int)
        position_impressions = defaultdict(int)
        
        for event in self.events:
            if event["clicked"]:
                pos = event["position_clicked"]
                position_clicks[pos] += 1
            for i, doc_id in enumerate(event["retrieved"][:10], start=1):
                position_impressions[i] += 1
        
        return {
            pos: position_clicks[pos] / position_impressions[pos]
            for pos in position_impressions
        }
    
    def calculate_session_success_rate(self) -> float:
        """计算会话成功率（用户是否找到需要的信息）"""
        successful_sessions = 0
        total_sessions = 0
        
        session_groups = defaultdict(list)
        for event in self.events:
            session_groups[(event["user_id"], event["query_id"])].append(event)
        
        for events in session_groups.values():
            if any(e["clicked"] for e in events):
                successful_sessions += 1
            total_sessions += 1
        
        return successful_sessions / total_sessions if total_sessions > 0 else 0.0
```

## 工程实践

### 评估流水线设计

```python
class RetrievalEvaluationPipeline:
    def __init__(self, retrieval_system: RetrievalSystem):
        self.system = retrieval_system
        self.evaluator = ClaudeCodeRetrievalEvaluator()
        self.report_generator = EvaluationReportGenerator()
    
    def run_offline_evaluation(
        self,
        test_dataset: RetrievalDataset
    ) -> EvaluationReport:
        """运行离线评估"""
        results = {
            "recall": [],
            "mrr": [],
            "ndcg": [],
            "coverage": []
        }
        
        for query in test_dataset.queries:
            retrieved = self.system.retrieve(query.text, top_k=50)
            expected = test_dataset.get_relevant_docs(query.id)
            
            results["recall"].append(
                recall_at_k([d.id for d in retrieved], expected, k=10)
            )
            results["mrr"].append(
                mean_reciprocal_rank_single(
                    [d.id for d in retrieved], expected
                )
            )
            results["ndcg"].append(
                self.evaluator.evaluate_ranking_quality(
                    query.text, retrieved,
                    [test_dataset.get_relevance(query.id, d.id) 
                     for d in retrieved]
                )
            )
        
        return EvaluationReport(
            recall_mean=np.mean(results["recall"]),
            recall_p50=np.median(results["recall"]),
            mrr_mean=np.mean(results["mrr"]),
            ndcg_mean=np.mean(results["ndcg"]),
            sample_results=results
        )
    
    def run_ab_test(
        self,
        variant_a: RetrievalSystem,
        variant_b: RetrievalSystem,
        traffic_split: float = 0.5,
        duration_days: int = 7
    ) -> ABTestResult:
        """运行A/B测试比较两个检索系统"""
        monitor = OnlineRetrievalMonitor()
        
        # 分配流量
        users = self.get_active_users(duration_days)
        group_a = users[:int(len(users) * traffic_split)]
        group_b = users[int(len(users) * traffic_split):]
        
        # 收集指标
        metrics_a = self.collect_metrics(variant_a, group_a, monitor)
        metrics_b = self.collect_metrics(variant_b, group_b, monitor)
        
        # 统计显著性检验
        return self.statistical_significance_test(metrics_a, metrics_b)
```

### 评估指标选择决策树

```
查询类型                  推荐指标              阈值建议
─────────────────────────────────────────────────────
事实查找类查询            Recall@5, HitRate     > 0.95
代码检索类查询            MRR, MAP              > 0.85
多文档综合任务            NDCG@10               > 0.75
开放式问答               Task Completion, CTR   > 0.70
长文档检索               Recall@20, Coverage   > 0.90
```

## 常见误区

- **只用单一指标**：只优化MRR可能导致结果过于相似（多样性问题），只优化Recall可能导致返回过多无关内容
- **离线指标和在线指标脱节**：离线NDCG提升但在线点击率下降，说明离线评估集可能过时
- **忽略相关度等级**：将所有相关文档等同处理，2分和3分都是"相关"，但排序时应该优先展示3分
- **评估集和生产分布不一致**：评估集太旧或太简单，无法反映真实用户查询

## 追问方向

- 如果Recall很高但用户反馈检索结果不有用，是哪里出了问题？
- 如何处理检索系统和排序系统的评估差异？
- 冷启动场景下如何评估新上线的检索系统？
- 如何建立评估-反馈-优化的闭环？

## 学习建议

建议结合具体项目理解评估指标的工程意义：在构建代码检索系统时，分别计算新旧系统的NDCG，用真实用户查询和标注数据验证改进效果。重点理解NDCG和MRR的区别：前者考虑所有相关文档的位置，后者只关心第一个。
