# 向量检索、关键词检索、混合检索如何选择？

## 面试问题

向量检索、关键词检索、混合检索如何选择？

## 考察重点

- 是否理解不同检索技术的底层原理和适用场景
- 能否根据业务需求选择合适的检索方案
- 是否能说明混合检索的融合策略
- 是否理解向量检索的局限性及应对方法

## 核心结论

三种检索方式各有优劣：向量检索擅长语义理解但对关键词精确匹配较弱，关键词检索擅长精确匹配但无法捕捉语义，混合检索结合两者优势。选择依据是业务场景对语义理解 vs 精确匹配的需求比例。

## 底层原理对比

### 向量检索

基于语义 embedding 的最近邻搜索：

```
Query: "如何修复汽车发动机"
    ↓ Embedding
Vector: [0.12, -0.34, 0.67, ...]
    ↓ ANN搜索
Top-K最近邻文档
```

核心算法：
- **HNSW**：分层可导航小世界图，高精度但内存占用大
- **IVF**：倒排索引加速，平衡精度与速度
- **PQ**：乘积量化，压缩向量降低内存

```python
class VectorSearchIndex:
    def __init__(self, dimension: int = 1536):
        self.dimension = dimension
        self.index = faiss.IndexHNSWFlat(dimension, 32)
        self.documents = []
    
    def add(self, doc_id: str, text: str, embedding: np.ndarray):
        self.index.add(embedding.reshape(1, -1))
        self.documents.append({'id': doc_id, 'text': text})
    
    def search(self, query_embedding: np.ndarray, k: int = 10) -> List[SearchResult]:
        distances, indices = self.index.search(query_embedding.reshape(1, -1), k)
        return [
            SearchResult(
                doc_id=self.documents[i]['id'],
                text=self.documents[i]['text'],
                score=1.0 / (1.0 + distances[0][j])
            )
            for j, i in enumerate(indices[0])
        ]
```

### 关键词检索（BM25）

基于词频和文档频率的统计排序：

```
Score = Σ IDF(term) × (tf(term) × (k1 + 1)) / (tf(term) + k1 × (1 - b + b × |d|/avgdl))

k1 = 1.5 (词频饱和参数)
b = 0.75 (文档长度归一化参数)
```

```python
class BM25Search:
    def __init__(self, k1: float = 1.5, b: float = 0.75):
        self.k1 = k1
        self.b = b
        self.documents = []
        self.avgdl = 0
        self.idf = {}
    
    def index(self, documents: List[str]):
        self.documents = documents
        doc_lengths = [len(doc.split()) for doc in documents]
        self.avgdl = sum(doc_lengths) / len(doc_lengths)
        
        # 计算IDF
        df = defaultdict(int)
        for doc in documents:
            for term in set(doc.split()):
                df[term] += 1
        
        N = len(documents)
        for term, df_t in df.items():
            self.idf[term] = math.log((N - df_t + 0.5) / (df_t + 0.5) + 1)
    
    def search(self, query: str, k: int = 10) -> List[SearchResult]:
        query_terms = query.split()
        scores = []
        
        for i, doc in enumerate(self.documents):
            doc_terms = doc.split()
            doc_len = len(doc_terms)
            score = 0.0
            
            for term in query_terms:
                if term in self.idf:
                    tf = doc_terms.count(term)
                    numerator = tf * (self.k1 + 1)
                    denominator = tf + self.k1 * (1 - self.b + self.b * doc_len / self.avgdl)
                    score += self.idf[term] * numerator / denominator
            
            scores.append((i, score))
        
        scores.sort(key=lambda x: x[1], reverse=True)
        return [SearchResult(doc_id=str(i), text=self.documents[i], score=s) for i, s in scores[:k]]
```

## 选择决策树

```
查询特点分析
    │
    ├─ 包含专有名词/代码/精确术语？
    │   └─ Yes → BM25权重 > 0.5
    │
    ├─ 需要语义理解（如"查找类似..."）？
    │   └─ Yes → 向量权重 > 0.6
    │
    ├─ 查询短且关键词明确？
    │   └─ Yes → BM25为主
    │
    └─ 查询长且语义复杂？
        └─ Yes → 向量为主
```

## 混合检索实现

### 融合策略

```python
class HybridSearch:
    def __init__(
        self,
        vector_weight: float = 0.5,
        keyword_weight: float = 0.5,
        fusion_method: str = "rrf"  # rrf: Reciprocal Rank Fusion
    ):
        self.vector_weight = vector_weight
        self.keyword_weight = keyword_weight
        self.fusion_method = fusion_method
    
    def search(
        self,
        query: str,
        vector_index,
        keyword_index,
        k: int = 10
    ) -> List[SearchResult]:
        # 1. 并行执行两种检索
        vector_results = vector_index.search(query, k * 2)
        keyword_results = keyword_index.search(query, k * 2)
        
        # 2. 分数归一化
        vector_scores = self.normalize_scores(vector_results)
        keyword_scores = self.normalize_scores(keyword_results)
        
        # 3. 融合结果
        if self.fusion_method == "rrf":
            return self.reciprocal_rank_fusion(vector_scores, keyword_scores, k)
        elif self.fusion_method == "weighted":
            return self.weighted_fusion(vector_scores, keyword_scores, k)
        else:
            return self.round_robin_fusion(vector_scores, keyword_scores, k)
    
    def reciprocal_rank_fusion(
        self,
        vector_scores: Dict[str, float],
        keyword_scores: Dict[str, float],
        k: int
    ) -> List[SearchResult]:
        rrf_scores = defaultdict(float)
        
        # 向量结果 RRF
        for rank, (doc_id, score) in enumerate(sorted(vector_scores.items(), key=lambda x: x[1], reverse=True)):
            rrf_scores[doc_id] += 1.0 / (60 + rank)
        
        # 关键词结果 RRF
        for rank, (doc_id, score) in enumerate(sorted(keyword_scores.items(), key=lambda x: x[1], reverse=True)):
            rrf_scores[doc_id] += 1.0 / (60 + rank)
        
        # 排序返回
        sorted_results = sorted(rrf_scores.items(), key=lambda x: x[1], reverse=True)
        return [SearchResult(doc_id=doc_id, score=score) for doc_id, score in sorted_results[:k]]
    
    def weighted_fusion(
        self,
        vector_scores: Dict[str, float],
        keyword_scores: Dict[str, float],
        k: int
    ) -> List[SearchResult]:
        combined = defaultdict(float)
        
        # 获取所有doc_id
        all_docs = set(vector_scores.keys()) | set(keyword_scores.keys())
        
        for doc_id in all_docs:
            v_score = vector_scores.get(doc_id, 0)
            k_score = keyword_scores.get(doc_id, 0)
            combined[doc_id] = self.vector_weight * v_score + self.keyword_weight * k_score
        
        sorted_results = sorted(combined.items(), key=lambda x: x[1], reverse=True)
        return [SearchResult(doc_id=doc_id, score=score) for doc_id, score in sorted_results[:k]]
```

### 动态权重调整

```python
class AdaptiveHybridSearch(HybridSearch):
    def calculate_weights(self, query: str) -> Tuple[float, float]:
        # 分析query特征
        has_exact_terms = self.has_exact_terms(query)
        is_semantic = self.is_semantic_query(query)
        query_length = len(query.split())
        
        if has_exact_terms:
            # 有精确关键词，增加BM25权重
            return 0.3, 0.7
        elif is_semantic or query_length > 10:
            # 语义查询或长查询，增加向量权重
            return 0.7, 0.3
        else:
            # 默认平衡
            return 0.5, 0.5
    
    def has_exact_terms(self, query: str) -> bool:
        exact_indicators = ['函数', '方法', '参数', '变量', 'API', 'SDK', 'class', 'def', 'function']
        return any(term in query for term in exact_indicators)
    
    def is_semantic_query(self, query: str) -> bool:
        semantic_indicators = ['类似', '相似', '怎么', '如何', '什么', '为什么', '意思']
        return any(term in query for term in semantic_indicators)
```

### 向量检索的优势场景

# 代码搜索示例
query = "用户认证的实现方式"

```

```

## 生产环境选择建议

### 决策矩阵

| 因素 | 向量检索 | BM25 | 混合 |
|------|---------|------|------|
| 数据规模 | >100万 | <1000万 | 任意 |
| 查询延迟 | 10-50ms | 1-10ms | 20-60ms |
| 内存占用 | 高 | 低 | 中 |
| 维护成本 | 中 | 低 | 高 |
| 语义理解 | ✅ | ❌ | ✅ |
| 精确匹配 | ❌ | ✅ | ✅ |
| 同义词 | ✅ | ❌ | ✅ |

### 实践建议

```python
# 小规模场景（<10万文档）
# 纯BM25即可，延迟低、维护简单

# 中等规模（10万-100万）
# 混合检索，使用轻量级向量索引

# 大规模场景（>100万）
# 分层检索：向量 → BM25二次验证
# 或者：向量 + 结果缓存
```
