# 如何设计元数据？

## 面试问题

如何设计元数据？

## 考察重点

- 是否理解元数据在检索系统中的过滤和精排作用
- 能否说清楚元数据的设计原则和常见类型
- 是否能结合业务场景设计合理的元数据体系
- 是否理解元数据对检索效果的影响

## 核心结论

元数据是检索系统的"过滤器"和"导航器"，设计原则是：高频过滤条件优先、枚举值优于连续值、层级结构优于扁平结构。

## 元数据的作用

```
检索流程：
用户Query → 向量检索 → 初排结果 → 元数据过滤 → 精排结果 → 输出

元数据在其中的作用：
1. 缩小检索范围（时间、类型、来源）
2. 提供可解释性（结果分类标签）
3. 支持精确匹配（ID精确查找）
4. 实现权限控制（可见性过滤）
```

## 元数据设计原则

### 原则1：高频过滤条件优先

```python
class MetadataSchema:
    # 常见高频过滤条件应作为顶级字段
    TOP_LEVEL_FIELDS = [
        'doc_type',      # 用户常按类型筛选
        'language',       # 多语言场景必筛
        'date_range',    # 时间范围筛选
        'category',      # 分类筛选
        'access_level',  # 权限筛选
    ]
    
    # 低频条件放二级
    SECONDARY_FIELDS = [
        'author',
        'tags',
        'quality_score',
    ]
```

### 原则2：枚举值优于连续值

```python
# 不好：连续值难以过滤
@dataclass
class BadMetadata:
    exact_date: datetime  # 精确日期
    
# 好：离散化便于过滤
@dataclass
class GoodMetadata:
    date_bucket: str  # "today", "this_week", "this_month", "older"
    
    @staticmethod
    def from_date(dt: datetime) -> str:
        now = datetime.now()
        delta = now - dt
        if delta.days == 0:
            return "today"
        elif delta.days < 7:
            return "this_week"
        elif delta.days < 30:
            return "this_month"
        else:
            return "older"
```

### 原则3：层级结构便于过滤

```python
class HierarchicalMetadata:
    # 不好的设计：扁平结构
    flat_tags = ["python", "machine-learning", "nlp", "beginner", "tutorial"]
    
    # 好的设计：层级结构
    hierarchical_metadata = {
        "language": "python",
        "domain": "machine-learning",
        "sub_domain": "nlp",
        "level": "beginner",
        "content_type": "tutorial"
    }
    
    # 支持逐层下钻过滤
    def filter_by_hierarchy(self, docs: List[Document], **filters) -> List[Document]:
        for key, value in filters.items():
            if value:  # 只过滤非空条件
                docs = [d for d in docs if d.metadata.get(key) == value]
        return docs
```

### 向量数据库元数据过滤

```python
class VectorDBWithMetadata:
    def __init__(self, index_name: str):
        self.client = PineconeClient(index_name)
    
    def upsert_with_metadata(self, chunks: List[ChunkWithMetadata]):
        for chunk in chunks:
            self.client.upsert(
                id=chunk.chunk_id,
                vector=chunk.embedding,
                metadata={
                    "doc_id": chunk.doc_id,
                    "doc_type": chunk.metadata.doc_type,
                    "language": chunk.metadata.language,
                    "category": chunk.metadata.category,
                    "access_level": chunk.metadata.access_level,
                    "created_at": chunk.metadata.created_at.isoformat(),
                    "tags": chunk.metadata.tags
                }
            )
    
    def search_with_filter(
        self,
        query_embedding: List[float],
        top_k: int,
        filter_conditions: dict
    ) -> List[SearchResult]:
        return self.client.query(
            vector=query_embedding,
            top_k=top_k,
            filter=filter_conditions,
            include_metadata=True
        )
```

## 元数据过滤实践

### 常见过滤模式

```python
class MetadataFilterBuilder:
    def build_filters(
        self,
        doc_type: str = None,
        language: str = None,
        date_range: Tuple[datetime, datetime] = None,
        category: str = None,
        tags: List[str] = None,
        access_level: str = None
    ) -> dict:
        filters = []
        
        if doc_type:
            filters.append({"field": "doc_type", "operator": "==", "value": doc_type})
        
        if language:
            filters.append({"field": "language", "operator": "==", "value": language})
        
        if date_range:
            filters.append({
                "field": "created_at",
                "operator": "range",
                "value": {"gte": date_range[0].isoformat(), "lte": date_range[1].isoformat()}
            })
        
        if category:
            filters.append({"field": "category", "operator": "==", "value": category})
        
        if tags:
            filters.append({"field": "tags", "operator": "in", "value": tags})
        
        if access_level:
            filters.append({"field": "access_level", "operator": "==", "value": access_level})
        
        return {"type": "and", "conditions": filters}
```

### 权限过滤实现

```python
class PermissionMetadataFilter:
    def filter_by_permission(
        self,
        docs: List[Document],
        user_id: str,
        user_access_level: str
    ) -> List[Document]:
        level_hierarchy = {
            "public": 0,
            "internal": 1,
            "confidential": 2,
            "restricted": 3
        }
        
        user_level = level_hierarchy.get(user_access_level, 99)
        
        filtered = []
        for doc in docs:
            doc_level = level_hierarchy.get(doc.metadata.access_level, 0)
            if doc_level <= user_level:
                # 进一步检查是否有具体用户权限
                if doc.metadata.access_level == "restricted":
                    if user_id in doc.metadata.allowed_users:
                        filtered.append(doc)
                else:
                    filtered.append(doc)
        
        return filtered
```

## 元数据质量保障

### 元数据自动提取

```python
class MetadataExtractor:
    def extract(self, doc: Document) -> DocumentMetadata:
        return DocumentMetadata(
            doc_id=doc.doc_id,
            title=doc.title,
            created_at=self.extract_created_at(doc),
            updated_at=self.extract_updated_at(doc),
            author=self.extract_author(doc),
            source=self.extract_source(doc),
            doc_type=self.classify_doc_type(doc),
            language=self.detect_language(doc.content),
            tags=self.extract_tags(doc),
            category=self.classify_category(doc),
            word_count=self.count_words(doc.content),
            reading_time_minutes=self.estimate_reading_time(doc.content),
            quality_score=self.estimate_quality(doc),
            verified=False
        )
    
    def classify_doc_type(self, doc: Document) -> str:
        # 基于内容特征分类
        if self.contains_code(doc.content):
            return "code"
        elif self.is_api_doc(doc.content):
            return "api_docs"
        elif self.is_tutorial(doc.content):
            return "tutorial"
        elif self.is_reference(doc.content):
            return "reference"
        else:
            return "article"
```

## 常见问题与解决

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 过滤后结果过少 | 条件太严格 | 放宽条件或使用OR逻辑 |
| 元数据缺失 | 提取不完整 | 补充默认值 + 人工审核 |
| 元数据不一致 | 多来源汇入 | 统一Schema + 规范化流程 |
| 过滤性能差 | 字段未索引 | 确保过滤字段有索引 |
