# 检索结果过多如何压缩？

## 面试问题

检索结果过多如何压缩？

## 考察重点

- 是否理解检索结果压缩的必要性和目的
- 是否掌握多种压缩策略的适用场景
- 是否有平衡信息保留和压缩的判断力
- 是否理解压缩对最终效果的影响

## 核心结论

检索结果压缩是在"信息完整性"和"上下文容量"之间找平衡。压缩不是简单裁剪，而是根据信息重要性和任务需求进行选择性保留。常用策略包括：基于相关性的截断、基于多样性的去重、基于重要性的加权、以及摘要生成。

## 参考回答

假设用户问"Python的异步编程"，检索系统返回了100个相关文档。全部塞进上下文？上下文窗口肯定不够。

这时候需要压缩。压缩的目标是：在有限的上下文容量内，保留最关键的信息。

**场景分析**：
- 100个文档可能有大量重复内容（比如多个博客都引用同一篇官方文档）
- 有些文档只回答了问题的一部分
- 有些文档质量高，有些质量低
- 用户可能只需要某个侧面的答案

**压缩思路**：
1. 先对结果去重（相似内容只保留一个）
2. 按相关性排序，截断到Top-K
3. 如果Top-K内容还是太长，对每个文档做摘要
4. 如果摘要还是太长，只保留核心句子

### 压缩策略分类

```python
class CompressionStrategy:
    """
    检索结果压缩策略分类
    """
    
    @dataclass
    class Strategy:
        name: str
        description: str
       适用场景: str
        优点: str
        缺点: str
    
    STRATEGIES = [
        Strategy(
            name="相关性截断",
            description="按相关性分数排序，保留Top-K",
            适用场景="初步压缩，保留最重要结果",
            优点="简单直接，保证核心相关结果",
            缺点="可能丢失长尾相关信息"
        ),
        Strategy(
            name="多样性去重",
            description="去除相似内容，保留不同观点",
            适用场景="多文档综合，需要全面覆盖",
            优点="保证信息多样性，避免重复",
            缺点="可能丢失高相关性的重复内容"
        ),
        Strategy(
            name="信息密度压缩",
            description="按信息量密度排序，保留高密度内容",
            适用场景="长文档处理",
            优点="保留最大信息量",
            缺点="可能破坏文档结构"
        ),
        Strategy(
            name="摘要压缩",
            description="对每个文档生成摘要",
            适用场景="需要保留原意但减少篇幅",
            优点="大幅减少token消耗",
            缺点="摘要质量依赖模型能力"
        )
    ]
```

### 多样性去重

```python
class DiversityDeduplicator:
    def __init__(self, similarity_threshold: float = 0.85):
        self.similarity_threshold = similarity_threshold
    
    def deduplicate(
        self,
        documents: List[RetrievedDocument]
    ) -> List[RetrievedDocument]:
        """
        多样性去重：保留不同观点的文档
        """
        if not documents:
            return []
        
        # 1. 按相关性排序
        sorted_docs = sorted(documents, key=lambda d: d.score, reverse=True)
        
        # 2. 选择代表性文档
        selected = [sorted_docs[0]]  # 第一个（最高相关）直接保留
        
        for doc in sorted_docs[1:]:
            # 检查与已选文档的相似度
            is_duplicate = False
            for selected_doc in selected:
                similarity = self._calculate_similarity(doc, selected_doc)
                if similarity > self.similarity_threshold:
                    is_duplicate = True
                    break
            
            if not is_duplicate:
                selected.append(doc)
        
        return selected
    
    def _calculate_similarity(
        self,
        doc1: RetrievedDocument,
        doc2: RetrievedDocument
    ) -> float:
        """计算两个文档的相似度"""
        # 使用向量余弦相似度
        vec1 = doc1.embedding
        vec2 = doc2.embedding
        return np.dot(vec1, vec2) / (np.linalg.norm(vec1) * np.linalg.norm(vec2))
```

### MMR最大边际相关性

```python
class MaxMarginalRelevance:
    """
    MMR (Maximal Marginal Relevance) 压缩策略
    同时考虑相关性和多样性
    """
    
    def __init__(
        self,
        lambda_param: float = 0.5,
        max_docs: int = 10
    ):
        self.lambda_param = lambda_param  # 相关性vs多样性权重
        self.max_docs = max_docs
    
    def select(
        self,
        query: str,
        candidates: List[RetrievedDocument]
    ) -> List[RetrievedDocument]:
        """
        MMR选择：在相关性和多样性之间平衡
        """
        selected = []
        remaining = candidates.copy()
        
        while len(selected) < self.max_docs and remaining:
            best_score = float('-inf')
            best_doc = None
            best_idx = None
            
            for i, doc in enumerate(remaining):
                # 相关性分数
                relevance = doc.score
                
                # 与已选文档的最大相似度（多样性惩罚）
                max_similarity = 0
                if selected:
                    similarities = [
                        self._cosine_similarity(doc.embedding, s.embedding)
                        for s in selected
                    ]
                    max_similarity = max(similarities)
                
                # MMR分数
                mmr_score = (
                    self.lambda_param * relevance -
                    (1 - self.lambda_param) * max_similarity
                )
                
                if mmr_score > best_score:
                    best_score = mmr_score
                    best_doc = doc
                    best_idx = i
            
            if best_doc:
                selected.append(best_doc)
                remaining.pop(best_idx)
        
        return selected
    
    def _cosine_similarity(self, vec1: np.ndarray, vec2: np.ndarray) -> float:
        return np.dot(vec1, vec2) / (np.linalg.norm(vec1) * np.linalg.norm(vec2))
```

### 分层压缩

```python
class HierarchicalCompression:
    """
    分层压缩：从粗到细逐层压缩
    """
    
    def __init__(
        self,
        max_tokens_per_doc: int = 500,
        max_total_tokens: int = 4000
    ):
        self.max_tokens_per_doc = max_tokens_per_doc
        self.max_total_tokens = max_total_tokens
    
    def compress(
        self,
        query: str,
        documents: List[RetrievedDocument]
    ) -> CompressionResult:
        """
        分层压缩流程
        """
        # 第一层：相关性截断
        top_docs = self._rough_cutoff(documents, limit=self.max_total_tokens // self.max_tokens_per_doc)
        
        # 第二层：去重
        deduped_docs = self._deduplicate(top_docs)
        
        # 第三层：每个文档压缩到限制长度
        compressed_docs = []
        remaining_tokens = self.max_total_tokens
        
        for doc in deduped_docs:
            if remaining_tokens <= 0:
                break
            
            doc_tokens = self._count_tokens(doc.content)
            
            if doc_tokens <= remaining_tokens:
                compressed_docs.append(doc)
                remaining_tokens -= doc_tokens
            else:
                # 需要压缩单个文档
                compressed = self._compress_single_doc(doc, remaining_tokens)
                compressed_docs.append(compressed)
                remaining_tokens = 0
        
        return CompressionResult(
            documents=compressed_docs,
            original_count=len(documents),
            compressed_count=len(compressed_docs),
            total_tokens=self._count_tokens_from_docs(compressed_docs)
        )
    
    def _compress_single_doc(
        self,
        doc: RetrievedDocument,
        max_tokens: int
    ) -> RetrievedDocument:
        """压缩单个文档到指定token数"""
        # 1. 尝试句子级别截断
        sentences = self._split_sentences(doc.content)
        selected = []
        token_count = 0
        
        for sentence in sentences:
            sentence_tokens = self._count_tokens(sentence)
            if token_count + sentence_tokens <= max_tokens:
                selected.append(sentence)
                token_count += sentence_tokens
            else:
                # 句子太长，截断
                selected.append(self._truncate_sentence(sentence, max_tokens - token_count))
                break
        
        compressed_content = " ".join(selected)
        
        return RetrievedDocument(
            id=doc.id,
            content=compressed_content,
            score=doc.score,
            metadata={**doc.metadata, "compressed": True}
        )
    
    def _split_sentences(self, text: str) -> List[str]:
        """拆分句子"""
        return re.split(r'[.!?]+', text)
    
    def _truncate_sentence(self, sentence: str, max_tokens: int) -> str:
        """截断句子到指定token数"""
        words = sentence.split()
        if len(words) <= max_tokens:
            return sentence
        return " ".join(words[:max_tokens])
    
    def _count_tokens(self, text: str) -> int:
        """估算token数（简单用空格分词）"""
        return len(text.split())
```

### 摘要压缩

```python
class SummarizationCompression:
    """
    基于摘要的压缩
    """
    
    def __init__(self, llm: LLM):
        self.llm = llm
    
    def compress(
        self,
        documents: List[RetrievedDocument],
        query: str,
        max_tokens_per_doc: int = 300
    ) -> List[RetrievedDocument]:
        """
        对每个文档生成query相关的摘要
        """
        compressed = []
        
        for doc in documents:
            # 构建摘要prompt
            prompt = f"""根据以下查询，对文档进行摘要：

查询：{query}

文档：{doc.content}

请生成一个简洁的摘要，保留与查询相关的信息，不超过{max_tokens_per_doc}个token。

摘要："""
            
            # 调用LLM生成摘要
            summary = self.llm.generate(prompt)
            
            compressed.append(RetrievedDocument(
                id=doc.id,
                content=summary,
                score=doc.score,
                metadata={**doc.metadata, "is_summary": True}
            ))
        
        return compressed
    
    def compress_batch(
        self,
        documents: List[RetrievedDocument],
        query: str,
        max_total_tokens: int = 4000
    ) -> List[RetrievedDocument]:
        """
        批量压缩，在总token限制内
        """
        # 先估算每个文档的摘要长度
        num_docs = len(documents)
        tokens_per_doc = min(300, max_total_tokens // num_docs)
        
        return self.compress(documents, query, tokens_per_doc)
```

### 信息密度排序

```python
class InformationDensityRanker:
    """
    按信息密度排序：保留信息量最大的内容
    """
    
    def __init__(self):
        self.stop_words = set(['the', 'a', 'an', 'is', 'are', 'was', 'were', ...])
    
    def rank(
        self,
        documents: List[RetrievedDocument],
        top_k: int = 10
    ) -> List[RetrievedDocument]:
        """
        按信息密度重新排序
        """
        scored_docs = []
        
        for doc in documents:
            density = self._calculate_density(doc.content)
            combined_score = (
                0.7 * doc.score +  # 原相关性分数
                0.3 * density      # 信息密度
            )
            scored_docs.append((combined_score, doc))
        
        # 排序并返回Top-K
        scored_docs.sort(key=lambda x: x[0], reverse=True)
        return [doc for _, doc in scored_docs[:top_k]]
    
    def _calculate_density(self, text: str) -> float:
        """
        计算信息密度
        信息密度 = 实义词数量 / 总词数
        """
        words = text.lower().split()
        if not words:
            return 0.0
        
        content_words = [w for w in words if w not in self.stop_words]
        return len(content_words) / len(words)
```

## Claude Code的压缩策略

```python
class ClaudeCodeRetrievalCompression:
    """
    Claude Code的检索结果压缩实现
    """
    
    def __init__(self):
        self.max_context_tokens = 8000  # 代码补全上下文限制
        self.code_ranker = CodeSpecificRanker()
    
    def compress_for_completion(
        self,
        retrieved: List[RetrievedSnippet],
        cursor_context: str,
        target_tokens: int
    ) -> CompressionResult:
        """
        为代码补全压缩检索结果
        """
        # 1. 代码特定排序（优先完整函数/类定义）
        ranked = self.code_ranker.rank_code_specific(
            retrieved,
            cursor_context
        )
        
        # 2. 长度感知截断
        truncated = self._length_aware_truncate(
            ranked,
            target_tokens
        )
        
        # 3. 去重（去除重复的import或函数定义）
        deduplicated = self._deduplicate_code_snippets(truncated)
        
        return deduplicated
    
    def _length_aware_truncate(
        self,
        documents: List[RetrievedSnippet],
        max_tokens: int
    ) -> List[RetrievedSnippet]:
        """
        长度感知的截断
        """
        selected = []
        current_tokens = 0
        
        for doc in documents:
            doc_tokens = self._estimate_code_tokens(doc.content)
            
            if current_tokens + doc_tokens <= max_tokens:
                selected.append(doc)
                current_tokens += doc_tokens
            else:
                # 尝试截断代码块（保留函数签名）
                truncated = self._truncate_code(doc, max_tokens - current_tokens)
                if truncated:
                    selected.append(truncated)
                break
        
        return selected
    
    def _truncate_code(
        self,
        snippet: RetrievedSnippet,
        max_tokens: int
    ) -> RetrievedSnippet:
        """
        截断代码片段，保留函数签名和核心逻辑
        """
        lines = snippet.content.split('\n')
        selected_lines = []
        current_tokens = 0
        
        for i, line in enumerate(lines):
            line_tokens = len(line.split())
            
            # 保留函数签名（前几行通常是函数定义）
            if i < 5 or line.startswith('def ') or line.startswith('class '):
                if current_tokens + line_tokens <= max_tokens:
                    selected_lines.append(line)
                    current_tokens += line_tokens
                else:
                    break
            else:
                # 跳过中间实现，只保留关键行
                if line.strip() and (
                    'return' in line or
                    'if ' in line or
                    'for ' in line or
                    'while ' in line
                ):
                    if current_tokens + line_tokens <= max_tokens:
                        selected_lines.append(line)
                        current_tokens += line_tokens
        
        return RetrievedSnippet(
            id=snippet.id,
            content='\n'.join(selected_lines),
            score=snippet.score,
            metadata={**snippet.metadata, "truncated": True}
        )
```

### 压缩决策框架

```python
class CompressionDecisionEngine:
    """
    压缩决策引擎：根据情况选择最佳压缩策略
    """
    
    def __init__(self):
        self.strategies = {
            "mmr": MaxMarginalRelevance(),
            "hierarchical": HierarchicalCompression(),
            "summarization": SummarizationCompression(llm),
            "density": InformationDensityRanker(),
            "diversity": DiversityDeduplicator()
        }
    
    def decide_compression(
        self,
        query: str,
        documents: List[RetrievedDocument],
        context_limit: int,
        task_type: str
    ) -> CompressionPlan:
        """
        决定压缩策略
        """
        original_tokens = sum(self._count_tokens(d.content) for d in documents)
        
        # 计算压缩比
        compression_ratio = context_limit / original_tokens if original_tokens > 0 else 1.0
        
        # 根据任务类型和压缩比选择策略
        if task_type == "diverse_answer":
            # 需要多角度答案，选择MMR
            return CompressionPlan(
                strategy="mmr",
                params={"lambda": 0.5, "max_docs": context_limit // 500}
            )
        elif compression_ratio < 0.3:
            # 压缩比很高，需要摘要
            return CompressionPlan(
                strategy="summarization",
                params={"max_tokens_per_doc": context_limit // len(documents)}
            )
        elif task_type == "code_generation":
            # 代码任务，用代码特定策略
            return CompressionPlan(
                strategy="hierarchical",
                params={"max_tokens_per_doc": 500, "max_total_tokens": context_limit}
            )
        else:
            # 默认用多样性去重
            return CompressionPlan(
                strategy="diversity",
                params={"similarity_threshold": 0.85}
            )
```

### 压缩质量评估

```python
class CompressionQualityEvaluator:
    """
    评估压缩质量
    """
    
    def evaluate(
        self,
        original: List[RetrievedDocument],
        compressed: List[RetrievedDocument],
        query: str
    ) -> CompressionMetrics:
        """
        评估压缩效果
        """
        # 1. 召回率：压缩后保留了多少相关信息
        recall = self._calculate_recall(original, compressed, query)
        
        # 2. 压缩比
        compression_ratio = (
            sum(self._count_tokens(d.content) for d in compressed) /
            sum(self._count_tokens(d.content) for d in original)
            if original else 1.0
        )
        
        # 3. 多样性：是否保留了不同观点
        diversity = self._calculate_diversity(compressed)
        
        # 4. 覆盖度：是否覆盖了query的各个方面
        coverage = self._calculate_coverage(original, compressed, query)
        
        return CompressionMetrics(
            recall=recall,
            compression_ratio=compression_ratio,
            diversity=diversity,
            coverage=coverage,
            overall_score=0.4 * recall + 0.2 * (1 - compression_ratio) + 0.2 * diversity + 0.2 * coverage
        )
```

## 常见误区

- **过度压缩**：为减少token牺牲太多信息
- **一刀切**：所有场景用同一种压缩策略
- **忽视多样性**：只保留最相关的，可能丢失不同角度
- **不考虑任务类型**：代码任务和问答任务的压缩需求不同

## 学习建议

建议动手实现一个简单的压缩器：对同一批检索结果分别使用不同策略压缩，对比压缩前后的效果差异。重点理解MMR算法：在保证相关性的同时如何通过惩罚相似文档来保证多样性。
