# 工具调用日志应该记录什么？

## 面试问题

工具调用日志应该记录什么？

## 考察重点

- 是否理解工具调用日志在可观测性体系中的核心地位
- 是否能设计完整的工具调用日志结构，覆盖输入、输出、耗时、错误
- 是否理解日志记录与性能影响、安全合规之间的平衡
- 是否能基于日志进行问题定位和根因分析

## 核心结论

工具调用日志是智能体系统可观测性的核心组件，既是问题定位的依据，也是安全审计的基础，更是模型行为分析的数据源。完整的工具调用日志应包含：调用元数据（时间、ID、工具名）、输入参数（处理敏感信息脱敏）、输出摘要（截断大对象）、执行耗时、错误信息（包含堆栈）。设计时需要平衡日志完整性和性能开销、安全合规。

## 日志设计的核心原则

### 日志完整性 vs 性能开销

| 记录内容 | 必要性 | 性能影响 | 备注 |
|---------|--------|---------|------|
| 调用时间戳 | 必须 | 极低 | 纳秒级 |
| 工具名称 | 必须 | 极低 | 字符串 |
| 输入参数 | 必须 | 中等 | 大对象序列化耗时 |
| 输出摘要 | 必须 | 中等 | 截断处理 |
| 执行耗时 | 必须 | 极低 | 整数 |
| 错误堆栈 | 必须 | 低 | 字符串格式化 |
| 上下文 ID | 必须 | 极低 | 用于关联 |

### 日志分层

```python
class LogLevel(Enum):
    DEBUG = "debug"      # 开发调试，包含完整参数和结果
    INFO = "info"       # 正常调用，包含摘要
    WARNING = "warning"  # 可恢复错误
    ERROR = "error"     # 严重错误

@dataclass
class ToolCallLog:
    # 元数据
    log_id: str
    timestamp: datetime
    log_level: LogLevel

    # 调用信息
    tool_name: str
    call_id: str              # 用于关联同一调用的各阶段
    parent_call_id: str        # 父调用 ID，用于嵌套调用

    # 输入输出
    input_params: dict         # 输入参数（脱敏后）
    output_summary: str        # 输出摘要
    output_size_bytes: int    # 输出大小

    # 性能数据
    latency_ms: float
    token_usage: TokenUsage   # 包含输入输出 token 数

    # 错误信息
    error: ErrorInfo = None

    # 上下文关联
    task_id: str
    user_id: str
    session_id: str
```

### 日志记录的最佳实践

```python
class ToolCallLogger:
    def __init__(self, storage: LogStorage, config: LoggingConfig):
        self.storage = storage
        self.config = config
        self.sensitive_field_patterns = [
            "password", "token", "secret", "api_key", "credential"
        ]

    async def log_tool_call(
        self,
        tool_name: str,
        input_params: dict,
        output: Any,
        context: ExecutionContext,
        error: Exception = None
    ) -> str:
        log_id = str(uuid4())

        # 1. 敏感信息脱敏
        sanitized_params = self.sanitize_params(input_params)

        # 2. 构建日志记录
        log_record = ToolCallLog(
            log_id=log_id,
            timestamp=datetime.now(),
            tool_name=tool_name,
            call_id=context.call_id,
            parent_call_id=context.parent_call_id,
            input_params=sanitized_params,
            output_summary=self.truncate_output(output),
            output_size_bytes=self.estimate_size(output),
            latency_ms=context.latency_ms,
            token_usage=context.token_usage,
            error=self.extract_error_info(error) if error else None,
            task_id=context.task_id,
            user_id=context.user_id,
            session_id=context.session_id
        )

        # 3. 确定日志级别
        log_level = self.determine_log_level(log_record)
        log_record.log_level = log_level

        # 4. 异步写入存储
        asyncio.create_task(self.storage.insert(log_record))

        return log_id

    def sanitize_params(self, params: dict) -> dict:
        """敏感信息脱敏"""
        sanitized = {}
        for key, value in params.items():
            if self.is_sensitive_field(key):
                sanitized[key] = "[REDACTED]"
            elif isinstance(value, dict):
                sanitized[key] = self.sanitize_params(value)
            elif isinstance(value, str) and len(value) > 1000:
                sanitized[key] = value[:1000] + "...[truncated]"
            else:
                sanitized[key] = value
        return sanitized

    def truncate_output(self, output: Any, max_length: int = 2000) -> str:
        """截断输出，防止日志过大"""
        if output is None:
            return "null"
        if isinstance(output, str):
            return output[:max_length] + "..." if len(output) > max_length else output
        if isinstance(output, (int, float, bool)):
            return str(output)
        # 其他类型转为字符串
        str_output = str(output)
        return str_output[:max_length] + "..." if len(str_output) > max_length else str_output

    def is_sensitive_field(self, field_name: str) -> bool:
        """判断是否为敏感字段"""
        field_lower = field_name.lower()
        return any(pattern in field_lower for pattern in self.sensitive_field_patterns)
```

## 上下文关联与链路追踪

### 调用链路构建

```python
class CallChainBuilder:
    def __init__(self):
        self.chains: dict[str, list[str]] = {}  # call_id -> [child_call_ids]

    def build_chain(self, root_call_id: str, logs: list[ToolCallLog]) -> CallChain:
        """从日志构建调用链路"""
        chain = CallChain(root_id=root_call_id)

        # 构建节点
        nodes = {}
        for log in logs:
            nodes[log.call_id] = CallNode(
                id=log.call_id,
                tool_name=log.tool_name,
                start_time=log.timestamp,
                end_time=log.timestamp + timedelta(milliseconds=log.latency_ms),
                status="error" if log.error else "success",
                input_params=log.input_params,
                output_summary=log.output_summary
            )

        # 构建边
        for log in logs:
            if log.parent_call_id:
                chain.add_edge(log.parent_call_id, log.call_id)

        chain.nodes = nodes
        return chain

class CallChain:
    def __init__(self, root_id: str):
        self.root_id = root_id
        self.nodes: dict[str, CallNode] = {}
        self.edges: list[tuple[str, str]] = []

    def add_edge(self, parent: str, child: str):
        self.edges.append((parent, child))

    def visualize(self) -> str:
        """生成调用链路图（文本格式）"""
        lines = [f"Call Chain (root: {self.root_id})"]
        for parent, child in self.edges:
            parent_node = self.nodes[parent]
            child_node = self.nodes[child]
            lines.append(f"  {parent_node.tool_name} -> {child_node.tool_name}")
        return "\n".join(lines)
```

### 上下文关联查询

```python
class LogQueryEngine:
    def __init__(self, storage: LogStorage):
        self.storage = storage

    async def query_by_task(self, task_id: str) -> list[ToolCallLog]:
        """查询某个任务的所有工具调用"""
        return await self.storage.query(task_id=task_id)

    async def query_by_session(self, session_id: str, time_range: TimeRange) -> list[ToolCallLog]:
        """查询某个会话在时间范围内的所有调用"""
        return await self.storage.query(
            session_id=session_id,
            start_time=time_range.start,
            end_time=time_range.end
        )

    async def query_errors(
        self,
        tool_name: str = None,
        time_range: TimeRange = None
    ) -> list[ToolCallLog]:
        """查询错误日志"""
        return await self.storage.query(
            tool_name=tool_name,
            has_error=True,
            start_time=time_range.start if time_range else None,
            end_time=time_range.end if time_range else None
        )

    async def query_slow_calls(self, threshold_ms: float = 1000) -> list[ToolCallLog]:
        """查询慢调用"""
        return await self.storage.query(
            min_latency_ms=threshold_ms
        )
```

## 日志分析与问题定位

### 常见问题定位场景

```python
class ToolCallAnalyzer:
    def __init__(self, logger: ToolCallLogger):
        self.logger = logger

    async def analyze_failure(self, call_id: str) -> FailureAnalysis:
        """分析单次调用失败"""
        log = await self.storage.get_call(call_id)

        analysis = FailureAnalysis(
            call_id=call_id,
            tool_name=log.tool_name,
            error_type=self.classify_error(log.error),
            root_cause=self.identify_root_cause(log)
        )

        # 查找相关调用
        related_calls = await self.find_related_calls(log)
        analysis.related_calls = related_calls

        return analysis

    def classify_error(self, error: ErrorInfo) -> str:
        """错误分类"""
        if not error:
            return "unknown"

        error_type = error.type.lower()

        if "timeout" in error_type:
            return "timeout"
        if "permission" in error_type or "denied" in error_type:
            return "permission_error"
        if "not found" in error_type:
            return "resource_not_found"
        if "invalid" in error_type:
            return "invalid_input"
        if "rate limit" in error_type:
            return "rate_limited"

        return "other"

    async def find_related_calls(self, log: ToolCallLog) -> list[ToolCallLog]:
        """查找相关调用（同任务、同用户等）"""
        return await self.storage.query(
            task_id=log.task_id,
            session_id=log.session_id,
            time_range=TimeRange(
                start=log.timestamp - timedelta(minutes=5),
                end=log.timestamp + timedelta(minutes=5)
            )
        )

    async def detect_patterns(self, logs: list[ToolCallLog]) -> list[Pattern]:
        """检测异常模式"""
        patterns = []

        # 检测频繁重试
        retry_counts = self.count_retries(logs)
        for tool_name, count in retry_counts.items():
            if count > 5:
                patterns.append(Pattern(
                    type="frequent_retry",
                    tool=tool_name,
                    detail=f"Retried {count} times"
                ))

        # 检测长时间运行
        slow_calls = [l for l in logs if l.latency_ms > 5000]
        if len(slow_calls) > 10:
            patterns.append(Pattern(
                type="slow_calls",
                detail=f"{len(slow_calls)} calls took > 5s"
            ))

        # 检测错误聚集
        error_by_tool = self.group_errors_by_tool(logs)
        for tool_name, errors in error_by_tool.items():
            if len(errors) > 20:
                patterns.append(Pattern(
                    type="error_cluster",
                    tool=tool_name,
                    detail=f"{len(errors)} errors in recent calls"
                ))

        return patterns
```

## 日志存储与成本优化

### 分层存储策略

```python
class HierarchicalLogStorage:
    def __init__(self):
        self.hot_storage: LogStorage    # SSD，本周数据
        self.warm_storage: LogStorage   # HDD，本月数据
        self.cold_storage: LogStorage  # 对象存储，历史数据

    async def insert(self, log: ToolCallLog):
        """根据时间热度选择存储层"""
        await self.hot_storage.insert(log)

        # 异步归档到温存储
        asyncio.create_task(self.archive_to_warm(log))

    async def archive_to_warm(self, log: ToolCallLog):
        """归档到温存储"""
        age = datetime.now() - log.timestamp
        if age > timedelta(days=7):
            await self.warm_storage.insert(log)

    async def query(self, **kwargs) -> list[ToolCallLog]:
        """查询时合并多层存储结果"""
        results = []

        # 先查热存储
        results.extend(await self.hot_storage.query(**kwargs))

        # 再查温存储
        results.extend(await self.warm_storage.query(**kwargs))

        # 最后查冷存储（如果需要）
        if self.need_cold_query(kwargs):
            results.extend(await self.cold_storage.query(**kwargs))

        return results
```

### 日志压缩与聚合

```python
class LogAggregator:
    def aggregate_hourly(self, logs: list[ToolCallLog]) -> HourlyAggregate:
        """按小时聚合日志"""
        hourly_stats = defaultdict(lambda: {
            "count": 0,
            "errors": 0,
            "total_latency": 0,
            "tools_used": set()
        })

        for log in logs:
            hour_key = log.timestamp.replace(minute=0, second=0, microsecond=0)
            stats = hourly_stats[hour_key]
            stats["count"] += 1
            stats["errors"] += 1 if log.error else 0
            stats["total_latency"] += log.latency_ms
            stats["tools_used"].add(log.tool_name)

        return HourlyAggregate(
            period_start=min(hourly_stats.keys()),
            period_end=max(hourly_stats.keys()),
            stats=dict(hourly_stats)
        )
```

## 工程实践 Checklist

设计工具调用日志时的 Checklist：

- [ ] **元数据完整**：是否记录了时间戳、调用 ID、工具名
- [ ] **参数脱敏**：敏感信息（密码、token、密钥）是否已脱敏
- [ ] **输出截断**：大输出对象是否已截断
- [ ] **错误信息完整**：错误堆栈、错误类型是否记录
- [ ] **性能数据**：延迟、token 消耗是否记录
- [ ] **上下文关联**：task_id、session_id、user_id 是否关联
- [ ] **调用链路**：嵌套调用的父子关系是否追踪
- [ ] **存储分层**：热数据、温数据、冷数据是否分层存储
- [ ] **查询支持**：是否支持按任务、会话、时间范围、错误类型查询

## 真实踩坑案例

### 案例：日志丢失导致问题无法定位

某系统在高并发时出现偶发错误，由于日志异步写入且没有缓冲，错误发生时的日志丢失，无法定位问题。

**修复方案**：
1. 增加日志缓冲队列（内存队列 + 磁盘 spillover）
2. 实现日志写入确认机制
3. 增加关键错误同步写入（同步 + 异步双写）

### 案例：日志过大导致存储成本爆炸

某团队没有对日志输出截断，单次工具调用返回了 10MB 的 JSON，导致日志存储成本急剧上升。

**修复方案**：
1. 增加输出大小限制（max 4KB）
2. 增加输出摘要字段（保留前 1KB + 总大小）
3. 实现日志轮转和过期自动删除

### 案例：日志中泄露敏感信息

某团队在日志中记录了完整的 API 请求参数，包含用户密码和支付信息，导致合规问题。

**修复方案**：
1. 增加敏感字段自动识别和脱敏
2. 增加日志审计流程
3. 实现 PII 数据检测告警
