# 如何设计数据库查询工具？

## 面试问题

如何设计数据库查询工具？

## 考察重点

- 是否理解数据库查询工具的安全风险：SQL 注入、权限过大、数据泄露
- 是否能设计安全的数据库查询工具，包含查询验证、结果限制、权限控制
- 是否能处理查询超时、大结果集、连接管理等边界情况
- 是否理解只读查询的实现方式和审计日志

## 核心结论

数据库查询工具让智能体能查询业务数据，是非常实用的能力，但也是高风险工具。设计不安全的数据库查询工具可能导致：
1. SQL 注入攻击
2. 数据泄露（查询敏感表）
3. 数据库压力（复杂查询、大结果集）
4. 权限滥用（UPDATE/DELETE 等危险操作）

安全的数据库查询工具设计核心是：**强制只读**（不允许修改操作）、**参数化查询**（防止注入）、**查询验证**（语法和语义检查）、**结果限制**（行数限制、分页）和**审计日志**（记录所有查询）。

## 安全风险分析

### 主要安全风险

| 风险类型 | 攻击方式 | 危害程度 | 防御措施 |
|---------|---------|---------|---------|
| SQL 注入 | `' OR '1'='1` | 极高 | 参数化查询 |
| 数据泄露 | `SELECT * FROM users` | 高 | 列权限控制 |
| 数据库压力 | `SELECT * FROM large_table` | 中 | 结果行数限制 |
| 敏感表访问 | `SELECT * FROM passwords` | 高 | 表级别权限 |
| 连接耗尽 | 复杂查询长时间占用连接 | 中 | 查询超时 + 连接池 |

## 安全防护实现

### 查询验证器

```python
class SQLQueryValidator:
    BLOCKED_KEYWORDS = {
        "WRITE": {"INSERT", "UPDATE", "DELETE", "DROP", "TRUNCATE", "ALTER", "CREATE", "REPLACE"},
        "DDL": {"CREATE", "ALTER", "DROP", "TRUNCATE"},
        "DCL": {"GRANT", "REVOKE"},
        "TCL": {"COMMIT", "ROLLBACK", "SAVEPOINT"}
    }

    def validate(self, query: str) -> ValidationResult:
        query_upper = query.upper()
        query_normalized = " ".join(query_upper.split())

        # 1. 检查是否为 SELECT
        if not query_normalized.strip().startswith("SELECT"):
            return ValidationResult(
                valid=False,
                reason="Only SELECT queries are allowed"
            )

        # 2. 检查危险关键字
        for category, keywords in self.BLOCKED_KEYWORDS.items():
            for keyword in keywords:
                if keyword in query_normalized:
                    return ValidationResult(
                        valid=False,
                        reason=f"Keyword '{keyword}' from {category} category not allowed"
                    )

        # 3. 语法检查（使用简单解析）
        if not self.check_syntax(query):
            return ValidationResult(valid=False, reason="Invalid SQL syntax")

        # 4. 检查是否有多语句（分号分隔）
        if ";" in query.strip().rstrip(";"):
            return ValidationResult(
                valid=False,
                reason="Multiple statements not allowed"
            )

        return ValidationResult(valid=True)

    def check_syntax(self, query: str) -> bool:
        # 简化的语法检查
        open_parens = query.count("(")
        close_parens = query.count(")")
        if open_parens != close_parens:
            return False

        single_quotes = query.count("'")
        if single_quotes % 2 != 0:
            return False

        return True
```

### 参数化查询执行

```python
class SafeDatabaseExecutor:
    def __init__(self, connection_pool: ConnectionPool):
        self.pool = connection_pool
        self.validator = SQLQueryValidator()

    async def execute_query(
        self,
        query: str,
        params: dict = None,
        max_rows: int = 1000,
        timeout_seconds: int = 30
    ) -> QueryResult:
        # 1. 查询验证
        validation = self.validator.validate(query)
        if not validation.valid:
            return QueryResult(
                success=False,
                error=validation.reason
            )

        # 2. 获取连接
        conn = await self.pool.acquire()

        try:
            # 3. 设置超时
            async with asyncio.timeout(timeout_seconds):
                async with conn.cursor() as cursor:
                    # 4. 执行参数化查询
                    if params:
                        await cursor.execute(query, params)
                    else:
                        await cursor.execute(query)

                    # 5. 获取结果（限制行数）
                    columns = [desc[0] for desc in cursor.description] if cursor.description else []
                    rows = []
                    async for row in cursor:
                        rows.append(dict(zip(columns, row)))
                        if len(rows) >= max_rows:
                            break

                    return QueryResult(
                        success=True,
                        columns=columns,
                        rows=rows,
                        total_rows=len(rows),
                        truncated=len(rows) >= max_rows
                    )

        except asyncio.TimeoutError:
            return QueryResult(
                success=False,
                error=f"Query timeout after {timeout_seconds} seconds"
            )
        except Exception as e:
            return QueryResult(
                success=False,
                error=f"Query execution error: {str(e)}"
            )
        finally:
            await self.pool.release(conn)
```

### 表级别权限控制

```python
class TablePermissionController:
    def __init__(self):
        # 允许查询的表及其别名
        self.allowed_tables = {
            "orders": {"columns": ["*"], "description": "订单表"},
            "products": {"columns": ["*"], "description": "产品表"},
            "customers": {"columns": ["id", "name", "email"], "description": "客户表（脱敏）"},
            "analytics_daily": {"columns": ["*"], "description": "每日分析数据"}
        }

        # 禁止访问的敏感表
        self.blocked_tables = {
            "users", "passwords", "credentials", "secrets",
            "admin", "permissions", "sessions"
        }

    def validate_table_access(self, query: str) -> ValidationResult:
        import re

        # 提取查询中的表名（简单实现）
        table_pattern = r'FROM\s+(\w+)|JOIN\s+(\w+)'
        tables = re.findall(table_pattern, query.upper())

        for table_match in tables:
            table_name = table_match[0] or table_match[1]

            # 检查是否在禁止列表
            if table_name in self.blocked_tables:
                return ValidationResult(
                    valid=False,
                    reason=f"Access to table '{table_name}' is not allowed"
                )

            # 检查是否在允许列表
            if table_name not in self.allowed_tables:
                return ValidationResult(
                    valid=False,
                    reason=f"Table '{table_name}' is not in the allowed list"
                )

        return ValidationResult(valid=True)
```

async function executeDatabaseQuery(
  query: string,
  params: Record<string, unknown>,
  config: DatabaseQueryConfig
): Promise<QueryResult> {
  // 1. 验证查询只包含 SELECT
  if (!query.trim().toUpperCase().startsWith('SELECT')) {
    return { success: false, error: 'Only SELECT queries are allowed' };
  }

  // 2. 检查表权限
  const tableAccess = validateTableAccess(query, config);
  if (!tableAccess.valid) {
    return { success: false, error: tableAccess.reason };
  }

  // 3. 使用参数化查询执行
  const result = await executeWithParams(query, params, {
    timeout: config.timeoutSeconds,
    maxRows: config.maxRows
  });

  return result;
}

## 工程实践 Checklist

设计数据库查询工具时的 Checklist：

- [ ] **强制只读**：是否只允许 SELECT 查询
- [ ] **参数化查询**：是否使用参数化查询防止注入
- [ ] **查询验证**：是否验证了 SQL 语法和关键字
- [ ] **结果限制**：是否限制了返回行数
- [ ] **查询超时**：是否设置了查询超时
- [ ] **表权限控制**：是否控制了表级别访问
- [ ] **列权限控制**：是否控制了列级别访问
- [ ] **审计日志**：是否记录了所有查询

## 真实踩坑案例

### 案例：参数化查询未使用导致注入

某团队的数据库工具虽然接受参数，但内部拼接 SQL 字符串，导致注入攻击。

**修复方案**：
1. 强制使用参数化查询
2. 增加 SQL 注入测试用例
3. 代码 review 时重点检查 SQL 拼接

### 案例：大结果集导致内存溢出

某查询返回了数百万行数据，导致服务内存溢出。

**修复方案**：
1. 强制设置 max_rows 限制
2. 实现流式查询（streaming）
3. 增加查询结果预估（EXPLAIN）
