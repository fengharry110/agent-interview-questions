# 工具返回值应该如何结构化？

## 面试问题

工具返回值应该如何结构化？

## 考察重点

- 是否理解工具返回值对模型后续决策的影响
- 是否能设计结构化返回值（成功/失败状态、错误信息、数据内容）
- 是否理解 Codex 和 Claude Code 的返回值实现差异
- 是否能处理大返回值的截断和压缩

## 核心结论

工具返回值是模型理解工具执行结果的唯一渠道。设计良好的返回值结构能让模型准确判断工具是否成功执行、返回了什么信息、是否需要重试或换策略。返回值应包含：`status`（执行状态）、`data`（返回数据）、`error`（错误信息）、`metadata`（元数据）。大返回值需要截断或压缩，避免超出上下文限制。

## 返回值结构设计

### 一、标准返回值结构

```python
@dataclass
class ToolResult:
    status: ToolStatus          # success | failure | partial
    data: Any                  # 返回数据
    error: ErrorInfo = None    # 错误信息
    metadata: ResultMetadata   # 元数据

    def to_model_format(self) -> str:
        """转换为模型可理解的格式"""
        if self.status == ToolStatus.SUCCESS:
            return f"[Success] {self.format_data()}"
        elif self.status == ToolStatus.FAILURE:
            return f"[Failed] {self.error.message if self.error else 'Unknown error'}"
        else:
            return f"[Partial] {self.format_data()}"

    def format_data(self, max_length: int = 2000) -> str:
        """格式化数据，截断大对象"""
        if self.data is None:
            return "No data returned"

        if isinstance(self.data, str):
            if len(self.data) > max_length:
                return f"{self.data[:max_length]}... [truncated, total {len(self.data)} chars]"
            return self.data

        if isinstance(self.data, (int, float, bool)):
            return str(self.data)

        # 结构化数据转为字符串
        str_repr = json.dumps(self.data, ensure_ascii=False, indent=2)
        if len(str_repr) > max_length:
            return f"{str_repr[:max_length]}... [truncated]"
        return str_repr

@dataclass
class ErrorInfo:
    type: str                  # error type
    message: str               # 错误消息
    recoverable: bool          # 是否可恢复
    suggestion: str = None     # 修复建议

@dataclass
class ResultMetadata:
    tool_name: str
    execution_time_ms: float
    token_cost: int = 0
    cache_hit: bool = False
```

#### 外部 API 工具

```python
class APIToolResult(ToolResult):
    def __init__(self, url: str, response: HTTPResponse, error: Exception = None):
        if error:
            super().__init__(
                status=ToolStatus.FAILURE,
                data=None,
                error=ErrorInfo(
                    type=type(error).__name__,
                    message=str(error),
                    recoverable=isinstance(error, (TimeoutError, ConnectionError))

                metadata=ResultMetadata(tool_name="APIRequest")
            )
        else:

                data={
                    "status_code": response.status_code,
                    "body": response.text[:5000],  # 截断响应体
                    "headers": dict(response.headers)
                },
                metadata=ResultMetadata(tool_name="APIRequest")
            )
```

## Claude Code 的返回值实现

### 工具返回值结构

Claude Code 在 `result.ts` 中定义了标准的工具返回值：

```typescript
// result.ts
interface ToolResult {
  tool: string;           // 工具名称
  result: string;         // 返回内容的字符串形式
  error: string | null;   // 错误信息

  // 状态信息
  metadata?: {
    truncated?: boolean;   // 是否被截断
    size?: number;        // 原始大小
    lines?: number;        // 行数（文件操作）
    duration?: number;     // 执行时长
  };
}
```

### 返回值截断机制

```typescript
// truncation.ts
const MAX_RESULT_LENGTH = 4000;
const TRUNCATION_SUFFIX = "... [Output truncated]";

function truncateResult(result: string, maxLength: number = MAX_RESULT_LENGTH): { truncated: string; wasTruncated: boolean } {
  if (result.length <= maxLength) {
    return { truncated: result, wasTruncated: false };
  }

  return {
    truncated: result.slice(0, maxLength - TRUNCATION_SUFFIX.length) + TRUNCATION_SUFFIX,
    wasTruncated: true
  };
}
```

### 多工具并行返回

Claude Code 支持多个工具并行执行，返回值聚合：

```typescript
// parallel.ts
interface ParallelToolResult {
  results: ToolResult[];
  summary: {
    total: number;
    successes: number;
    failures: number;
    totalDuration: number;
  };
}

function aggregateResults(results: ToolResult[]): ParallelToolResult {
  return {
    results,
    summary: {
      total: results.length,
      successes: results.filter(r => !r.error).length,
      failures: results.filter(r => r.error).length,
      totalDuration: results.reduce((sum, r) => sum + (r.metadata?.duration || 0), 0)
    }
  };
}
```

## 返回值对模型行为的影响

### 模型如何解析返回值

```python
class ResultParser:
    def parse(self, result: ToolResult, model_context: ModelContext) -> ParseResult:
        """模型如何解析返回值"""

        # 1. 检查是否成功
        if result.status != ToolStatus.SUCCESS:
            return ParseResult(
                action="retry_or_abort",
                reason=f"Tool failed: {result.error.message if result.error else 'unknown'}",
                suggestion=result.error.suggestion if result.error else None
            )

        # 2. 检查数据是否为空
        if result.data is None or result.data == "":
            return ParseResult(
                action="continue",
                reason="Tool returned empty result, trying next step"
            )

        # 3. 检查是否被截断
        if result.metadata.get("truncated"):
            return ParseResult(
                action="consider_more_targeted_query",
                reason="Result was truncated, might need more specific query"
            )

        # 4. 检查数据量
        if isinstance(result.data, (list, dict)) and len(result.data) == 0:
            return ParseResult(
                action="modify_parameters",
                reason="Result is empty, should modify query parameters"
            )

        return ParseResult(
            action="proceed",
            reason="Tool executed successfully"
        )
```

### 返回值质量评估

```python
class ResultQualityEvaluator:
    def evaluate(self, result: ToolResult, expected: Any) -> QualityScore:
        scores = {}

        # 1. 完整性评分
        if result.status == ToolStatus.SUCCESS:
            scores["completeness"] = 1.0 if result.data else 0.5
        else:
            scores["completeness"] = 0.0

        # 2. 准确性评分（如果能验证）
        if expected:
            scores["accuracy"] = self.calculate_accuracy(result.data, expected)

        # 3. 可用性评分
        if result.metadata.get("truncated"):
            scores["usability"] = 0.7  # 被截断会降低可用性
        else:
            scores["usability"] = 1.0

        return QualityScore(
            overall=(scores["completeness"] * 0.4 + scores["accuracy"] * 0.4 + scores["usability"] * 0.2),
            details=scores
        )
```

## 工程实践 Checklist

设计工具返回值时的 Checklist：

- [ ] **状态明确**：success/failure/partial 状态是否清晰
- [ ] **错误信息完整**：错误类型、消息、建议是否完整
- [ ] **数据截断**：大返回值是否截断并告知模型
- [ ] **元数据丰富**：执行时间、大小、行数等是否记录
- [ ] **格式一致**：不同工具的返回值格式是否一致
- [ ] **可解析性**：返回值是否容易被模型理解和解析

## 真实踩坑案例

### 案例：返回值格式不一致导致模型困惑

某团队的不同工具返回格式不一致，有的返回字符串，有的返回 JSON 对象，有的返回列表。模型在解析时频繁出错。

**修复方案**：
1. 统一所有工具的返回值格式为 `ToolResult` 结构
2. 增加返回值类型验证
3. 在文档中明确每种工具的返回值格式

### 案例：返回值过大导致上下文溢出

某工具返回了 10MB 的文件内容，导致上下文溢出，任务直接失败。

**修复方案**：
1. 增加返回值大小限制
2. 实现智能截断（保留关键信息）
3. 增加分页读取机制
