# 智能体项目如何写进简历？

## 面试问题

你做过智能体项目，简历上应该怎么写才能让面试官觉得有技术含量，而不是"调了个 API"？

## 考察重点

- 是否能区分"调 API"和"做系统"的表述差异
- 是否知道简历上应该突出哪些技术维度
- 是否能用量化指标证明项目价值
- 是否能体现个人在项目中的独特贡献
- 是否了解当前行业对智能体工程师的能力预期

## 核心结论

简历上写智能体项目的核心不是描述功能，而是证明你构建了一个**有工程深度的系统**，核心体现在架构设计、工具工程、评测体系、安全控制和可观测性五个维度。

## 参考回答

写智能体项目进简历，要避开"调 API"陷阱。面试官每天看几十份简历，如果简历上写的是"基于 GPT-4 开发了智能客服系统"，这等于什么都没说。关键是把"调 API"这件事拆解成多个工程维度。

### 简历五层结构

```
第一层：项目背景与目标（1-2 行）
第二层：系统架构设计（2-3 行）
第三层：核心技术难点（3-4 行）
第四层：量化成果指标（2-3 行）
第五层：个人角色与贡献（1-2 行）
```

### 示例对比

| 版本 | 写法 | 效果 |
|-----|------|------|
| 差 | 基于 GPT-4 开发智能客服，回答用户问题 | 调 API |
| 中 | 基于 LangChain 搭建 Agent 系统，支持多轮对话 | 套框架 |
| 好 | 自研 Agent Loop + 动态工具编排，支持 20+ 工具热插拔，设计三级权限体系，线上 P95 延迟 3.2s | 有工程深度 |

### 具体写法示例

```text
智能客服 Agent 系统
- 技术栈：Python / FastAPI / Redis / 自研 Agent Framework
- 自研 Agent Loop 引擎，支持 Plan-and-Execute 与 ReAct 双模式切换，覆盖简单问答和复杂多步任务
- 设计动态工具注册与编排框架，支持 20+ 工具热插拔，工具调用成功率 97.3%
- 构建三级权限体系（只读 / 确认 / 自动执行），月均拦截异常工具调用 200+ 次
- 落地端到端评测流水线，200+ 回归测试用例，每次发布前自动化评测
- 线上 P95 延迟 3.2s，用户满意度从 72% 提升至 89%
```

## 深度展开

### 维度的工程化拆解

面试官看智能体项目简历时，内心在验证五个维度：

**维度一：Agent Loop**
你是不是自己写了控制循环，还是直接用框架的默认实现？控制循环的停止条件如何设置？如何处理循环失控？

**维度二：工具工程**
工具注册是手动硬编码还是动态发现？工具描述如何写才能让模型正确选择？工具参数校验和错误处理怎么做？

**维度三：上下文管理**
上下文窗口满了怎么处理？是简单截断还是智能压缩？多轮交互中如何保持状态一致性？

**维度四：评测体系**
你如何知道改了一个 Prompt 是变好还是变坏？有没有自动化评测流水线？评测覆盖哪些维度？

**维度五：安全控制**
工具调用有没有权限校验？用户的敏感数据如何隔离？有没有审计日志？

## 工程实践

### 简历自查清单

```python
def check_resume_quality(resume_section: str) -> dict:
    checks = {
        "has_architecture": "架构" in resume_section or "设计" in resume_section,
        "has_metrics": any(word in resume_section for word in ["%", "ms", "qps", "延迟"]),
        "has_tech_stack": "Python" in resume_section or "TypeScript" in resume_section,
        "has_difficulty": "难点" in resume_section or "挑战" in resume_section,
        "has_personal_contribution": "负责" in resume_section or "主导" in resume_section,
        "avoids_buzzwords": "精通" not in resume_section and "熟练" not in resume_section,
    }
    return checks
```

### STAR 法则在智能体项目中的体现

```
S (Situation): 业务方需要自动处理用户工单，人工处理成本高
T (Task): 构建智能体系统自动分析、分类并回复工单
A (Action):
  - 设计 Agent Loop 框架，支持 ReAct 和 Plan-and-Execute 双模式
  - 构建工具注册系统，集成查询、修改、审批等 20+ 工具
  - 设计三级权限体系，高风险操作需要人工确认
  - 搭建评测流水线，覆盖准确率、召回率、延迟、成本、安全性
R (Result): 自动处理率 85%，人工介入减少 60%，用户满意度提升 17%
```

## 常见误区

- **功能堆砌**：把"支持 XX 功能"罗列十几条，没有重点。面试官记不住
- **缺少量化**：全文没有一个数字，全是"提高效率""降低成本"
- **夸大其词**：写上"智能""AI""自主"等词但没有技术细节支撑
- **忽视失败**：只写成功的路径，从不提遇到了什么困难、怎么解决的
- **角色模糊**：多人项目不写自己的具体贡献，面试官无法判断你做了什么
- **技术栈过时**：还在写 TensorFlow，智能体领域的核心技术栈是 Agent Loop、Tool Registry、MCP

## 追问方向

- **"你说工具调用成功率 97.3%，剩下 2.7% 失败是什么原因？"**
  要准备好讲述失败案例分类、根因和优化过程。

- **"你们每天 5000 请求，并发量多大？系统怎么扛的？"**
  考察系统设计和工程落地能力。

- **"评测流水线的 200 个用例是怎么设计的？谁写的？"**
  考察评测体系的完善程度和团队协作分工。

- **"你自研 Agent Framework 替代 LangChain 的理由是什么？"**
  准备清楚技术选型的工程权衡。

- **"如果让你重新做这个项目，有哪些地方你会做得不一样？"**
  考察复盘能力和技术成长。

## 学习建议

准备智能体项目简历时，先列出你做过的所有技术动作，然后按"架构设计 > 技术难点 > 量化成果 > 个人角色"的顺序重新组织。找一位有经验的工程师帮你 review 简历，让他指出哪些地方像"调 API"。建议读一下 Anthropic 和 OpenAI 发布的 Agent 最佳实践文档，了解行业对 Agent 工程师的能力定义。不要写"熟练使用 ChatGPT"，这在简历上是一个减分项。

最后记住一个原则：简历上的每句话，都要能引出面试官的一个追问，而你对这个追问有充分的回答准备。
