# 智能体的核心组件有哪些？

## 面试问题

智能体的核心组件有哪些？

## 考察重点

- 是否能从系统架构角度理解智能体，而不是简单罗列概念
- 是否能说清楚每个组件的作用和相互关系
- 是否了解 Claude Code 和 Codex 在组件设计上的差异
- 是否有实际实现某个组件的经验

## 核心结论

智能体由四大核心组件构成：**Agent Loop**（控制循环）、**Tool Registry**（工具注册）、**Context Manager**（上下文管理）、**Planning Module**（规划模块）。这四个组件的交互方式决定了智能体的行为模式。

---

## 四大核心组件

### 组件一：Agent Loop（控制循环）

**作用**：控制整个智能体的执行流程，决定什么时候停止。

**实现方式**：

```typescript
// Claude Code 风格：while(true) + 状态机
while (true) {
  state = processStep(state);
  if (shouldStop(state)) break;
}

// Codex 风格：事件驱动
for await (const event of stream) {
  if (event.type === 'function_call') {
    await executeTool(event);
  } else if (event.type === 'output_text.done') {
    return event.text;
  }
}
```

**关键设计点**：
- 停止条件（turnCount、tokenBudget、用户中断）
- 状态持久化（支持中断恢复）
- Circuit Breaker（防止异常情况死循环）

### 组件二：Tool Registry（工具注册）

**作用**：管理智能体可用的所有工具。

**核心功能**：

```typescript
interface Tool {
  name: string;
  description: string;
  parameters: JSONSchema;
  handler: (args: any) => Promise<ToolResult>;
  permission?: PermissionLevel;
}

class ToolRegistry {
  private tools: Map<string, Tool>;

  register(tool: Tool): void;
  unregister(name: string): void;
  getTool(name: string): Tool | undefined;
  listTools(): Tool[];  // 用于传递给模型
}
```

**关键设计点**：
- 工具描述如何写（影响模型选择）
- 权限分级（安全控制）
- 参数校验（防止注入）

### 组件三：Context Manager（上下文管理）

**作用**：管理对话历史、任务状态、工具返回结果。

**核心功能**：

```typescript
interface ContextManager {
  // 添加用户输入
  addUserMessage(msg: string): void;

  // 添加模型输出
  addModelMessage(msg: Message): void;

  // 添加工具返回
  addToolResult(result: ToolResult): void;

  // 上下文压缩
  compress(strategy: CompressionStrategy): void;

  // 获取当前上下文
  getContext(): Message[];

  // token 预算管理
  getTokenBudget(): { used: number; remaining: number };
}
```

**Claude Code 的 4 阶段压缩**：
1. 摘要重写（Summarize）
2. 选择性保留（Selective Keep）
3. 目标重置（Goal Reset）
4. 强制截断（Force Truncate）

### 组件四：Planning Module（规划模块）

**作用**：决定下一步做什么。

**两种策略**：

| 策略 | 实现 | 适用场景 |
|-----|------|---------|
| **ReAct** | 边做边想，每步都输出思考 | 简单任务、交互式 |
| **Plan-and-Execute** | 先生成完整计划，再执行 | 复杂任务、需要全局规划 |

```typescript
// ReAct 实现
async function reactLoop(task: Task) {
  while (!isComplete(task)) {
    const thought = await model.think(getContext());
    const action = await model.decideAction(thought, getTools());
    const result = await executeTool(action);
    context.addObservation(thought, action, result);
  }
}

// Plan-and-Execute 实现
async function planAndExecute(task: Task) {
  const plan = await model.createPlan(task);
  for (const step of plan.steps) {
    const result = await executeStep(step);
    if (needsReplan(result)) {
      plan = await model.updatePlan(plan, result);
    }
  }
}
```

## Claude Code 的组件实现

### Agent Loop
- 文件：`query.ts`
- 实现：1421 行 while(true) 循环，10 步迭代
- 特点：不是递归，用状态机避免栈溢出

### Tool Registry
- 内置工具：shell, edit, glob, grep, read, write, submit
- MCP 工具：动态发现和注册
- 实现：codex-rs/core/src/tools/spec.rs

### Context Manager
- 实现：4 阶段压缩
- 关键状态：`tokenBudget`, `autoCompactTracking`
- Circuit Breaker：`hasAttemptedReactiveCompact`

### Planning Module
- 实现：`update_plan` 工具
- 策略：Plan-and-Execute
- 特点：计划是模型的输出，不是系统输入

## Codex 的组件实现

### Agent Loop
- 实现：事件驱动 SSE 流
- 循环控制：function_call 循环直到 output_text.done
- API：Responses API

### Tool Registry
- 三类工具：内置（shell）、MCP、Responses API
- 命名：`mcp__<server>__<method>`
- 实现：codex-rs/core/src/tools/

### Context Manager
- 实现：HTTP 无状态，每次调用携带完整上下文
- 预算控制：通过 max_tokens 参数

### Planning Module
- 实现：`update_plan` 工具
- 特点：模型可以随时更新计划

## 工程实践：组件设计模式

### 1. 依赖注入

```typescript
interface AgentDependencies {
  llm: LLM;
  toolRegistry: ToolRegistry;
  contextManager: ContextManager;
  planningModule: PlanningModule;
}

class Agent {
  constructor(deps: AgentDependencies) {
    this.llm = deps.llm;
    this.toolRegistry = deps.toolRegistry;
    // ...
  }
}
```

### 2. 中间件/拦截器

```typescript
// 工具调用拦截器
const toolInterceptor = {
  async beforeCall(tool: string, args: any) {
    // 权限检查
    // 参数校验
    // 日志记录
  },
  async afterCall(tool: string, args: any, result: any) {
    // 结果处理
    // 异常记录
  }
};

// 上下文压缩拦截器
const contextInterceptor = {
  beforeCompress(context: Message[]) {
    // 决定保留什么
  }

```

### 3. 可观测性

```typescript
// 每个组件都要有日志和指标
interface AgentMetrics {
```

## 面试追问方向

1. **"你的智能体项目是如何组织这四个组件的？有没有把某些组件合并或拆分？"**

   考察候选人的架构设计能力。

2. **"如果 Context Manager 和 Tool Registry 冲突（比如工具描述和上下文中的信息不一致），你怎么处理？"**

   考察候选人对组件协作的理解。

3. **"为什么 Claude Code 选择 while(true) 而不是递归实现 Agent Loop？"**

   考察候选人对系统设计细节的理解。

4. **"如果要给你的智能体加一个新组件（比如 Memory Module），你会怎么设计？"**

   考察候选人的扩展性思维。
