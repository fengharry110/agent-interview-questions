# 智能体如何理解用户真实意图？

## 面试问题

智能体如何理解用户真实意图？

## 考察重点

- 是否理解表面意图和真实意图的区别
- 是否了解多轮对话、上下文理解的重要性
- 是否能说清楚意图澄清的策略
- 是否有处理隐含意图、用户真实目标的经验

## 核心结论

用户表达的意图（Surface Intent）往往不是他的真实意图（True Intent）。智能体需要通过**多轮澄清**、**上下文推理**、**隐含意图挖掘**来理解用户的真实目标。核心挑战是用户不会直接说出全部信息，需要智能体主动询问和推断。

---

## 表面意图 vs 真实意图

### 常见差异

```
用户说："帮我查一下订单"
表面意图：query_order
真实意图：可能是：
  - 想取消订单（但还没说出来）
  - 想了解物流进度（但只说了订单）
  - 想退货（但还没说出来）

用户说："这个多少钱"
表面意图：query_price
真实意图：可能是：
  - 只是看看价格
  - 想买但觉得贵
  - 帮别人问的
```

### 隐含意图的类型

| 类型 | 示例 | 挖掘方法 |
|-----|------|---------|
| **关联意图** | "查完订单后想取消" | 多轮对话 |
| **情感意图** | 用户抱怨时不只是要解决方案 | 情感分析 |
| **潜在意图** | 用户不知道自己想要什么 | 引导式询问 |
| **隐式意图** | 用户不会直接说"我是来投诉的" | 意图分类 |

## 理解真实意图的技术手段

### 1. 多轮对话上下文管理

```python
class ConversationContext:
    def __init__(self):
        self.history: list[Turn] = []
        self.current_goal: str = None
        self.slots: dict = {}
        self.inferred_intents: list[str] = []

    def add_turn(self, user_input: str, system_response: str):
        self.history.append(Turn(user_input, system_response))

    def get_recent_context(self, n: int = 5) -> str:
        """获取最近n轮对话"""
        return "\n".join([
            f"用户: {t.user_input}\n系统: {t.system_response}"
            for t in self.history[-n:]
        ])

    def infer_true_intent(self) -> str:
        """从历史中推断真实意图"""
        # 如果最近有投诉相关的词
        if any(word in self.history[-1].user_input
               for word in ["不满", "太差", "投诉"]):
            return "complaint"

        # 如果用户连续查询了同一实体的不同属性
        if self.has_multiple_queries_about_same_entity():
            return "exploratory"

        return self.current_goal
```

### 2. 意图澄清策略

```python
class IntentClarifier:
    def __init__(self):
        self.clarification_templates = {
            "query_order": [
                "您是想查询订单的什么信息？物流进度还是订单详情？",
                "请问是哪个平台的订单？"
            ],
            "refund": [
                "请问您是想全额退款还是部分退款？",
                "退款原因是什么呢？"
            ]
        }

    def should_clarify(self, intent: str, slots: dict) -> bool:
        """判断是否需要澄清"""
        # 参数不完整时需要澄清
        required_slots = self.get_required_slots(intent)
        missing_slots = set(required_slots) - set(slots.keys())
        return len(missing_slots) > 0

    def get_clarification_question(self, intent: str, missing_slots: set) -> str:
        """生成澄清问题"""
        questions = []
        for slot in missing_slots:
            questions.append(self.slot_descriptions.get(slot, slot))

        return f"请问{''.join(questions)}？"
```

### 3. 隐含意图识别

```python
def infer_true_intent(user_input: str, context: ConversationContext) -> dict:
    prompt = f"""分析用户输入，推断真实意图。

    用户输入：{user_input}

    对话历史：
    {context.get_recent_context()}

    1. 用户说的表面意图是什么？
```

## 真实意图理解的挑战

### 挑战一：用户不知道自己要什么

```
用户："帮我看看"  → 真的不知道该问什么
```

**策略**：主动推荐 + 场景引导

```python
def handle_vague_request(user_input: str) -> str:
    if is_vague(user_input):
        return """您可以选择：
1. 查询订单状态
2. 咨询产品问题
3. 了解优惠政策
4. 联系人工客服

请问您想了解哪方面的内容？"""
```

### 挑战二：用户表达不准确

```
用户："我要那个"  → "那个"指代不明
用户："上次那个订单"  → 需要从历史推断
```

**策略**：实体链接 + 上下文指代消解

```python
def resolve_reference(text: str, context: ConversationContext) -> str:
    # 替换指代词
    resolved = text

    # "那个"需要从历史中找
    if "那个" in text:
        entity = context.last_mentioned_entity("order")
        if entity:
            resolved = text.replace("那个", f"订单{entity.id}")

    # "上次"需要确定时间
    if "上次" in text:
        last_order = context.get_last_order()
        if last_order:
            resolved = text.replace("上次", f"订单号{last_order.id}")

    return resolved
```

### 挑战三：用户隐瞒关键信息

```
用户："这个能用吗？"（指着某个功能）
→ 用户想知道但不确定能不能用
→ 真实意图：想确认功能是否对自己可用
```

**策略**：主动确认 + 功能解释

```python
def handle_implied_question(user_input: str, context: dict) -> str:
    # 分析用户可能在问什么
    inferred_question = infer_implied_question(user_input, context)

    # 直接回答隐含问题
    return f"""关于您的问题"{user_input}"：

    {answer_implied_question(inferred_question)}

    请问还有其他问题吗？"""
```

## 真实意图理解在 Claude Code 中的体现

Claude Code 通过多轮交互来理解用户的真实编程意图：

```typescript
async function understandTrueIntent(
    userInput: string,
    context: ConversationContext
): Promise<IntentUnderstanding> {
    // 1. 先理解字面意思
    const surfaceIntent = classifyIntent(userInput);

    // 2. 结合上下文推断真实意图
    const historicalGoals = context.getRecentGoals();
    const currentFile = context.getCurrentFile();

    // 3. 如果用户修改了之前的内容，可能是想迭代
    if (userInput.includes("change") && historicalGoals.length > 0) {
        return {
            trueIntent: "iterate_on_previous",
            target: historicalGoals[0],
            confidence: 0.85
        };
    }

    // 4. 如果用户在问"能不能"，可能是想确认某个功能
    if (userInput.includes("can I") || userInput.includes("could")) {
        return {
            trueIntent: "capability_check",
            capability: extractCapability(userInput),
            confidence: 0.75
        };
    }

    return {
        trueIntent: surfaceIntent,
        confidence: 0.9
    };
}
```

## 意图澄清的最佳实践

### 1. 澄清的时机

```
需要澄清的情况：
- 必填参数缺失
- 意图模糊（多个可能）
- 用户表达了多个意图
- 用户情绪异常

不需要澄清的情况：
- 高频简单意图（可以直接执行）
- 用户明确拒绝澄清
- 紧急情况（快速通道）
```

### 2. 澄清的方式

```python
# 好例子：具体、有选择
"您是想查询哪个订单？最近的有XXX订单，或者您可以提供订单号。"

# 坏例子：太开放、不知道问什么
"您想问什么？"
```

### 3. 澄清的次数限制

```python
MAX_CLARIFICATION_ROUNDS = 3

def should_stop_clarifying(context: ConversationContext) -> bool:
    if context.clarification_rounds >= MAX_CLARIFICATION_ROUNDS:
        return True

    if context.user_seems_frustrated():
        return True

    return False
```

## 面试追问方向

1. **"用户说'我要投诉'但不说是投诉什么，你怎么处理？"**

   考察候选人对投诉处理流程的理解。

2. **"你怎么判断用户是真的要完成任务，还是只是随便问问？"**

   考察候选人对用户意图深度的理解。

3. **"如果用户在多轮对话中逐渐暴露真实意图，你怎么设计状态管理？"**

   考察候选人对对话状态设计的理解。

4. **"真实意图理解和意图识别有什么区别？"**

   考察候选人对两个概念混淆的识别。
