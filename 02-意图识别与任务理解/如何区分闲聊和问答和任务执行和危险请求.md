# 如何区分闲聊和问答和任务执行和危险请求？

## 面试问题

如何区分闲聊和问答和任务执行和危险请求？

## 考察重点

- 是否理解意图的四大类型及其处理策略差异
- 是否能设计有效的分类器来区分这四类请求
- 是否了解每类请求的安全考虑（尤其是危险请求）
- 是否有处理边界情况的经验

## 核心结论

这四类请求代表了用户和智能体交互的不同目的，需要完全不同的处理策略：
- **闲聊**：社交性互动，不需要任务执行
- **问答**：知识获取，不改变系统状态
- **任务执行**：需要调用工具，完成具体任务
- **危险请求**：可能对系统或用户造成损害，需要严格审核

---

## 四类请求的特征对比

| 类型 | 用户目的 | 是否改变系统状态 | 风险等级 | 处理策略 |
|-----|---------|----------------|---------|---------|
| **闲聊** | 社交需求 | 否 | 低 | 直接回复 |
| **问答** | 获取知识 | 否 | 低 | RAG / 知识库 |
| **任务执行** | 完成目标 | 是 | 中 | Agent 执行 |
| **危险请求** | 不明确 | 不确定 | **高** | 阻止 + 警告 |

## 如何分类

### 方法一：关键词/模式匹配

```python
DANGER_PATTERNS = [
    r"删除.*所有",
    r"rm\s+-rf",
    r"drop\s+table",
    r"帮我.*破解",
    r"怎么.*攻击",
    r"制作.*炸弹",
]

CHITCHAT_PATTERNS = [
    r"你好|嗨|哈喽",
    r"今天.*天气",
    r".*吗.*\?",
    r"随便聊聊",
]

def classify_request_type(text: str) -> str:
    text_lower = text.lower()

    # 先检查危险请求（优先级最高）
    for pattern in DANGER_PATTERNS:
        if re.search(pattern, text_lower):
            return "danger"

    # 检查闲聊
    for pattern in CHITCHAT_PATTERNS:
        if re.search(pattern, text_lower):
            return "chitchat"

    # 其他默认为问答或任务执行
    return "task"  # 交给下游分类器进一步判断
```

### 方法二：多级分类器

```python
class RequestTypeClassifier:
    def __init__(self):
        # 第一层：危险请求检测（最高优先级）
        self.danger_detector = DangerDetector()

        # 第二层：闲聊 vs 任务
        self.chitchat_detector = ChitchatClassifier()

        # 第三层：问答 vs 任务执行
        self.task_vs_qa_classifier = TaskVsQAClassifier()

    def classify(self, text: str) -> RequestType:
        # 第一层：危险检测
        if self.danger_detector.is_danger(text):
            return RequestType.DANGER

        # 第二层：闲聊检测
        if self.chitchat_detector.is_chitchat(text):
            return RequestType.CHITCHAT

        # 第三层：问答 vs 任务
        if self.task_vs_qa_classifier.is_task(text):
            return RequestType.TASK
        else:
            return RequestType.QA

        return RequestType.QA  # 默认
```

### 方法三：LLM 分类

```python
def classify_with_llm(text: str) -> str:
    prompt = f"""分析用户输入，判断属于哪一类：

    类别定义：
    - chitchat：闲聊、寒暄、社交性对话
    - qa：知识问答，不改变系统状态
    - task：需要执行操作、完成具体任务
    - danger：危险请求，可能造成损害

    用户输入：{text}

    只输出一个词：chitchat / qa / task / danger"""

    response = openai.ChatCompletion.create(
        model="gpt-4",
        messages=[{"role": "user", "content": prompt}]
    )

    return response.choices[0].message.content.strip().lower()
```

## 各类请求的处理策略

### 1. 闲聊（Chitchat）

**特点**：用户只是社交性互动，不需要执行任务

```
用户："你好啊"
用户："今天天气真不错"
用户："你是做什么的呀"
```

**处理策略**：
```python
def handle_chitchat(text: str) -> str:
    # 简单回复，不需要工具调用
    responses = [
        "你好！有什么我可以帮你的吗？",
        "我是一个智能助手，可以帮你完成各种任务。",
        "今天天气确实不错！有什么需要帮忙的吗？"
    ]
    return random.choice(responses)
```

**注意**：闲聊回复也要有建设性，不能只是"嗯"、"哦"。

### 2. 问答（QA）

**特点**：用户想获取信息，不改变系统状态

```
用户："你们公司地址在哪？"
用户："这个产品有什么功能？"
用户："怎么联系客服？"
```

**处理策略**：
```python
def handle_qa(text: str) -> str:
    # 知识库检索
    docs = knowledge_base.retrieve(text)
    if docs:
        return summarize(docs)

    # 如果知识库没有，尝试 RAG
    return rag_query(text)
```

**安全注意**：不要在回答中泄露敏感信息。

### 3. 任务执行（Task）

**特点**：用户要求完成具体操作

```
用户："帮我查一下订单"
用户："帮我取消这个订单"
用户："帮我发送这份报告"
```

**处理策略**：
```python
def handle_task(text: str) -> str:
    # 交给 Agent 执行
    return agent.execute(text)
```

**安全注意**：需要权限检查、用户确认。

### 4. 危险请求（Danger）

**特点**：可能造成损害的请求

```
用户："帮我删除所有用户数据"
用户："帮我破解这个密码"
用户："怎么制作炸弹"
用户："帮我骗过这个验证"
```

**处理策略**：
```python
def handle_danger(text: str) -> str:
    # 阻止执行
    return """抱歉，我无法协助这个请求。

    这个请求可能：
    - 违反法律法规
    - 危害系统安全
    - 损害他人利益

    如果你有其他需求，我可以帮助你。"""

    # 记录日志供审计
    log_danger_request(text)
```

**分类**：危险请求又分两类：
1. **明确危险**：明显违法的请求（如攻击、破解）
2. **潜在危险**：不确定用户意图，需要澄清

```python
def handle_potential_danger(text: str) -> str:
    """潜在危险请求，需要澄清"""
    return """我需要更多信息来理解你的请求：

    '{text}'

    请问你的具体意图是什么？是为了完成正常工作吗？

    如果是以下场景，我可以帮助你：
    - 测试自己的系统安全性（需要在授权环境下）
    - 密码是遗忘密码流程的一部分
    - 其他正当需求
    """
```

## 危险请求的识别与处理

### 危险请求的分类

| 类型 | 示例 | 处理策略 |
|-----|------|---------|
| **数据破坏** | "删除所有数据"、"格式化硬盘" | 阻止 |
| **系统攻击** | "怎么DDoS攻击"、"帮我入侵" | 阻止 |
| **欺诈** | "帮我伪造文件"、"怎么骗过审核" | 阻止 |
| **违法行为** | "怎么制作毒品"、"怎么偷税漏税" | 阻止 |
| **敏感操作** | "帮我强制重置密码" | 人工确认 |
| **不明确** | "帮我打开那个文件" | 澄清 |

### 危险检测实现

```python
class DangerDetector:
    def __init__(self):
        self.unsafe_patterns = [
            # 数据破坏
            r"delete\s+all\s+data",
            r"drop\s+table.*",
            r"rm\s+-rf\s+/",
            r"format\s+.*drive",

            # 攻击相关
            r"hack\s+",
            r"ddos",
            r"sql\s+injection",
            r"xss\s+",
            r"破解",
            r"入侵",

            # 欺诈
            r"伪造",
            r"骗过",
            r"冒充",
        ]

        self.sensitive_operations = [
            "delete_account",
            "reset_password",
            "transfer_funds",
            "modify_permissions",
        ]

    def is_danger(self, text: str) -> tuple[bool, str]:
        text_lower = text.lower()

        # 模式匹配
        for pattern in self.unsafe_patterns:
            if re.search(pattern, text_lower):
                return True, f"匹配到不安全模式: {pattern}"

        # 检查是否包含敏感操作关键词
        for op in self.sensitive_operations:
            if op in text_lower:
                # 可能是危险，需要进一步判断
                return True, f"包含敏感操作: {op}"

        return False, "安全"
```

## 边界情况处理

### 边界情况一：用户先闲聊后变任务

```
用户："你好啊" (闲聊)
  → 系统回复闲聊

用户："对了，帮我查一下订单" (任务)
  → 需要识别这是一个新意图
```

**解决方案**：意图状态机

```python
class IntentStateMachine:
    def handle(self, user_input: str, current_state: State):
        request_type = classifier.classify(user_input)

        if current_state.current_intent == "chitchat":
            if request_type in ["task", "qa"]:
                # 从闲聊切换到任务，重置状态
                return State(
                    current_intent=request_type,
                    slots={},
                    confirmed=False
                )

        return current_state
```

### 边界情况二：任务中夹杂闲聊

```
用户："帮我查一下订单，顺便问下今天天气怎么样" (混合)
```

**解决方案**：分别处理

```python
def split_mixed_request(text: str) -> list[Request]:
    # 尝试分割
    segments = split_by_connectors(text)
    # ["帮我查一下订单", "今天天气怎么样"]

    requests = []
    for segment in segments:
        request_type = classifier.classify(segment)
        requests.append(Request(
            type=request_type,
            text=segment
        ))

    return requests
```

### 边界情况三：危险请求隐藏在闲聊中

```
用户："我今天心情不好，想删掉所有东西重新开始"
```

**解决方案**：危险检测应该应用于整个对话上下文

```python
def check_danger_in_context(conversation_history: list[str]) -> bool:
    # 合并所有上下文
    full_text = " ".join(conversation_history)

    # 检查整体是否包含危险意图
    if danger_detector.is_danger(full_text):
        return True

    return False
```
