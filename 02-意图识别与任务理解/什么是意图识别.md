# 什么是意图识别？

## 面试问题

什么是意图识别？

## 考察重点

- 是否理解意图识别是"把用户说的话转成机器能理解的结构化表示"
- 是否能说清楚意图识别在智能体架构中的位置（入口处理）
- 是否了解意图识别的技术方案：规则、ML分类、BERT/LLM
- 是否有处理模糊意图、多意图、意图漂移的经验

## 核心结论

意图识别是智能体的"入口过滤器"，负责把用户的自然语言转成结构化的意图标签，决定后续走哪条处理路径。好意图识别的标准是：**高准确率**（不误判）、**高召回率**（不漏判）、**快速响应**（低延迟）。

## 意图的层次结构

### 第一层：领域意图（Domain Intent）

用户属于哪个业务领域？

```
用户："帮我查一下账户余额"
  领域：金融/账户
  意图：查询余额

用户："帮我订一张机票"
  领域：旅游/出行
  意图：订机票
```

### 第二层：动作意图（Action Intent）

用户想做什么操作？

```
意图类型：
  - query（查询）：查余额、查航班、查天气
  - action（执行）：下单、取消、退款
  - control（控制）：暂停、继续、取消
  - chat（聊天）：闲聊、问候、告别
```

### 第三层：参数意图（Slot Intent）

用户提供了哪些关键信息？

```
意图：订机票
参数：
  - 出发地：北京
  - 目的地：上海
  - 日期：明天
  - 缺失：航班偏好（直飞/经停）
```

## 意图识别的技术方案

### 方案一：规则匹配

适用于：固定模式的意图（如命令式语句）

```python
# 规则匹配示例
rules = [
    (r".*查.*余额.*", "query_balance"),
    (r".*订.*机票.*", "book_flight"),
    (r".*取消.*订单.*", "cancel_order"),
]

def classify_intent(text: str) -> str:
    for pattern, intent in rules:
        if re.match(pattern, text):
            return intent
    return "unknown"
```

**优点**：快速、可解释、成本低
**缺点**：覆盖率低，无法处理变体表达

### 方案二：机器学习分类

适用于：有足够训练数据的场景

```python
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.linear_model import LogisticRegression

# 特征提取
vectorizer = TfidfVectorizer(max_features=5000)
X_train = vectorizer.fit_transform(training_texts)

# 训练分类器
classifier = LogisticRegression()
classifier.fit(X_train, training_labels)

# 预测
def classify_intent(text: str) -> str:
    X = vectorizer.transform([text])
    return classifier.predict(X)[0]
```

**优点**：准确率较高，可处理变体
**缺点**：需要训练数据，无法处理新意图

### 方案三：BERT/Transformer 模型

适用于：对准确率要求高的场景

```python
from transformers import pipeline

classifier = pipeline(
    "zero-shot-classification",
    model="facebook/bart-large-mnli"
)

def classify_intent(text: str) -> str:
    labels = [
        "查询余额", "订机票", "取消订单",
        "投诉", "退款", "其他"
    ]
    result = classifier(text, labels)
    return result["labels"][0]
```

**优点**：准确率高，支持零样本
**缺点**：延迟高、成本高

### 方案四：LLM（大语言模型）

适用于：需要理解复杂意图、处理模糊表达

```python
def classify_intent_with_llm(user_input: str) -> dict:
    prompt = f"""分析用户输入，输出JSON格式的意图：
    {{
        "intent": "意图名称",
        "confidence": 0.0-1.0,
        "slots": {{"参数名": "参数值"}},
        "requires_clarification": true/false
    }}

    用户输入：{user_input}"""

    response = llm.generate(prompt)
    return json.parse(response)
```

**优点**：理解能力强，可处理复杂意图
**缺点**：延迟高、成本高、需要prompt工程

## 意图识别的工程实践

### 1. 多级分类器架构

```
第一级：快速过滤（规则 + 小模型）
  - 如果命中已知模式，直接返回
  - 毫秒级响应

第二级：精确分类（BERT/LLM）
  - 用于规则无法处理的case
  - 100-500ms 响应

第三级：兜底处理（转人工/默认回复）
  - 处理识别失败的情况
```

### 2. 意图置信度处理

```python
def classify_intent(text: str) -> IntentResult:
    # 规则匹配
    rule_result = match_rules(text)
    if rule_result.confidence > 0.9:
        return rule_result

    # ML 分类
    ml_result = ml_classify(text)
    if ml_result.confidence > 0.8:
        return ml_result

    # 低置信度：请求澄清
    if ml_result.confidence < 0.5:
        return IntentResult(
            intent="needs_clarification",
            confidence=ml_result.confidence,
            prompt="您是想查询余额还是其他操作？"
        )

    return ml_result
```

### 3. 意图序列标注

当用户表达中包含多个意图时：

```
用户："帮我查一下余额，然后取消上次的订单"
    ↓
意图序列：
  [
    { intent: "query_balance", action: "查余额" },
    { intent: "cancel_order", action: "取消订单" }
  ]
```


## 意图识别的评估指标

| 指标 | 计算方式 | 理想值 |
|-----|---------|-------|
| **准确率** | 预测正确的意图 / 总预测数 | > 95% |
| **召回率** | 预测正确的意图 / 实际存在的意图 | > 95% |
| **F1 分数** | 2 * 准确率 * 召回率 / (准确率 + 召回率) | > 95% |
| **延迟** | P99 响应时间 | < 100ms |
| **覆盖率** | 能识别多少种意图 / 总意图数 | > 90% |

## 常见问题与解决方案

### 问题一：用户表达模糊

```
用户："帮我处理一下"
  → 意图不明确
  → 需要澄清
```

**解决方案**：多轮澄清

```python
if intent.requires_clarification:
    return Response(
        message="请问您是想：\n1. 查询账户余额\n2. 取消订单\n3. 其他操作？",
        next_action="clarification"
    )
```

### 问题二：多意图

```
用户："帮我取消订单并退款"
  → 包含两个意图：cancel_order + refund
```

**解决方案**：意图序列，返回多个意图

### 问题三：意图漂移

```
用户："帮我查余额"
  → 意图：query_balance
  用户："不对，是另一张卡"
  → 意图漂移：补充参数，而不是新意图
```

**解决方案**：意图状态机

```python
class IntentState:
    current_intent: str
    slots: dict
    confirmed: bool

def handle_modification(state: IntentState, new_input: str) -> IntentState:
    if is_slot_update(new_input, state.current_intent):
        # 只更新参数
        state.slots.update(extract_slots(new_input))
        return state
    else:
        # 新意图
        return IntentState(current_intent=new_input, ...)
```
