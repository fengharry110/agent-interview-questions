# 多意图请求如何拆解？

## 面试问题

多意图请求如何拆解？

## 考察重点

- 是否理解多意图的类型（顺序执行、并行执行、依赖执行）
- 是否能设计有效的意图拆分策略
- 是否了解意图之间的依赖关系处理
- 是否有处理复杂多意图的经验

## 核心结论

多意图请求拆解的核心是**识别 + 排序 + 执行**。首先识别出所有意图，然后判断依赖关系（哪些可以并行、哪些必须顺序），最后按正确顺序执行。

---

## 多意图的类型

### 类型一：顺序意图

一个意图的执行结果影响下一个意图。

```
用户："帮我查一下余额，如果超过1000就转账到卡A"

意图序列：
1. query_balance → 返回余额
2. 判断：余额 > 1000？
3. 如果是：transfer_to_card_A

执行必须顺序进行
```

### 类型二：并行意图

多个意图之间没有依赖，可以同时执行。

```
用户："帮我查一下账户余额和最近的消费记录"

意图序列：
1. query_balance
2. query_recent_transactions

执行可以并行（都只是查询）
```

### 类型三：混合意图

既有并行也有顺序的部分。

```
用户："帮我查余额，然后转账到卡A，再发个通知"

意图序列：
1. query_balance (查询)
2. transfer_to_card_A (依赖1的结果)
3. send_notification (可以和2并行)
```

## 意图依赖分析

```python
from enum import Enum

class DependencyType(Enum):
    NONE = "none"           # 无依赖，可并行
    DEPENDS_ON_PREV = "depends_on_prev"  # 依赖前一个
    DEPENDS_ON_SPECIFIC = "depends_on_specific"  # 依赖特定意图

class IntentNode:
    def __init__(self, intent: str, action: callable):
        self.intent = intent
        self.action = action
        self.dependencies: list[str] = []
        self.results: dict = {}

    def add_dependency(self, other_intent: str):
        self.dependencies.append(other_intent)

def build_intent_graph(intents: list[Intent]) -> list[IntentNode]:
    nodes = []
    for i, intent in enumerate(intents):
        node = IntentNode(intent.name, intent.action)

        # 检查是否依赖前一个
        if intent.depends_on_previous:
            node.add_dependency(intents[i-1].name)

        # 检查是否有显式依赖
        if intent.depends_on:
            for dep in intent.depends_on:
                node.add_dependency(dep)

        nodes.append(node)

    return nodes
```

## 执行计划生成

```python
from collections import defaultdict

def generate_execution_plan(nodes: list[IntentNode]) -> ExecutionPlan:
    # 构建依赖图
    graph = defaultdict(list)
    in_degree = defaultdict(int)

    for node in nodes:
        for dep in node.dependencies:
            graph[dep].append(node.name)
            in_degree[node.name] += 1

    # 拓扑排序
    plan = []
    parallel_groups = []

    while nodes:
        # 找出所有入度为0的节点（无依赖）
        ready = [n for n in nodes if in_degree[n.name] == 0]

        if not ready:
            # 循环依赖，错误
            raise ValueError("Circular dependency detected")

        # 这一轮可以并行执行
        parallel_groups.append(ready)

        # 移除这一轮的节点
        for node in ready:
            nodes.remove(node)
            # 更新依赖图
            for dependent in graph[node.name]:
                in_degree[dependent] -= 1

    return ExecutionPlan(parallel_groups=parallel_groups)
```

## 执行引擎

```python
async def execute_plan(plan: ExecutionPlan, context: dict) -> list[Result]:
    results = []

    for group in plan.parallel_groups:
        # 这一组的意图可以并行执行
        tasks = []
        for node in group:
            # 注入依赖的结果
            node_context = inject_dependencies(context, node, results)
            tasks.append(node.action(node_context))

        # 等待所有任务完成
        group_results = await asyncio.gather(*tasks)
        results.extend(group_results)

    return results

def inject_dependencies(context: dict, node: IntentNode, prior_results: list[Result]) -> dict:
    """把依赖的结果注入到上下文中"""
    enriched_context = context.copy()

    for dep in node.dependencies:
        # 找到依赖的结果
        dep_result = find_result(dep, prior_results)
        if dep_result:
            enriched_context[f"{dep}_result"] = dep_result

    return enriched_context
```

## 多意图处理的边界情况

### 情况一：意图冲突

```
用户："帮我取消订单，但别退款"
```

**处理**：识别冲突，询问确认

```python
def detect_conflicts(intents: list[Intent]) -> list[Conflict]:
    conflicts = []

    for i, intent1 in enumerate(intents):
        for j, intent2 in enumerate(intents[i+1:], i+1):
            if are_conflicting(intent1, intent2):
                conflicts.append(Conflict(intent1, intent2))

    return conflicts

def handle_conflicts(conflicts: list[Conflict]) -> str:
    return f"""您的请求中存在冲突：
    - {conflicts[0].intent1.name} 和 {conflicts[0].intent2.name}

    请问您想怎么处理？
    1. 只执行 {conflicts[0].intent1.name}
    2. 只执行 {conflicts[0].intent2.name}
    3. 其他"""
```

### 情况二：部分执行失败

```
用户："帮我查余额、转账、发通知"
执行到转账时失败了
```

**处理**：回滚 + 错误报告

```python
async def execute_with_rollback(plan: ExecutionPlan, context: dict) -> ExecutionResult:
    executed = []
    results = []

    try:
        for group in plan.parallel_groups:
            tasks = []
            for node in group:
                node_context = inject_dependencies(context, node, results)
                tasks.append(node.action(node_context))

            group_results = await asyncio.gather(*tasks)

            # 检查是否有失败
            for node, result in zip(group, group_results):
                if result.is_failure:
                    # 执行回滚
                    await rollback(executed)
                    return ExecutionResult(
                        status="failed",
                        failed_at=node.intent,
                        error=result.error,
                        partial_results=results
                    )

                executed.append(node)
                results.append(result)

    except Exception as e:
        await rollback(executed)
        return ExecutionResult(status="error", error=str(e))

    return ExecutionResult(status="success", results=results)
```

### 情况三：意图变更

```
用户："帮我查余额然后转账
    （还没执行完）
用户："算了，不转了，帮我看看有什么优惠"
```

**处理**：取消当前计划，开始新计划

```python
def handle_intent_change(context: ExecutionContext, new_intent: str):
    # 如果有正在执行的任务
    if context.current_plan:
        # 询问用户是否确定取消
        return """您还有任务在进行中：
        {正在执行的任务}

        确定要取消并开始新任务吗？"""

    # 如果可以取消，启动新计划
    context.cancel_current_plan()
    context.start_new_plan(new_intent)
```

## 多意图执行示例

### 完整代码

```python
class MultiIntentExecutor:
    def __init__(self):
        self.intent_classifier = IntentClassifier()
        self.executor = ExecutionEngine()

    async def handle(self, user_input: str) -> str:
        # 1. 识别多意图
        intents = self.intent_classifier.identify(user_input)

        if len(intents) == 1:
            # 单意图，直接执行
            return await self.executor.execute_single(intents[0])

        # 2. 分析依赖
        plan = build_execution_plan(intents)

        # 3. 执行计划
        results = await self.executor.execute_plan(plan)

        # 4. 汇总结果
        return self.format_results(results)

    def format_results(self, results: list[Result]) -> str:
        if not results:
            return "任务已完成"

        response_parts = []
        for i, result in enumerate(results):
            response_parts.append(f"步骤{i+1}：{result.summary}")

        return "\n".join(response_parts)
```
