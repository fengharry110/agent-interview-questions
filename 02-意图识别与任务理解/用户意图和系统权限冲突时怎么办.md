# 用户意图和系统权限冲突时怎么办？

## 面试问题

用户意图和系统权限冲突时怎么办？

## 考察重点

- 是否理解权限控制的重要性
- 是否能设计合理的权限冲突处理策略
- 是否了解何时该拒绝、何时该升级
- 是否有处理权限相关边界情况的经验

## 核心结论

用户意图和权限冲突时，核心处理原则是**安全优先 + 透明沟通 + 给出替代方案**。不要直接拒绝后就没有下文，要告诉用户为什么被拒绝，并提供可行的替代方案。

---

## 权限冲突的类型

### 类型一：操作权限不足

用户想执行某个操作，但没有权限。

```
用户："帮我删掉所有用户数据"
系统：当前用户是普通客服，只能查看数据，不能删除
```

### 类型二：数据访问权限不足

用户想访问某些数据，但没有权限。

```
用户："帮我看看其他同事的工资"
系统：工资数据只有 HR 和本人可以查看
```

### 类型三：业务规则限制

用户想执行的操作虽然有权限，但违反业务规则。

```
用户："帮我取消这个3年前的订单并退款"
系统：订单超过30天不能退款
```

### 类型四：安全风险

用户想执行的操作可能存在安全风险。

```
用户："帮我把账户密码改成 123456"
系统：密码不符合安全策略
```

## 权限检查流程

```python
class PermissionChecker:
    def __init__(self):
        self.user_roles = load_user_roles()
        self.operation_permissions = load_operation_permissions()
        self.business_rules = load_business_rules()

    def check_permission(
        self,
        user: User,
        intent: Intent,
        context: dict
    ) -> PermissionResult:
        # 1. 检查操作权限
        if not self.has_operation_permission(user, intent):
            return PermissionResult(
                allowed=False,
                reason="operation_not_allowed",
                message="您没有执行此操作的权限"
            )

        # 2. 检查数据访问权限
        if not self.has_data_permission(user, intent, context):
            return PermissionResult(
                allowed=False,
                reason="data_not_accessible",
                message="您没有访问相关数据的权限"
            )

        # 3. 检查业务规则
        rule_violation = self.check_business_rules(intent, context)
        if rule_violation:
            return PermissionResult(
                allowed=False,
                reason="business_rule_violation",
                message=rule_violation.message,
                alternatives=rule_violation.alternatives
            )

        return PermissionResult(allowed=True)

    def has_operation_permission(self, user: User, intent: Intent) -> bool:
        user_role = self.user_roles.get(user.id)
        allowed_operations = self.operation_permissions.get(user_role, [])
        return intent.operation in allowed_operations
```

## 冲突处理策略

### 策略一：透明拒绝 + 原因说明

```python
def handle_permission_conflict(
    user: User,
    intent: Intent,
    conflict: PermissionResult
) -> str:
    # 构建清晰的拒绝消息
    message = f"抱歉，无法执行您请求的操作。\n\n"
    message += f"原因：{conflict.message}\n\n"

    # 提供更多信息
    if conflict.reason == "operation_not_allowed":
        message += "说明：此操作需要更高权限。\n"
        message += "如果您确实需要执行此操作，请联系管理员申请权限。"

    elif conflict.reason == "business_rule_violation":
        message += f"业务规则：{conflict.business_rule}\n"
        if conflict.alternatives:
            message += "\n您可以尝试：\n"
            for alt in conflict.alternatives:
                message += f"- {alt}\n"

    return message
```

### 策略二：提供替代方案

```python
def generate_alternatives(
    intent: Intent,
    conflict: PermissionResult
) -> list[Alternative]:
    alternatives = []

    # 如果没有操作权限，提供只读版本
    if conflict.reason == "operation_not_allowed":
        if intent.type == "delete":
            alternatives.append(Alternative(
                action="query",
                description="查看数据（只读）",
                command="帮我看看这些数据"
            ))
        elif intent.type == "modify":
            alternatives.append(Alternative(
                action="request_change",
                description="提交修改申请",
                command="帮我提交一个修改申请"
            ))

    # 如果违反业务规则，提供合规的替代
    if conflict.reason == "business_rule_violation":
        if "order_too_old" in conflict.business_rule:
            alternatives.append(Alternative(
                action="manual_review",
                description="申请人工审核",
                command="帮我申请人工审核处理"
            ))

    return alternatives
```

### 策略三：权限升级

```python
class PermissionEscalation:
    def __init__(self):
        self.pending_requests = []

    def request_escalation(
        self,
        user: User,
        intent: Intent,
        reason: str
    ) -> str:
        # 创建权限请求
        request = PermissionRequest(
            user_id=user.id,
            intent=intent,
            reason=reason,
            status="pending",
            created_at=datetime.now()
        )

        self.pending_requests.append(request)

        # 通知相关人员
        self.notify_managers(request)

        return f"""您的权限申请已提交。

        申请ID：{request.id}
        申请操作：{intent.description}
        当前状态：等待审批

        我们会通过站内信通知您审批结果。"""
```

## 权限模型设计

### RBAC 模型

```python
class Role:
    ADMIN = "admin"
    MANAGER = "manager"
    OPERATOR = "operator"
    GUEST = "guest"

class Permission:
    READ = "read"
    WRITE = "write"
    DELETE = "delete"
    APPROVE = "approve"

ROLE_PERMISSIONS = {
    Role.ADMIN: [Permission.READ, Permission.WRITE, Permission.DELETE, Permission.APPROVE],
    Role.MANAGER: [Permission.READ, Permission.WRITE, Permission.APPROVE],
    Role.OPERATOR: [Permission.READ, Permission.WRITE],
    Role.GUEST: [Permission.READ]
}

def check_permission(user: User, required_permission: Permission) -> bool:
    user_permissions = ROLE_PERMISSIONS.get(user.role, [])
    return required_permission in user_permissions
```

### ABAC 模型（基于属性的访问控制）

```python
class ABACPolicy:
    def __init__(self):
        self.policies = []

    def add_policy(self, policy: Policy):
        self.policies.append(policy)

    def evaluate(
        self,
        user: User,
        action: str,
        resource: Resource,
        context: dict
    ) -> bool:
        for policy in self.policies:
            if policy.matches(user, action, resource, context):
                return policy.effect == "allow"

        return False  # 默认拒绝

# 示例策略
policy = Policy(
    name="own_data_only",
    condition=lambda user, action, resource, ctx:
        resource.owner_id == user.id or user.role == Role.ADMIN,
    effect="allow"
)
```

## 边界情况处理

### 情况一：权限随时效变化

```
用户："帮我审批这个申请"
审批操作有时效：只有在工作时间才能审批
```

**处理**：检查当前时间是否允许

```python
def check_temporal_permission(intent: Intent, context: dict) -> PermissionResult:
    current_hour = datetime.now().hour

    if intent.operation == "approve":
        if current_hour < 9 or current_hour >= 18:
            return PermissionResult(
                allowed=False,
                reason="time_restriction",
                message="审批操作仅在工作时间内可用（9:00-18:00）",
                alternatives=[
                    "您可以提前提交申请，系统会在工作时间自动处理",
                    "如有紧急情况，请联系值班管理员"
                ]
            )

    return PermissionResult(allowed=True)
```

### 情况二：批量操作权限

```
用户："帮我删除这1000条数据"
批量操作有额外限制
```

**处理**：分级权限 + 确认

```python
def check_batch_permission(
    user: User,
    intent: Intent,
    batch_size: int
) -> PermissionResult:
    # 小批量：普通权限
    if batch_size <= 10:
        return check_basic_permission(user, intent)

    # 中批量：需要确认
    if batch_size <= 100:
        if not intent.confirmed:
            return PermissionResult(
                allowed=False,
                reason="confirmation_required",
                message=f"您即将操作 {batch_size} 条数据，请确认是否继续。",
                requires_confirmation=True
            )

    # 大批量：需要更高权限
    if batch_size > 100:
        if user.role not in [Role.ADMIN, Role.MANAGER]:
            return PermissionResult(
                allowed=False,
                reason="insufficient_permission",
                message=f"批量操作超过100条需要管理员权限。",
                alternatives=[
                    "分批操作，每批不超过100条",
                    "联系管理员协助处理"
                ]
            )

    return PermissionResult(allowed=True)
```

### 情况三：委托权限

```
用户A想让用户B帮忙执行某个操作
需要委托机制
```

**处理**：临时授权

```python
class DelegationManager:
    def create_delegation(
        self,
        from_user: User,
        to_user: User,
        operation: str,
        valid_until: datetime
    ) -> Delegation:
        delegation = Delegation(
            from_user=from_user.id,
            to_user=to_user.id,
            operation=operation,
            valid_until=valid_until,
            created_at=datetime.now()
        )

        self.save_delegation(delegation)
        return delegation

    def check_delegation(
        self,
        user: User,
        operation: str
    ) -> bool:
        delegations = self.get_valid_delegations(user.id)

        for delegation in delegations:
            if delegation.operation == operation:
                if datetime.now() < delegation.valid_until:
                    return True

        return False
```

1. **"用户说'这是我老板让我操作的'，你怎么处理？"**

   考察候选人对权限和责任边界的理解。

2. **"权限检查的性能怎么保证？"**

   考察候选人对权限系统性能优化的理解。

3. **"如果权限系统本身有漏洞，你怎么发现和防止？"**

   考察候选人对安全的深入理解。

4. **"你怎么向用户解释权限冲突而不显得冷漠？"**

   考察候选人对用户体验的关注。
