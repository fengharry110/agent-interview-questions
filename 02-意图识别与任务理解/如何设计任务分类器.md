# 如何设计任务分类器？

## 面试问题

如何设计任务分类器？

## 考察重点

- 是否理解任务分类器的层次结构：领域分类 → 意图分类 → 槽位填充
- 是否能说清楚分类器的训练数据构建方法
- 是否了解分类器的评估指标和调优策略
- 是否有处理类别不平衡、漏判误判的经验

## 核心结论

任务分类器设计的关键是**分层的Taxonomy** + **足够的训练数据** + **持续优化机制**。不要试图用单一模型解决所有问题，应该按业务领域分层，每层用合适的模型。

---

## 任务分类器的分层设计

### 第一层：领域分类（Domain Classification）

划分用户属于哪个业务领域。

```
用户输入 → 领域分类 → 金融 / 电商 / 物流 / 客服 / 其他
```

| 领域 | 示例意图 |
|-----|---------|
| 金融 | 查询余额、转账、信用卡还款 |
| 电商 | 下单、退款、物流查询 |
| 物流 | 查快递、修改地址、签收确认 |
| 客服 | 投诉、建议、咨询 |

### 第二层：意图分类（Intent Classification）

在领域内进一步分类具体意图。

```
电商领域 → 意图分类 → 下单 / 退款 / 换货 / 投诉 / 评价 / 查询
```

### 第三层：槽位填充（Slot Filling）

提取意图相关的关键参数。

```
退款意图 → 槽位填充 → 订单号、原因、退款金额
```

## 分类器架构

```python
class HierarchicalClassifier:
    def __init__(self):
        self.domain_classifier = DomainClassifier()      # 领域分类器
        self.intent_classifiers = {                      # 意图分类器（按领域）
            "finance": FinanceIntentClassifier(),
            "ecommerce": EcommerceIntentClassifier(),
            # ...
        }
        self.slot_fillers = {                            # 槽位填充器（按意图）
            "refund": RefundSlotFiller(),
            "order_query": OrderQuerySlotFiller(),
            # ...
        }

    def classify(self, text: str, context: list[str] = None) -> ClassificationResult:
        # 第一步：领域分类
        domain = self.domain_classifier.predict(text)

        # 第二步：领域内意图分类
        intent_classifier = self.intent_classifiers[domain]
        intent = intent_classifier.predict(text)

        # 第三步：槽位填充
        slot_filler = self.slot_fillers[intent]
        slots = slot_filler.extract(text)

        return ClassificationResult(
            domain=domain,
            intent=intent,
            slots=slots,
            confidence=min(
                domain.confidence,
                intent.confidence
            )
        )
```

### 方法二：数据增强

```python
from data_augmentation import SynonymReplacement, RandomInsertion, BackTranslation

def augment_training_data(original_data: list[dict], multiplier: int = 5) -> list[dict]:
    augmented = []
    for item in original_data:
        augmented.append(item)

        # 同义词替换
        for _ in range(multiplier):
            augmented.append({
                **item,
                "text": SynonymReplacement.replace(item["text"])
            })

        # 回译增强
        augmented.append({
            **item,
            "text": BackTranslation.translate(item["text"])
        })

    return augmented
```

## 类别不平衡处理

### 问题

```
训练数据分布：
- query_balance: 5000 条（50%）
- cancel_order: 2000 条（20%）
- refund: 1500 条（15%）
- complaint: 500 条（5%）
- other: 1000 条（10%）
```

模型会倾向于预测高频类别。

### 解决方案

#### 1. 过采样少数类

```python
from imblearn.over_sampling import SMOTE

X_resampled, y_resampled = SMOTE().fit_resample(
    X_train, y_train
)
```

#### 2. 欠采样多数类

```python
from imblearn.under_sampling import RandomUnderSampler

X_resampled, y_resampled = RandomUnderSampler().fit_resample(
    X_train, y_train
)
```

#### 3. 类别权重

```python
# 训练时给少数类更高的权重
class_weights = {
    "query_balance": 1.0,
    "cancel_order": 1.0,
    "refund": 1.5,
    "complaint": 3.0,  # 高权重
    "other": 2.0
}

model.fit(X_train, y_train, class_weight=class_weights)
```

#### 4. Focal Loss

```python
class FocalLoss(nn.Module):
    def __init__(self, alpha=1, gamma=2):
        super().__init__()
        self.alpha = alpha
        self.gamma = gamma

    def forward(self, inputs, targets):
        ce_loss = nn.functional.cross_entropy(inputs, targets)
        pt = torch.exp(-ce_loss)
        focal_loss = self.alpha * (1 - pt) ** self.gamma * ce_loss
        return focal_loss
```

## 评估指标

### 基础指标

```python
from sklearn.metrics import classification_report, confusion_matrix

def evaluate_classifier(classifier, X_test, y_test):
    y_pred = classifier.predict(X_test)

    # 分类报告
    print(classification_report(y_test, y_pred))

    # 混淆矩阵
    print(confusion_matrix(y_test, y_pred))

    # 每个类别的指标
    metrics = {}
    for intent in INTENT_LABELS:
        tp = sum((y_test == intent) & (y_pred == intent))
        fp = sum((y_test != intent) & (y_pred == intent))
        fn = sum((y_test == intent) & (y_pred != intent))

        metrics[intent] = {
            "precision": tp / (tp + fp) if (tp + fp) > 0 else 0,
            "recall": tp / (tp + fn) if (tp + fn) > 0 else 0,
            "support": sum(y_test == intent)
        }

    return metrics
```

## 持续优化机制

### 1. 线上回流

```python
def collect_feedback(result: ClassificationResult, user_feedback: str):
    """收集用户纠正反馈"""
    if user_feedback == "correct":
        log_correct(result)
    else:
        log_incorrect(
            text=result.original_text,
            predicted_intent=result.intent,
            corrected_intent=user_feedback
        )

# 定期用纠正数据重新训练
def retrain_with_feedback():
    corrections = load_corrections_since_last_training()
    new_training_data = augment_with_corrections(corrections)
    model.retrain(new_training_data)
```

### 2. A/B 测试

def ab_test_classifier():
