# 如何评估意图识别是否准确？

## 面试问题

如何评估意图识别是否准确？

## 考察重点

- 是否理解意图识别的评估指标体系
- 是否能设计有效的测试集和评估流程
- 是否了解在线评估和离线评估的区别
- 是否有持续监控意图识别效果的经验

## 核心结论

意图识别评估需要**多维度指标** + **分层测试集** + **持续监控**。不能只看准确率，要根据业务场景权衡 precision/recall，以及不同意图类别的表现差异。

## 评估指标体系

### 基础指标

```python
from sklearn.metrics import precision_recall_fscore_support, confusion_matrix

def evaluate_intent_classification(
    y_true: list[str],
    y_pred: list[str],
    intent_labels: list[str]
) -> dict:
    # 整体指标
    overall_precision, overall_recall, overall_f1, _ = \
        precision_recall_fscore_support(y_true, y_pred, average='weighted')

    # 每个类别的指标
    per_class_metrics = {}
    for label in intent_labels:
        y_true_binary = [1 if y == label else 0 for y in y_true]
        y_pred_binary = [1 if y == label else 0 for y in y_pred]

        tp = sum([1 for t, p in zip(y_true_binary, y_pred_binary) if t == 1 and p == 1])
        fp = sum([1 for t, p in zip(y_true_binary, y_pred_binary) if t == 0 and p == 1])
        fn = sum([1 for t, p in zip(y_true_binary, y_pred_binary) if t == 1 and p == 0])

        precision = tp / (tp + fp) if (tp + fp) > 0 else 0
        recall = tp / (tp + fn) if (tp + fn) > 0 else 0
        f1 = 2 * precision * recall / (precision + recall) if (precision + recall) > 0 else 0

        per_class_metrics[label] = {
            "precision": precision,
            "recall": recall,
            "f1": f1,
            "support": sum(y_true_binary)
        }

    # 混淆矩阵
    cm = confusion_matrix(y_true, y_pred, labels=intent_labels)

    return {
        "overall": {
            "precision": overall_precision,
            "recall": overall_recall,
            "f1": overall_f1
        },
        "per_class": per_class_metrics,
        "confusion_matrix": cm
    }
```

## 测试集设计

### 分层采样

```python
def create_evaluation_dataset(
    conversations: list[Conversation],
    intent_distribution: dict[str, float]
) -> EvaluationDataset:
    """
    确保测试集覆盖各种场景
    """
    dataset = {
        "intents": {},  # 按意图分层采样
        "domains": {},  # 按领域采样
        "difficulty": {},  # 按难度分级
        "edge_cases": []  # 边界 case
    }

    # 1. 按意图分布采样
    for intent, ratio in intent_distribution.items():
        intent_samples = [c for c in conversations if c.intent == intent]
        n_samples = int(len(conversations) * ratio * 0.8)  # 80% 用于测试
        dataset["intents"][intent] = random.sample(intent_samples, n_samples)

    # 2. 边界 case 收集
    edge_cases = [
        # 模糊表达
        "帮我看看",
        "处理一下",
        "那个东西",

        # 多意图
        "帮我查一下余额然后转账",

        # 异常表达
        "查一下订单啦啦啦啦啦",

        # 上下文依赖
        "就是上次那个订单",
        "和刚才一样的"
    ]
    dataset["edge_cases"] = edge_cases

    return dataset
```

### 测试集质量保证

```python
def validate_test_dataset(dataset: EvaluationDataset) -> ValidationResult:
    """检查测试集是否符合要求"""
    issues = []

    # 检查每个意图是否有足够样本
    for intent, samples in dataset["intents"].items():
        if len(samples) < 30:
            issues.append(f"意图 {intent} 样本不足：{len(samples)}")

    # 检查是否覆盖边界 case
    if len(dataset["edge_cases"]) < 10:
        issues.append("边界 case 覆盖不足")

    # 检查类别平衡
    all_samples = [s for samples in dataset["intents"].values() for s in samples]
    distribution = Counter([s.intent for s in all_samples])

    max_ratio = max(distribution.values()) / len(all_samples)
    if max_ratio > 0.5:
        issues.append(f"类别严重不平衡，最大占比：{max_ratio:.2%}")

    return ValidationResult(
        valid=len(issues) == 0,
        issues=issues
    )
```

## 在线评估

### A/B 测试

```python
class OnlineEvaluator:
    def __init__(self, control_model, treatment_model):
        self.control = control_model
        self.treatment = treatment_model

    def assign(self, user_id: str) -> str:
        """用户随机分配到实验组或对照组"""
        bucket = hash(user_id) % 100
        if bucket < 50:
            return "control"
        else:
            return "treatment"

    def evaluate(self, results: list[PredictionResult]) -> dict:
        """计算在线指标"""
        control_results = [r for r in results if r.group == "control"]
        treatment_results = [r for r in results if r.group == "treatment"]

        # 计算各项指标
        control_metrics = self.compute_metrics(control_results)
        treatment_metrics = self.compute_metrics(treatment_results)

        # 统计显著性检验
        significance = self.statistical_test(
            control_results,
            treatment_results
        )

        return {
            "control": control_metrics,
            "treatment": treatment_metrics,
            "lift": compute_lift(control_metrics, treatment_metrics),
            "significance": significance
        }
```

### 持续监控

```python
class IntentMonitor:
    def __init__(self, metrics_client):
        self.metrics = metrics_client

    def record_prediction(
        self,
        intent: str,
        confidence: float,
        user_feedback: str,
        latency_ms: float
    ):
        # 记录预测
        self.metrics.inc("intent.predicted", tags={"intent": intent})

        # 记录置信度分布
        self.metrics.histogram("intent.confidence", confidence, tags={"intent": intent})

        # 记录用户反馈
        if user_feedback == "correct":
            self.metrics.inc("intent.correct", tags={"intent": intent})
        elif user_feedback == "wrong":
            self.metrics.inc("intent.wrong", tags={"intent": intent})

        # 记录延迟
        self.metrics.histogram("intent.latency", latency_ms)

    def get_dashboard(self) -> dict:
        """生成监控仪表板"""
        return {
            "accuracy_by_intent": self.compute_accuracy_by_intent(),
            "avg_confidence_by_intent": self.compute_avg_confidence(),
            "low_confidence_rate": self.compute_low_confidence_rate(),
            "latency_p99": self.compute_latency_p99(),
            "user_correction_rate": self.compute_correction_rate()
        }
```

## 评估流程

### 离线评估流程

```
1. 准备测试集
   - 分层采样，确保覆盖各种场景
   - 标注 ground truth
   - 验证测试集质量

2. 运行评估
   - 用待测模型预测测试集
   - 对比预测结果和 ground truth
   - 计算各项指标

3. 分析结果
   - 找出表现差的意图
   - 分析错误案例
   - 识别改进方向

4. 生成报告
   - 整体指标
   - 分类别指标
   - 典型错误案例
```

### 在线评估流程

```
1. 部署实验
   - A/B 测试 或 Canary Release
   - 记录所有预测和反馈

2. 数据收集
   - 收集足够的样本量（通常需要 7-14 天）
   - 清洗异常数据

3. 效果评估
   - 计算业务指标
   - 统计显著性检验
   - 输出结论

4. 决策
   - 提升显著 → 全量发布
   - 提升不显著 → 继续实验
   - 下降 → 回滚
```

## 常见问题与解决

### 问题一：测试集和线上分布不一致

**症状**：离线指标很好，线上效果差

**解决**：
```python
def check_distribution_shift(
    training_data: list,
    test_data: list,
    online_data: list
) -> bool:
    """检测分布漂移"""
    train_dist = compute_distribution(training_data)
    test_dist = compute_distribution(test_data)
    online_dist = compute_distribution(online_data)

    # 计算分布差异
    train_test_diff = kl_divergence(train_dist, test_dist)
    train_online_diff = kl_divergence(train_dist, online_dist)

    if train_online_diff > train_test_diff * 2:
        logger.warning("检测到分布漂移！")
        return True

    return False
```

### 问题二：用户反馈噪声大

**症状**：用户纠正反馈不可靠

**解决**：多轮确认才计入

```python
def get_reliable_feedback(
    initial_feedback: str,
    confirmation_feedback: str = None
) -> str | None:
    if initial_feedback in ["correct", "wrong"]:
        if confirmation_feedback is None:
            return None  # 需要确认

        if confirmation_feedback == initial_feedback:
            return initial_feedback

    return None  # 反馈不一致，忽略
```
