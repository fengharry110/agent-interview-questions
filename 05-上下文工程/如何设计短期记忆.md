# 如何设计短期记忆？

## 面试问题

如何设计短期记忆？

## 考察重点

- 是否理解工作记忆的容量限制和衰减机制
- 能否说清楚信息选择、衰减和持久化的策略
- 是否能结合具体源码（Claude Code/Codex）说明实现细节
- 是否理解短期记忆与长期记忆的交互边界

## 核心结论

短期记忆是智能体在执行过程中临时保持信息的机制，其核心挑战是在有限容量下做出高效的取舍决策。Claude Code通过动态窗口管理实现工作记忆，Codex通过事件驱动的消息队列维护会话状态。

## Claude Code实现分析

Claude Code的短期记忆机制基于一个有限容量的消息缓冲区：

```python
class MessageBuffer:
    def __init__(self, max_tokens: int = 200000):
        self.max_tokens = max_tokens
        self.messages = []
        self.token_count = 0
    
    def add(self, message: dict):
        self.messages.append(message)
        self.token_count += self._count_tokens(message)
        while self.token_count > self.max_tokens:
            removed = self.messages.pop(0)
            self.token_count -= self._count_tokens(removed)
```

关键设计点：
1. **容量硬限制**：到达上限后FIFO淘汰旧消息
2. **语义保留策略**：优先保留最近对话和关键决策点
3. **分层衰减**：区分用户意图、工具调用、执行结果的重要性层级

```python
class PriorityMessageBuffer(MessageBuffer):
    PRIORITY_LEVELS = {
        "user_intent": 100,
        "tool_definitions": 80,
        "recent_history": 60,
        "execution_results": 40,
        "intermediate_thinking": 20
    }
    
    def evict_low_priority(self):
        while self.token_count > self.max_tokens * 0.9:
            low_priority = self._find_lowest_priority_message()
            if low_priority:
                self.messages.remove(low_priority)
                self.token_count -= self._count_tokens(low_priority)
```

## Codex实现分析

Codex采用事件驱动的短期记忆架构，将交互分解为离散事件：

```typescript
interface ConversationEvent {
    type: 'user_message' | 'assistant_message' | 'tool_call' | 'tool_result';
    timestamp: number;
    content: string;
    importance: number;
    metadata?: Record<string, any>;
}

class EventStore {
    private events: ConversationEvent[] = [];
    private readonly maxEvents = 1000;
    
    add(event: ConversationEvent): void {
        this.events.push(event);
        if (this.events.length > this.maxEvents) {
            this.events = this.compress(this.events);
        }
    }
    
    getRelevant(task: string, limit: number): ConversationEvent[] {
        const embeddings = this.computeEmbeddings(task);
        return this.rankByRelevance(embeddings).slice(0, limit);
    }
}
```

Codex的短期记忆有两个关键特性：
1. **事件溯源**：所有交互作为不可变事件存储，支持回放和重放
2. **相关性检索**：基于语义相似度而非时间顺序选择上下文

## 短期记忆的核心机制

### 1. 容量管理

人类工作记忆容量约为4-7个chunk，LLM的"工作记忆"受token限制：

| 模型 | 最大上下文 | 有效工作记忆比例 |
|------|-----------|----------------|
| GPT-4 32K | 32K tokens | ~60% (留空间给输出) |
| Claude 100K | 100K tokens | ~70% |
| Gemini 1M | 1M tokens | ~80% |

### 2. 信息选择策略

```python
class WorkingMemorySelector:
    def select(self, task: str, available_context: List[str]) -> List[str]:
        scores = {}
        for item in available_context:
            relevance = self.embedding_similarity(task, item)
            recency = self.recency_weight(item)
            importance = self.inherent_importance(item)
            scores[item] = relevance * 0.5 + recency * 0.3 + importance * 0.2
        return sorted(scores.items(), key=lambda x: x[1], reverse=True)
```

### 3. 衰减与强化

```python
class DecayableMemory:
    def __init__(self):
        self.access_count = defaultdict(int)
        self.last_access = defaultdict(float)
    
    def record_access(self, key: str):
        self.access_count[key] += 1
        self.last_access[key] = time.time()
    
    def get_decay_score(self, key: str) -> float:
        age = time.time() - self.last_access[key]
        frequency = self.access_count[key]
        return math.exp(-age / 3600) * math.log(frequency + 1)
```

## 工程实践

### Claude Code的会话管理

```python
class ClaudeCodeSession:
    def __init__(self, model: str = "claude-3-opus"):
        self.model = model
        self.max_tokens = 200000
        self.tools = load_tool_definitions()
        
    def build_context(self, task: str) -> List[dict]:
        context = []
        # 1. 系统提示（工具定义）
        context.extend(self.load_system_prompt())
        # 2. 用户意图摘要
        context.append(self.summarize_user_intent(task))
        # 3. 相关历史（语义检索）
        context.extend(self.retrieve_relevant_history(task))
        # 4. 当前任务
        context.append({"role": "user", "content": task})
        return context
    
    def load_system_prompt(self) -> List[dict]:
        return [
            {"role": "system", "content": SYSTEM_PROMPT},
            {"role": "system", "content": self._format_tools(self.tools)}
        ]
```

### Codex的状态管理

```typescript
class CodexSession {
    private state: Map<string, any> = new Map();
    private eventLog: Event[] = [];
    
    updateState(toolName: string, result: ToolResult): void {
        this.state.set(`last_${toolName}`, result);
        this.eventLog.push({
            type: 'state_update',
            tool: toolName,
            result: this.summarizeResult(result),
            timestamp: Date.now()
        });
    }
    
    getContextForTask(task: string): Message[] {
        const relevant = this.eventLog
            .filter(e => this.isRelevant(e, task))
            .slice(-50); // 保留最近50个相关事件
        return this.buildMessages(relevant);
    }
}
```

## 失败模式与处理

| 失败模式 | 表现 | 解决方案 |
|---------|------|---------|
| 容量溢出 | 丢失关键历史信息 | 分层记忆 + 语义压缩 |
| 重要性误判 | 保留噪音，丢弃关键 | 动态校准 + 用户反馈 |
| 衰减过度 | 短期重复任务表现差 | 多级缓存 + 任务识别 |
| 状态不一致 | 多轮对话逻辑混乱 | 事件溯源 + 状态快照 |
