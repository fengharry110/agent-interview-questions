# Agent 项目如何写进简历？

## 面试问题

如果你做过 Agent 项目，你会如何把它写进简历？

## 考察重点

- 是否能把项目写成工程成果，而不是 API 调用
- 是否能突出 Agent 核心能力
- 是否知道简历要体现结果、难点和指标

## 核心结论

Agent 项目写简历时，要写清楚问题、系统能力、工程设计、评测结果和业务价值，避免只写“调用大模型实现智能助手”。

## 参考回答

我会用“背景-方案-能力-结果”的方式写。比如：实现一个代码分析 Agent，支持仓库文件检索、错误定位、工具调用、补丁生成和测试验证；设计了工具权限控制、执行 trace 和任务评测集；在 30 个测试任务上达到某个成功率。

简历中要突出具体技术点，例如 Tool Calling、RAG、任务规划、权限控制、评测、日志、成本优化、部署。最好能有 GitHub 链接、架构图、demo 截图或评测数据。

## 深度展开

面试官最怕看到“套壳项目”。如果你只写“接入 ChatGPT 实现问答”，区分度很低。你要证明自己做的是系统：有哪些工具、如何保证安全、如何调试失败、如何评估效果、如何处理成本和延迟。

## 工程实践

- README 写清楚架构和使用方式
- 提供任务样例和评测结果
- 展示 trace 或日志截图
- 写明你解决过的失败案例

## 常见误区

- 只写模型名字，不写系统设计
- 没有指标，只有主观描述
- 项目无法运行或 README 不完整

## 追问方向

- 没上线的项目如何体现价值？
- 如何避免被认为是调包？
- 面试时如何讲项目难点？

## 学习建议

准备一套项目故事：为什么做、怎么设计、遇到什么坑、怎么评测、结果如何。这比背概念更能打动面试官。
