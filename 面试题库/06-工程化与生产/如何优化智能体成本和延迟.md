# 如何优化 Agent 的成本和延迟？

## 面试问题

Agent 系统通常成本和延迟较高，你会如何优化？

## 考察重点

- 是否理解多步调用会放大 token 成本和等待时间
- 是否能从模型、上下文、工具、缓存和并行多个角度优化
- 是否知道优化不能牺牲核心成功率

## 核心结论

优化 Agent 成本和延迟要减少不必要的模型调用、控制上下文、选择合适模型、缓存稳定结果，并并行执行独立工具。

## 参考回答

Agent 成本高通常来自多轮模型调用、长上下文、重复检索和重复工具调用。首先可以做任务分级：简单任务用规则或小模型，复杂任务再调用强模型。其次要减少无意义步骤，让 Agent 在真正需要时才调用工具。

上下文也要控制。不要把所有历史都塞给模型，而是保留与当前决策相关的信息。稳定的检索结果、工具结果和中间分析可以缓存。对于互不依赖的工具调用，可以并行执行。

但优化不能只看便宜和快，还要看任务成功率。如果用小模型导致失败率显著上升，总成本可能反而更高。

## 深度展开

Agent 优化经常是系统工程问题，而不是单纯换模型。一次任务里可能有 8 次模型调用，每次都带大量历史上下文。只要减少两次不必要调用，收益就很明显。相反，盲目压缩上下文可能让 Agent 丢失关键状态。

## 工程实践

- 使用 router 判断任务复杂度
- 对上下文做摘要和状态化管理
- 对检索和工具结果加缓存
- 记录每个任务的 cost breakdown

## 常见误区

- 只通过换便宜模型降成本
- 盲目增加 top-k 导致上下文膨胀
- 不记录成本，优化没有依据

## 追问方向

- 什么时候可以用小模型？

- 如何判断 Agent 步骤过多？
