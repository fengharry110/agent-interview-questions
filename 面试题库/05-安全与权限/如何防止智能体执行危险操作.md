# 如何防止 Agent 执行危险操作？

## 面试问题

如何防止 Agent 执行危险操作？

## 考察重点

- 是否理解危险操作要由系统拦截
- 是否能设计权限分级和审批流程
- 是否关注可审计和可回滚

## 核心结论

防止危险操作的关键是最小权限、工具分级、执行前校验、人工确认和审计日志。

## 参考回答

我会先对工具和操作分级。低风险操作如读取公开文档可以默认允许；中风险操作如写文件、发送草稿、修改非生产数据需要确认；高风险操作如删除数据、转账、修改生产配置、发送外部邮件应默认禁止或强审批。

其次，工具执行前要做参数校验和权限检查。比如文件路径必须在工作区内，数据库操作必须限制表和语句类型，网络请求必须限制域名。

最后，所有高风险动作要记录审计日志，最好支持撤销或回滚。

## 深度展开

危险操作不能交给模型自行判断，因为模型可能被诱导，也可能误判业务影响。安全边界必须在工具执行层和业务系统层实现。Agent 可以提出操作意图，但系统决定是否允许执行。

## 工程实践

- 建立工具风险等级
- 对 delete、send、pay、deploy、write 等动作强确认
- 使用沙箱环境执行不确定命令
- 对外部影响操作生成预览

## 常见误区

- 只在 prompt 里写“不要做危险事”
- 缺少操作日志，出问题后无法追责
- 把所有工具都开放给所有用户

## 追问方向

- 如何设计工具权限模型？
- 人工确认信息应该展示什么？
- Agent 操作失败后如何回滚？

## 学习建议

安全题的核心态度是：模型可以建议，系统负责约束。
