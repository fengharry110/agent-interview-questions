# 如何保护密钥和敏感数据？

## 面试问题

Agent 系统中如何保护 API Key、密钥和敏感数据？

## 考察重点

- 是否知道模型上下文不是安全存储
- 是否理解密钥应由工具后端持有
- 是否关注日志、trace 和外部请求泄露

## 核心结论

密钥不应该进入模型上下文。Agent 可以请求能力，但不应该直接看到或持有秘密。

## 参考回答

API Key、数据库密码、用户 token 不应该放进 prompt，也不应该出现在模型可见上下文。正确方式是后端工具持有密钥，模型只生成受限操作请求，例如“查询订单状态”，由工具后端完成鉴权和调用。

敏感数据要最小化暴露。只有完成任务所需的信息才传给模型，日志和 trace 中要脱敏。工具报错也要过滤，避免把环境变量、请求头或密钥打印出来。

如果 Agent 能访问文件或数据库，还要做权限隔离，防止它读取与任务无关的数据。

## 深度展开

Agent 系统有多个泄露面：模型上下文、工具输入输出、日志、向量库、缓存、第三方模型 API。保护敏感数据需要端到端设计，而不是只管 prompt。

## 工程实践

- 密钥放在服务端环境变量或密钥管理服务
- 工具返回前做敏感字段脱敏
- trace 存储设置访问权限和保留周期
- 对外部请求做域名和数据类型限制

## 常见误区

- 为了方便把 API Key 写进 prompt
- 日志里保存完整请求和响应
- 把用户隐私写入长期记忆且不可删除

## 追问方向

- 工具异常里包含密钥怎么办？
- Agent 如何防止敏感信息外发？
- 长期记忆里的隐私如何删除？

## 学习建议

记住一句话：模型看到的信息越少，系统越安全。
