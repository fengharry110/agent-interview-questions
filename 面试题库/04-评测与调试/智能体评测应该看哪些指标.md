# Agent 评测应该看哪些指标？

## 面试问题

Agent 系统评测应该关注哪些指标？

## 考察重点

- 是否能从质量、过程、成本、安全多个维度回答
- 是否知道不同 Agent 场景指标不同
- 是否能避免只看准确率

## 核心结论

Agent 指标应覆盖任务结果、执行过程、工程效率和安全合规四类。

## 参考回答

结果指标包括任务成功率、答案正确率、格式合规率和用户满意度。过程指标包括工具调用准确率、平均步骤数、重试次数、规划失败率和上下文丢失率。工程指标包括平均延迟、token 成本、工具执行成本、并发稳定性。安全指标包括越权操作次数、敏感信息泄露次数、危险操作拦截率和人工确认覆盖率。

不同场景要选择核心指标。代码 Agent 关注测试通过率和 patch 质量；客服 Agent 关注解决率和人工转接率；数据分析 Agent 关注计算正确性和结论可追溯性。

## 深度展开

指标之间会冲突。比如提高 top-k 可能提升召回，但增加成本和噪声；增加人工确认能提高安全性，但降低效率；用更强模型提升成功率，但成本和延迟上升。因此面试中要体现取舍意识。

## 工程实践

- 每个任务记录 success、steps、tokens、latency、cost
- 工具调用记录 expected_tool 和 actual_tool
- 安全相关操作单独统计
- 用 dashboard 观察长期趋势

## 常见误区

- 只说准确率，不说成本和安全
- 不区分离线评测和线上指标
- 没有按业务场景定义成功标准

## 追问方向

- 如何衡量工具调用是否正确？
- 成本和成功率冲突时怎么办？
- 哪些指标适合自动化评测？

## 学习建议

回答指标题时先分层，再结合一个具体场景，会比直接罗列名词更强。
