# RAG 和长期记忆有什么区别？

## 面试问题

RAG 和 Agent 的长期记忆有什么区别？

## 考察重点

- 是否能区分外部知识和用户状态
- 是否理解记忆涉及写入、更新、遗忘和权限
- 是否知道二者可能使用相似技术但语义不同

## 核心结论

RAG 主要检索外部知识，长期记忆主要管理用户偏好、历史任务和持续状态。二者都可能用向量检索，但产品语义和安全要求不同。

## 参考回答

RAG 面向的是已有知识库，比如文档、手册、网页、代码说明。它解决的是“回答或决策时应该参考哪些资料”。长期记忆面向的是 Agent 与用户长期交互产生的信息，比如用户偏好、项目背景、常用约束、之前任务结论。

长期记忆不仅是检索，还涉及写入和维护。系统要判断什么信息值得记住，什么时候更新，什么时候删除，如何让用户查看和纠正。错误记忆会长期影响 Agent 行为，因此风险更高。

## 深度展开

从技术上看，二者可能都用向量数据库，但 RAG 更像知识索引，memory 更像用户状态管理。RAG 的数据通常由管理员或文档流程维护；memory 的数据可能来自对话，需要更强的隐私、授权和可解释机制。

## 工程实践

- 明确哪些信息允许写入长期记忆
- 记忆写入前可以让用户确认
- 给记忆加来源、时间和置信度
- 提供查看、编辑和删除能力

## 常见误区

- 把所有历史对话都塞进向量库当记忆
- 不区分事实、偏好和临时上下文
- 记忆写错后没有纠正机制

## 追问方向

- 什么信息不应该写入长期记忆？
- 记忆如何过期？
- 用户隐私如何保护？

## 学习建议

一句话记忆：RAG 管知识，memory 管状态和偏好。
