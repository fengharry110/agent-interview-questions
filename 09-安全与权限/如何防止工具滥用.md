# 如何防止工具滥用？

## 面试问题

如何防止工具滥用？

## 考察重点

- 是否理解工具滥用的多种形态：频率滥用、参数滥用、组合滥用、后果滥用
- 是否能设计多维度的滥用防护策略
- 是否了解基于风险评分的动态工具授权机制
- 是否有过工具滥用防护的实际工程经验

## 核心结论

工具滥用不是单一维度的防护，而是**四道闸门**的协同：**权限闸门**（谁可以用）、**参数闸门**（怎么用）、**频率闸门**（用多少次）、**后果闸门**（用了之后做什么）。智能体系统的特殊性在于工具调用是模型自主发起的，因此防护不仅要看"工具本身"，还要看"调用的上下文和意图"。

## 参考回答

### 四层防护架构

```
┌──────────────────────────────────────────────────────────────┐
│ 第一层：权限闸门                                              │
│  检查: 角色、会话、上下文是否允许使用该工具                     │
│  失败: 直接拒绝 + 记录审计日志                                 │
├──────────────────────────────────────────────────────────────┤
│ 第二层：参数闸门                                              │
│  检查: 参数是否符合白名单/黑名单/格式约束                      │
│  失败: 拒绝 + 提示正确的参数格式                                │
├──────────────────────────────────────────────────────────────┤
│ 第三层：频率闸门                                              │
│  检查: 调用频率/总量/并发是否超过限制                          │
│  失败: 限速/排队/拒绝 + 告警                                   │
├──────────────────────────────────────────────────────────────┤
│ 第四层：后果闸门                                              │
│  检查: 操作后果风险评估（影响范围、可逆性、数据敏感性）         │
│  失败: 需要用户确认 / 阻断 + 告警                              │
└──────────────────────────────────────────────────────────────┘
```

### 工具注册与权限配置

```python
class ToolRegistry:
    """
    工具注册中心——所有工具必须先注册才能使用
    """
    
    def __init__(self):
        self.tools: Dict[str, ToolDefinition] = {}
    
    def register_tool(self, tool: ToolDefinition):
        """注册工具及其安全策略"""
        if tool.name in self.tools:
            raise DuplicateToolError(f"工具 {tool.name} 已注册")
        self.tools[tool.name] = tool
    
    def get_tool_definition(self, name: str) -> ToolDefinition:
        if name not in self.tools:
            raise ToolNotFoundError(f"工具 {name} 未注册")
        return self.tools[name]


class ToolDefinition:
    """工具定义——包含安全策略"""
    
    def __init__(
        self,
        name: str,
        allowed_roles: List[str],
        param_validators: List[ParamValidator],
        rate_limits: RateLimitConfig,
        risk_level: str,
        requires_confirmation: bool = False,
        max_call_count: int = None
    ):
        self.name = name
        self.allowed_roles = allowed_roles
        self.param_validators = param_validators
        self.rate_limits = rate_limits
        self.risk_level = risk_level
        self.requires_confirmation = requires_confirmation
        self.max_call_count = max_call_count
```

### 参数验证器

```python
class ParamValidators:
    """
    参数验证器集合
    """
    
    @staticmethod
    def path_in_allowed_dirs(allowed_dirs: List[str]) -> ParamValidator:
        """路径必须在允许的目录内"""
        def validate(param_value: str) -> ValidationResult:
            abs_path = os.path.abspath(param_value)
            for allowed in allowed_dirs:
                allowed_abs = os.path.abspath(allowed)
                if abs_path.startswith(allowed_abs):
                    return ValidationResult(valid=True)
            return ValidationResult(
                valid=False,
                reason=f"路径不在允许目录内: {allowed_dirs}"
            )
        return validate
    
    @staticmethod
    def command_in_allowlist(allowed_commands: List[str]) -> ParamValidator:
        """命令必须在白名单内"""
        def validate(param_value: str) -> ValidationResult:
            cmd = param_value.strip().split()[0] if param_value else ""
            if cmd in allowed_commands:
                return ValidationResult(valid=True)
            return ValidationResult(
                valid=False,
                reason=f"命令不在白名单内: {allowed_commands}"
            )
        return validate
    
    @staticmethod
    def regex_match(pattern: str, error_msg: str = None) -> ParamValidator:
        """参数必须匹配正则表达式"""
        def validate(param_value: str) -> ValidationResult:
            if re.match(pattern, param_value):
                return ValidationResult(valid=True)
            return ValidationResult(
                valid=False,
                reason=error_msg or f"参数不匹配格式: {pattern}"
            )
        return validate
    
    @staticmethod
    def max_length(limit: int) -> ParamValidator:
        """参数长度限制"""
        def validate(param_value: str) -> ValidationResult:
            if len(param_value) <= limit:
                return ValidationResult(valid=True)
            return ValidationResult(
                valid=False,
                reason=f"参数长度超过限制: {limit}"
            )
        return validate
```

### 频率控制和限流

```python
class RateLimiter:
    """
    工具调用频率控制器——防止频率滥用
    """
    
    def __init__(self):
        self.windows: Dict[str, List[float]] = {}
    
    def check_rate_limit(
        self,
        tool_name: str,
        session_id: str,
        config: RateLimitConfig
    ) -> bool:
        """
        检查是否超过频率限制
        """
        now = time.time()
        key = f"{session_id}:{tool_name}"
        
        if key not in self.windows:
            self.windows[key] = []
        
        # 清理过期记录
        self.windows[key] = [
            t for t in self.windows[key]
            if now - t < config.window_seconds
        ]
        
        # 检查次数限制
        if len(self.windows[key]) >= config.max_calls:
            return False
        
        # 检查全局限制
        if config.global_max_calls:
            global_key = f"global:{tool_name}"
            global_count = len([
                t for t in self.windows.get(global_key, [])
                if now - t < config.window_seconds
            ])
            if global_count >= config.global_max_calls:
                return False
        
        self.windows[key].append(now)
        return True


class RateLimitConfig:
    """频率限制配置"""
    
    def __init__(
        self,
        max_calls: int,
        window_seconds: int = 60,
        global_max_calls: int = None
    ):
        self.max_calls = max_calls
        self.window_seconds = window_seconds
        self.global_max_calls = global_max_calls
```

### 风险评分引擎

```python
class ToolRiskScorer:
    """
    工具调用风险评分引擎——综合评估每次调用的风险
    """
    
    RISK_FACTORS = {
        "shell": {
            "base_risk": 0.7,
            "param_risk": {
                "rm": 0.9,
                "sudo": 0.9,
                "chmod": 0.8,
                "dd": 0.9,
                ">": 0.6,
                ">>": 0.5,
                "curl": 0.5,
                "wget": 0.5
            }
        },
        "write_file": {
            "base_risk": 0.4,
            "param_risk": {
                "overwrite_existing": 0.6,
                "write_to_system_path": 0.8
            }
        },
        "delete_file": {
            "base_risk": 0.8,
            "param_risk": {
                "delete_directory": 0.7,
                "batch_delete": 0.8
            }
        },
        "network_request": {
            "base_risk": 0.5,
            "param_risk": {
                "external_domain": 0.6,
                "post_data": 0.5,
                "file_upload": 0.8
            }
        }
    }
    
    def score(
        self,
        tool_name: str,
        params: Dict,
        context: ToolContext
    ) -> float:
        """
        计算工具调用的风险评分 (0.0 - 1.0)
        """
        if tool_name not in self.RISK_FACTORS:
            return 0.3  # 未知工具的默认风险
        
        config = self.RISK_FACTORS[tool_name]
        score = config["base_risk"]
        
        # 参数风险加成
        for param_key, param_risk in config["param_risk"].items():
            if param_key in str(params).lower():
                score = max(score, param_risk)
        
        # 上下文调整
        if context.has_recent_failures:
            score += 0.1  # 最近有失败，风险增加
        
        if context.is_retry:
            score += 0.1  # 重试操作风险增加
        
        # 后果评估
        if self._has_persistent_effect(tool_name, params):
            score += 0.15
        
        return min(score, 1.0)
    
    def _has_persistent_effect(self, tool_name: str, params: Dict) -> bool:
        """判断操作是否有持久性影响"""
        persistent_ops = {"write", "delete", "update", "create", "drop"}
        return any(op in tool_name.lower() for op in persistent_ops)
```

## 工程实践

### 工具调用决策流程

```
工具调用请求
    ↓
┌──────────────────┐
│ 权限检查          │ ← 角色/会话/上下文
├──────────────────┤
│ 通过? ──否──→ 拒绝 │
└────────┬─────────┘
         ↓ 是
┌──────────────────┐
│ 参数验证          │ ← 白名单/格式/长度
├──────────────────┤
│ 通过? ──否──→ 拒绝 │
└────────┬─────────┘
         ↓ 是
┌──────────────────┐
│ 频率检查          │ ← 调用次数/窗口
├──────────────────┤
│ 通过? ──否──→ 限速 │
└────────┬─────────┘
         ↓ 是
┌──────────────────┐
│ 风险评分          │ ← BASE + PARAM + CONTEXT
├──────────────────┤
│ < 0.3 → 自动执行    │
│ 0.3-0.6 → 用户确认  │
│ > 0.6 → 阻断       │
└────────┬─────────┘
         ↓ 自动执行/确认后执行
┌──────────────────┐
│ 执行 + 审计日志    │
└──────────────────┘
```

### 防护策略对比

| 策略 | 防护对象 | 误报率 | 实现成本 | 用户体验影响 |
|------|---------|--------|---------|------------|
| 角色白名单 | 工具越权使用 | 低 | 低 | 无 |
| 参数验证 | 危险参数输入 | 中 | 中 | 低（返回提示） |
| 频率限制 | 循环调用 | 低 | 低 | 低（限速透明） |
| 风险评分 | 综合滥用 | 中高 | 高 | 中（需确认时） |
| 后果确认 | 高风险操作 | 低 | 中 | 高（频繁确认） |
| 调用次数上限 | 无限循环 | 低 | 低 | 低（限制合理） |

```python
class ToolCallAudit:
    """工具调用审计日志结构"""
    
    def log_call(
        self,
        tool_name: str,
        params: Dict,
        user: User,
        session_id: str,
        risk_score: float,
        decision: str,
        result: Any = None,
        error: str = None
    ):
        log_entry = {
            "timestamp": datetime.utcnow().isoformat(),
            "event_type": "tool_call",
            "tool_name": tool_name,
            "params": self._sanitize_params(params),
            "user_id": user.id,
            "user_role": user.role,
            "session_id": session_id,
            "risk_score": risk_score,
            "decision": decision,  # approved/denied/confirmed/blocked
            "result_summary": self._summarize_result(result),
            "error": error,
            "trace_id": self._get_trace_id()
        }
        self._write_log(log_entry)
    
    def _sanitize_params(self, params: Dict) -> Dict:
        """脱敏工具参数中的敏感字段"""
        sensitive_params = {"password", "token", "api_key", "secret"}
        sanitized = {}
        for key, value in params.items():
            if key.lower() in sensitive_params:
                sanitized[key] = "***REDACTED***"
            else:
                sanitized[key] = value
        return sanitized
```

## 常见误区

- **只看工具名称不看参数**：允许了 shell 工具但没限制具体命令，rm -rf / 和 ls 一样被执行
- **一次授权用到底**：会话开始时授权了所有工具，之后智能体在什么上下文用都不再检查
- **忽略工具组合风险**：read_file + network_request 可以组合成数据外泄通道
- **频率限制只限单工具**：智能体交替调用不同工具绕过单个工具的限制
- **风险评分过于简单**：只用关键词匹配判断风险，没有考虑上下文和操作意图
- **阻断后不给反馈**：模型不知道被阻断的原因，可能换个方式继续尝试同个操作

## 追问方向

- 如何防止智能体通过工具链组合绕过单个工具的防护？（例如用 read_file 读取私钥然后用 write_file 写入另一位置）
- 工具调用的限流策略如何避免误伤合法用例？比如批量测试需要大量调用 shell
- 你如何区分"用户明确要求的操作"和"智能体自作主张的操作"？
- 如果第三方工具集成到系统中，如何确保它不会成为滥用的入口？
- 风险评分中的参数风险如何动态更新？新的危险模式出现后如何快速覆盖？

## 学习建议

先定义每个工具的"安全使用边界"：谁可以用、参数有哪些限制、调用频率上限是多少。然后实现集中的调用拦截层——所有工具调用都经过这个层做安全检查。最后加上风险评分机制，对高风险调用触发用户确认或阻断。面试中关键要讲清楚"参数级防护"——不是限制工具本身，而是限制工具怎么用。
