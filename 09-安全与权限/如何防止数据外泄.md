# 如何防止数据外泄？

## 面试问题

如何防止数据外泄？

## 考察重点

- 是否理解智能体系统中数据外泄的多种渠道
- 是否能说清楚输入过滤、输出过滤、上下文限制三层防护
- 是否了解数据分类和分级访问控制的工程实践
- 是否有过实际的数据外泄防护方案设计经验

## 核心结论

防止智能体数据外泄的核心是**建立多层数据边界**：第一层限制智能体能接触到什么数据（数据范围控制），第二层限制智能体能输出什么数据（输出过滤），第三层监控数据流向（审计追踪）。不能只依赖模型"不主动泄露"——系统层面必须有强制执行的数据围栏。

---

## 数据外泄渠道全景

```
用户输入数据
    ↓
┌──────────────────────┐
│   智能体系统          │
│                      │
│  ┌──────┐  ┌──────┐  │
│  │ 模型  │→│ 工具  │  │  → 工具调用结果可能包含敏感数据
│  └──┬───┘  └──────┘  │
│     │                 │
│     ↓                 │
│  上下文（包含所有数据） │  → 全部数据对模型可见
│                      │
└──────────┬───────────┘
           │
           ↓
    ┌──────────────┐
    │  模型输出     │  → 可能包含原始数据
    └──────────────┘
    
外泄渠道：
  - 模型直接在回答中输出敏感数据
  - 工具调用参数中携带了不应暴露的数据
  - 智能体将数据写入外部文件或 API
  - 上下文被其他会话共享导致数据泄露
  - 日志系统记录了完整的敏感数据
```

### 数据范围控制

```python
class DataScopeEnforcer:
    """
    数据范围执行器——限制智能体可以访问的数据范围
    """
    
    def __init__(self):
        self.scope_rules = []
    
    def add_scope_rule(self, rule: ScopeRule):
        self.scope_rules.append(rule)
    
    def check_data_access(
        self,
        session: Session,
        data_source: str,
        operation: str
    ) -> bool:
        """
        检查智能体是否有权从该数据源读取/写入数据
        """
        user_role = session.user.role
        task_context = session.current_task
        
        for rule in self.scope_rules:
            if rule.matches(data_source, user_role, task_context):
                return operation in rule.allowed_operations
        
        return False  # 默认拒绝
    
    def get_accessible_data_sources(
        self,
        session: Session
    ) -> List[str]:
        """
        获取智能体在当前会话中可访问的数据源列表
        """
        accessible = []
        for rule in self.scope_rules:
            if rule.matches_session(session):
                accessible.append(rule.data_source)
        return accessible


class ScopeRule:
    """数据范围规则"""
    
    def __init__(
        self,
        data_source: str,
        allowed_roles: List[str],
        allowed_operations: List[str],
        context_conditions: Dict = None
    ):
        self.data_source = data_source
        self.allowed_roles = allowed_roles
        self.allowed_operations = allowed_operations
        self.context_conditions = context_conditions or {}
```

### 上下文数据最小化

```python
class ContextDataMinimizer:
    """
    上下文数据最小化——只向模型上下文注入必要的数据
    """
    
    def prepare_context(
        self,
        task: Task,
        available_data: Dict[str, Any]
    ) -> Context:
        """
        准备模型上下文，只包含任务所需的最小数据
        """
        # 1. 分析任务需要哪些数据
        required_data_keys = self._analyze_task_requirements(task)
        
        # 2. 从可用数据中选择子集
        selected_data = {
            key: available_data[key]
            for key in required_data_keys
            if key in available_data
        }
        
        # 3. 对敏感字段做脱敏
        sanitized_data = self._sanitize_sensitive_fields(selected_data)
        
        # 4. 数据量限制（避免上下文过大）
        truncated_data = self._truncate_large_fields(sanitized_data)
        
        return Context(data=truncated_data)
    
    def _sanitize_sensitive_fields(self, data: Dict) -> Dict:
        """脱敏敏感字段"""
        sensitive_patterns = [
            "email", "phone", "ssn", "credit_card",
            "password", "address", "birth_date"
        ]
        
        sanitized = {}
        for key, value in data.items():
            if any(p in key.lower() for p in sensitive_patterns):
                sanitized[key] = "***MASKED***"
            else:
                sanitized[key] = value
        
        return sanitized
    
    def _truncate_large_fields(
        self,
        data: Dict,
        max_field_length: int = 5000
    ) -> Dict:
        """截断过大的字段"""
        truncated = {}
        for key, value in data.items():
            if isinstance(value, str) and len(value) > max_field_length:
                truncated[key] = value[:max_field_length] + "...[truncated]"
            else:
                truncated[key] = value
        return truncated
```

## 深度展开

### 输出数据过滤层

```
模型输出
    ↓
┌─────────────────────────────┐
│ 第一层：敏感数据检测          │
│ - 正则匹配（API Key、密码等）  │
│ - ML 分类器（判断是否包含 PII）│
└──────────┬──────────────────┘
           ↓
┌─────────────────────────────┐
│ 第二层：数据分类检查          │
│ - 如果是 confidential 以上   │
│ - 直接替换为脱敏版本           │
└──────────┬──────────────────┘
           ↓
┌─────────────────────────────┐
│ 第三层：外泄可能性评分        │
│ - 基于内容和上下文            │
│ - 高风险时阻断输出 + 告警     │
└──────────┬──────────────────┘
           ↓
     安全输出
```

```python
class OutputDataProtector:
    """
    输出数据保护——三层过滤机制
    """
    
    def __init__(self):
        self.pattern_detector = PatternDetector()
        self.classification_checker = ClassificationChecker()
        self.risk_scorer = RiskScorer()
    
    async def protect_output(
        self,
        raw_output: str,
        context: OutputContext
    ) -> ProtectedOutput:
        """
        保护输出内容，防止数据外泄
        """
        # 第一层：敏感模式检测
        findings = self.pattern_detector.detect(raw_output)
        if findings.has_sensitive_data:
            self._record_event(context, "sensitive_pattern_detected", findings)
        
        # 第二层：数据分类检查
        classification = self.classification_checker.classify(raw_output)
        if classification.level in ("confidential", "restricted"):
            protected = self._apply_masking(raw_output, classification)
            
            if classification.level == "restricted":
                self._trigger_alert(context, "attempted_restricted_output")
                return ProtectedOutput(
                    safe_output="[此输出包含受限数据，已被拦截]",
                    original_classification=classification,
                    blocked=True
                )
            
            return ProtectedOutput(
                safe_output=protected,
                original_classification=classification,
                masked_fields=classification.masked_fields
            )
        
        # 第三层：风险评分
        risk_score = self.risk_scorer.score(raw_output, context)
        if risk_score > 0.8:
            self._trigger_alert(context, "high_risk_output")
            return ProtectedOutput(
                safe_output=self._generate_safe_summary(raw_output),
                risk_score=risk_score,
                blocked_partially=True
            )
        
        return ProtectedOutput(safe_output=raw_output)
```

### 数据外泄检测指标

| 指标 | 检测内容 | 告警阈值 | 响应动作 |
|------|---------|---------|---------|
| 敏感数据密度 | 输出中敏感数据占比 | > 20% | 告警 + 输出阻断 |
| 数据量异常 | 单次输出数据量 | > 100KB | 审查 + 截断 |
| 重复数据提取 | 同一数据的多次输出 | > 3次/分钟 | 限速 + 调查 |
| 跨域访问 | 访问非任务相关数据源 | 任何一次 | 告警 + 记录 |
| 输出目的地 | 数据写入外部地址 | 非白名单地址 | 阻断 + 告警 |

### 数据外泄实时监控

```python
class DataLeakMonitor:
    """
    数据外泄实时监控
    """
    
    def __init__(self):
        self.access_log = []
        self.alert_thresholds = {
            "sensitive_output_count": 5,
            "data_volume_per_minute": 10 * 1024 * 1024,
            "unauthorized_access_attempts": 3
        }
    
    def record_access(self, event: DataAccessEvent):
        """记录数据访问事件"""
        self.access_log.append(event)
        
        # 实时检查是否达到告警阈值
        if self._should_alert(event):
            self._trigger_alert(event)
    
    def _should_alert(self, event: DataAccessEvent) -> bool:
        """判断是否触发告警"""
        recent_events = self._get_recent_events(
            event.session_id,
            window_seconds=60
        )
        
        # 敏感输出次数
        sensitive_count = sum(
            1 for e in recent_events
            if e.event_type == "sensitive_output"
        )
        if sensitive_count >= self.alert_thresholds["sensitive_output_count"]:
            return True
        
        # 数据量检查
        total_volume = sum(
            e.data_volume for e in recent_events
        )
        if total_volume > self.alert_thresholds["data_volume_per_minute"]:
            return True
        
        return False
```

## 常见误区

- **只防输出不防输入**：认为只要过滤模型输出就够了，忽略了智能体可能通过工具调用将数据写入外部系统
- **上下文塞入全部数据**：把数据库全部内容塞进上下文让模型"自己判断"，大幅增加了外泄风险
- **忽略工具本身的外泄能力**：允许智能体调用网络请求工具（curl、requests）时没有限制目标域名
- **数据分级缺失**：所有数据一视同仁，没有区分公开数据和机密数据的不同保护策略
- **只靠模型自觉**：在 prompt 中要求模型"不要泄露敏感数据"但没有系统级强制执行
- **日志记录原始数据**：审计日志中包含完整的敏感数据，日志系统本身变成了泄露渠道

## 学习建议

从数据分类开始——先盘点你的智能体系统接触了哪些数据，给每类数据打上密级标签。然后针对每类数据制定"智能体能做什么"和"智能体能输出什么"两个策略。最后实现输出过滤环路——在模型输出返回用户之前，检测并阻断敏感数据的泄露。面试中重点强调"数据围栏"概念：不是防止模型泄露数据，而是让数据根本没机会进入模型上下文。
