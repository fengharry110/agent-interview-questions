# 如何设计沙箱？

## 面试问题

如何设计沙箱？

## 考察重点

- 是否理解沙箱在智能体系统中的核心作用：限制模型行为的副作用范围
- 是否能说清楚沙箱的不同层级（进程级/容器级/虚拟机级）及其取舍
- 是否了解沙箱的文件系统、网络、资源限制三大隔离维度
- 是否有过沙箱方案设计和实际部署的经验

## 核心结论

智能体沙箱的核心不是"完全隔离"——完全隔离的沙箱会让智能体无法完成有意义的任务。沙箱设计的本质是**在隔离性和功能性之间找到平衡**：限制危险操作的权限，同时允许安全操作正常工作。好的沙箱设计遵循"最小隔离"原则：只隔离需要的系统资源，不隔离的不多余限制。

## 沙箱隔离层级

```
隔离层级                          功能完整性
    ↑                               ↓
┌───────────────────────────────────────────────┐
│  第三层：虚拟机沙箱                             │
│  - 完整 OS 隔离                                │
│  - 硬件虚拟化                                  │
│  - 开销大，启动慢                               │
├───────────────────────────────────────────────┤
│  第二层：容器沙箱                               │
│  - 进程 + 文件系统隔离                          │
│  - namespace 隔离                               │
│  - 常用方案：Docker 容器                        │
├───────────────────────────────────────────────┤
│  第一层：进程沙箱                               │
│  - seccomp / AppArmor / SELinux                │
│  - 文件系统只读 + 网络限制                      │
│  - 低开销，适合大多数场景                        │
├───────────────────────────────────────────────┤
│  第零层：语言级沙箱                             │
│  - Python Jail / JS Sandbox                   │
│  - 仅限制代码执行                              │
│  - 轻量但容易被绕过                             │
└───────────────────────────────────────────────┘
```

### 沙箱管理器设计

```python
class SandboxManager:
    """
    沙箱管理器——管理智能体的执行沙箱
    """
    
    def __init__(self, config: SandboxConfig):
        self.config = config
        self.active_sandboxes: Dict[str, Sandbox] = {}
    
    async def create_sandbox(
        self,
        session_id: str,
        user_role: str,
        task_requirements: TaskRequirements
    ) -> Sandbox:
        """
        为会话创建沙箱
        """
        # 1. 根据任务需求确定沙箱级别
        level = self._determine_level(task_requirements, user_role)
        
        # 2. 创建沙箱实例
        sandbox = Sandbox(
            session_id=session_id,
            level=level,
            config=self.config.get_level_config(level)
        )
        
        # 3. 初始化沙箱
        await sandbox.initialize()
        
        # 4. 注册并返回
        self.active_sandboxes[session_id] = sandbox
        return sandbox
    
    def _determine_level(
        self,
        requirements: TaskRequirements,
        user_role: str
    ) -> SandboxLevel:
        """根据任务需求确定沙箱隔离级别"""
        if user_role == "admin":
            return SandboxLevel.PROCESS
        if requirements.requires_network:
            return SandboxLevel.CONTAINER
        if requirements.requires_system_access:
            return SandboxLevel.CONTAINER
        return SandboxLevel.PROCESS
    
    async def destroy_sandbox(self, session_id: str):
        """销毁沙箱，清理所有资源"""
        if session_id in self.active_sandboxes:
            sandbox = self.active_sandboxes[session_id]
            await sandbox.cleanup()
            del self.active_sandboxes[session_id]
    
    async def cleanup_all(self):
        """清理所有沙箱（系统关闭时调用）"""
        for session_id in list(self.active_sandboxes.keys()):
            await self.destroy_sandbox(session_id)
```

            "/dev/*", "/proc/*", "/sys/*",
            "~/.ssh/*", "~/.aws/*", "~/.config/*"
        ]
    )

    # 网络限制
    network_allowed: bool = False
    allowed_domains: List[str] = field(
        default_factory=lambda: [
            "*.github.com", "*.npmjs.org", "pypi.org",
            "*.internal.company.com"
        ]
    )
    blocked_domains: List[str] = field(
        default_factory=lambda: ["*.malicious.com", "10.*", "172.*", "192.168.*"]
    )
    
    # 资源限制

    max_memory_mb: int = 512

### 沙箱文件系统隔离

```python
class SandboxFilesystem:
    """
    沙箱文件系统——限制智能体的文件访问范围
    """
    
    def __init__(self, config: SandboxConfig, sandbox_dir: str):
        self.config = config
        self.sandbox_dir = sandbox_dir
        self._setup_sandbox_dir()
    
    def _setup_sandbox_dir(self):
        """设置沙箱工作目录"""
        os.makedirs(self.sandbox_dir, exist_ok=True)
        os.makedirs(os.path.join(self.sandbox_dir, "temp"), exist_ok=True)
        os.makedirs(os.path.join(self.sandbox_dir, "output"), exist_ok=True)
    
    def resolve_path(self, requested_path: str) -> PathResolution:
        """
        解析路径，将请求路径映射到沙箱内的实际路径
        """
        abs_path = os.path.abspath(os.path.expanduser(requested_path))
        
        # 检查黑名单路径
        for blocked in self.config.blocked_paths:
            if fnmatch.fnmatch(abs_path, os.path.abspath(os.path.expanduser(blocked))):
                return PathResolution(
                    allowed=False,
                    reason=f"路径在黑名单中: {blocked}",
                    sandbox_path=None
                )
        
        # 检查读权限
        read_allowed = False
        for allowed in self.config.allowed_read_paths:
            if fnmatch.fnmatch(abs_path, os.path.abspath(os.path.expanduser(allowed))):
                read_allowed = True
                break
        
        # 检查写权限
        write_allowed = False
        for allowed in self.config.allowed_write_paths:
            if fnmatch.fnmatch(abs_path, os.path.abspath(os.path.expanduser(allowed))):
                write_allowed = True
                break
        
        if not read_allowed and not write_allowed:
            return PathResolution(
                allowed=False,
                reason="路径不在允许范围内",
                sandbox_path=None
            )
        
        # 路径映射：在沙箱内创建对应的目录结构
        # 这样可以防止智能体直接访问宿主机文件系统
        mapped_path = abs_path.replace(":", "_").replace("\\", "/")
        if mapped_path.startswith("/"):
            mapped_path = mapped_path[1:]
        sandbox_path = os.path.join(self.sandbox_dir, mapped_path)
        
        return PathResolution(
            allowed=True,
            reason="",
            sandbox_path=sandbox_path,
            is_read_allowed=read_allowed,
            is_write_allowed=write_allowed
        )


@dataclass
class PathResolution:
    """路径解析结果"""
    allowed: bool
    reason: str
    sandbox_path: Optional[str]
    is_read_allowed: bool = False
    is_write_allowed: bool = False
```

### 网络隔离

```python
class SandboxNetwork:
    """
    沙箱网络——限制智能体的网络访问
    """
    
    def __init__(self, config: SandboxConfig):
        self.config = config
        self.proxy_server = None
    
    async def create_network_proxy(self) -> ProxyConfig:
        """
        创建沙箱网络代理（正向代理模式）
        """
        if not self.config.network_allowed:
            return ProxyConfig(blocked=True)
        
        # 创建代理服务器，只允许访问白名单域名
        self.proxy_server = await self._start_proxy(
            allowed_domains=self.config.allowed_domains,
            blocked_domains=self.config.blocked_domains
        )
        
        return ProxyConfig(
            blocked=False,
            proxy_host="127.0.0.1",
            proxy_port=self.proxy_server.port,
            allowed_domains=self.config.allowed_domains
        )
    
    def check_url(self, url: str) -> bool:
        """
        检查 URL 是否允许访问
        """
        from urllib.parse import urlparse
        
        if not self.config.network_allowed:
            return False
        
        parsed = urlparse(url)
        hostname = parsed.hostname
        
        # 检查黑名单
        for blocked in self.config.blocked_domains:
            if fnmatch.fnmatch(hostname, blocked):
                return False
        
        # 检查白名单（如果有）
        if self.config.allowed_domains:
            for allowed in self.config.allowed_domains:
                if fnmatch.fnmatch(hostname, allowed):
                    return True
            return False  # 有白名单但不在内
        
        return True  # 无白名单，允许所有非黑名单的域名
    
    async def cleanup(self):
        """清理网络代理"""
        if self.proxy_server:
            await self.proxy_server.stop()
```

### 资源限制

```python
class SandboxResourceLimiter:
    """
    沙箱资源限制——防止智能体消耗过多资源
    """
    
    def __init__(self, config: SandboxConfig):
        self.config = config
        self.usage: Dict[str, ResourceUsage] = {}
    
    def track_process_start(self, session_id: str) -> bool:
        """
        跟踪进程启动，检查是否超过进程数限制
        """
        if session_id not in self.usage:
            self.usage[session_id] = ResourceUsage()
        
        usage = self.usage[session_id]
        if usage.process_count >= self.config.max_process_count:
            return False
        
        usage.process_count += 1
        return True
    
    def track_process_end(self, session_id: str):
        """进程结束时释放计数"""
        if session_id in self.usage:
            self.usage[session_id].process_count -= 1
    
    def check_memory(self, session_id: str, additional_mb: int) -> bool:
        """检查内存使用是否超限"""
        if session_id not in self.usage:
            return True
        
        usage = self.usage[session_id]
        return (usage.memory_mb + additional_mb) <= self.config.max_memory_mb
    
    def check_disk(self, session_id: str, additional_mb: int) -> bool:
        """检查磁盘使用是否超限"""
        if session_id not in self.usage:
            return True
        
        usage = self.usage[session_id]
        return (usage.disk_mb + additional_mb) <= self.config.max_disk_usage_mb
    
    def get_usage_report(self, session_id: str) -> Dict:
        """获取资源使用报告"""
        usage = self.usage.get(session_id, ResourceUsage())
        config = self.config
        
        return {
            "processes": f"{usage.process_count}/{config.max_process_count}",
            "memory_mb": f"{usage.memory_mb}/{config.max_memory_mb}",
            "disk_mb": f"{usage.disk_mb}/{config.max_disk_usage_mb}",
            "cpu_seconds": f"{usage.cpu_seconds}/{config.max_cpu_seconds}",
            "execution_seconds": f"{usage.execution_seconds}/{config.max_execution_seconds}"
        }
```

### 沙箱生命周期管理

```
沙箱创建
    ↓
┌─────────────────┐
│ 初始化           │ ← 创建工作目录，设置隔离
├─────────────────┤
│ 就绪             │ ← 等待任务
└────────┬────────┘
         ↓ 接收到任务
┌─────────────────┐
│ 活跃             │ ← 执行任务中
├─────────────────┤
│ 子状态:          │
│ - 空闲 (无任务)   │
│ - 忙碌 (执行中)   │
│ - 超时警告        │
└────────┬────────┘
         ↓ 任务完成/超时/异常
┌─────────────────┐
│ 清理             │ ← 删除临时文件，回收资源
├─────────────────┤
│ 销毁             │ ← 完全移除沙箱
└──────────────────┘
```

### 沙箱级别对比

| 特性 | 进程沙箱 | 容器沙箱 | 虚拟机沙箱 |
|------|---------|---------|-----------|
| 启动时间 | < 100ms | 1-5s | 30-60s |
| 内存开销 | 5-20MB | 50-200MB | 512MB+ |
| 隔离性 | 中 | 高 | 极高 |
| 文件系统隔离 | 部分 | 完整 | 完整 |
| 网络隔离 | 需额外配置 | 原生支持 | 原生支持 |
| 系统调用限制 | seccomp | seccomp + namespace | 硬件级 |
| 逃逸难度 | 中 | 高 | 极高 |
| 适用场景 | 代码执行、文件操作 | 网络请求、包管理 | 高风险操作 |
| 清理速度 | 快 | 中 | 慢 |

## 常见误区

- **沙箱不是"银弹"**：以为用了 Docker 就万事大吉，忽略了容器自身的逃逸漏洞
- **过度隔离影响功能**：完全禁止网络访问导致智能体无法安装依赖、调用 API
- **忽略沙箱清理**：沙箱使用后不清理，临时文件堆积、敏感数据残留
- **沙箱级别一刀切**：所有用户和所有任务用同一个隔离级别，要么过度隔离要么隔离不足
- **忽略沙箱开销**：每个会话启动一个完整容器，资源消耗巨大
- **沙箱逃逸检测缺失**：没有监控沙箱内的异常行为，逃逸了也不知道
- **无日志记录**：沙箱内的操作没有审计日志，出问题后无法追溯

## 追问方向

- 你的沙箱如何防止数据在多个会话之间串扰？容器级别的隔离是否足够？
- 沙箱需要访问用户私有的 git 仓库或 npm 包，如何在保持隔离的同时允许访问？
- 如果沙箱内的进程变成了僵尸进程，你的系统怎么处理？
- 沙箱逃逸后的应急响应流程是什么？如何最小化逃逸的损害范围？
- 在 serverless 环境中如何实现沙箱？Lambda 函数本身是否可以作为沙箱？

## 学习建议

从最简单的进程级沙箱开始——使用 chroot、seccomp 和资源限制。不需要一上来就上 Docker 或虚拟机。先把文件系统隔离和资源限制做好，这已经能覆盖 90% 的风险场景。当智能体需要网络访问时再升级到容器沙箱。面试中关键要讲清楚"分级别隔离"的思路——不同风险等级的任务用不同级别的沙箱。
