# 为什么提示词注入对智能体更危险？

## 面试问题

为什么提示词注入对智能体更危险？

## 考察重点

- 是否理解智能体系统的"工具调用能力"是注入危害放大的根本原因
- 是否能说清楚注入在聊天机器人和智能体中的危害差异
- 是否了解智能体特有的注入攻击面（工具调用、权限提升）
- 是否有过智能体安全事件的实际处理经验

## 核心结论

提示词注入对智能体更危险，因为智能体**能调用工具**。聊天机器人被注入最多输出错误信息，但智能体被注入可能触发工具执行——删文件、发邮件、改数据库、执行命令。危害从"信息层面"升级到了"操作层面"。

### 注入检测

```python
class InjectionDetector:
    """
    注入攻击检测
    """
    
    def __init__(self):
        self.suspicious_patterns = self._load_patterns()
    
    def check_input(self, user_input: str) -> DetectionResult:
        """检测用户输入是否包含注入企图"""
        score = 0
        reasons = []
        
        # 检查系统指令覆盖
        if self._has_instruction_override(user_input):
            score += 0.4
            reasons.append("检测到指令覆盖尝试")
        
        # 检查工具操控
        if self._has_tool_manipulation(user_input):
            score += 0.3
            reasons.append("检测到工具操控尝试")
        
        # 检查角色扮演诱导
        if self._has_roleplay_induction(user_input):
            score += 0.2
            reasons.append("检测到角色扮演诱导")
        
        return DetectionResult(
            is_suspicious=score > 0.5,
            confidence=score,
            reasons=reasons,
            action="block" if score > 0.7 else "flag"
        )
```

## 常见误区

- **觉得注入只是理论问题**：已经有多个实际案例证明智能体可以被成功注入并造成危害
- **只防用户输入不防工具返回**：间接注入（通过工具返回的内容）比直接注入更隐蔽、更难防御
- **低估注入的自动化风险**：手动注入可能被发现，但脚本化的批量注入可以在短时间内造成大范围危害
- **认为 prompt 加固就安全了**：没有绝对安全的 prompt，防御需要多层（输入过滤 + 权限控制 + 审计）

## 学习建议

列一个你智能体系统所有工具的危险等级清单。对每个"高危险"工具（能写文件、能执行命令、能发网络请求），设计至少两种防御措施。然后模拟一次注入攻击，验证你的防御是否有效。安全不是一次性的工作，而需要持续测试和改进。
