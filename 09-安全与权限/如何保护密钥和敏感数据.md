# 如何保护密钥和敏感数据？

## 面试问题

如何保护密钥和敏感数据？

## 考察重点

- 是否理解智能体系统中的密钥面临哪些特殊的泄露风险
- 是否能说清楚密钥存储、传递、使用的安全原则
- 是否了解智能体在读取文件或执行命令时可能意外暴露密钥的场景
- 是否有过密钥泄露防护的实际经验

## 核心结论

智能体系统的密钥保护面临一个特殊挑战：智能体需要调用工具（访问数据库、调用 API），因此**必须能接触到密钥**，但同时又要防止密钥被误输出或泄露。核心策略是"三不"：**不写进代码**（从环境变量/密钥管理服务读取）、**不传进上下文**（用变量引用而非明文）、**不输出到响应**（在输出层做过滤）。

## 密钥风险全景图

```
┌─────────────────────────────────────────────────────────┐
│                   密钥泄露渠道                            │
├─────────────┬───────────────────────────────────────────┤
│ 模型上下文  │ 密钥被直接写入 prompt 或 system message    │
├─────────────┼───────────────────────────────────────────┤
│ 工具参数    │ API Key 作为工具参数明文传递               │
├─────────────┼───────────────────────────────────────────┤
│ 文件读取    │ 智能体读取 .env 或配置文件并输出           │
├─────────────┼───────────────────────────────────────────┤
│ 环境变量    │ 用户要求"列出所有环境变量"导致密钥泄露     │
├─────────────┼───────────────────────────────────────────┤
│ 执行日志    │ 工具调用的入参和返回结果中包含密钥         │
├─────────────┼───────────────────────────────────────────┤
│ 版本控制    │ 智能体将含密钥的代码提交到 git 仓库        │
├─────────────┼───────────────────────────────────────────┤
│ API 响应    │ 第三方 API 返回的响应中包含新密钥          │
└─────────────┴───────────────────────────────────────────┘
```

## 密钥的三道防线

```
第一道防线：存储层
  密钥管理服务 (Vault/AWS Secrets Manager/K8s Secrets)
  环境变量注入（不在代码中硬编码）
  
第二道防线：传递层
  引用替换（${{SECRET:KEY_NAME}}）
  不在模型上下文中出现明文
  不在日志中出现明文
  
第三道防线：输出层
  正则匹配过滤
  机器学习检测
  预提交钩子（pre-commit hook）
```

### 密钥管理策略

```python
class SecretManager:
    """
    密钥管理器
    """
    
    def __init__(self):
        self.secrets: Dict[str, str] = {}
        self._load_from_env()
    
    def _load_from_env(self):
        """从环境变量加载密钥"""
        required_secrets = [
            'OPENAI_API_KEY',
            'DATABASE_URL',
            'EXTERNAL_API_KEY'
        ]
        for secret_name in required_secrets:
            value = os.environ.get(secret_name)
            if value:
                self.secrets[secret_name] = value
    
    def get_secret(self, name: str) -> str:
        """
        获取密钥（返回引用而非明文）
        """
        if name not in self.secrets:
            raise SecretNotFound(f"密钥 {name} 不存在")
        
        # 返回一个引用标记
        return f"${{SECRET:{name}}}"
    
    def resolve_secret(self, reference: str) -> str:
        """
        解析密钥引用为实际值（仅在执行层使用）
        """
        match = re.match(r'\$\{SECRET:(\w+)\}', reference)
        if match:
            return self.secrets.get(match.group(1), '')
        return reference
```

## 深度展开

### 密钥在智能体流程中的流转

```python
class SecretFlow:
    """
    密钥在智能体系统中的安全流转
    """
    
    def __init__(self):
        self.secret_manager = SecretManager()
        self.output_filter = OutputFilter()
    
    async def execute_tool_with_secret(
        self,
        tool_call: ToolCall,
        context: Context
    ) -> ToolResult:
        """
        使用密钥执行工具调用（安全版本）
        """
        args = tool_call.args
        
        # 1. 在参数中使用引用而非明文
        if args.get('api_key'):
            args['api_key'] = self.secret_manager.get_secret('API_KEY')
        
        # 2. 在工具执行层解析引用
        resolved_args = {
            k: self.secret_manager.resolve_secret(v) if isinstance(v, str) else v
            for k, v in args.items()
        }
        
        # 3. 执行工具（使用实际密钥）
        result = await execute_tool(tool_call.name, resolved_args)
        
        # 4. 对返回结果做密钥过滤
        result.output = self.output_filter.filter_secrets(result.output)
        
        # 5. 记录日志时使用引用而不是真实值
        log_entry = {
            'tool': tool_call.name,
            'args': args,
            'result': result.output
        }
        
        return result
```

### 输出过滤

```python
class OutputFilter:
    """
    输出过滤器——防止密钥出现在模型输出中
    """
    
    PATTERNS = [
        (r'sk-[a-zA-Z0-9]{32,}', '[API_KEY_REDACTED]'),
        (r'AKIA[0-9A-Z]{16}', '[AWS_KEY_REDACTED]'),
        (r'eyJ[A-Za-z0-9_-]{10,}\.[A-Za-z0-9_-]{10,}\.[A-Za-z0-9_-]{10,}', '[JWT_REDACTED]'),
        (r'-----BEGIN (RSA|EC|OPENSSH) PRIVATE KEY-----', '[PRIVATE_KEY_REDACTED]'),
        (r'ghp_[a-zA-Z0-9]{36}', '[GITHUB_TOKEN_REDACTED]'),
        (r'db_(?:user|pass|name)\s*[:=]\s*["\']?[^"\'\s]+["\']?', '[DB_CREDENTIAL_REDACTED]'),
    ]
    
    def filter_secrets(self, text: str) -> str:
        """过滤文本中的密钥"""
        for pattern, replacement in self.PATTERNS:
            text = re.sub(pattern, replacement, text, flags=re.IGNORECASE)
        return text
    
    def filter_messages(self, messages: List[Message]) -> List[Message]:
        """过滤对话消息中的密钥"""
        filtered = []
        for msg in messages:
            if msg.content:
                msg.content = self.filter_secrets(msg.content)
            filtered.append(msg)
        return filtered
```

### 安全配置文件读取

```python
class SafeConfigReader:
    """
    安全的配置文件读取
    """
    
    SENSITIVE_CONFIG_PATTERNS = [
        "*password*", "*secret*", "*token*",
        "*key*", "*credential*", ".env",
        "*.pem", "*.key", "config.json"
    ]
    
    BLOCKED_PATHS = [
        "/etc/passwd", "/etc/shadow",
        "/etc/ssl/", "~/.ssh/",
        "*/credentials*"
    ]
    
    @staticmethod
    def is_path_allowed(path: str) -> bool:
        """检查路径是否允许读取"""
        abs_path = os.path.abspath(os.path.expanduser(path))
        
        for blocked in SafeConfigReader.BLOCKED_PATHS:
            if fnmatch.fnmatch(abs_path, os.path.expanduser(blocked)):
                return False
        
        return True
    
    @staticmethod
    def should_warn(filename: str) -> bool:
        """是否应该警告用户正在读取可能敏感的文件"""
        for pattern in SafeConfigReader.SENSITIVE_CONFIG_PATTERNS:
            if fnmatch.fnmatch(filename, pattern):
                return True
        return False
```

### 密钥检测器

```python
class SecretDetector:
    """
    自动检测密钥泄露
    """
    
    def __init__(self):
        self.detectors = [
            ("API Key", r'(?:api[_-]?key|apikey)\s*[:=]\s*["\']?\w{16,}'),
            ("AWS Key", r'AKIA[0-9A-Z]{16}'),
            ("OpenAI Key", r'sk-[a-zA-Z0-9]{32,}'),
            ("JWT Token", r'eyJ[A-Za-z0-9_-]{10,}\.[A-Za-z0-9_-]{10,}\.'),
            ("Private Key", r'-----BEGIN.*PRIVATE KEY-----'),
            ("GitHub Token", r'gh[pousr]_[A-Za-z0-9_]{36,}'),
            ("Password Field", r'"password"\s*:\s*"[^"]+"'),
        ]
    
    def scan_text(self, text: str) -> List[SecretFinding]:
        """扫描文本中的密钥"""
        findings = []
        for name, pattern in self.detectors:
            matches = re.finditer(pattern, text, re.IGNORECASE)
            for match in matches:
                findings.append(SecretFinding(
                    type=name,
                    position=match.span(),
                    preview=text[
                        max(0, match.start()-20):
                        min(len(text), match.end()+20)
                    ]
                ))
        return findings
    
    def scan_before_send(
        self,
        messages: List[Dict]
    ) -> ScanResult:
        """在发送给模型前扫描消息"""
        full_text = json.dumps(messages)
        findings = self.scan_text(full_text)
        
        return ScanResult(
            has_secrets=len(findings) > 0,
            findings=findings,
            suggestion="检测到可能的密钥泄露，请检查消息内容"
        )
```

### 密钥保护策略对比

| 策略 | 安全性 | 实现复杂度 | 性能影响 | 适用场景 |
|------|--------|-----------|---------|---------|
| 环境变量注入 | 中 | 低 | 无 | 开发环境 |
| 密钥管理服务 | 高 | 高 | 低（缓存后） | 生产环境 |
| 引用替换 | 高 | 中 | 低 | 智能体上下文传递 |
| 输出过滤 | 中 | 低 | 低 | 防御性措施 |
| 日志脱敏 | 高 | 中 | 低 | 合规要求 |
| pre-commit hook | 中 | 低 | 开发时 | 代码仓库防护 |

### 日志脱敏实现

```python
class LogSanitizer:
    """
    日志脱敏——在写入日志前过滤密钥
    """
    
    SENSITIVE_KEYS = [
        "api_key", "apikey", "api-key",
        "password", "passwd", "secret",
        "token", "access_token", "refresh_token",
        "private_key", "credential",
        "authorization", "x-api-key"
    ]
    
    @classmethod
    def sanitize_dict(cls, data: dict) -> dict:
        """递归脱敏字典中的敏感字段"""
        sanitized = {}
        for key, value in data.items():
            if any(s in key.lower() for s in cls.SENSITIVE_KEYS):
                sanitized[key] = "***REDACTED***"
            elif isinstance(value, dict):
                sanitized[key] = cls.sanitize_dict(value)
            elif isinstance(value, list):
                sanitized[key] = [
                    cls.sanitize_dict(item) if isinstance(item, dict) else item
                    for item in value
                ]
            else:
                sanitized[key] = value
        return sanitized
```

## 常见误区

- **在 prompt 中写密钥**：直接在 system prompt 中写入数据库密码或 API Key，模型输出时可能外泄
- **以为日志系统不存敏感数据**：日志中记录了完整的工具调用参数，包含密钥
- **忽略工具返回中的密钥**：API 返回的数据中可能包含新的密钥（如 token 刷新）
- **密钥硬编码在代码中**：密钥放在代码仓库里，智能体可能读取并输出
- **只做一层防护**：只有输出过滤没有引用替换，或者只有环境变量没有日志脱敏。需要多层防护
- **忽略密钥轮换**：密钥长期不轮换，泄露后危害范围大
- **密钥权限过大**：一个密钥拥有所有权限，没有按需分配最小权限

## 学习建议

先检查你的智能体系统中有多少处"密钥明文传递"——搜索代码、配置文件和 prompt 中是否有 API Key、密码等。然后用环境变量或密钥管理服务替换。最后加上输出过滤——即使模型不小心输出了密钥，也能被过滤掉。这三步做完后，再考虑更复杂的密钥防护方案。面试中关键是讲清楚"引用替换"机制——让密钥对模型不可见，只在执行层解析。
