# 如何处理越权请求？

## 面试问题

如何处理越权请求？

## 考察重点

- 是否理解越权请求在智能体系统中的特殊性：模型可能主动发出越权请求
- 是否能设计前置检查 + 运行时检查 + 事后审计的三段式防护
- 是否了解越权的不同类型：水平越权、垂直越权、上下文越权
- 是否有过越权防护的实际工程经验

## 核心结论

越权请求的核心处理原则是**在请求执行前就阻断**，不要依赖执行后的追溯。智能体系统中的越权有三个来源：用户主动的越权（用户要求做权限之外的事）、智能体自主的越权（模型误判自己有权限）、以及注入导致的越权（攻击者通过 prompt 注入让模型执行越权操作）。针对每种来源需要不同的防护策略。

---

## 越权类型全景

```
越权请求类型
    │
    ├── 水平越权 (Horizontal Privilege Escalation)
    │    用户 A 访问用户 B 的数据
    │    例: 普通用户请求查看其他用户的私密文件
    │
    ├── 垂直越权 (Vertical Privilege Escalation)
    │    低权限角色执行高权限操作
    │    例: 访客尝试执行删除操作
    │
    ├── 上下文越权 (Contextual Privilege Escalation)
    │    在当前会话上下文中执行了越界操作
    │    例: 本应操作项目 A 却操作了项目 B
    │
    └── 工具组合越权 (Tool Composition Escalation)
        通过组合低风险工具完成高风险操作
        例: read_file + network_request → 数据外泄
```

## 参考回答

### 三段式防护

```
┌─────────────────────────────────────────────────────────────┐
│ 第一段：前置检查                                              │
│ 请求到达时立即做权限验证                                       │
│                                                             │
│  用户输入 → 意图识别 → 权限匹配 → 允许/拒绝                   │
│                                                             │
│ 第二段：运行时检查                                            │
│ 执行过程中的每次工具调用都要验证                                │
│                                                             │
│  模型选择工具 → 检查权限 → 检查参数 → 执行/阻断                │
│                                                             │
│ 第三段：事后审计                                              │
│ 执行完成后的审计和异常检测                                     │
│                                                             │
│  执行完成 → 记录审计 → 异常检测 → 必要时告警 / 回滚            │
└─────────────────────────────────────────────────────────────┘
```

### 权限检查器

```python
class AuthorizationChecker:
    """
    权限检查器——前置 + 运行时双重检查
    """
    
    def __init__(self):
        self.pre_checks: List[PreCheck] = []
        self.runtime_checks: List[RuntimeCheck] = []
    
    def add_pre_check(self, check: PreCheck):
        """添加前置检查"""
        self.pre_checks.append(check)
    
    def add_runtime_check(self, check: RuntimeCheck):
        """添加运行时检查"""
        self.runtime_checks.append(check)
    
    async def pre_authorize(
        self,
        request: AgentRequest,
        user: User,
        session: Session
    ) -> AuthorizationResult:
        """
        前置检查：请求进入时执行
        """
        # 1. 解析用户意图
        intent = await self._parse_intent(request)
        
        # 2. 逐项检查
        for check in self.pre_checks:
            result = await check.check(intent, user, session)
            if not result.allowed:
                return AuthorizationResult(
                    allowed=False,
                    reason=result.reason,
                    suggestion=result.suggestion,
                    check_point="pre_authorize",
                    intent=intent
                )
        
        return AuthorizationResult(
            allowed=True,
            intent=intent
        )
    
    async def runtime_authorize(
        self,
        tool_call: ToolCall,
        user: User,
        session: Session,
        context: ExecutionContext
    ) -> AuthorizationResult:
        """
        运行时检查：每次工具调用时执行
        """
        for check in self.runtime_checks:
            result = await check.check(tool_call, user, session, context)
            if not result.allowed:
                return AuthorizationResult(
                    allowed=False,
                    reason=result.reason,
                    suggestion=result.suggestion,
                    check_point="runtime",
                    tool_call=tool_call
                )
        
        return AuthorizationResult(allowed=True)
```

### 前置检查实现

```python
class PreChecks:
    """
    前置检查规则集合
    """
    
    @staticmethod
    def role_based_access() -> PreCheck:
        """基于角色的访问检查"""
        async def check(
            intent: Intent,
            user: User,
            session: Session
        ) -> CheckResult:
            required_role = intent.required_role
            if required_role and user.role not in REQUIRED_ROLES[required_role]:
                return CheckResult(
                    allowed=False,
                    reason=f"操作需要 {required_role} 角色，当前用户为 {user.role}",
                    suggestion=f"请联系管理员升级到 {required_role} 角色"
                )
            return CheckResult(allowed=True)
        return check
    
    @staticmethod
    def scope_check() -> PreCheck:
        """操作范围检查"""
        async def check(
            intent: Intent,
            user: User,
            session: Session
        ) -> CheckResult:
            # 检查操作的资源是否在用户的范围内
            resource_scope = intent.resource_scope
            user_scope = session.data_scope
            
            if resource_scope and user_scope:
                if not self._scope_overlap(resource_scope, user_scope):
                    return CheckResult(
                        allowed=False,
                        reason=f"操作目标 {resource_scope} 不在用户范围 {user_scope} 内",
                        suggestion="请确认操作目标是否正确"
                    )
            return CheckResult(allowed=True)
        return check
    
    @staticmethod
    def operation_time_check() -> PreCheck:
        """操作时间检查"""
        async def check(
            intent: Intent,
            user: User,
            session: Session
        ) -> CheckResult:
            if intent.operation_type == "destructive":
                current_hour = datetime.now().hour
                if current_hour < 6 or current_hour > 22:
                    return CheckResult(
                        allowed=False,
                        reason="破坏性操作在非工作时间被限制",
                        suggestion="请在 6:00-22:00 之间执行此操作，或联系管理员"
                    )
            return CheckResult(allowed=True)
        return check
```

### 运行时越权检测

```python
class RuntimeAuthorization:
    """
    运行时授权——实时检测越权操作
    """
    
    def __init__(self):
        self.tool_permission_cache = {}
    
    async def check_tool_call(
        self,
        tool_call: ToolCall,
        session: Session,
        execution_context: ExecutionContext
    ) -> AuthorizationResult:
        """
        检查工具调用是否越权
        """
        # 1. 检查工具权限
        tool_check = await self._check_tool_permission(
            tool_call.name, session
        )
        if not tool_check.allowed:
            return tool_check
        
        # 2. 检查参数权限
        param_check = await self._check_param_permission(
            tool_call.name, tool_call.params, session
        )
        if not param_check.allowed:
            return param_check
        
        # 3. 检查上下文权限
        context_check = await self._check_context_boundary(
            tool_call, execution_context
        )
        if not context_check.allowed:
            return context_check
        
        # 4. 检查组合操作权限（工具链）
        chain_check = await self._check_tool_chain(
            tool_call, session.tool_call_history
        )
        if not chain_check.allowed:
            return chain_check
        
        return AuthorizationResult(allowed=True)
    
    async def _check_context_boundary(
        self,
        tool_call: ToolCall,
        context: ExecutionContext
    ) -> AuthorizationResult:
        """
        检查操作是否在任务上下文边界内
        """
        if context.task_boundary:
            # 检查操作目标是否在任务定义的范围内
            if not self._is_within_boundary(
                tool_call.params,
                context.task_boundary
            ):
                return AuthorizationResult(
                    allowed=False,
                    reason=f"操作超出任务边界: {context.task_boundary.name}",
                    suggestion="请确认此操作是否与当前任务相关"
                )
        return AuthorizationResult(allowed=True)
    
    async def _check_tool_chain(
        self,
        tool_call: ToolCall,
        history: List[ToolCall]
    ) -> AuthorizationResult:
        """
        检查工具链组合是否越权
        防止: read_file 读取敏感文件 → curl 将数据发送到外部
        """
        if not history:
            return AuthorizationResult(allowed=True)
        
        # 检查前后工具的组合风险
        last_call = history[-1]
        
        dangerous_combinations = [
            ("read_file", "network_request"),
            ("read_file", "write_file"),
            ("shell", "network_request"),
            ("database_query", "write_file"),
        ]
        
        for prev_tool, curr_tool in dangerous_combinations:
            if last_call.name == prev_tool and tool_call.name == curr_tool:
                return AuthorizationResult(
                    allowed=False,
                    reason=f"检测到高风险工具组合: {prev_tool} → {curr_tool}",
                    suggestion="此操作序列已被安全策略限制，如需执行请联系管理员",
                    risk_score=0.8
                )
        
        return AuthorizationResult(allowed=True)
```

### 越权请求处理策略

```python
class AuthorizationHandler:
    """
    越权请求处理器——统一的越权处理策略
    """
    
    async def handle_unauthorized(
        self,
        result: AuthorizationResult,
        user: User,
        request: Any
    ) -> Response:
        """
        处理越权请求
        """
        # 记录安全事件
        await self._log_security_event(result, user, request)
        
        # 根据越权严重程度分级处理
        if result.risk_score and result.risk_score > 0.8:
            return await self._handle_critical_violation(result, user)
        elif result.risk_score and result.risk_score > 0.5:
            return await self._handle_moderate_violation(result, user)
        else:
            return await self._handle_minor_violation(result, user)
    
    async def _handle_critical_violation(
        self,
        result: AuthorizationResult,
        user: User
    ) -> Response:
        """
        严重越权：立即阻断 + 告警 + 可能冻结会话
        """
        # 触发安全告警
        await self._trigger_alert(
            level="critical",
            message=f"严重越权尝试: {result.reason}",
            user_id=user.id
        )
        
        # 返回阻断信息
        return Response(
            content="安全策略：此操作已被系统阻止。",
            detail="该操作已被记录并通知安全团队。如有疑问请联系管理员。",
            suggestion="请确认您的操作权限，或联系管理员申请授权。"
        )
    
    async def _handle_moderate_violation(
        self,
        result: AuthorizationResult,
        user: User
    ) -> Response:
        """
        中等越权：拒绝 + 提供替代方案
        """
        alternatives = self._generate_alternatives(result)
        
        response_text = f"无法执行请求的操作。\n\n原因：{result.reason}\n"
        if alternatives:
            response_text += "\n您可以尝试以下替代方案：\n"
            for alt in alternatives:
                response_text += f"- {alt}\n"
        
        if result.suggestion:
            response_text += f"\n建议：{result.suggestion}"
        
        return Response(content=response_text, detail=result.reason)
    
    async def _handle_minor_violation(
        self,
        result: AuthorizationResult,
        user: User
    ) -> Response:
        """
        轻微越权：拒绝 + 给出提示
        """
        return Response(
            content=f"此操作当前不可用。\n\n原因：{result.reason}\n\n您可以在获得授权后重试。",
            detail=result.reason,
            suggestion=result.suggestion
        )
```

### 越权响应策略对比

| 越权级别 | 风险评分 | 处理动作 | 用户体验 | 安全强度 |
|---------|---------|---------|---------|---------|
| 轻微 | 0.0-0.3 | 拒绝 + 提示 | 友好 | 低 |
| 中等 | 0.3-0.6 | 拒绝 + 替代方案 | 中 | 中 |
| 严重 | 0.6-0.8 | 拒绝 + 告警 + 记录 | 差 | 高 |
| 极严重 | 0.8-1.0 | 阻断 + 告警 + 冻结 | 极差 | 极高 |

### 越权实时监控

```python
class AuthorizationMonitor:
    """
    越权实时监控——检测和统计越权模式
    """
    
    def __init__(self):
        self.violation_counts: Dict[str, int] = {}
        self.violation_history: List[ViolationEvent] = []
    
    def record_violation(self, event: ViolationEvent):
        """记录越权事件"""
        self.violation_history.append(event)
        
        # 统计用户级别的越权次数
        key = f"user:{event.user_id}"
        self.violation_counts[key] = self.violation_counts.get(key, 0) + 1
        
        # 检查是否达到告警阈值
        if self.violation_counts[key] >= 3:
            self._trigger_user_alert(event.user_id)
        
        # 统计 IP 级别的越权
        ip_key = f"ip:{event.ip_address}"
        self.violation_counts[ip_key] = self.violation_counts.get(ip_key, 0) + 1
        if self.violation_counts[ip_key] >= 5:
            self._trigger_ip_ban(event.ip_address)
    
    def get_user_violation_count(self, user_id: str) -> int:
        """获取用户的越权次数"""
        return self.violation_counts.get(f"user:{user_id}", 0)
    
    def check_if_blocked(self, user_id: str) -> bool:
        """检查用户是否应被临时封禁"""
        return self.get_user_violation_count(user_id) >= 5
```

## 常见误区

- **越权检测只做一次**：只在请求入口处做了一次权限检查，忽略了运行时模型自主调用的工具可能越权
- **忽略工具组合越权**：单看每个工具调用都在权限范围内，但组合起来实现了越权操作
- **过度暴露拒绝原因**：返回了过多的系统信息（"没有 /etc/passwd 的读取权限"），为攻击者提供信息
- **拒绝后没有替代方案**：只说了"不行"不告诉用户"可以做什么"，用户体验差且模型会反复尝试
- **越权和意图分离**：不结合用户意图判断是否越权，导致合法的跨项目操作被误阻止
- **日志记录不完整**：只记录了被拒绝的越权请求，没记录上下文和原因，事后分析困难

## 追问方向

- 当模型判断失误，将一个合法请求误判为越权时，你的系统如何让用户绕过这个误判？
- 水平越权（用户 A 访问用户 B 的数据）在智能体系统中如何检测？单纯靠角色检查不够
- 你的系统如何处理"边界情况"——一个操作部分越权部分合法？比如批量操作中部分成功部分失败
- 越权检测和成本控制如何联动？高风险操作通常也是高成本操作
- 多智能体协同场景下，一个智能体产生的越权请求如何追溯责任？

## 学习建议

越权防护的关键不是"拒绝请求"本身，而是"正确判断并优雅处理"。先实现前置检查——在请求进入时就判断用户是否有权限做某件事。然后加上运行时检查——每次工具调用都验证是否在权限范围内。最后加上工具链检测——防止通过组合工具实现越权。面试中重点强调"三段式防护"和"分级处理策略"。
