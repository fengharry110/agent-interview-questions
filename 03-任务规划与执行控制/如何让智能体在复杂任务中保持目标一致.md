# 如何让智能体在复杂任务中保持目标一致？

## 面试问题

如何让智能体在复杂任务中保持目标一致？

## 考察重点

- 是否理解目标一致性问题在智能体系统中的具体表现：目标漂移、目标腐蚀、上下文污染
- 是否能从 Codex 和 Claude Code 源码中找到目标保持的具体机制
- 是否能设计可落地的目标检测、恢复和防御方案
- 是否理解"目标一致"和"计划正确"的区别

## 核心结论

目标一致性（Goal Consistency）是智能体系统在复杂任务中的核心挑战。不同于"计划正确性"（计划是否能达成目标），目标一致性关注的是**系统在执行过程中是否始终朝向同一个目标**。主要问题包括目标漂移（gradual drift）、目标腐蚀（goal corruption）和上下文污染。解决方案包括周期性检查点、目标锚定、上下文隔离和降级策略。

## 问题分析：什么是目标一致性问题

### 目标一致性问题在智能体中的表现

| 问题类型 | 表现 | 典型场景 |
|---------|------|---------|
| 目标漂移 | 模型在执行过程中逐渐偏离原始目标 | "帮我重构 auth 模块"执行到一半变成"优化数据库查询" |
| 目标腐蚀 | 上下文被污染，原始目标被覆盖 | 多轮对话后模型忘记了最初的任务 |
| 计划失效 | 生成了正确计划但执行时走偏 | 计划说要读 A 文件，实际读了 B 文件 |
| 资源迷失 | 在次要目标上消耗大量资源 | 花 30 分钟优化一个不重要的函数 |

### 为什么复杂任务更容易出现目标一致性问题

1. **长周期**：复杂任务执行时间长，上下文不断累积，有效信息被稀释
2. **多步骤**：步骤之间可能存在冲突，后面的步骤可能否定前面的
3. **外部干扰**：用户可能在任务执行过程中修改目标
4. **错误累积**：每一步的小误差累积成大偏差

## 目标保持的核心机制

### 机制一：目标锚定（Goal Anchoring）

目标锚定是指在系统的关键位置保存原始目标的引用，执行过程中定期检查当前状态是否与目标一致。

```python
class GoalAnchoring:
    def __init__(self, check_interval: int = 5):
        self.original_goal: str = None
        self.check_interval = check_interval
        self.goal_embedding = None

    def set_goal(self, goal: str):
        self.original_goal = goal
        self.goal_embedding = self.embed(goal)

    def check_drift(self, current_state: ExecutionState) -> DriftResult:
        current_description = current_state.summarize()
        current_embedding = self.embed(current_description)

        # 计算当前状态与原始目标的语义距离
        distance = cosine_distance(self.goal_embedding, current_embedding)

        if distance > self.drift_threshold:
            return DriftResult(
                is_drifted=True,
                distance=distance,
                current_summary=current_description,
                recommended_action="replan"
            )
        return DriftResult(is_drifted=False, distance=distance)

    def get_anchor_context(self) -> str:
        return f"原始目标: {self.original_goal}\n保持目标一致，避免偏离。"
```

### 机制二：检查点与回滚（Checkpoint and Rollback）

在关键步骤后保存执行状态，当检测到目标漂移时可以回滚到上一个检查点。

```python
class CheckpointManager:
    def __init__(self, checkpoint_interval: int = 3):
        self.checkpoints: list[Checkpoint] = []
        self.checkpoint_interval = checkpoint_interval
        self.steps_since_last_checkpoint = 0

    def maybe_checkpoint(self, state: ExecutionState) -> Optional[Checkpoint]:
        self.steps_since_last_checkpoint += 1

        if self.steps_since_last_checkpoint >= self.checkpoint_interval:
            checkpoint = Checkpoint(
                id=uuid4(),
                timestamp=datetime.now(),
                goal=self.extract_goal(state),
                completed_steps=self.extract_completed_steps(state),
                context_snapshot=self压缩_context(state.context),
                tool_results=list(state.recent_results[-5:])
            )
            self.checkpoints.append(checkpoint)
            self.steps_since_last_checkpoint = 0
            return checkpoint
        return None

    def rollback_to(self, checkpoint_id: uuid) -> ExecutionState:
        checkpoint = next(c for c in self.checkpoints if c.id == checkpoint_id)
        return self.restore_from_checkpoint(checkpoint)
```

### 机制三：上下文隔离（Context Isolation）

将原始目标、执行计划和中间结果隔离存储，避免上下文污染。

```python
class ContextIsolation:
    def __init__(self):
        self.goal_store: GoalStore           # 原始目标（长期保留）
        self.plan_store: PlanStore          # 当前计划（定期更新）
        self.execution_store: ExecutionStore # 执行历史（可压缩）
        self.immune_regions: list[str]       # 压缩免疫区

    def build_input_for_model(self, task: Task, additional_context: list[str]) -> list[Message]:
        messages = []

        # 第一位：系统提示（包含目标）
        messages.append(SystemMessage(content=self.goal_store.get_prompt(task.id)))

        # 第二位：Immutable 上下文（压缩时不删除）
        if self.immune_regions:
            messages.append(SystemMessage(content=self.immune_regions[0]))

        # 第三位：近期执行结果
        recent_results = self.execution_store.get_recent(task.id, limit=10)
        messages.extend([ResultMessage(r) for r in recent_results])

        # 第四位：用户补充上下文
        messages.extend([UserMessage(c) for c in additional_context])

        return messages

    def add_immune_region(self, content: str, reason: str):
        self.immune_regions.append(f"[压缩免疫] {content} (原因: {reason})")
```

### 2. 目标状态追踪

Claude Code 在每次循环迭代中追踪目标相关状态：

```typescript
// 目标状态追踪
interface GoalTracking {
  originalGoal: string;          // 原始目标
  currentFocus: string;           // 当前聚焦
  goalAlignmentScore: number;     // 目标对齐分数 (0-1)
  lastGoalCheck: number;         // 上次检查的 turn 数
}

function calculateGoalAlignment(
  currentContext: Context,
  originalGoal: string
): number {
  // 1. 计算上下文中与目标相关的 token 比例
  const goalTokens = extractGoalRelatedTokens(currentContext, originalGoal);
  const totalTokens = countTokens(currentContext);

  // 2. 计算当前步骤描述与目标的语义相似度
  const currentStepSimilarity = semanticSimilarity(
    currentContext.lastStepDescription,
    originalGoal
  );

  // 3. 综合评分
  return (goalTokens / totalTokens) * 0.4 + currentStepSimilarity * 0.6;
}
```

### 3. 目标检查点

Claude Code 的规划模块会定期保存目标检查点：

```python
# plan_manager.py — 目标检查点
class PlanManager:
    def __init__(self):
        self.goal_checkpoints: dict[str, GoalCheckpoint] = {}

    def save_goal_checkpoint(self, task_id: str, state: ExecutionState):
        self.goal_checkpoints[task_id] = GoalCheckpoint(
            goal=state.original_goal,
            plan=self.extract_plan(state),
            completed_steps=self.get_completed_steps(state),
            timestamp=datetime.now(),
            context_hash=self.hash_context(state.context)
        )

    def detect_goal_drift(self, task_id: str, state: ExecutionState) -> bool:
        checkpoint = self.goal_checkpoints.get(task_id)
        if not checkpoint:
            return False

        # 检查上下文哈希是否变化（可能存在上下文污染）
        if self.hash_context(state.context) != checkpoint.context_hash:
            return True

        # 检查当前计划是否与原计划有重大偏离
        current_plan = self.extract_plan(state)
        if self.plan_similarity(current_plan, checkpoint.plan) < 0.5:
            return True

        return False
```

## Codex 的目标保持实现

### 1. update_plan 工具的目标校验

Codex 的 `update_plan` 工具允许模型随时更新计划，但也需要校验更新是否保持了目标一致性：

```rust
// plan.rs — 计划更新校验
pub fn validate_plan_update(
    old_plan: &Plan,
    new_plan: &Plan,
    original_goal: &str
) -> PlanValidationResult {
    let mut issues = Vec::new();

    // 1. 检查新计划是否覆盖了原目标的所有关键点
    let goal_coverage = calculate_goal_coverage(new_plan, original_goal);
    if goal_coverage < 0.8 {
        issues.push(ValidationIssue {
            severity: Severity::High,
            message: format!("计划只覆盖了原目标的 {:.0}%", goal_coverage * 100)
        });
    }

    // 2. 检查计划是否有明显的目标转移
    let first_steps_match = steps_match_intent(&new_plan.steps[0], original_goal);
    if !first_steps_match {
        issues.push(ValidationIssue {
            severity: Severity::Medium,
            message: "计划的第一步似乎与原目标不匹配".to_string()
        });
    }

    // 3. 检查是否有重复或循环步骤
    let duplicates = find_duplicate_steps(&new_plan.steps);
    if !duplicates.is_empty() {
        issues.push(ValidationIssue {
            severity: Severity::Low,
            message: format!("发现重复步骤: {:?}", duplicates)
        });
    }

    PlanValidationResult { valid: issues.is_empty(), issues }
}
```

## 目标一致性评测

### 评测指标

```python
@dataclass
class GoalConsistencyMetrics:
    goal_drift_rate: float          # 目标漂移率：发生漂移的任务 / 总任务
    average_drift_turns: float      # 平均漂移发生轮次
    recovery_rate: float           # 恢复率：漂移后恢复的任务 / 漂移任务
    goal_corruption_rate: float     # 目标腐蚀率：目标被污染的任务 / 总任务
    final_goal_match_rate: float    # 最终目标匹配率：完成时目标一致的任务 / 总任务

class GoalConsistencyEvaluator:
    def evaluate(self, test_tasks: list[TestTask], execution_traces: list[Trace]) -> GoalConsistencyMetrics:
        drifts = []
        corruptions = 0
        recoveries = 0

        for task, trace in zip(test_tasks, execution_traces):
            drift_events = self.detect_drift_events(task, trace)
            drifts.extend(drift_events)

            if self.is_goal_corrupted(task, trace):
                corruptions += 1

            if drift_events and self.was_recovered(drift_events):
                recoveries += 1

        total_tasks = len(test_tasks)
        return GoalConsistencyMetrics(
            goal_drift_rate=len(set(d.events[0].task_id for d in drifts)) / total_tasks,
            average_drift_turns=sum(d.detection_turn for d in drifts) / len(drifts) if drifts else 0,
            recovery_rate=recoveries / len(set(d.task_id for d in drifts)) if drifts else 0,
            goal_corruption_rate=corruptions / total_tasks,
            final_goal_match_rate=self.count_final_match(test_tasks, execution_traces) / total_tasks
        )
```

### 漂移检测方法

```python
class DriftDetector:
    def __init__(self, threshold: float = 0.7):
        self.threshold = threshold

    def detect_drift_events(self, task: TestTask, trace: Trace) -> list[DriftEvent]:
        events = []
        goal_embedding = task.goal_embedding

        for turn in trace.turns:
            # 计算当前上下文与原始目标的相似度
            context_embedding = self.embed(turn.context_summary)
            similarity = cosine_similarity(goal_embedding, context_embedding)

            if similarity < self.threshold:
                events.append(DriftEvent(
                    turn=turn.number,
                    similarity=similarity,
                    context_summary=turn.context_summary,
                    detected_behavior=turn.model_behavior
                ))
        return events

    def detect_gradual_drift(self, task: TestTask, trace: Trace) -> bool:
        # 检测渐进式漂移（每次偏移一点，最终大幅偏离）
        scores = []
        for turn in trace.turns:
            context_embedding = self.embed(turn.context_summary)
            scores.append(cosine_similarity(task.goal_embedding, context_embedding))

        # 检查趋势是否为持续下降
        if len(scores) < 5:
            return False

        # 线性回归检测趋势
        slope = linear_regression_slope(range(len(scores)), scores)
        return slope < -0.1  # 负斜率表示渐进偏离
```

## 工程实践 Checklist

设计目标保持机制时的 Checklist：

- [ ] **目标提取**：任务开始时是否提取并存储了原始目标？
- [ ] **周期性检查**：是否定期检查当前状态与原始目标的一致性？
- [ ] **降级策略**：检测到漂移后是否有降级处理（回滚、重规划、询问用户）？
- [ ] **上下文隔离**：原始目标是否存储在压缩免疫区？
- [ ] **日志记录**：目标变更是否记录到执行轨迹？
- [ ] **用户干预**：用户是否可以中途修改或确认目标？
- [ ] **评测覆盖**：是否有目标一致性的评测指标？

## 真实踩坑案例

### 案例：多轮对话中的目标腐蚀

某团队实现客服智能体时发现：用户最初说要"查询订单"，但经过 10 轮对话后，模型开始处理"退货请求"——用户从未明确提出过这个请求。

**问题分析**：
1. 模型把"查询订单"过程中的中间结果（订单状态：已收货）当作了新目标
2. 上下文中的"订单状态"信息被模型解释为"需要处理的问题"
3. 没有目标锚定机制，模型根据上下文"自由发挥"

**修复方案**：
1. 添加目标锚定层，每次模型输出前检查是否偏离原目标
2. 将"查询订单"和"处理退货"明确标记为不同意图，需要用户确认切换
3. 增加目标一致性评分，低于阈值时要求用户确认
