# 显式规划和隐式规划有什么区别？

## 面试问题

显式规划和隐式规划有什么区别？

## 考察重点

- 是否理解显式规划和隐式规划的机制差异
- 是否能根据场景选择合适的规划方式
- 是否了解两者的优缺点
- 是否有实际应用经验

## 核心结论

**显式规划**是结构化的决策过程（模型输出明确的计划），**隐式规划**是通过模型内在能力实现的（如思维链、推理）。显式规划可解释性强、易于干预；隐式规划更自然、但难以控制。

---

## 显式规划

### 机制

```
用户输入 → 模型推理 → 输出结构化计划 → 执行 → 验证
                     ↑
                计划可见、可干预
```

### 实现

```python
class ExplicitPlanner:
    async def plan(self, task: Task) -> Plan:
        # 模型输出显式的计划结构
        prompt = f"""为以下任务制定执行计划：

        任务：{task.description}

        请输出JSON格式的计划：
        {{
            "steps": [
                {{"id": 1, "action": "...", "expected_outcome": "..."}},
                {{"id": 2, "action": "...", "expected_outcome": "..."}}
            ],
            "dependencies": {{"step_2": ["step_1"]}},
            "contingencies": {{"step_1": {{"failure": "alternative_1"}}}}
        }}"""

        response = await llm.generate(prompt)
        return Plan.parse(response)

    async def execute_plan(self, plan: Plan, executor: Executor) -> Result:
        for step in plan.ordered_steps():
            # 执行前可以检查和干预
            if self.user_should_confirm(step):
                await self.request_confirmation(step)

            result = await executor.execute(step)

            # 执行后验证
            if not self.validate_step(step, result):
                # 处理失败
                await self.handle_failure(step, result, plan)
```

### 特点

| 优点 | 缺点 |
|-----|------|
| 可解释性强 | 规划开销大 |
| 可人工干预 | 依赖模型规划能力 |
| 易于调试 | 可能过度规划 |
| 适合复杂任务 | 灵活性较低 |

## 隐式规划

### 机制

```
用户输入 → 模型内在推理（无显式计划输出）→ 直接执行
                     ↑
                计划隐藏在模型权重中
```

### 实现

```python
class ImplicitPlanner:
    async def execute(self, task: Task, executor: Executor) -> Result:
        # 没有显式的计划输出
        # 模型直接决定下一步做什么

        while not task.is_complete():
            # 模型决定下一步（内部推理）
            action = await model.decide(
                task=task,
                context=executor.get_context()
            )

            # 直接执行
            result = await executor.execute(action)

            # 更新上下文
            executor.update_context(action, result)

        return executor.final_result
```

### 思维链（Chain-of-Thought）

隐式规划的一种形式是思维链推理：

```python
def implicit_plan_with_cot(task: str) -> str:
    prompt = f"""任务：{task}

    思考过程（你的内部推理）：
    1. 理解任务目标...
    2. 分析当前状态...
    3. 决定下一步...

    执行：
    [实际执行内容]

    观察结果：
    [结果]

    下一步决定：
    [继续推理]"""

    return llm.generate(prompt)
```

### 特点

| 优点 | 缺点 |
|-----|------|
| 灵活性高 | 难以干预 |
| 自然流畅 | 难以调试 |
| 开销小 | 不可预测 |
| 适合简单任务 | 无法保证计划完整性 |

---

## 对比总结

| 维度 | 显式规划 | 隐式规划 |
|-----|---------|---------|
| **透明度** | 高（计划可见） | 低（计划隐藏） |
| **控制粒度** | 精细 | 粗粒度 |
| **规划开销** | 高 | 低 |
| **适应性** | 需要重新规划 | 自然适应 |
| **可靠性** | 可验证 | 依赖模型 |
| **适用场景** | 复杂任务 | 简单任务 |
| **用户交互** | 可中途干预 | 难以干预 |
| **调试难度** | 低 | 高 |

## 选择策略

### 用显式规划

```
✓ 任务需要多个步骤
✓ 需要人工审核计划
✓ 计划失败需要明确原因
✓ 需要记录完整执行轨迹
✓ 高风险操作需要确认
```

### 用隐式规划

```
✓ 简单、单一操作
✓ 用户不需要了解计划
✓ 需要快速响应
✓ 任务路径相对固定
✓ 模型能力足够强
```

### 混合使用

```python
class HybridPlanner:
    def __init__(self):
        self.explicit_planner = ExplicitPlanner()
        self.implicit_planner = ImplicitPlanner()

    async def plan(self, task: Task) -> ExecutionMode:
        # 分析任务
        complexity = self.analyze_complexity(task)
        risk_level = self.assess_risk(task)
        needs_audit = self.requires_audit(task)

        # 高风险或需要审计 → 显式规划
        if risk_level == "high" or needs_audit:
            return ExecutionMode.EXPLICIT

        # 低风险且简单 → 隐式规划
        if complexity == "low" and risk_level == "low":
            return ExecutionMode.IMPLICIT

        # 中等复杂度 → 混合
        return ExecutionMode.HYBRID
```

## Claude Code 的实现

Claude Code 结合了两种方式：

```typescript
async function planWithHybrid(task: Task): Promise<Execution> {
  // 1. 高层目标用显式规划
  const highLevelPlan = await model.explicitPlan({
    goal: task.description,
    maxSteps: 10
  });

  // 2. 每个高层步骤内部用隐式规划
  for (const step of highLevelPlan.steps) {
    // 验证步骤前置条件
    if (!await validatePreconditions(step)) {
      // 前置条件不满足，重新显式规划
      highLevelPlan = await model.replan(task);
      break;
    }

    // 执行步骤（内部用思维链）
    await model.executeWithImplicitPlanning(step);

    // 验证步骤结果
    if (!await validateResult(step)) {
      // 结果不符合预期，重新规划
      await model.explicitReplan(task);
    }
  }
}
```

## 面试追问方向

1. **"显式规划的开销有多大？怎么优化？"**

   考察候选人对规划效率的理解。

2. **"隐式规划怎么调试？你怎么知道模型在想什么？"**

   考察候选人对模型可解释性的理解。

3. **"如果用户想看计划但又不想等太久，怎么办？"**

   考察候选人对用户体验的权衡。

   考察候选人对自适应规划的理解。
