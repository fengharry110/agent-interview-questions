# 一次性规划和逐步规划如何选择？

## 面试问题

一次性规划和逐步规划如何选择？

## 考察重点

- 是否理解两种规划方式的本质区别
- 是否能根据任务特点选择规划策略
- 是否了解混合使用的场景
- 是否有处理复杂任务的经验

## 核心结论

一次性规划和逐步规划没有绝对优劣，关键看任务特性：**简单任务**用一次性规划效率高；**复杂任务**用逐步规划更稳定；**混合策略**是常见最佳实践。

---

## 两种规划方式对比

| 维度 | 一次性规划 | 逐步规划 |
|-----|-----------|---------|
| **适用场景** | 简单、明确的任务 | 复杂、不确定的任务 |
| **规划时间** | 前期花费多 | 每步花费少 |
| **执行速度** | 快（不需要频繁规划） | 慢（需要多轮交互） |
| **容错性** | 低（一步错可能全错） | 高（可以随时调整） |
| **资源消耗** | 前置资源多 | 持续资源多 |
| **用户交互** | 少 | 多 |

## 一次性规划（First-Principles）

### 适用场景

```
✓ 任务目标明确
✓ 执行路径清晰
✓ 子任务依赖关系明确
✓ 环境相对稳定
✓ 需要快速执行
```

### 实现示例

```python
class OneShotPlanner:
    def plan(self, task: Task) -> ExecutionPlan:
        # 1. 理解目标
        goal = self.parse_goal(task.user_input)

        # 2. 生成完整计划
        steps = self.decompose_task(goal)

        # 3. 验证计划完整性
        if not self.validate_plan(steps):
            raise PlanningError("无法生成完整计划")

        # 4. 优化执行顺序
        optimized_steps = self.optimize_order(steps)

        return ExecutionPlan(steps=optimized_steps)

    def decompose_task(self, goal: Goal) -> list[Step]:
        """一次性分解整个任务"""
        # LLM 生成完整计划
        prompt = f"""分解以下任务为具体步骤：

        任务：{goal.description}

        要求：
        1. 列出所有必要步骤
        2. 明确每个步骤的前置依赖
        3. 考虑可能的失败点和回退策略

        输出JSON格式的步骤列表"""

        response = llm.generate(prompt)
        return parse_steps(response)
```

## 逐步规划（Step-by-Step）

### 适用场景

```
✓ 任务目标模糊，需要澄清
✓ 环境复杂，需要探索
✓ 任务有多个可能方向
✓ 用户可能随时修改目标
✓ 需要频繁验证中间结果
```

### 实现示例

```python
class StepByStepPlanner:
    def __init__(self):
        self.max_steps = 50

    async def plan_and_execute(
        self,
        task: Task,
        executor: TaskExecutor
    ) -> ExecutionResult:
        steps_completed = 0
        current_plan = []

        while not task.is_complete() and steps_completed < self.max_steps:
            # 1. 基于当前状态规划下一步
            next_step = await self.plan_next_step(
                task=task,
                completed_steps=current_plan
            )

            if not next_step:
                break

            # 2. 执行步骤
            result = await executor.execute(next_step)

            # 3. 观察结果
            observation = await self.observe(result)

            # 4. 检查是否需要重新规划
            if self.should_replan(observation, task):
                current_plan = await self.replan(task, current_plan)
            else:
                current_plan.append(next_step)

            steps_completed += 1

        return ExecutionResult(steps=current_plan)
```

### Claude Code 的逐步规划

Claude Code 使用逐步规划，循环执行：

```typescript
async function agenticLoop(task: Task): Promise<Result> {
  let state = initializeState(task);

  while (!shouldStop(state)) {
    // 1. 规划下一步
    const nextAction = await model.decide({
      task: state.goal,
      context: compressContext(state.history),
      tools: availableTools
    });

    // 2. 执行
    const result = await executeAction(nextAction);

    // 3. 观察
    state.history.push({ action: nextAction, result });

    // 4. 评估是否需要重新规划
    if (result.type === 'unexpected_outcome') {
      state.plan = await model.replan(state);  // 重新规划
    }
  }

  return state.finalResult;
}
```

## 混合策略

### 自适应规划

```python
class AdaptivePlanner:
    def __init__(self):
        self.one_shot_planner = OneShotPlanner()
        self.step_by_step_planner = StepByStepPlanner()
        self.task_analyzer = TaskComplexityAnalyzer()

    async def plan(self, task: Task) -> ExecutionPlan:
        # 1. 分析任务复杂度
        complexity = self.task_analyzer.analyze(task)

        if complexity.level == "simple":
            # 简单任务用一次性规划
            return await self.one_shot_planner.plan(task)

        elif complexity.level == "medium":
            # 中等复杂度：一次性规划 + 逐步执行验证
            plan = await self.one_shot_planner.plan(task)
            return await self.step_by_step_planner.refine_and_execute(plan)

        else:
            # 复杂任务用逐步规划
            return await self.step_by_step_planner.plan_and_execute(task)
```

### 分层混合

```python
class HierarchicalHybridPlanner:
    """
    高层：一次性规划（确定大致方向）
    低层：逐步执行（每步灵活调整）
    """

    async def plan(self, task: Task) -> HybridPlan:
        # 高层：粗粒度计划
        high_level_plan = await self.one_shot_planner.plan_high_level(task)

        # 对每个高层步骤，生成细粒度执行计划
        detailed_steps = []
        for step in high_level_plan.steps:
            if step.complexity == "high":
                # 复杂步骤用逐步规划
                sub_plan = await self.step_by_step_planner.plan(step)
                detailed_steps.extend(sub_plan.steps)
            else:
                # 简单步骤直接执行
                detailed_steps.append(step)

        return HybridPlan(
            high_level=high_level_plan,
            detailed=detailed_steps
        )
```

## 决策框架

```
任务特点 → 选择策略

目标明确 + 路径清晰 + 环境稳定
  → 一次性规划

目标模糊 + 需要探索 + 不确定性高
  → 逐步规划

复杂任务
  → 混合策略（高层一次性，低层逐步）

用户交互多
  → 逐步规划（每步确认）

执行速度要求高
  → 一次性规划（减少交互）
```

## 实践建议

### 1. 从逐步规划开始

```python
# 先用逐步规划验证任务可行性
planner = StepByStepPlanner(max_steps=5)
result = await planner.quick_validate(task)

if result.success_rate > 0.8:
    # 验证通过，用一次性规划加速
    return await one_shot_planner.plan(task)
else:
    # 验证失败，继续逐步规划
    return result.plan
```

### 2. 设置检查点

```python
class CheckpointedPlanner:
    async def plan_with_checkpoints(self, task: Task) -> Plan:
        # 一次性生成完整计划
        full_plan = await self.one_shot_planner.plan(task)

        # 在关键步骤插入检查点
        checkpointed_plan = []
        for i, step in enumerate(full_plan.steps):
            checkpointed_plan.append(step)

            if self.is_critical_step(step):
                checkpointed_plan.append(CheckpointStep(
                    validate_before_proceed=True
                ))

        return Plan(steps=checkpointed_plan)
```
