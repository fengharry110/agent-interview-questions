# 规划错误如何评测和修复？

## 面试问题

规划错误如何评测和修复？

## 考察重点

- 是否理解规划错误的类型和原因
- 是否能设计有效的评测方法
- 是否了解常见的修复策略
- 是否有持续改进规划能力的经验

## 核心结论

规划错误的评测需要**多维度指标** + **真实场景测试**，修复需要**根因分析** + **针对性优化**。核心思路是：评测不是为了惩罚，而是为了持续改进规划能力。

## 规划错误的类型

### 类型一：规划不足

```
问题：模型没有生成足够的步骤来完成目标
症状：
- 计划提前结束
- 目标未完全实现
- 关键步骤缺失
```

### 类型二：规划过度

```
问题：模型生成了不必要的步骤
症状：
- 执行时间过长
- 步骤冗余
- 资源浪费
```

### 类型三：规划错误

```
问题：步骤顺序或内容错误
症状：
- 步骤依赖不满足
- 执行失败
- 结果不符合预期
```

### 类型四：规划不稳定

```
问题：同样的输入产生不同的计划
症状：
- 相同任务多次运行结果不同
- 计划质量波动大
```

## 评测方法

### 离线评测

```python
class PlanningEvaluator:
    def __init__(self):
        self.metrics_calculator = MetricsCalculator()

    def evaluate_plans(
        self,
        test_cases: list[TestCase],
        planner: Planner
    ) -> EvaluationResult:
        results = []

        for case in test_cases:
            plan = planner.plan(case.task)

            # 计算各项指标
            metrics = self.metrics_calculator.calculate(plan, case.expected)

            results.append(PlanEvaluation(
                case_id=case.id,
                plan=plan,
                metrics=metrics,
                errors=self.identify_errors(plan, case.expected)
            ))

        return self.aggregate_results(results)

    def calculate_metrics(self, plan: Plan, expected: ExpectedOutcome) -> Metrics:
        return Metrics(
            # 正确性
            correctness=self.calculate_correctness(plan, expected),

            # 完整性
            completeness=self.calculate_completeness(plan, expected),

            # 效率
            efficiency=self.calculate_efficiency(plan, expected),

            # 稳定性
            stability=self.calculate_stability(plan, expected),

            # 可执行性
            executability=self.calculate_executability(plan)
        )

    def calculate_correctness(self, plan: Plan, expected: ExpectedOutcome) -> float:
        """计算正确性"""
        correct_steps = 0
        total_steps = len(plan.steps)

        for step in plan.steps:
            if step.action in expected.actions:
                if self.is_step_correct(step, expected):
                    correct_steps += 1

        return correct_steps / total_steps if total_steps > 0 else 0
```

class OnlinePlanningEvaluator:

        self.feedback_collector = FeedbackCollector()

    async def evaluate_execution(self, execution: Execution) -> OnlineMetrics:
        # 收集用户反馈
        feedback = await self.feedback_collector.collect(execution.task_id)

        trajectory_analysis = self.analyze_trajectory(execution)

            execution.outcome,

            user_satisfaction=feedback.average_rating,
            efficiency=trajectory_analysis.step_efficiency,
            error_rate=trajectory_analysis.error_rate
        )

### 规划质量评估维度

| 维度 | 指标 | 计算方式 |
|-----|------|---------|
| **正确性** | 步骤准确率 | 正确步骤数 / 总步骤数 |
| **完整性** | 目标达成率 | 达成目标数 / 总任务数 |
| **效率** | 规划长度比 | 实际步数 / 最优步数 |
| **稳定性** | 计划方差 | 相同任务多次计划的一致性 |
| **可执行性** | 执行成功率 | 成功执行数 / 计划步骤数 |

## 错误分析

### 错误分类

```python
class ErrorClassifier:
    def classify_errors(self, evaluation: PlanEvaluation) -> list[ErrorCategory]:
        errors = []

        # 缺失错误
        if evaluation.metrics.completeness < 0.8:
            errors.append(ErrorCategory(
                type="missing_steps",
                severity="high",
                description="计划缺少必要步骤"
            ))

        # 顺序错误
        if self.has_ordering_errors(evaluation):
            errors.append(ErrorCategory(
                type="ordering_error",
                severity="medium",
                description="步骤顺序不正确"
            ))

        # 冗余错误
        if evaluation.metrics.efficiency < 0.7:
            errors.append(ErrorCategory(
                type="redundancy",
                severity="low",
                description="存在冗余步骤"
            ))

        return errors

    def analyze_root_cause(self, error: ErrorCategory) -> RootCause:
        """分析错误根因"""
        if error.type == "missing_steps":
            return self.analyze_missing_steps(error)
        elif error.type == "ordering_error":
            return self.analyze_ordering(error)
        elif error.type == "redundancy":
            return self.analyze_redundancy(error)

        return RootCause(unknown=True)
```

### 错误模式识别

```python
class ErrorPatternRecognizer:
    def __init__(self):
        self.patterns = []

    def learn_patterns(self, error_history: list[PlanningError]) -> list[Pattern]:
        """从错误历史中学习模式"""
        # 聚类相似错误
        clusters = self.cluster_errors(error_history)

        # 提取模式
        patterns = []
        for cluster in clusters:
            pattern = self.extract_pattern(cluster)
            patterns.append(pattern)

        return patterns

    def predict_failure_points(self, plan: Plan) -> list[FailurePoint]:
        """预测计划的潜在失败点"""
        failure_points = []

        for i, step in enumerate(plan.steps):
            # 检查是否有已知错误模式
            if self.matches_known_pattern(step):
                failure_points.append(FailurePoint(
                    step_index=i,
                    risk_level="high",
                    pattern=self.get_matched_pattern(step)
                ))

        return failure_points
```

### 策略一：Prompt 优化

```python
class PromptOptimizer:
    def __init__(self):
        self.llm = LLM()

    def optimize_planning_prompt(
        self,
        error_analysis: ErrorAnalysis
    ) -> str:
        """根据错误分析优化 prompt"""
        base_prompt = load_base_planning_prompt()

        # 添加针对性指导
        if "missing_steps" in error_analysis.error_types:
            base_prompt += "\n\n注意：请确保计划包含所有必要的前置步骤。"

        if "ordering_error" in error_analysis.error_types:
            base_prompt += "\n\n注意：请仔细考虑步骤之间的依赖关系。"

        if "redundancy" in error_analysis.error_types:
            base_prompt += "\n\n注意：请避免不必要的冗余步骤。"

        return base_prompt
```

### 策略二：Few-Shot 示例

```python
class FewShotExampleSelector:
    def select_examples(
        self,
        error_type: str,
        num_examples: int = 3
    ) -> list[PlanningExample]:
        """选择针对特定错误类型的示例"""
        all_examples = self.load_examples()

        # 过滤出解决该错误类型的正面示例
        relevant = [
            ex for ex in all_examples
            if error_type in ex.solved_errors
        ]

        # 选择与当前任务相似的示例
        selected = self.select_similar(relevant, current_task, num_examples)

        return selected

    def build_few_shot_prompt(
        self,
        base_prompt: str,
        examples: list[PlanningExample]
    ) -> str:
        prompt = base_prompt + "\n\n示例：\n"
        for ex in examples:
            prompt += f"\n输入：{ex.input}\n"
            prompt += f"计划：{ex.correct_plan}\n"

        return prompt
```

### 策略三：模型微调

```python
class PlanningFinetuner:
    def __init__(self):
        self.training_data_builder = TrainingDataBuilder()

    def prepare_training_data(
        self,
        error_cases: list[ErrorCase]
    ) -> Dataset:
        """准备训练数据"""
        # 从错误中学习：错误的计划 → 正确的计划
        training_pairs = []

        for case in error_cases:
            if case.corrected_plan:
                training_pairs.append({
                    "input": case.task_description,
                    "incorrect_plan": case.incorrect_plan,
                    "correct_plan": case.corrected_plan,
                    "feedback": case.error_feedback
                })

        return self.training_data_builder.build(training_pairs)

    def finetune(self, dataset: Dataset) -> Model:
        """微调模型"""
        model = load_base_model()

        training_config = TrainingConfig(
            learning_rate=1e-5,
            batch_size=16,
            epochs=3,
            loss_function="planning_loss"
        )

        return model.finetune(dataset, training_config)
```

## 持续改进机制

### 反馈循环

```
收集错误案例
    ↓
分析错误原因
    ↓
制定改进策略
    ↓
实施改进
    ↓
验证效果
    ↓
如果效果不好，重复
```

```python
class ContinuousImprovementLoop:
    def __init__(self):
        self.error_collector = ErrorCollector()
        self.analyzer = ErrorAnalyzer()
        self.optimizer = PromptOptimizer()
        self.evaluator = PlanningEvaluator()

    async def run_improvement_cycle(self) -> ImprovementReport:
        # 1. 收集最近的错误
        recent_errors = self.error_collector.get_recent_errors(days=7)

        if len(recent_errors) < 10:
            return ImprovementReport(status="insufficient_data")

        # 2. 分析错误
        analysis = self.analyzer.analyze(recent_errors)

        # 3. 生成改进建议
        improvements = self.generate_improvements(analysis)

        # 4. 实施最有效的改进
        best_improvement = self.select_best(improvements)
        await self.implement(best_improvement)

        # 5. 验证效果
        new_metrics = await self.evaluator.evaluate()

        return ImprovementReport(
            improvements_made=[best_improvement],
            metrics_before=analysis.baseline_metrics,
            metrics_after=new_metrics,
            status="improved" if new_metrics.better_than(analysis.baseline) else "no_improvement"
        )
```
