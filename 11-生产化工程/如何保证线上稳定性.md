# 如何保证线上稳定性？

## 面试问题

如何保证线上稳定性？

## 考察重点

- 是否理解智能体系统的稳定性挑战与传统 Web 服务不同
- 是否能列出智能体系统特有的故障模式及其应对策略
- 是否了解熔断、降级、限流、重试等稳定性模式的智能体适配方案
- 是否有智能体线上稳定性保障的实际经验

## 核心结论

智能体系统的稳定性保障比传统服务多了一个维度：不仅要保证系统不崩溃，还要保证智能体的行为不失控。核心策略是"三层防护"：**基础设施层**（限流、熔断、容灾）、**智能体层**（循环检测、超时控制、降级策略）、**业务层**（灰度发布、监控告警、人工兜底）。其中智能体层的稳定性是最容易被忽视但最容易出问题的。

## 深度展开

### 三层防护架构

```python
class ThreeLayerDefense:
    """
    三层稳定性防护
    """
    
    def __init__(self):
        self.infrastructure = InfrastructureLayer()
        self.agent_layer = AgentLayerGuard()
        self.business_layer = BusinessLayerGuard()
    
    async def execute_with_guarantee(self, task: Task) -> Result:
        """三层防护下执行任务"""
        # 1. 基础设施检查
        if not self.infrastructure.can_accept():
            return Result.failure("系统负载过高，请稍后重试")
        
        # 2. 智能体层执行（带保护）
        execution = self.agent_layer.create_execution(task)
        
        async for step in execution:
            # 循环检测
            if self.agent_layer.detect_loop(step):
                await self.business_layer.alert("检测到智能体循环")
                return self.agent_layer.recover(execution)
            
            # 超时检查
            if self.agent_layer.is_timeout(execution):
                await self.business_layer.alert("执行超时")
                return self.agent_layer.fallback(execution)
            
            # 上下文检查
            if self.agent_layer.is_context_overflow(execution):
                await self.agent_layer.compress_context(execution)
        
        # 3. 业务层验证
        result = execution.result()
        if not self.business_layer.validate(result):
            await self.business_layer.escalate(result)
        
        return result
```

### 基础设施层

```python
class InfrastructureLayer:
    """
    基础设施稳定性保障
    """
    
    def __init__(self):
        self.rate_limiter = RateLimiter(
            max_rpm=100,
            max_concurrent=20
        )
        self.circuit_breaker = CircuitBreaker(
            failure_threshold=5,
            recovery_timeout=30
        )
        self.bulkhead = Bulkhead(
            max_concurrent_tasks=10
        )
    
    def can_accept(self) -> bool:
        """检查能否接受新请求"""
        return all([
            self.rate_limiter.allow(),
            not self.circuit_breaker.is_open(),
            self.bulkhead.has_capacity()
        ])
```

### 智能体层

```python
class AgentLayerGuard:
    """
    智能体层稳定性保障
    """
    
    def __init__(self):
        self.max_steps = 30
        self.max_tokens = 100_000
        self.max_duration = 300
        self.loop_detector = LoopDetector()
    
    def detect_loop(self, trace: Trace) -> bool:
        return self.loop_detector.detect(trace)
    
    def is_timeout(self, execution: Execution) -> bool:
        return execution.duration > self.max_duration
    
    def is_context_overflow(self, execution: Execution) -> bool:
        return execution.total_tokens > self.max_tokens * 0.8
    
    async def compress_context(self, execution: Execution):
        execution.messages = await self.summarize(
            execution.messages[:5],
            execution.messages[-10:]
        )
    
    def fallback(self, execution: Execution) -> Result:
        return Result(
            status="fallback",
            message="智能体执行遇到问题，已切换为简化模式",
            partial_result=execution.partial_result()
        )
```

### 业务层

```python
class BusinessLayerGuard:
    """
    业务层稳定性保障
    """
    
    def __init__(self):
        self.alert_manager = AlertManager()
    
    async def alert(self, message: str, severity: str = "warning"):
        await self.alert_manager.send(
            message=message,
            severity=severity,
            channel="oncall"
        )
    
    def validate(self, result: Result) -> bool:
        checks = [
            self._check_result_completeness(result),
            self._check_result_safety(result),
            self._check_result_consistency(result)
        ]
        return all(checks)
    
    async def escalate(self, result: Result):
        await self.alert(
            f"智能体自动处理失败，需要人工介入。"
            f"会话ID: {result.session_id}",
            severity="critical"
        )
```

### 故障模式与应对

| 故障模式 | 检测方法 | 自动恢复 | 人工介入条件 |
|---------|---------|---------|------------|
| 无限循环 | 步骤计数器 + 模式匹配 | 中断循环，返回部分结果 | 连续 3 次触发循环 |
| API 超时 | 请求超时检测 | 重试 3 次 | 重试全部失败 |
| 上下文溢出 | Token 计数 | 自动压缩 | 压缩后仍溢出 |
| 工具错误 | 异常捕获 | 重试 + 替代工具 | 关键工具全部不可用 |
| 模型输出异常 | Schema 校验 | 重新生成 | 连续 3 次格式错误 |

        "fallback_rate": {
            "description": "降级触发率",

            "action": "超过 5% 需要排查根因"

            "threshold": 60000,
            "action": "超过 60 秒需要优化"
        }
    }
    
    def check_health(self) -> HealthReport:
        statuses = []
        for metric, config in self.METRICS.items():
            current = self._get_current(metric)
            statuses.append({
                "metric": metric,
                "current": current,
                "threshold": config['threshold'],
                "healthy": current < config['threshold']
            })
        
        return HealthReport(
            overall=all(s['healthy'] for s in statuses),
            details=statuses
        )

### 熔断器实现

```python
class AgentCircuitBreaker:
    """
    智能体专用的熔断器
    """
    
    def __init__(self):
        self.state = "closed"
        self.failures = 0
        self.threshold = 5
        self.last_failure_time = 0
        self.recovery_timeout = 30
    
    def call(self, func):
        if self.state == "open":
            if time.time() - self.last_failure_time > self.recovery_timeout:
                self.state = "half-open"
            else:
                raise CircuitBreakerOpen("熔断器已打开")
        
        try:
            result = func()
            if self.state == "half-open":
                self.state = "closed"
                self.failures = 0
            return result
        except Exception as e:
            self.failures += 1
            self.last_failure_time = time.time()
            if self.failures >= self.threshold:
                self.state = "open"
            raise
```

## 常见误区

- **只关注系统稳定性不关注智能体行为稳定性**：系统没崩但智能体在死循环，对用户来说就是不可用
- **熔断后没有降级方案**：模型 API 挂了智能体就完全不可用。需要准备本地小模型或固定回复作为降级
- **重试策略太激进**：模型 API 返回限流错误后立即重试，反而加重限流。需要用指数退避
- **没有人工兜底机制**：所有自动化保护都失效时，需要能无缝切换到人工处理

## 学习建议

从最基础的限制开始：给智能体加最大步数限制和超时控制。这两条能防止 80% 的稳定性问题。然后逐步增加循环检测、熔断、降级等机制。每加一个保护机制，都要测试它在真实场景下的表现——保护机制本身也可能有 bug。
