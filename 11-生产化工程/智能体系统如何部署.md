# 智能体系统如何部署？

## 面试问题

智能体系统如何部署？需要考虑哪些环境和配置？

## 考察重点

- 是否理解智能体部署和传统后端部署的核心差异
- 是否能说明模型接入层、业务逻辑层、基础设施层的分层部署策略
- 是否关注部署的可复现性、可回滚、可观测
- 是否考虑多模型、多环境、多版本下的部署复杂度

## 核心结论

智能体部署的本质是管理不确定性：模型输出的不确定性、工具执行的不确定性、用户输入的不确定性。部署方案需要同时覆盖模型路由、服务编排、监控告警和快速回滚。

### 部署架构总览

### Docker 部署方案

```dockerfile
# Dockerfile - Agent Runtime
FROM python:3.11-slim

WORKDIR /app

RUN apt-get update && apt-get install -y --no-install-recommends \
    curl ca-certificates && \
    rm -rf /var/lib/apt/lists/*

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

ENV PYTHONUNBUFFERED=1
ENV LOG_LEVEL=INFO
ENV CONFIG_PATH=/app/config/production.yaml

EXPOSE 8080

HEALTHCHECK --interval=30s --timeout=10s --retries=3 \
    CMD curl -f http://localhost:8080/health || exit 1

CMD ["python", "-m", "agent_server"]
```

### CI/CD 流水线

```yaml
# .github/workflows/deploy-agent.yml
name: Deploy Agent

on:
  push:
    branches: [main]
    paths:
      - "agent/**"
      - "Dockerfile"
      - "deploy/**"

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Run unit tests
        run: |
          pip install -r requirements.txt
          pip install -r requirements-dev.txt
          pytest tests/unit/ --cov=agent --cov-report=xml
      - name: Run integration tests
        run: pytest tests/integration/ --timeout=120
      - name: Run evaluate
        run: python scripts/evaluate.py --test-set tests/eval/regression.json

  build:
    needs: test
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v2
      - name: Build and push
        uses: docker/build-push-action@v4
        with:
          push: true
          tags: |
            registry.internal/agent-runtime:${{ github.sha }}
            registry.internal/agent-runtime:latest
          cache-from: type=gha
          cache-to: type=gha

  deploy-staging:
    needs: build
    runs-on: ubuntu-latest
    environment: staging
    steps:
      - name: Deploy to staging
        run: |
          kubectl set image deployment/agent-runtime \
            agent=registry.internal/agent-runtime:${{ github.sha }} \
            -n agent-staging
          kubectl rollout status deployment/agent-runtime \
            -n agent-staging --timeout=5m

  deploy-production:
    needs: deploy-staging
    runs-on: ubuntu-latest
    environment: production
    steps:
      - name: Deploy to production (10% canary)
        run: |
          kubectl set image deployment/agent-runtime-canary \
            agent=registry.internal/agent-runtime:${{ github.sha }} \
            -n agent-prod
      - name: Wait and monitor canary
        run: |
          sleep 300
          python scripts/check-canary-metrics.py \
            --deployment agent-runtime-canary \
            --threshold-error 0.01
      - name: Rollout to full production
        if: success()
        run: |
          kubectl set image deployment/agent-runtime \
            agent=registry.internal/agent-runtime:${{ github.sha }} \
            -n agent-prod
          kubectl rollout status deployment/agent-runtime \
            -n agent-prod --timeout=10m
```

## 深度展开

### 第一层：智能体部署的特殊性

智能体部署和传统后端部署有三个核心差异：

| 维度 | 传统后端 | 智能体系统 |
|------|---------|-----------|
| 输出确定性 | 输入确定，输出确定 | 同一输入可能不同输出 |
| 依赖链路 | 请求-响应，通常无状态 | 多轮工具调用，有状态 |
| 失败模式 | HTTP 500 / 超时 | 模型幻觉、工具失败、循环 |
| 性能瓶颈 | CPU / IO / DB | Token 消耗、模型延迟 |
| 测试验证 | 单元测试覆盖率高 | 需要语义评测 + 回归测试 |

### 第二层：模型路由层部署策略

```python
class ModelGateway:
    def __init__(self, config: dict):
        self.providers = {
            "openai": OpenAIProvider(config["openai"]),
            "claude": AnthropicProvider(config["claude"]),
            "local": LocalProvider(config["local"]),
        }
        self.router = ModelRouter(config["router"])
        self.circuit_breaker = CircuitBreaker(
            failure_threshold=5,
            recovery_timeout=60,
        )

    async def complete(
        self,
        request: CompletionRequest,
    ) -> CompletionResponse:
        with self.circuit_breaker:
            model = self.router.select(request)

            if model == "local" and request.requires_reasoning:
                model = "claude"

            provider = self.providers[model]
            try:
                return await provider.complete(request)
            except RateLimitError:
                return await self._fallback(request, model)
            except TimeoutError:
                return await self._retry_with_backoff(request)

    async def _fallback(
        self,
        request: CompletionRequest,
        failed_model: str,
    ) -> CompletionResponse:
        fallback_order = ["claude", "openai", "local"]
        for model in fallback_order:
            if model != failed_model:
                try:
                    return await self.providers[model].complete(request)
                except Exception:
                    continue
        raise AllProvidersFailed("No available model provider")
```

```python
class EnvironmentManager:
    ENVIRONMENTS = {
        "dev": {
            "replicas": 1,
            "model": "gpt-3.5-turbo",
            "cache_enabled": False,
            "log_level": "DEBUG",
            "eval_on_deploy": False,
        },
        "staging": {
            "replicas": 2,
            "model": "gpt-4o-mini",
            "cache_enabled": True,
            "log_level": "INFO",
            "eval_on_deploy": True,
        },
        "production": {
            "replicas": 5,
            "model": "claude-sonnet-4",
            "cache_enabled": True,
            "log_level": "WARNING",
            "eval_on_deploy": False,
        },
    }

    def deploy_to_environment(
        self,
        version: str,
        env: str,
    ):
        config = self.ENVIRONMENTS[env]
        namespace = f"agent-{env}"

        if config["eval_on_deploy"]:
            eval_result = self.run_regression_eval(version)
            if eval_result.score < 0.85:
                raise DeploymentBlocked(
                    f"Eval score {eval_result.score} below threshold 0.85"
                )

        self.k8s_client.apply(
            self._build_deployment_spec(version, config, namespace)
        )
```

## 工程实践

```python
class AgentDeploymentManager:
    def __init__(self):
        self.health_checker = HealthChecker()
        self.metric_collector = MetricCollector()
        self.version_manager = VersionManager()

    async def deploy(self, version: str, strategy: str = "rolling"):
        pre_checks = await self._run_pre_deployment_checks()
        if not pre_checks.all_passed:
            raise DeploymentFailed(pre_checks.failures)

        if strategy == "rolling":
            await self._rolling_deploy(version)
        elif strategy == "blue_green":
            await self._blue_green_deploy(version)
        elif strategy == "canary":
            await self._canary_deploy(version)

        post_checks = await self._run_post_deployment_checks()
        if not post_checks.all_passed:
            await self._rollback(version)

    async def _rolling_deploy(self, version: str):
        old_replicas = await self.k8s.get_replica_count("agent-runtime")
        for i in range(old_replicas):
            await self.k8s.rolling_update(
                deployment="agent-runtime",
                image=f"agent:{version}",
                max_unavailable=1,
            )
            await self._wait_for_stable(old_replicas)

    async def _blue_green_deploy(self, version: str):
        await self.k8s.create_deployment(
            name="agent-runtime-green",
            image=f"agent:{version}",
            replicas=5,
        )
        await self._wait_for_healthy("agent-runtime-green")

        await self.k8s.update_service(
            name="agent-runtime-svc",
            selector="agent-runtime-green",
        )

        await asyncio.sleep(300)
        await self.k8s.delete_deployment("agent-runtime-blue")

    async def _canary_deploy(self, version: str):
        await self.k8s.create_deployment(
            name="agent-runtime-canary",
            image=f"agent:{version}",
            replicas=1,
        )

        await asyncio.sleep(600)
        metrics = await self.metric_collector.get_canary_metrics()

        thresholds = {
            "error_rate": 0.01,
            "p95_latency": 5000,
            "task_completion_rate": 0.95,
        }

        for metric, threshold in thresholds.items():
            if metrics[metric] > threshold:
                await self._rollback(version)
                return

        await self.k8s.scale_deployment("agent-runtime-canary", replicas=5)
        await self.k8s.update_service(
            "agent-runtime-svc",
            selector="agent-runtime-canary",
        )

    async def _run_pre_deployment_checks(self):
        checks = [
            self.health_checker.check_database(),
            self.health_checker.check_redis(),
            self.health_checker.check_model_api(),
            self.health_checker.check_disk_space(),
        ]
        results = await asyncio.gather(*checks)
        return PreDeploymentResult(results)

    async def _run_post_deployment_checks(self):
        checks = [
            self.health_checker.check_endpoint("/health"),
            self.health_checker.check_endpoint("/ready"),
            self.health_checker.run_test_task("test_simple_query"),
        ]
        results = await asyncio.gather(*checks)
        return PostDeploymentResult(results)

    async def _rollback(self, failed_version: str):
        previous_version = self.version_manager.get_previous(failed_version)
        await self.k8s.set_image(
            "agent-runtime",
            f"agent:{previous_version}",
        )
        await self.k8s.rollout_status("agent-runtime", timeout=300)

        alert_payload = {
            "severity": "critical",
            "message": f"Rollback from {failed_version} to {previous_version}",
        }
        await self.alerting.send(alert_payload)
```

## 常见误区

- **把智能体当无状态服务部署**：智能体通常有对话上下文、记忆状态，需要仔细设计 session 的存储和恢复策略
- **忽略模型 API 的限流和延迟**：模型 API 的限流、延迟抖动远比传统 API 严重，需要做客户端限流和重试
- **健康检查只检查进程存活**：智能体需要检查的核心是模型 API 是否可用、工具链是否通、核心链路是否能跑通
- **一刀切的副本数配置**：不同 Agent 的资源消耗差异很大，按 Agent 类型配置不同的资源规格更合理
- **缺少模型层面的灰度**：只做代码灰度不够，提示词变更和模型切换更要灰度，因为影响面更大
