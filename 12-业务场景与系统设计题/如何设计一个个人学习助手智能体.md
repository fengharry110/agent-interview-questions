# 如何设计一个个人学习助手智能体？

## 面试问题

如何设计一个个人学习助手智能体？

## 考察重点

- 是否理解"个性化学习"和"通用问答"的本质区别
- 是否能说清楚学习助手需要追踪用户的知识状态并自适应调整
- 是否了解间隔重复、知识追踪、自适应学习等技术
- 是否有教育类产品或学习系统的设计经验

## 核心结论

个人学习助手智能体和通用问答机器人的核心区别在于**个性化**和**长期记忆**：它需要知道用户已经掌握了什么、什么还不懂、什么即将遗忘。设计核心是"用户知识模型"——一个持续更新的数据结构，追踪用户对每个知识点的掌握程度。智能体基于这个模型来决定教什么、怎么教、什么时候复习。

```python
class LearningAssistantSystem:
    """
    个人学习助手系统
    """
    
    def __init__(self):
        self.knowledge_tracer = KnowledgeTracer()
        self.lesson_planner = LessonPlanner()
        self.review_scheduler = ReviewScheduler()
        self.content_generator = ContentGenerator()
    
    async def learn(self, user: User, topic: str) -> LearningSession:
        knowledge_state = await self.knowledge_tracer.get_state(user.id, topic)
        
        lesson = await self.lesson_planner.plan(
            user=user, topic=topic, knowledge_state=knowledge_state
        )
        
        content = await self.content_generator.generate(
            user=user, lesson=lesson, learning_style=user.learning_style
        )
        
        session = LearningSession(user=user, content=content, exercises=lesson.exercises)
        await self._process_feedback(session)
        
        return session
```

### 知识追踪

```python
class KnowledgeTracer:
    async def get_state(self, user_id: str, topic: str) -> KnowledgeState:
        knowledge_graph = await self._build_graph(topic)
        
        states = {}
        for concept in knowledge_graph.concepts:
            states[concept.id] = await self._estimate_mastery(
                user_id=user_id, concept=concept
            )
        
        return KnowledgeState(
            user_id=user_id, topic=topic, concept_states=states,
            overall_mastery=mean(s.values()),
            weak_points=[c.id for c, s in states.items() if s.mastery < 0.4],
            ready_for_review=[c.id for c, s in states.items() if s.needs_review]
        )
```

### 自适应学习规划

```python
class LessonPlanner:
    async def plan(self, user: User, topic: str, knowledge_state: KnowledgeState) -> Lesson:
        weak_points = knowledge_state.weak_points
        review_points = knowledge_state.ready_for_review
        
        if review_points and random.random() < 0.3:
            return await self._create_review_lesson(knowledge_state, review_points)
        elif weak_points:
            return await self._create_focus_lesson(topic, weak_points)
        else:
            return await self._create_new_lesson(user, topic, knowledge_state)
```

### 间隔重复

```python
class SpacedRepetitionScheduler:
    def __init__(self):
        self.intervals = [1, 3, 7, 14, 30, 90]
    
    def next_review(self, concept: Concept, performance: float) -> datetime:
        if performance >= 0.9:
            level = min(concept.review_level + 1, len(self.intervals) - 1)
        elif performance >= 0.7:
            level = concept.review_level
        else:
            level = 0
        
        days = self.intervals[level]
        return datetime.now() + timedelta(days=days)
```

```python
class LearningStyleAdapter:
```

## 常见误区

- **把学习助手当成问答机器人**：用户问什么答什么不是学习，学习需要主动规划、引导和复习
- **忽视遗忘曲线**：教一遍就完，不安排复习，用户一周后忘掉 70%
- **内容不个性化**：所有人学同样的内容，不考虑用户当前的知识水平和学习风格
- **缺乏动机激励**：学习是反人性的，智能体需要鼓励、进度反馈和成就感设计

## 追问方向

- 如何判断用户是真的掌握了还是在假装知道（比如背答案）？
- 当用户对某个知识点反复出错时，智能体应该换种方式教还是判断用户不适合学？
- 学习助手的知识图谱怎么构建和维护？谁来更新？

## 学习建议

先不急着做完整的系统。选择一个你熟悉的小知识点（比如 Python 的装饰器），写一个简单的"教学智能体"：它先问用户三个问题评估掌握程度，然后根据结果决定教什么。跑几轮后你会发现用户在交互中暴露的知识盲区往往出乎意料——这就是你改进知识模型的基础。
