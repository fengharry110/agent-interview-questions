# 如何设计一个销售跟进智能体

## 面试问题

设计一个销售跟进智能体，能够自动筛查潜在客户、安排跟进计划、发送个性化沟通内容、并辅助销售完成商机推进。你会如何设计这个系统？

## 考察重点

- 是否理解销售流程的漏斗模型（Awareness → Interest → Decision → Action）和每个阶段的智能体策略
- 是否能说清楚销售跟进智能体和营销自动化的区别：个性化 vs 批量、关系维护 vs 触达
- 是否了解如何平衡自动化效率和人脉关系维护的温度
- 是否有实际的 CRM 系统集成经验，理解销售业务的数据模型

## 核心结论

销售跟进智能体的本质是一个"关系管理系统"而非"触达系统"，核心不是发更多消息，而是在**正确的时间用正确的方式与正确的人沟通**，核心能力是客户意向预测、个性化沟通生成和跟进节奏优化。

```typescript
class LeadScorer {
  private model: ScoringModel;

  async score(lead: LeadProfile): Promise<LeadScore> {
    // 结合规则和模型评分
    const ruleScore = this.ruleBasedScore(lead);
    const modelScore = await this.model.predict(lead);

    return {
      totalScore: (ruleScore * 0.3 + modelScore * 0.7),
      factors: {
        engagement: modelScore.factors.engagement,
        fit: modelScore.factors.fit,     // 与目标客户画像匹配度
        intent: modelScore.factors.intent, // 购买意向
        timing: modelScore.factors.timing  // 近期购买可能性
      },
      updatedAt: new Date()
    };
  }

  private ruleBasedScore(lead: LeadProfile): number {
    let score = 50;  // 基础分
    if (lead.source === 'referral') score += 15;
    if (lead.interactions.length > 5) score += 10;
    if (lead.stage === 'negotiation') score += 20;
    // 负面信号
    if (lead.tags.includes('unsubscribed')) score -= 50;
    if (lead.tags.includes('bounced')) score -= 20;
    return Math.max(0, Math.min(100, score));
  }
}
```

#### 2. 跟进策略引擎

```typescript
interface FollowUpPlan {
  leadId: string;
  actions: FollowUpAction[];
  priority: 'high' | 'medium' | 'low';
  reason: string;
}

interface FollowUpAction {
  type: 'email' | 'call' | 'sms' | 'social_message' | 'meeting';
  scheduledAt: Date;
  contentTemplate: string;
  personalizationData: Record<string, string>;
  condition?: string;  // 触发条件
}

class FollowUpStrategyEngine {
  private rules: FollowUpRule[];

  constructor() {
    this.rules = [
      { name: 'new_lead_greeting', condition: 'stage==new', action: 'email_within_24h' },
      { name: 'after_demo', condition: 'last_action==demo', action: 'follow_up_email_48h' },
      { name: 'no_response_7d', condition: 'last_contact>7d && no_response', action: 'reengagement_sms' },
      { name: 'high_score_call', condition: 'score>80 && stage!=negotiation', action: 'schedule_call' }
    ];
  }

  async generatePlan(lead: LeadProfile): Promise<FollowUpPlan> {
    // 1. 匹配规则
    const matchedRules = this.rules.filter(r =>
      this.evaluateCondition(r.condition, lead)
    );

    // 2. 生成具体行动
    const actions: FollowUpAction[] = [];
    for (const rule of matchedRules) {
      const action = await this.createAction(rule, lead);
      if (action) actions.push(action);
    }

    // 3. 排序和去重
    return {
      leadId: lead.id,
      actions: this.prioritizeActions(actions),
      priority: this.calculatePriority(lead),
      reason: this.generateReason(lead, matchedRules)
    };
  }

  async smartSchedule(
    actions: FollowUpAction[],
    preferences: ContactPreference
  ): Promise<FollowUpAction[]> {
    // 智能排程：考虑最佳联系时间
    return actions.map(action => {
      const bestSlot = this.findBestTimeSlot(preferences.bestTime, action.type);
      return {
        ...action,
        scheduledAt: bestSlot ?? action.scheduledAt
      };
    });
  }

  private findBestTimeSlot(
    slots: TimeSlot[],
    actionType: string
  ): Date | null {
    // 根据历史数据找到该类动作的最佳时间
    const channelPatterns = {
      email: { bestDays: [2, 3, 4], bestHour: 10 }, // 周二到周四 10am
      call: { bestDays: [2, 3, 4], bestHour: 14 },   // 周二到周四 2pm
      sms: { bestDays: [1, 2, 3, 4, 5], bestHour: 11 } // 工作日 11am
    };

    const pattern = channelPatterns[actionType];
    if (!pattern) return null;

    // 找到下一个符合条件的日期时间
    return this.findNextSlot(pattern);
  }
}
```

#### 4. 互动响应分析

```typescript
interface InteractionResult {
  type: 'email_open' | 'email_click' | 'email_reply'
      | 'call_answered' | 'call_no_answer'
      | 'link_visit' | 'meeting_booked';
  timestamp: Date;
  details: Record<string, any>;
}

class InteractionAnalyzer {
  async processInteraction(
    result: InteractionResult,
    lead: LeadProfile
  ): Promise<LeadProfile> {
    // 更新客户画像和意向评分
    const updated = await this.updateLeadFromInteraction(lead, result);

    // 分析意图信号
    const signals = await this.extractSignals(result, updated);

    // 更新跟进计划
    if (signals.length > 0) {
      updated.plan = await this.adjustFollowUpPlan(updated, signals);
    }

    return updated;
  }

  private async extractSignals(
    result: InteractionResult,
    lead: LeadProfile
  ): Promise<IntentSignal[]> {
    const signals: IntentSignal[] = [];

    // 正面信号
    if (result.type === 'email_reply') {
      const sentiment = await this.analyzeSentiment(result.details.replyContent);
      if (sentiment === 'positive') {
        signals.push({ type: 'positive_reply', strength: 0.7 });
      }
      // 提取关键词（预算、时间、决策人）
      const keywords = this.matchBuyingKeywords(result.details.replyContent);
      signals.push(...keywords.map(k => ({
        type: 'buying_signal',
        keyword: k,
        strength: 0.5
      })));
    }

    // 负面信号
    if (result.type === 'email_reply' && result.details.replyContent.includes('unsubscribe')) {
      signals.push({ type: 'unsubscribe_request', strength: 0.9 });
    }

    return signals;
  }

  private async adjustFollowUpPlan(
    lead: LeadProfile,
    signals: IntentSignal[]
  ): Promise<FollowUpPlan> {
    if (signals.some(s => s.type === 'unsubscribe_request')) {
      return {
        leadId: lead.id,
        actions: [],
        priority: 'low',
        reason: '客户请求取消订阅'
      };
    }

    if (signals.some(s => s.type === 'buying_signal')) {
      // 加速跟进：从培育转向推进
      return {
        leadId: lead.id,
        actions: [
          {
            type: 'call',
            scheduledAt: this.nextBusinessHour(),
            contentTemplate: 'proposal_call',
            personalizationData: {}
          }
        ],
        priority: 'high',
        reason: '检测到购买信号'
      };
    }

    return lead.plan;  // 维持现有计划
  }

  private analyzeSentiment(text: string): Promise<'positive' | 'negative' | 'neutral'> {
    return this.model.analyze(`分析以下回复的情感倾向: ${text}`, {
      categories: ['positive', 'negative', 'neutral']
    });
  }
}
```

```typescript
class MultiChannelCoordinator {
  // 防骚扰：相同内容不通过多个渠道发送
  async coordinate(plan: FollowUpPlan, history: Interaction[]): Promise<FollowUpPlan> {
    const lastChannels = history.slice(-3).map(h => h.channel);
    const usedContent = history.slice(-5).map(h => h.contentId);

    return {
      ...plan,
      actions: plan.actions
        .filter(a => !lastChannels.includes(a.type as any))  // 避免同渠道连续
        .filter(a => !usedContent.includes(a.contentTemplate))  // 避免重复内容
    };
  }
}
```

### 2. 合规与隐私

```typescript
class ComplianceManager {
  private readonly CONSENT_REQUIRED = true;
  private readonly OPT_OUT_WINDOW = 30;  // days

  async checkCompliance(
    lead: LeadProfile,
    action: FollowUpAction
  ): Promise<ComplianceResult> {
    // 1. 检查是否同意接收
    if (!lead.consentGiven && this.CONSENT_REQUIRED) {
      return { allowed: false, reason: 'no_consent' };
    }

    // 2. 检查退订状态
    if (lead.tags.includes('unsubscribed')) {
      return { allowed: false, reason: 'unsubscribed' };
    }

    // 3. 检查退订时间窗口
    const unsubscribedAt = lead.tags.find(t => t.startsWith('unsubscribed_at'));
    if (unsubscribedAt) {
      const date = new Date(unsubscribedAt.split(':')[1]);
      if (Date.now() - date.getTime() < this.OPT_OUT_WINDOW * 24 * 60 * 60 * 1000) {
        return { allowed: false, reason: 'opt_out_window' };
      }
    }

    // 4. 频率限制
    const recentContacts = lead.interactions.filter(i =>
      Date.now() - i.timestamp.getTime() < 7 * 24 * 60 * 60 * 1000
    );
    if (recentContacts.length > 3) {
      return { allowed: false, reason: 'frequency_limit' };
    }

    return { allowed: true };
  }
}
```

## 常见误区

### 误区 1：把所有客户同等对待

所有客户都走相同的跟进流程是最大的误区。高意向客户需要加速推进，低意向客户需要培育，垃圾线索需要快速过滤。不分层会导致资源浪费和机会错失。

### 误区 2：过度自动化破坏关系

每次互动都想让模型写一封"完美"的个性化邮件，但客户能感觉到这是自动生成的。真正的个性化不在于措辞，而在于在正确的时间用正确的话题触达。

### 误区 3：只关注发送不关注接收

系统设计时往往只关注"怎么发消息"，但忽略了对客户回复的响应。客户回复后 24 小时内必须有合理响应，否则之前的跟进全部白费。

### 误区 4：忽略负面信号的及时处理

客户说"不要再联系了"就应该立即停止跟进而不是继续发消息。系统要有立即响应负面信号并更新的机制。

## 追问方向

1. **"如何设计销售智能体的'学习'能力——它如何从成功的跟进中学习并改进策略？"**

    建立"赢单复盘"流程：将赢单和输单的跟进序列做对比分析，找出高频获胜路径（比如某些话术、某些时间节点更有效），然后自动调整跟进策略。

2. **"如何处理多个销售员共享客户池的情况？如何避免竞争和冲突？"**

    通过客户锁定机制（谁先联系谁拥有）、自动分配算法（按技能匹配、按负载均衡）、以及冲突检测（两个智能体同时联系同一个客户）来管理。

3. **"如果客户的回应超出了智能体的处理范围（如要求谈价格、要求见高层），如何升级？"**

    根据回应内容自动判断需要人工介入的节点。设定升级规则（如价格讨论、合同条款、投诉等自动标记为"需人工处理"），同时整理对话摘要辅助销售快速接手。

4. **"如何衡量销售智能体的 ROI？是帮销售更多地沟通，还是帮销售更聪明地沟通？"**

    核心指标应该是"有效触达率"（有回复的触达/总触达）和"销售时间节省"（销售花在处理重复任务上的时间减少）。好的智能体不是让销售发更多消息，而是让销售把时间花在最有价值的事情上。
