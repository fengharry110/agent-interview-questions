# 如何设计一个招聘筛选智能体？

## 面试问题

如何设计一个招聘筛选智能体？

## 考察重点

- 是否理解招聘筛选的核心流程（JD 分析→简历筛选→面试评估→决策支持）
- 是否能说清楚智能体能做哪些、哪些必须由人决定
- 是否了解招聘场景的特殊约束（公平性、偏见避免、合规性）
- 是否有招聘系统或 HR 系统的设计经验

## 核心结论

招聘筛选智能体的核心价值是提升效率而非替代决策：智能体负责简历初筛、技能匹配、面试问题生成等重复性工作，但最终的录用决策必须由人做出。最关键的设计考量是**公平性和偏见避免**——智能体必须被持续审计，确保不会因性别、年龄、地域等因素产生系统性偏见。

## 参考回答

### 系统架构

```python
class RecruitmentAgentSystem:
    """
    招聘筛选智能体系统
    """
    
    def __init__(self):
        self.jd_analyzer = JDAnalyzer()
        self.resume_screener = ResumeScreener()
        self.interview_prep = InterviewPreparer()
        self.bias_detector = BiasDetector()
    
    async def process_position(self, position: Position) -> HiringPipeline:
        jd_analysis = await self.jd_analyzer.analyze(position)
        
        pipeline = HiringPipeline(position_id=position.id)
        for resume in position.received_resumes:
            score = await self.resume_screener.screen(
                resume=resume,
                jd_analysis=jd_analysis
            )
            pipeline.add_candidate(resume, score)
        
        bias_report = await self.bias_detector.audit(pipeline)
        if bias_report.has_bias:
            pipeline = await self._adjust_for_fairness(pipeline, bias_report)
        
        shortlisted = pipeline.get_shortlisted()
        for candidate in shortlisted:
            interview_kit = await self.interview_prep.prepare(
                candidate=candidate,
                jd_analysis=jd_analysis
            )
            candidate.interview_kit = interview_kit
        
        return pipeline
```

### JD 分析器

```python
class JDAnalyzer:
    async def analyze(self, position: Position) -> JDAnalysis:
        prompt = f"""
        职位：{position.title}
        JD 描述：{position.description}
        
        请提取：
        1. 必备技能（硬性要求）
        2. 加分技能（软性要求）
        3. 经验要求
        4. 学历要求
        5. 隐性要求
        
        对每个要求标注重要性：must / preferred / plus
        """
        response = await self.llm.complete(prompt)
        return self._parse_analysis(response)
```

## 深度展开

### 简历筛选

```python
class ResumeScreener:
    SCORING_DIMENSIONS = [
        "skill_match",
        "experience_match",
        "education_match",
        "achievement_score",
        "culture_fit",
    ]
    
    async def screen(self, resume: Resume, jd_analysis: JDAnalysis) -> CandidateScore:
        parsed = await self._parse_resume(resume)
        
        scores = {}
        for dimension in self.SCORING_DIMENSIONS:
            scores[dimension] = await self._score_dimension(
                parsed, jd_analysis, dimension
            )
        
        total = sum(scores.values())
        bias_flags = await self._check_bias(parsed, scores)
        
        return CandidateScore(
            candidate_id=resume.candidate_id,
            total_score=total,
            dimension_scores=scores,
            bias_flags=bias_flags,
            recommendation=self._recommendation(total, bias_flags)
        )
```

### 偏见检测

```python
class BiasDetector:
    """
    偏见检测——招聘智能体最关键的安全机制
    """
    
    BIAS_TYPES = ["gender_bias", "age_bias", "region_bias", "school_bias", "name_bias"]
    
    async def audit(self, pipeline: HiringPipeline) -> BiasReport:
        findings = []
        for bias_type in self.BIAS_TYPES:
            detector = self._get_detector(bias_type)
            result = await detector.check(pipeline)
            if result.has_bias:
                findings.append(BiasFinding(
                    type=bias_type,
                    severity=result.severity,
                    description=result.description
                ))
        
        distribution = self._check_distribution(pipeline)
        if distribution.imbalanced:
            findings.append(BiasFinding(
                type="distribution_bias",
                severity="high",
                description=f"某群体通过率显著偏低: {distribution.detail}"
            ))
        
        return BiasReport(has_bias=len(findings) > 0, findings=findings)
```

```python
class InterviewPreparer:
    async def prepare(self, candidate: Candidate, jd_analysis: JDAnalysis) -> InterviewKit:
        summary = await self._summarize_resume(candidate.resume)
        gaps = self._analyze_gaps(candidate.skills, jd_analysis.required_skills)
        questions = await self._generate_questions(candidate=candidate, jd=jd_analysis, gaps=gaps)
        
        return InterviewKit(
            candidate_summary=summary,
            skill_gaps=gaps,
            suggested_questions=questions,
            red_flags=candidate.bias_flags
        )
```

## 常见误区

- **让智能体做最终决策**：录用决策涉及复杂的人为判断，智能体只能提供参考评分
- **忽略偏见问题**：模型可能从训练数据中继承了偏见，需要持续监控和校正
- **只看技能匹配不看文化匹配**：技术最强的候选人不一定是最合适的
- **JD 关键词硬匹配**：候选人可能换过行业但能力可迁移，死板的关键词匹配会漏掉优质候选人

## 追问方向

- 如何确保智能体不会因为候选人的性别、年龄、地域而产生评分偏差？
- 如果候选人的简历是用 AI 优化的（含大量关键词），如何防止评分虚高？
- 如果招聘智能体筛掉了一个后来在其他公司表现极好的候选人，怎么发现和改进？

## 学习建议

先从小规模开始：让智能体辅助筛选 50 份简历，人工复核所有结果。重点不是看智能体筛得对不对，而是看是否有系统性偏见。确认无偏见后再逐步扩大自动化范围。公平性审计应该是招聘智能体最优先的功能。
