# 如何设计一个企业知识库智能体

## 面试问题

设计一个企业知识库智能体，员工可以用自然语言查询企业内部文档、规范、知识，支持问答、交叉引用和多轮追问。你会如何设计这个系统？

## 考察重点

- 是否理解企业知识库的复杂性：不只是 RAG，还需要权限控制、版本管理、多源融合
- 是否能说清楚检索增强生成在企业场景下的特殊挑战：引用可信度、数据新鲜度、知识冲突
- 是否了解企业知识管理的方法论：知识分类、知识图谱、知识生命周期
- 是否有实际的企业知识库系统设计经验，包括文档解析和权限集成

## 核心结论

企业知识库智能体不只是"百科问答"，而是一个**面向企业的知识服务平台**，核心挑战是：在权限约束下，从多源异构的知识中检索出最相关、最新、最可信的信息，并给出可追溯的答案。

#### 1. 知识接入与处理 Pipeline

```typescript
interface KnowledgeDocument {
  id: string;
  title: string;
  content: string;
  source: 'wiki' | 'confluence' | 'sharepoint' | 'code_repo' | 'local_file';
  metadata: {
    author: string;
    createdAt: Date;
    updatedAt: Date;
    version: number;
    tags: string[];
    department: string[];
    accessLevel: 'public' | 'internal' | 'confidential' | 'restricted';
  };
  chunks: DocumentChunk[];
}

interface DocumentChunk {
  id: string;
  content: string;
  embedding: number[];
  position: number;  // 在文档中的位置
  headings: string[];  // 章节路径
  metadata: Partial<KnowledgeDocument['metadata']>;
}

class KnowledgeIngestionPipeline {
  private parser: DocumentParser;
  private chunker: DocumentChunker;
  private embedder: EmbeddingService;
  private indexer: VectorIndex;

  async processDocument(doc: RawDocument): Promise<KnowledgeDocument> {
    // 1. 格式解析
    const parsed = await this.parser.parse(doc, doc.format);

    // 2. 智能分块
    const chunks = await this.chunker.chunk(parsed, {
      strategy: 'semantic',  // 按语义边界切分
      maxChunkSize: 512,
      overlapTokens: 50,
      // 保持标题层级结构
      preserveStructure: true
    });

    // 3. 生成嵌入
    const embeddings = await this.embedder.embed(chunks.map(c => c.content));

    // 4. 写入索引（带元数据过滤）
    const docChunks = chunks.map((chunk, i) => ({
      ...chunk,
      embedding: embeddings[i],
      metadata: { ...parsed.metadata }
    }));

    await this.indexer.index(docChunks, parsed.metadata);

    return { ...parsed, chunks: docChunks };
  }
}
```

#### 2. 智能分块策略

```typescript
class DocumentChunker {
  chunk(parsed: ParsedDocument, config: ChunkConfig): DocumentChunk[] {
    switch (config.strategy) {
      case 'semantic':
        return this.semanticChunk(parsed, config);
      case 'fixed_size':
        return this.fixedSizeChunk(parsed, config);
      case 'recursive':
        return this.recursiveChunk(parsed, config);
    }
  }

  private semanticChunk(
    parsed: ParsedDocument,
    config: ChunkConfig
  ): DocumentChunk[] {
    // 按 Markdown 标题、代码块、段落等语义边界切分
    const segments = parsed.segments
      .filter(s => s.type !== 'empty')
      .reduce((chunks: Segment[], segment) => {
        const last = chunks[chunks.length - 1];
        if (!last || this.shouldSplit(last, segment, config)) {
          chunks.push({ segments: [segment] });
        } else {
          last.segments.push(segment);
        }
        return chunks;
      }, []);

    // 确保每个 chunk 不超过最大长度
    return segments.flatMap(s => this.ensureMaxSize(s, config));
  }

  private shouldSplit(
    current: SegmentGroup,
    next: Segment,
    config: ChunkConfig
  ): boolean {
    // 在标题层级变化时分割
    if (next.type === 'heading' && next.level <= 2) return true;
    // 在代码块开始/结束时分割
    if (next.type === 'code_block' && current.segments.some(s => s.type === 'code_block')) return true;
    // 长度超限时分割
    if (current.totalLength > config.maxChunkSize) return true;
    return false;
  }
}
```

| 分块策略 | 适用文档 | 优点 | 缺点 |
|---------|---------|------|------|
| 固定大小 | 日志、代码 | 简单可控 | 可能切断语义 |
| 语义分块 | 技术文档、规范 | 保持语义完整性 | 块大小不均 |
| 递归分块 | 混合内容 | 灵活适应内容 | 实现复杂 |
| 按结构分块 | 官网、手册 | 结构清晰 | 依赖文档格式 |

#### 3. 混合检索

```typescript
interface SearchRequest {
  query: string;
  filters: {
    departments?: string[];
    tags?: string[];
    dateRange?: { from: Date; to: Date };
    source?: string[];
  };
  userPermissions: string[];
  topK: number;
}

interface SearchResult {
  chunkId: string;
  content: string;
  score: number;
  source: string;
  metadata: Record<string, any>;
  documentTitle: string;
  highlightRanges: [number, number][];
}

class HybridSearcher {
  private vectorDB: VectorIndex;
  private fullTextIndex: FullTextIndex;
  private reranker: Reranker;

  async search(request: SearchRequest): Promise<SearchResult[]> {
    // 并行执行多种检索
    const [vectorResults, keywordResults, graphResults] = await Promise.all([
      this.vectorSearch(request),
      this.keywordSearch(request),
      this.graphSearch(request),
    ]);

    // 融合排序
    const fused = this.fuseResults([vectorResults, keywordResults, graphResults]);

    // 重排
    const reranked = await this.reranker.rerank(fused, request.query);

    // 权限过滤
    return this.filterByPermissions(reranked, request.userPermissions);
  }

  private async vectorSearch(request: SearchRequest): Promise<SearchResult[]> {
    const queryEmbedding = await this.embedder.embed(request.query);
    return this.vectorDB.search(queryEmbedding, {
      topK: request.topK * 2,
      filters: request.filters
    });
  }

  private fuseResults(results: SearchResult[][]): SearchResult[] {
    // Reciprocal Rank Fusion
    const scores = new Map<string, number>();
    for (const list of results) {
      for (let i = 0; i < list.length; i++) {
        const id = list[i].chunkId;
        scores.set(id, (scores.get(id) || 0) + 1 / (60 + i));
      }
    }

    // 按融合分数排序
    return Array.from(scores.entries())
      .sort((a, b) => b[1] - a[1])
      .slice(0, 20)
      .map(([id]) => this.findById(id));
  }
}
```

#### 4. 权限过滤层

```typescript
class PermissionFilter {
  private permissionMatrix: Map<string, Set<string>>;
  // userRole -> 可访问的 accessLevel 集合

  async filterByPermissions(
    results: SearchResult[],
    userPermissions: string[]
  ): Promise<SearchResult[]> {
    // 1. 检查每个结果的访问级别
    const accessible = results.filter(r => {
      const requiredLevel = r.metadata.accessLevel;
      return userPermissions.includes(requiredLevel)
        || userPermissions.includes('admin');
    });

    // 2. 确保结果数量充足
    if (accessible.length < 3) {
      // 记录权限不足的查询（用于审计）
      await this.auditLog.record({
        userId: userPermissions[0],
        query: results.map(r => r.metadata.documentTitle),
        blockedCount: results.length - accessible.length
      });
    }

    return accessible;
  }
}
```

```typescript
interface AnswerWithCitations {
  answer: string;
  citations: Citation[];
  confidence: number;
  disclaimers: string[];
}

class AnswerGenerator {
  async generate(
    query: string,
    context: SearchResult[],
    history: Message[]
  ): Promise<AnswerWithCitations> {
    const prompt = `
你是一个企业知识库助手。基于以下信息回答用户问题。

## 检索到的参考资料
${context.map((doc, i) => `
### [${i + 1}] ${doc.documentTitle}
${doc.content}
来源: ${doc.source}
更新时间: ${doc.metadata.updatedAt}
`).join('\n')}

## 用户问题
${query}

## 对话历史
${history.slice(-5).map(m => `${m.role}: ${m.content}`).join('\n')}

## 要求
- 仅使用参考资料中的信息回答
- 对每个关键信息标注引用编号 [1][2]
- 如果资料不足，明确说明"参考资料中没有相关信息"
- 标注不同资料之间的冲突（如果有）
- 输出格式：答案正文 + 参考文献列表
`;

    const response = await this.model.generate(prompt, {
      temperature: 0.1,  // 低温度保证事实性
      maxTokens: 2000
    });

    return this.parseResponse(response, context);
  }

  private parseResponse(
    raw: string,
    context: SearchResult[]
  ): AnswerWithCitations {
    // 提取引用的文档编号
    const citationPattern = /\[(\d+)\]/g;
    const citedDocs = new Set<number>();
    let match;

    while ((match = citationPattern.exec(raw)) !== null) {
      citedDocs.add(parseInt(match[1]));
    }

    return {
      answer: raw,
      citations: Array.from(citedDocs).map(i => ({
        index: i,
        title: context[i - 1]?.documentTitle ?? 'unknown',
        source: context[i - 1]?.source ?? 'unknown',
        url: context[i - 1]?.metadata?.url
      })),
      confidence: citedDocs.size > 0 ? 0.8 : 0.3,
      disclaimers: this.generateDisclaimers(context)
    };
  }
}
```

### 知识冲突处理

```typescript
class ConflictDetector {
  async detectConflicts(context: SearchResult[]): Promise<Conflict[]> {
    // 检测不同文档中的矛盾信息
    const conflicts: Conflict[] = [];

    // 按主题聚类
    const clusters = this.clusterByTopic(context);

    for (const cluster of clusters) {
      // 提取每个文档中的断言
      const claims = await this.extractClaims(cluster);

      // 检测矛盾的断言
      for (const [topic, assertions] of claims.entries()) {
        const contradictory = this.findContradictions(assertions);
        if (contradictory.length > 0) {
          conflicts.push({
            topic,
            documents: contradictory.map(c => ({
              id: c.docId,
              version: c.version,
              updatedAt: c.updatedAt
            })),
            description: `${topic} 在不同文档中有矛盾: ${contradictory.map(c => c.claim).join(' vs ')}`
          });
        }
      }
    }

    return conflicts;
  }
}
```

### 知识新鲜度管理

```typescript
class FreshnessManager {
  async checkFreshness(
    doc: KnowledgeDocument,
    query: string
  ): Promise<FreshnessStatus> {
    // 1. 检查文档年龄
    const age = Date.now() - doc.metadata.updatedAt.getTime();
    if (age > this.getMaxAge(doc.source)) {
      return { status: 'stale', action: 'reindex' };
    }

    // 2. 检查引用文档的更新
    if (doc.metadata.references) {
      const refsUpdated = await this.checkReferencesFreshness(doc.metadata.references);
      if (refsUpdated.some(r => r.newerVersion)) {
        return { status: 'needs_update', action: 'notify_author' };
      }
    }

    // 3. 检测知识空白 - 用户频繁问但没有文档覆盖的问题
    const gapScore = await this.calculateGapScore(query);
    if (gapScore > 0.8) {
      return { status: 'knowledge_gap', action: 'create_document' };
    }

    return { status: 'fresh', action: 'none' };
  }

  private getMaxAge(source: string): number {
    const maxAgeBySource: Record<string, number> = {
      'wiki': 90 * 24 * 60 * 60 * 1000,  // 90 天
      'policy': 30 * 24 * 60 * 60 * 1000, // 30 天
      'code_repo': 7 * 24 * 60 * 60 * 1000 // 7 天
    };
    return maxAgeBySource[source] ?? 365 * 24 * 60 * 60 * 1000;
  }
}
```

```typescript
class DocSyncService {
  async syncFromConfluence(space: string): Promise<SyncResult> {
    // 增量同步：只更新变更的文档
    const recentChanges = await this.confluenceAPI.getChanges(space, {
      since: this.lastSyncTime
    });

    for (const change of recentChanges) {
      if (change.type === 'deleted') {
        await this.indexer.delete(change.id);
      } else {
        const doc = await this.confluenceAPI.getDocument(change.id);
        await this.ingestion.processDocument(doc);
      }
    }

    this.lastSyncTime = new Date();
    return {
      updated: recentChanges.filter(c => c.type === 'updated').length,
      deleted: recentChanges.filter(c => c.type === 'deleted').length,
      new: recentChanges.filter(c => c.type === 'created').length
    };
  }
}
```

## 常见误区

### 误区 1：把所有文档无差别地放入知识库

企业文档质量参差不齐。旧版文档、废弃的规范、个人笔记都会降低检索质量。需要建立文档质量筛选机制，只有经过审核的文档才进入知识库。

### 误区 2：认为向量检索可以解决所有搜索问题

对于精确匹配（如版本号、订单号、代码变量名），关键词/全文检索比向量检索更准确。混合检索（向量 + 关键词 + 知识图谱）效果远好于单一检索方式。

### 误区 3：忽视权限对搜索体验的影响

权限过滤可能导致某些用户搜索不到应该存在的信息。但更糟糕的是，用户可能不知道"有答案但自己看不到"，导致以为知识库没有该内容。应该在无结果时提示"你可能没有查看相关文档的权限"。

### 误区 4：知识库和业务系统的文档割裂

员工在知识库中看到的信息和业务系统中看到的实际数据可能不一致。需要建立"知识"和"数据"的连接，例如在流程文档中嵌入到实际系统的链接或数据快照。

## 追问方向

1. **"如何确保模型在回答时严格引用知识库而不自己编造？"**

    设计多层约束：第一，在 prompt 中明确要求只基于参考资料回答；第二，对模型输出做引用校验，检测回答中的事实性陈述是否都有引用标注；第三，在展示时把引用高亮显示，让用户可以点开验证。

2. **"如何处理不同文档中对同一个问题的矛盾表述？"**

    检测矛盾后，根据文档的权威性（官方文档 > 团队 Wiki > 个人笔记）和时效性（新版 > 旧版）做排序，在回答中同时标注矛盾信息并说明系统倾向哪个版本。

3. **"员工提问'我的项目文档在哪里'，系统需要知道用户是谁以及有哪些项目。这类个性化检索如何实现？"**

    需要集成身份认证系统，在检索时自动注入用户相关的上下文过滤器。例如根据用户所属部门、参与项目自动添加过滤条件，实现"同一个问题不同人看到不同答案"。

4. **"如何量化评估企业知识库智能体的效果？"**

    核心指标包括：检索命中率（用户问题能否检索到相关文档）、回答采纳率（用户是否采纳智能体的答案）、知识查找时间节省（对比使用前后查询时长）。辅助指标：知识库覆盖率（未被覆盖的问题比例）。
