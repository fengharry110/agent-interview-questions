# 如何设计一个浏览器操作智能体

## 面试问题

设计一个浏览器操作智能体，用户通过自然语言指令控制浏览器完成网页操作，如填写表单、抓取数据、执行操作流程。你会如何设计这个系统？

## 考察重点

- 是否理解浏览器自动化的核心技术：DOM 操作、事件模拟、页面状态同步
- 是否能说清楚"视觉理解"和"DOM 理解"两种路线的优劣，以及如何结合
- 是否了解浏览器操作的安全风险：CSRF、数据泄露、账户安全
- 是否有实际的浏览器自动化项目经验（Playwright/Puppeteer/Selenium）

## 核心结论

浏览器操作智能体的本质是一个"视觉-结构-语义"三层对齐系统：先把眼睛（视觉截图）和骨架（DOM 树）对齐，再用大模型做语义理解，最后通过 Playwright 精确操作。

#### 1. 页面理解层

```typescript
interface PageState {
  url: string;
  title: string;
  screenshot: Buffer;
  domTree: DOMNode;
  interactiveElements: InteractiveElement[];
  elementMap: Map<string, ElementInfo>;  // accessibilityId -> element
}

interface InteractiveElement {
  id: string;           // CSS 选择器或 XPath
  type: 'button' | 'input' | 'link' | 'select' | 'checkbox'
      | 'radio' | 'textarea' | 'dropdown';
  text: string;
  attributes: Record<string, string>;
  boundingBox: { x: number; y: number; width: number; height: number };
  isVisible: boolean;
  isEnabled: boolean;
  isChecked?: boolean;
  value?: string;
  label: string;         // 可访问性标签
  ariaLabel?: string;
}

class PageUnderstandingEngine {
  private domParser: DOMParser;
  private visionAnalyzer: VisionAnalyzer;

  async understandPage(page: Page): Promise<PageState> {
    // 1. 获取 DOM 树
    const domTree = await this.domParser.parseDOM(page);

    // 2. 提取可交互元素
    const interactiveElements = await this.extractInteractiveElements(domTree);

    // 3. 截图
    const screenshot = await page.screenshot({ fullPage: false });

    // 4. 视觉分析：截图 → 元素定位（辅助 DOM 定位）
    const visionElements = await this.visionAnalyzer.analyze(screenshot);

    // 5. DOM 和视觉结果对齐
    const aligned = this.alignDOMAndVision(interactiveElements, visionElements);

    return {
      url: page.url(),
      title: await page.title(),
      screenshot,
      domTree,
      interactiveElements: aligned,
      elementMap: this.buildElementMap(aligned)
    };
  }

  private async extractInteractiveElements(
    domNode: DOMNode
  ): Promise<InteractiveElement[]> {
    const elements: InteractiveElement[] = [];

    // 递归遍历 DOM 树，找出所有可交互元素
    const traverse = (node: DOMNode) => {
      const tag = node.tagName?.toLowerCase();
      const role = node.attributes?.role;

      if (this.isInteractive(tag, role)) {
        elements.push({
          id: this.generateSelector(node),
          type: this.mapElementType(tag, role),
          text: node.text?.trim() ?? '',
          attributes: node.attributes,
          boundingBox: node.boundingBox,
          isVisible: this.isElementVisible(node),
          isEnabled: !node.attributes?.disabled,
          label: node.attributes?.['aria-label']
            ?? node.attributes?.label
            ?? node.text?.trim()
            ?? ''
        });
      }

      for (const child of node.children ?? []) {
        traverse(child);
      }
    };

    traverse(domNode);
    return elements;
  }

  private isInteractive(tag: string, role?: string): boolean {
    const interactiveTags = ['a', 'button', 'input', 'select',
      'textarea', 'checkbox', 'radio'];
    const interactiveRoles = ['button', 'link', 'combobox',
      'textbox', 'option', 'checkbox', 'radio', 'tab'];
    return interactiveTags.includes(tag) || interactiveRoles.includes(role ?? '');
  }
}
```

#### 2. 元素选择器生成

```typescript
class SelectorGenerator {
  generate(element: InteractiveElement, state: PageState): string[] {
    // 生成多个备选选择器，按优先级排序
    const selectors: string[] = [];

    // 1. 通过 aria-label 或 text
    if (element.ariaLabel) {
      selectors.push(`[aria-label="${element.ariaLabel}"]`);
    }
    if (element.label && element.type === 'link') {
      selectors.push(`text="${element.label}"`);
    }

    // 2. 通过元素类型和文本
    if (element.text) {
      selectors.push(`${element.type}:has-text("${element.text}")`);
    }

    // 3. 通过 role 和 name
    selectors.push(`role=${element.type}[name="${element.label}"]`);

    // 4. 通过 ID
    if (element.attributes.id) {
      selectors.push(`#${element.attributes.id}`);
    }

    // 5. 通过唯一文本匹配（Playwright 文本选择器）
    if (element.text && element.text.length > 2) {
      selectors.push(`text="${element.text}"`);
    }

    return selectors;
  }

  async trySelectors(page: Page, selectors: string[]): Promise<string | null> {
    for (const selector of selectors) {
      try {
        const el = await page.$(selector);
        if (el) return selector;
      } catch {
        continue;
      }
    }
    return null;
  }
}
```

```typescript
class BrowserActionDecider {
  private model: LLM;

  async decideNextAction(context: DecisionContext): Promise<Action> {
    const prompt = this.buildDecisionPrompt(context);
    const response = await this.model.generate(prompt, {
      temperature: 0.2,
      format: 'json'

    const action = JSON.parse(response);
```

#### 4. 安全沙箱层

```typescript
interface BrowserPermission {
  allowedDomains: string[];
  maxActions: number;
  allowedActions: ActionType[];
  sensitiveDataProtection: boolean;
  humanConfirmActions: ActionType[];
}

class BrowserSecuritySandbox {
  private permission: BrowserPermission;
  private actionCount = 0;
  private visitedUrls: string[] = [];

  async checkAction(action: Action, pageState: PageState): Promise<SecurityVerdict> {
    // 1. 操作次数限制
    this.actionCount++;
    if (this.actionCount > this.permission.maxActions) {
      return { allowed: false, reason: 'Max actions exceeded' };
    }

    // 2. URL 白名单
    const url = new URL(pageState.url);
    if (!this.permission.allowedDomains.some(d => url.hostname.endsWith(d))) {
      return { allowed: false, reason: `Domain not allowed: ${url.hostname}` };
    }

    // 3. 敏感操作确认
    if (this.permission.humanConfirmActions.includes(action.type)) {
      return { allowed: true, requiresHumanConfirm: true };
    }

    // 4. 防止操作敏感元素
    if (action.selector) {
      const elementInfo = pageState.elementMap.get(action.selector);
      if (elementInfo && this.isSensitiveElement(elementInfo)) {
        return { allowed: false, reason: 'Sensitive element detected' };
      }
    }

    return { allowed: true };
  }

  private isSensitiveElement(element: InteractiveElement): boolean {
    const sensitivePatterns = [
      /password/i, /secret/i, /token/i, /credit/i,
      /ssn/i, /social/i, /security/i
    ];
    return sensitivePatterns.some(p =>
      p.test(element.label) || p.test(element.attributes?.name ?? '')
    );
  }

  async maskSensitiveData(page: Page): Promise<void> {
    // 在截图中遮挡敏感信息
    const sensitiveSelectors = [
      'input[type="password"]',
      '[data-sensitive="true"]',
      '.pii-field'
    ];

    for (const selector of sensitiveSelectors) {
      const elements = await page.$$(selector);
      for (const el of elements) {
        await el.evaluate((node: HTMLElement) => {
          node.style.filter = 'blur(5px)';
        });
      }
    }
  }
}
```

#### 5. 执行与确认层

```typescript
class BrowserExecutor {
  private page: Page;
  private confirm: HumanConfirmCallback;

  async execute(action: Action): Promise<ActionResult> {
    let result: ActionResult;

    switch (action.type) {
      case 'navigate':
        result = await this.executeNavigate(action.value!);
        break;
      case 'click':
        result = await this.executeClick(action.selector!);
        break;
      case 'type':
        result = await this.executeType(action.selector!, action.value!);
        break;
      case 'select':
        result = await this.executeSelect(action.selector!, action.value!);
        break;
      case 'scroll':
        result = await this.executeScroll(action.value);
        break;
      case 'extract':
        result = await this.executeExtract(action.selector);
        break;
      case 'wait':
        await this.page.waitForTimeout(2000);
        result = { success: true, message: 'Waited 2s' };
        break;
    }

    // 确认操作是否生效
    const confirmation = await this.confirmAction(result, action);
    return { ...result, confirmed: confirmation };
  }

  private async executeClick(selector: string): Promise<ActionResult> {
    try {
      const element = await this.waitForElement(selector);
      await element.click();
      await this.page.waitForLoadState('networkidle');
      return { success: true, message: 'Clicked successfully' };
    } catch (error) {
      // 如果 CSS 选择器失败，尝试文本选择或坐标点击
      return this.fallbackClick(selector);
    }
  }

  private async waitForElement(
    selector: string,
    timeout = 5000
  ): Promise<ElementHandle> {
    return this.page.waitForSelector(selector, {
      state: 'visible',
      timeout
    });
  }

  private async fallbackClick(selector: string): Promise<ActionResult> {
    // 用文本匹配尝试
    try {
      await this.page.click(`text="${selector}"`);
      return { success: true, message: 'Clicked via text selector' };
    } catch {
      return { success: false, error: `Cannot find element: ${selector}` };
    }
  }

  private async confirmAction(
    result: ActionResult,
    action: Action
  ): Promise<boolean> {
    // 截图比对
    const screenshot = await this.page.screenshot();
    // 检查页面是否按预期变化
    const verification = await this.verifyActionEffect(action, screenshot);
    return verification;
  }
}
```

### 元素定位策略对比

| 策略 | 准确率 | 鲁棒性 | 速度 | 适用场景 |
|-----|-------|--------|------|---------|
| CSS 选择器 | 高 | 低 | 快 | 静态页面 |
| XPath | 高 | 低 | 中 | 复杂 DOM 结构 |
| 文本匹配 | 中 | 中 | 快 | 文案固定的元素 |
| 视觉定位 | 低 | 高 | 慢 | DOM 难以定位时 |
| Playwright 语义选择器 | 高 | 中 | 中 | 推荐首选 |
| AI 视觉 + DOM 融合 | 最高 | 最高 | 最慢 | 终极方案 |

```typescript
// 路线一：纯 DOM 操作（Playwright 原生）
class DOMOnlyStrategy {
  async findElement(instruction: string, page: Page): Promise<ElementHandle> {
    // 将自然语言转为 DOM 查询
    const selector = await this.nl2Selector(instruction);
    return page.$(selector);
  }
}

// 路线二：截图 + 视觉理解
class VisionOnlyStrategy {
  async findElement(instruction: string, page: Page): Promise<ElementHandle> {
    const screenshot = await page.screenshot();
    const coordinates = await this.visionModel.locateElement(instruction, screenshot);
    // 将坐标映射到 DOM 元素
    return page.evaluateHandle(({ x, y }) => {
      return document.elementFromPoint(x, y);
    }, coordinates);
  }
}

// 路线三：DOM + 视觉融合（推荐）
class HybridStrategy {
  async findElement(instruction: string, page: Page): Promise<ElementHandle> {
    // 1. 先用 DOM 获取候选元素
    const domCandidates = await this.getDOMCandidates(instruction, page);

    // 2. 用视觉定位缩小范围
    const screenshot = await page.screenshot();
    const visualTargets = await this.visionModel.locateAreas(instruction, screenshot);

    // 3. 求交集
    const matched = this.intersect(domCandidates, visualTargets);
    return matched.length > 0 ? matched[0] : domCandidates[0];
  }
}
```

### 多步操作的错误恢复

```typescript
class ErrorRecovery {
  async recover(
    failedAction: Action,
    context: DecisionContext,
    attemptCount: number
  ): Promise<RecoveryResult> {
    if (attemptCount >= 3) {
      return { strategy: 'abort', message: 'Max retries exceeded' };
    }

    // 恢复策略集
    const strategies = [
      // 策略 1: 刷新页面重试
      async () => {
        await this.page.reload();
        return this.execute(failedAction);
      },
      // 策略 2: 用替代选择器
      async () => {
        const altSelector = this.generateAltSelector(failedAction.selector!);
        return this.execute({ ...failedAction, selector: altSelector });
      },
      // 策略 3: 滚动到目标位置
      async () => {
        await this.page.evaluate((sel: string) => {
          document.querySelector(sel)?.scrollIntoView();
        }, failedAction.selector);
        return this.execute(failedAction);
      },
      // 策略 4: 等待更长时间
      async () => {
        await this.page.waitForTimeout(3000);
        return this.execute(failedAction);
      }
    ];

    for (const strategy of strategies) {
      try {
        const result = await strategy();
        if (result.success) {
          return { strategy: 'recovered', result };
        }
      } catch {
        continue;
      }
    }

    return { strategy: 'abort', message: 'All recovery strategies failed' };
  }
}
```

```typescript
class BrowserPoolManager {
```

### 3. 视觉锚点定位

```typescript
class VisualAnchorLocator {
  async locateByVisual(
    instruction: string,
    screenshot: Buffer,
    domElements: InteractiveElement[]
  ): Promise<InteractiveElement | null> {
    // 用视觉模型理解截图，找到指令对应的视觉区域
    const visionResult = await this.visionModel.detect(screenshot, instruction);

    // 找到该视觉区域对应的 DOM 元素
    for (const el of domElements) {
      if (this.overlap(visionResult.bbox, el.boundingBox)) {
        return el;
      }
    }

    return null;
  }

  private overlap(
    a: { x: number; y: number; width: number; height: number },
    b: { x: number; y: number; width: number; height: number }
  ): boolean {
    const overlapX = Math.max(0,
      Math.min(a.x + a.width, b.x + b.width) - Math.max(a.x, b.x));
    const overlapY = Math.max(0,
      Math.min(a.y + a.height, b.y + b.height) - Math.max(a.y, b.y));
    const overlapArea = overlapX * overlapY;
    const minArea = Math.min(a.width * a.height, b.width * b.height);
    return overlapArea / minArea > 0.5;
  }
}
```

## 常见误区

### 误区 1：完全依赖视觉定位

纯截图+视觉模型定位速度慢、精度不稳定，在元素重叠、动态内容时容易失败。DOM 定位更快更准，视觉定位只应该作为 DOM 定位的补充。

### 误区 2：忽略页面的动态加载

很多页面内容是通过 AJAX/WebSocket 动态加载的。操作前需要等待特定元素出现，而不是等待固定时间。使用 Playwright 的 `waitForSelector` 和 `waitForResponse` 更可靠。

### 误区 3：不做操作前后的状态确认

执行"点击"操作后，需要确认页面是否真的发生了变化。缺少确认机制时，操作可能被忽略（如按钮被禁用）而智能体仍以为操作成功了。

### 误区 4：允许操作敏感页面

智能体应该限制访问的域名范围，对登录、支付、修改密码等敏感操作必须要求人工确认。否则一次 prompt injection 就可能导致严重的安全问题。

## 追问方向

1. **"如果目标网页使用了 Shadow DOM，你的智能体如何处理？"**

    Shadow DOM 中的元素对常规 DOM 查询不可见。需要通过 Playwright 的 `page.evaluate` 直接访问 shadowRoot，或者使用 `>>` 穿透选择器。设计时需要在 DOM 解析层统一处理常规 DOM 和 Shadow DOM。

2. **"如何处理验证码、弹窗通知、Cookie 同意对话框等干扰？"**

    模式识别：预定义常见弹窗类型的关闭策略（如点击"同意"、"关闭"、"X"按钮）。在执行用户任务前，先做一次页面"清洁"，自动关闭所有干扰弹窗。

3. **"如果页面使用了 canvas 渲染而非标准 DOM 元素，如何操作？"**

    Canvas 渲染的应用（如游戏、绘图工具、数据可视化）无法通过 DOM 操作。需要通过坐标定位 + 鼠标事件模拟来实现。这种情况下，视觉理解是唯一可行的方案。

4. **"多个浏览器标签页之间的操作如何协调？"**

    需要维护标签页的上下文关系。例如"在搜索结果页打开新标签页查看详情，然后回到原标签页"。设计一个标签页管理器来追踪标签页的父子关系和切换路径。
