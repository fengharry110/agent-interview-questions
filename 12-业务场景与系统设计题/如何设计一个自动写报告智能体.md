# 如何设计一个自动写报告智能体

## 面试问题

设计一个自动写报告智能体，能够根据用户提供的数据、文档和要求，自动生成结构完整、内容准确的分析报告。你会如何设计这个系统？

## 考察重点

- 是否理解长文档生成的特殊挑战：篇章结构一致性、信息准确性、避免重复
- 是否能说清楚报告智能体和通用文本生成的区别：报告需要结构化、可验证、可迭代
- 是否了解如何控制报告质量：事实校验、引用追溯、格式规范
- 是否有实际的项目经验，能说清楚多章节长报告生成的 Pipeline

## 核心结论

自动写报告智能体的核心不是"一次生成整篇报告"，而是**分层构建**——先搭结构（大纲），再填充内容（逐段写），最后精修（风格统一）。关键能力是"写什么"（内容规划）和"不写什么"（信息过滤）。

#### 1. 需求分析与大纲生成

```typescript
interface ReportRequirement {
  title: string;
  purpose: string;       // 报告目的
  audience: 'executive' | 'technical' | 'general' | 'client';
  tone: 'formal' | 'semi_formal' | 'casual';
  maxLength: number;
  sections: SectionRequirement[];
  dataSources: string[];
  template?: string;
  brandVoice?: string;   // 企业品牌语调
}

interface Outline {
  title: string;
  sections: Section[];
  version: number;
  metadata: {
    generatedAt: Date;
    estimatedLength: number;
  };
}

interface Section {
  id: string;
  title: string;
  level: number;        // 1: 一级标题, 2: 二级标题
  description: string;  // 本节要传达的核心信息
  requiredData: string[];
  estimatedLength: 'short' | 'medium' | 'long';
  subsections?: Section[];
}

class OutlineGenerator {
  async generateOutline(requirement: ReportRequirement): Promise<Outline> {
    const prompt = `
生成报告大纲。

## 报告信息
标题: ${requirement.title}
目的: ${requirement.purpose}
受众: ${requirement.audience}
语调: ${requirement.tone}
字数: ${requirement.maxLength}

## 大纲要求
- 逻辑递进：背景 → 分析 → 结论 → 建议
- 每个章节有明确的核心信息
- 标注每个章节需要的数据
- 平衡各章节长度

## 输出格式
JSON 格式的大纲，包含层级结构。
`;

    const result = await this.model.generate(prompt, {
      temperature: 0.3,
      format: 'json'
    });

    // 大纲校验
    return this.validateOutline(JSON.parse(result));
  }

  private validateOutline(outline: Outline): Outline {
    // 检查是否有足够的章节
    if (outline.sections.length < 2) {
      throw new Error('Outline must have at least 2 sections');
    }

    // 检查章节深度
    const maxDepth = this.maxSectionDepth(outline);
    if (maxDepth > 3) {
      throw new Error('Outline depth exceeds maximum (3 levels)');
    }

    return outline;
  }
}
```

```typescript
class DataCollector {

  private knowledgeBase: RAGEngine;

    outline: Outline,
    userData: RawData

    const data = new Map<string, SectionData>();

      data.set(section.id, sectionData);

      sections: data,
      globalData: await this.collectGlobalData(outline, userData),
      sources: this.buildSourceList(data)
    };
  }

    section: Section,
    userData: RawData

    const [facts, figures, knowledgeItems] = await Promise.all([

        topK: 5,
        filters: section.requiredData
      })
    ]);

    return {
      sectionId: section.id,
      facts,
      figures,
      quotes: [],
      charts: await this.generateCharts(figures, section),
      keyFindings: await this.extractFindings(facts, figures)
    };
  }
}
```

#### 3. 逐节生成引擎

```typescript
interface SectionContent {
  sectionId: string;
  content: string;
  citations: Citation[];
  generatedAt: Date;
  wordCount: number;
  qualityScore: number;
}

class SectionWriter {
  private static readonly MAX_RETRIES = 3;

  async writeSection(
    section: Section,
    data: SectionData,
    previouslyWrittenSections: SectionContent[],
    reportRequirement: ReportRequirement
  ): Promise<SectionContent> {
    let attempt = 0;

    while (attempt < SectionWriter.MAX_RETRIES) {
      try {
        const content = await this.generateWithPrompt(
          section, data, previouslyWrittenSections, reportRequirement
        );

        // 校验
        const validation = await this.validateSection(content, data, section);
        if (validation.passed) {
          return content;
        }

        attempt++;
        // 基于校验结果修复
        await this.fixSection(content, validation);
      } catch {
        attempt++;
      }
    }

    throw new Error(`Failed to write section: ${section.id} after ${SectionWriter.MAX_RETRIES} attempts`);
  }

  private async generateWithPrompt(
    section: Section,
    data: SectionData,
    previous: SectionContent[],
    requirement: ReportRequirement
  ): Promise<SectionContent> {
    const prompt = this.buildSectionPrompt(section, data, previous, requirement);
    const rawContent = await this.model.generate(prompt, {
      temperature: 0.4,
      maxTokens: this.estimateTokens(section.estimatedLength)
    });

    return {
      sectionId: section.id,
      content: rawContent,
      citations: this.extractCitations(rawContent, data),
      generatedAt: new Date(),
      wordCount: this.countWords(rawContent),
      qualityScore: 0  // 稍后计算
    };
  }

  private buildSectionPrompt(
    section: Section,
    data: SectionData,
    previous: SectionContent[],
    requirement: ReportRequirement
  ): string {
    return `
## 任务
撰写报告章节。

## 章节信息
标题: ${section.title}
层级: ${section.level}
核心信息: ${section.description}
目标字数: ${section.estimatedLength}

## 可用数据
事实:
${data.facts.map(f => `- ${f.text}${f.source ? ` [src: ${f.source}]` : ''}`).join('\n')}

关键数据:
${data.figures.map(f => `- ${f.label}: ${f.value}`).join('\n')}

图表: ${data.charts.map(c => `[Chart: ${c.type}, ${c.title}]`).join(', ')}

## 上下文
上一章: ${previous[previous.length - 1]?.sectionId ?? '无'}
报告语调: ${requirement.tone}
受众: ${requirement.audience}

## 写作要求
- 仅使用提供的数据，不要编造
- 每个关键数据点标注引用
- 与上一章节自然过渡
- 段落长度 3-5 句
- 语言：中文
- 避免：
  * 重复前面的内容
  * 过度使用形容词
  * 未经数据支持的结论
`;
  }
}
```

```typescript
class ReportIntegrator {
  async integrate(
    sections: SectionContent[],
    outline: Outline,
    requirement: ReportRequirement
  ): Promise<DraftReport> {
    // 1. 按章节顺序排列
    const ordered = this.orderSections(sections, outline);

    // 2. 添加过渡段落
    const withTransitions = await this.addTransitions(ordered);

    // 3. 统一风格和格式
    const unified = await this.unifyStyle(withTransitions, requirement);

    // 4. 生成目录
    const toc = this.generateTOC(outline);

    // 5. 添加前言和摘要
    const executiveSummary = await this.generateExecutiveSummary(unified, requirement);
    const introduction = await this.generateIntroduction(unified, requirement);

    return {
      title: outline.title,
      executiveSummary,
      introduction,
      tableOfContents: toc,
      sections: unified,
      citations: this.collectAllCitations(sections),
      appendices: [],
      generatedAt: new Date(),
      wordCount: this.totalWordCount(unified)
    };
  }

  private async unifyStyle(
    sections: SectionContent[],
    requirement: ReportRequirement
  ): Promise<SectionContent[]> {
    // 统一语气、术语、数字格式、引用格式
    const styleGuide = this.getStyleGuide(requirement);

    return Promise.all(sections.map(async (section, index) => {
      const unified = await this.model.generate(`
统一以下文本的风格。

要求:
- 语调: ${requirement.tone}
- 数字格式: ${styleGuide.numberFormat}
- 术语规范: ${styleGuide.terminology}
- 引用格式: [数字]

文本:
${section.content}

请保持内容不变，只调整风格:
      `, { temperature: 0.1 });

      return { ...section, content: unified };
    }));
  }
}
```

#### 5. 质量审查

```typescript
interface QualityReport {
  overallScore: number;      // 0-100
  checks: QualityCheck[];
  issues: QualityIssue[];
  suggestions: string[];
}

interface QualityCheck {
  name: string;
  passed: boolean;
  score: number;
  details: string;
}

class QualityReviewer {
  async review(draft: DraftReport): Promise<QualityReport> {
    const checks: QualityCheck[] = [];

    // 1. 事实性检查
    checks.push({
      name: 'factual_accuracy',
      passed: await this.checkFactualAccuracy(draft),
      score: await this.scoreFactualAccuracy(draft),
      details: 'Checking each claim against data sources'
    });

    // 2. 逻辑连贯性
    checks.push({
      name: 'logical_coherence',
      passed: await this.checkCoherence(draft),
      score: await this.scoreCoherence(draft),
      details: 'Checking logical flow between sections'
    });

    // 3. 重复检测
    checks.push({
      name: 'duplication',
      passed: await this.checkDuplication(draft),
      score: await this.scoreDuplication(draft),
      details: 'Checking for repeated content'
    });

    // 4. 格式合规
    checks.push({
      name: 'format_compliance',
      passed: await this.checkFormat(draft),
      score: await this.scoreFormat(draft),
      details: 'Checking style guide compliance'
    });

    // 5. 引用完整性
    checks.push({
      name: 'citation_integrity',
      passed: await this.checkCitations(draft),
      score: await this.scoreCitations(draft),
      details: 'All claims have proper citations'
    });

    return {
      overallScore: checks.reduce((s, c) => s + c.score, 0) / checks.length,
      checks,
      issues: this.findIssues(checks, draft),
      suggestions: await this.generateSuggestions(checks, draft)
    };
  }

  private async checkFactualAccuracy(draft: DraftReport): Promise<boolean> {
    // 对报告中的每个数据点做交叉验证
    const claims = this.extractClaims(draft);
    for (const claim of claims) {
      const verified = await this.verifyClaim(claim);
      if (!verified) {
        return false;
      }
    }
    return true;
  }

  private async verifyClaim(claim: Claim): Promise<boolean> {
    // 检查 claim 的来源是否支持该陈述
    if (!claim.source) return false;
    return claim.source.value === claim.statedValue;
  }
}
```

```typescript
class LongReportGenerator {
  async generate(requirement: ReportRequirement): Promise<FinalReport> {
    // Map: 并行生成各章节
    const sectionResults = await Promise.all(
      requirement.sections.map(section =>
        this.generateSection(section, requirement)

    // Reduce: 合并并精修
    const merged = await this.mergeSections(sectionResults);
```

## 常见误区

### 误区 1：一次生成整篇长报告

一次生成长报告会导致：后段内容质量下降、主题漂移、内容重复。应该分节生成，每节作为一个独立任务，最后统一整合。

### 误区 2：让模型自己决定写什么

模型倾向于写它"知道"的东西而不是用户需要的东西。必须通过明确的大纲和数据约束来控制内容方向。

### 误区 3：忽略事实准确性校验

模型生成的报告看起来很专业但可能包含严重的事实错误。每个数值、日期、引用都需要交叉验证。

### 误区 4：认为报告越长越好

好的报告是"准确、简洁、有力"的。模型天然倾向于啰嗦。需要设置字数上限，并要求模型优先用数据说话。

## 追问方向

1. **"如何确保自动生成的报告在不同章节之间风格一致？"**

    逐节生成后需要进行"风格统一"的后处理步骤。将各章节输入模型，要求在不改变内容的前提下统一语气、术语和格式。也可以预先定义风格指南作为 prompt 的固定部分。

2. **"如果用户提供的数据不足以支撑某些章节的结论，系统怎么做？"**

    在数据收集阶段检测数据缺口，在相应章节标注"数据不足以支持结论"或生成图表占位。也可以引导用户补充数据，而不是让模型编造。

3. **"自动生成的报告如何在保持结构的同时适应不同的输出格式（PDF/Word/HTML）？"**

    采用"内容-展示"分离架构。内部使用 JSON/Markdown 格式存储结构化内容，然后通过不同的渲染器输出为目标格式。

4. **"如何防止自动生成报告时泄露机密信息？"**

    在数据收集阶段做权限过滤，在质量审查阶段做敏感信息检测（正则 + LLM 检测）。对于有权限分级的数据，生成时自动降级或脱敏。
