# 多智能体如何评测？

## 面试问题

多智能体如何评测？

## 考察重点

- 是否能区分单智能体评测和多智能体评测的差异
- 是否理解多智能体系统需要评测的维度远多于单智能体
- 是否能说清楚协调效率、通信开销、冲突频率等特有指标
- 是否有实际的评测体系和回归测试经验

## 核心结论

多智能体评测比单智能体复杂得多，除了要评测每个智能体的个体能力，还要评测**协调效率、通信质量、冲突频率、系统稳定性**四个系统级维度；评测体系必须包含离线评测、在线评测和回归测试三层。

## 评测维度

### 维度一：个体智能体质量

每个智能体独立工作的能力。

```python
class IndividualEvaluation:
    def __init__(self, agent: Agent, test_cases: list[TestCase]):
        self.agent = agent
        self.test_cases = test_cases

    async def evaluate(self) -> IndividualMetrics:
        results = []
        for test_case in self.test_cases:
            result = await self.agent.execute(test_case.task, tools=self.agent.tools)
            results.append(EvaluationResult(
                task_id=test_case.id,
                success=self.check_success(result, test_case.expected),
                quality_score=self.calculate_quality(result, test_case.criteria),
                tokens=result.tokens_used,
                latency_ms=result.latency_ms,
                tool_accuracy=self.check_tool_usage(result, test_case.expected_tools)
            ))
        return IndividualMetrics(
            success_rate=sum(r.success for r in results) / len(results),
            avg_quality=sum(r.quality_score for r in results) / len(results),
            avg_tokens=sum(r.tokens for r in results) / len(results),
            avg_latency=sum(r.latency_ms for r in results) / len(results),
            tool_accuracy=sum(r.tool_accuracy for r in results) / len(results)
        )
```

### 维度二：协调效率

智能体之间协作的好坏。

```
关键指标：
  规划准确率：Orchestrator 分配任务的准确率
    正确分配的任务数 / 总任务数

  协调开销占比：
    协调消耗的 token / 总 token
    目标：< 20%

  任务等待时间：
    智能体从分配到开始执行的平均等待时间
    目标：< 1s

  决策收敛速度：
    从开始讨论到达成一致的平均轮次
    目标：< 3 轮
```

```python
class CoordinationEvaluation:
    def __init__(self, system: MultiAgentSystem):
        self.system = system

    async def evaluate(self, test_cases: list) -> CoordinationMetrics:
        results = []
        for test in test_cases:
            trace = await self.system.execute_with_trace(test.task)
            results.append({
                "plan_accuracy": self.calc_plan_accuracy(trace.plan, test.expected_distribution),
                "coordination_overhead": trace.coordination_tokens / trace.total_tokens,
                "decision_rounds": trace.decision_rounds,
                "idle_time": trace.total_idle_time
            })
        return CoordinationMetrics(
            avg_plan_accuracy=sum(r["plan_accuracy"] for r in results) / len(results),
            avg_coordination_overhead=sum(r["coordination_overhead"] for r in results) / len(results),
            avg_decision_rounds=sum(r["decision_rounds"] for r in results) / len(results),
            avg_idle_time=sum(r["idle_time"] for r in results) / len(results)
        )
```

### 维度三：通信质量

智能体之间信息交换的质量。

```
指标              计算方式                理想值
信息传递准确率    正确传递的信息 / 总信息     > 95%
信息传递完整率    保留的信息量 / 原始信息量   > 80%
误解率            导致错误的信息传递次数      < 5%
通信效率          有用信息 token / 总通信 token  > 60%
```

### 维度四：冲突频率

智能体之间产生冲突的频率和严重程度。

```python
class ConflictEvaluation:
    async def evaluate(self, system: MultiAgentSystem, test_cases: list) -> ConflictMetrics:
        total_conflicts = 0
        conflict_severity = []
        resolution_success = 0
        resolution_time = []

        for test in test_cases:
            conflict_log = await system.execute_with_logging(test.task)
            conflicts = conflict_log.conflicts
            total_conflicts += len(conflicts)
            for conflict in conflicts:
                conflict_severity.append(conflict.severity)
                if conflict.resolved:
                    resolution_success += 1
                resolution_time.append(conflict.resolution_time_ms)

        return ConflictMetrics(
            conflict_rate=total_conflicts / len(test_cases),
            avg_severity=sum(conflict_severity) / max(len(conflict_severity), 1),
            resolution_rate=resolution_success / max(total_conflicts, 1),
            avg_resolution_time=sum(resolution_time) / max(len(resolution_time), 1),
            total_conflicts=total_conflicts
        )
```

## 评测方法

### 1. 离线评测（单元测试级别）

对单个智能体或特定交互进行测试。

```python
class OfflineEvaluator:
    def __init__(self):
        self.test_suite = {
            "planner": [
                ("做一个用户登录系统", {"expected_steps": 5, "expected_components": ["前端", "后端", "数据库"]}),
                ("部署一个静态网站", {"expected_steps": 3, "expected_components": ["服务器", "域名"]})
            ],
            "executor": [
                ("用Python写一个冒泡排序", {"expected_output": "排序函数", "expected_tools": ["write_file"]}),
            ],
            "critic": [
                ("检查以下代码的安全性", {"expected_issues": ["SQL注入", "XSS"], "min_issues_found": 2})
            ],
            "orchestrator": [
                ("分析销售数据并生成报告", {"expected_agents": ["data", "analysis", "report"]})
            ]
        }

    async def evaluate_all(self) -> OfflineReport:
        results = {}
        for component, tests in self.test_suite.items():
            component_results = []
            for task, expected in tests:
                result = await self.run_test(component, task, expected)
                component_results.append(result)
            results[component] = {
                "pass_rate": sum(r.passed for r in component_results) / len(component_results),
                "details": component_results
            }
        return OfflineReport(
            overall_pass_rate=sum(r["pass_rate"] for r in results.values()) / len(results),
            component_results=results
        )
```

### 2. 集成评测（端到端测试）

对整个多智能体系统进行端到端测试。

```python
class IntegrationEvaluator:
    def __init__(self):
        self.end_to_end_tests = [
            EndToEndTest(
                name="全流程开发测试",
                task="开发一个TODO应用的REST API",
                expected_outputs={"has_api_spec": True, "has_implementation": True, "has_tests": True, "has_docs": True},
                max_cost=0.5, max_latency=30000, min_quality=0.7
            ),
            EndToEndTest(
                name="电商售后处理",
                task="处理一个退货请求",
                expected_outputs={"found_order": True, "initiated_refund": True, "notified_user": True},
                max_cost=0.3, max_latency=20000, min_quality=0.8
            )
        ]

    async def evaluate(self) -> IntegrationReport:
        results = []
        for test in self.end_to_end_tests:
            result = await self.execute_test(test)
            results.append(result)
        return IntegrationReport(
            tests_passed=sum(1 for r in results if r.passed),
            total_tests=len(results),
            avg_cost=sum(r.cost for r in results) / len(results),
            avg_latency=sum(r.latency for r in results) / len(results),
            avg_quality=sum(r.quality for r in results) / len(results),
            details=results
        )
```

### 3. 回归测试（持续集成级别）

每次修改系统后自动运行，防止回归。

```python
class RegressionTestSuite:
    def __init__(self):
        self.regression_tests = self.load_regression_tests()
        self.baseline_scores = self.load_baseline()

    def load_regression_tests(self) -> list[RegressionTest]:
        failed_cases = self.load_failed_cases()
        return [
            RegressionTest(id=case.id, task=case.task, expected_output=case.expected,
                           known_failure_pattern=case.failure_pattern, importance=case.importance)
            for case in failed_cases
        ]

    async def run_regression(self, system: MultiAgentSystem) -> bool:
        regressions = []
        for test in self.regression_tests:
            result = await system.execute(test.task)
            passed = self.check_result(result, test.expected_output)
            if not passed:
                regressions.append(Regression(test_id=test.id, expected=test.expected_output, actual=result))
        if regressions:
            self.alert_regressions(regressions)
            return False
        return True
```

## 评测指标汇总

| 维度 | 指标 | 计算方式 | 理想值 |
|------|------|---------|-------|
| 任务完成 | 完成率 | 成功任务 / 总任务 | > 90% |
| 任务质量 | 质量评分 | 专家评分 / 自动评分 | > 0.8 |
| 成本 | 平均 Token | 总 Token / 任务数 | < 30K |
| 延迟 | P95 响应时间 | - | < 15s |
| 协调效率 | 协调开销占比 | 协调 Token / 总 Token | < 20% |
| 冲突 | 冲突率 | 冲突次数 / 任务数 | < 0.5 |
| 通信 | 信息准确率 | 正确传递 / 总信息 | > 95% |
| 稳定性 | 故障容忍 | 容忍故障的智能体比例 | > 50% |
| 回归 | 回归通过率 | 通过测试 / 总回归测试 | > 95% |

## 评测框架

```python
class EvaluationFramework:
    def __init__(self):
        self.offline = OfflineEvaluator()
        self.integration = IntegrationEvaluator()
        self.regression = RegressionTestSuite()

    async def full_evaluation(self, system: MultiAgentSystem, config: EvaluationConfig) -> EvaluationReport:
        report = EvaluationReport(timestamp=time.time(), config=config)

        if config.run_offline:
            report.offline = await self.offline.evaluate_all()
        if config.run_integration:
            report.integration = await self.integration.evaluate()
        if config.run_regression:
            report.regression_passed = await self.regression.run_regression(system)
        if config.baseline:
            report.comparison = self.compare_with_baseline(report, config.baseline)

        return report
```

## 常见误区

- 只评测最终结果质量，忽略协调开销、冲突频率等系统级指标
- 在线评测时混入离线测试用例，导致评测数据不准确
- 没有建立基线（baseline），无法判断每次修改是进步还是退步
- 回归测试只覆盖老的成功案例，不关注新引入的失败模式
- 评测成本本身被忽略——跑一次全量评测可能比系统运行成本还高

## 追问方向

1. **"你的评测集是怎么构建的？覆盖了哪些边界情况？"**

   考察候选人对评测集设计的系统性思考。

2. **"协调效率太低时，你怎么定位是哪个智能体的问题？"**

   考察候选人对细粒度可观测性的理解。

3. **"如果线下评测分数很高但线上效果很差，你会怎么排查？"**

   考察候选人对线上线下差异的处理经验。

4. **"多智能体系统的评测和单智能体评测最大的不同是什么？"**

   考察候选人是否理解系统级评测的挑战。

---

## 学习建议

为一个多智能体系统建立完整的评测体系，至少包含 20 个测试用例，覆盖正常流程、边界情况、异常场景、冲突场景。运行基线版本和优化版本，记录所有指标的变化。重点关注"协调开销占比"和"冲突率"这两个多智能体特有的指标——它们往往是性能瓶颈的关键指示器。
