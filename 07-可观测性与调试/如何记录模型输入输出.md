# 如何记录模型输入输出？

## 面试问题

如何记录模型输入输出？

## 考察重点

- 是否理解记录模型输入输出的核心挑战（数据量大、敏感信息多、会影响性能）
- 是否有切实可行的分级记录策略
- 是否考虑过记录本身的成本和安全性
- 是否有从模型 IO 记录中定位问题的实际经验

## 核心结论

记录模型输入输出的核心挑战不在于"怎么记"，而在于"记多少"和"怎么保证不记到敏感信息"。策略是分级记录：日常只记元数据和摘要，调试时才记完整 IO。同时必须在记录入口和出口做脱敏处理，否则模型 IO 记录本身就是安全风险。

## 参考回答

### 记什么

模型调用的输入输出包含以下内容，但不是所有内容都需要每次记录：

```typescript
interface ModelIO {
  // 必须记录（元信息）
  timestamp: string;
  model: string;            // 模型名称 + 版本
  duration_ms: number;      // 推理耗时
  tokens_in: number;        // 输入 token 数
  tokens_out: number;       // 输出 token 数
  
  // 按需记录（核心内容）
  system_prompt?: string;
  messages?: Message[];     // 完整的 messages 数组
  tools?: ToolDefinition[]; // 此时可用的工具
  tool_choice?: string;
  
  // 输出
  response?: ModelResponse; // 模型返回的完整响应
  
  // 可选（用于成本分析）
  cost_usd?: number;
  cache_hit?: boolean;
  retry_count?: number;
}
```

### 三种记录模式

```python
class ModelIORecorder:
    """分级记录模型输入输出"""
    
    def __init__(self, mode: str = 'normal'):
        self.mode = mode  # 'minimal' | 'normal' | 'full'
    
    def record(self, call: ModelCall):
        if self.mode == 'minimal':
            return self._record_minimal(call)
        elif self.mode == 'normal':
            return self._record_normal(call)
        else:
            return self._record_full(call)
    
    def _record_minimal(self, call: ModelCall):
        """只记元数据，适合大规模生产环境"""
        return {
            'model': call.model,
            'tokens_in': call.usage.prompt_tokens,
            'tokens_out': call.usage.completion_tokens,
            'duration_ms': call.duration_ms,
            'timestamp': call.timestamp,
            'status': call.status
        }
    
    def _record_normal(self, call: ModelCall):
        """记录元数据 + 消息摘要 + 完整响应"""
        return {
            **self._record_minimal(call),
            'messages_summary': self._summarize_messages(call.messages),
            'response': call.response,
            # 不记录完整 messages，但记录长度和结构
            'messages_count': len(call.messages),
            'messages_total_chars': sum(
                len(m.content) for m in call.messages
            ),
            'system_prompt_version': call.system_prompt_version
        }
    
    def _record_full(self, call: ModelCall):
        """记录所有内容，适合调试"""
        return {
            **self._record_normal(call),
            'messages': call.messages,
            'tools': call.tools,
            'tool_choice': call.tool_choice,
            'raw_request': call.raw_request,
            'raw_response': call.raw_response
        }
    
    def _summarize_messages(self, messages):
        """对消息列表做摘要，保留结构但不保留完整内容"""
        summaries = []
        for msg in messages:
            content = msg.content or ''
            summary = {
                'role': msg.role,
                'length': len(content),
                'preview': content[:200] + '...' if len(content) > 200 else content
            }
            if msg.role == 'tool':
                summary['tool_name'] = msg.tool_name
            summaries.append(summary)
        return summaries
```

### 敏感信息脱敏

这是记录模型 IO 时最容易忽略但最重要的事情：

```python
class IOSanitizer:
    """模型输入输出的脱敏处理"""
    
    SENSITIVE_PATTERNS = [
        (r'api[_-]?key["\']?\s*[:=]\s*["\']?\w{16,}', 'API_KEY_REDACTED'),
        (r'AKIA[0-9A-Z]{16}', 'AWS_KEY_REDACTED'),
        (r'sk-[a-zA-Z0-9]{32,}', 'OPENAI_KEY_REDACTED'),
        (r'password["\']?\s*[:=]\s*["\'][^"\']+["\']', 'PASSWORD_REDACTED'),
        (r'token["\']?\s*[:=]\s*["\'][^"\']+["\']', 'TOKEN_REDACTED'),
        (r'bearer\s+[a-zA-Z0-9._-]+', 'BEARER_REDACTED', re.I),
    ]
    
    def sanitize(self, data: Any) -> Any:
        """递归脱敏"""
        if isinstance(data, str):
            return self._sanitize_string(data)
        elif isinstance(data, dict):
            return {k: self.sanitize(v) for k, v in data.items()}
        elif isinstance(data, list):
            return [self.sanitize(item) for item in data]
        return data
    
    def _sanitize_string(self, text: str) -> str:
        for pattern, replacement in self.SENSITIVE_PATTERNS:
            text = re.sub(pattern, replacement, text, flags=re.I)
        return text
```

## 深度展开

### 记录模型 IO 的核心挑战

#### 挑战一：数据量爆炸

一次智能体执行可能包含 10-30 次模型调用。每次调用的输入（messages + tools）可能几十 KB。如果全部记录，一个会话就可能产生数 MB 的 IO 数据。

```
一次 Agent 执行：
  - 10 次模型调用
  - 每次输入 ~20KB（含历史上下文和工具定义）
  - 每次输出 ~1KB
  - 总 IO 数据：~210KB
  
1000 次执行/天：
  - 210MB/天的模型 IO 数据
  - 6.3GB/月
  - 需要存储、检索、清理
```

**解决方案**：
1. 分级记录（minimal/normal/full）
2. 采样记录（全量记录对选中的 session）
3. 自动清理（TTL 策略）
4. 存储时压缩

#### 挑战二：敏感信息无处不在

模型输入输出中可能包含：

```python
class SensitivityDetector:
    """检测模型 IO 中的敏感信息类型"""
    
    SENSITIVITY_TYPES = {
        "credentials": [

            "authorization", "bearer"
        ],
        "pii": [
```

#### 挑战三：记录影响性能

每次模型调用后记录 IO 会增加延迟。如果记录是同步的，会延长用户等待时间。

```python
class AsyncIORecorder:
    """异步 IO 记录，不阻塞主流程"""
    
    def __init__(self):
        self.queue = asyncio.Queue()
        self.worker_task = asyncio.create_task(self._worker())
    
    async def record(self, io_data: ModelIO):
        # 立即放入队列，不阻塞
        sanitized = await self.sanitizer.sanitize(io_data)
        await self.queue.put(sanitized)
    
    async def _worker(self):
        """后台批量写入存储"""
        batch = []
        while True:
            try:
                item = await asyncio.wait_for(
                    self.queue.get(), timeout=5.0
                )
                batch.append(item)
                
                if len(batch) >= 100:
                    await self._flush(batch)
                    batch = []
            except asyncio.TimeoutError:
                if batch:
                    await self._flush(batch)
                    batch = []
```

### 实用记录策略

```python
class ModelIOStrategy:
    """
    完整的模型 IO 记录策略
    """
    
    @staticmethod
    def should_record_full(session) -> bool:
        """判断是否记录完整 IO"""
        return any([
            session.user.is_internal,
            session.has_errors,
            random.random() < 0.01,  # 1% 采样
            session.tags.intersects({'debug', 'test'})
        ])
    
    @staticmethod
    def retention_policy(level: str) -> Dict:
        """不同级别的保留策略"""
        policies = {
            'minimal': {
                'retention_days': 90,
                'storage': 'clickhouse',
                'compression': 'zstd'
            },
            'normal': {
                'retention_days': 30,
                'storage': 'clickhouse',
                'compression': 'zstd'
            },
            'full': {
                'retention_days': 7,
                'storage': 'object_store',
                'compression': 'gzip'
            }
        }
        return policies.get(level, policies['minimal'])
```

## 工程实践

### 最小可行实现

```python
class SimpleModelLogger:
    """
    最简单的模型 IO 记录器
    适合在开发阶段使用
    """
    
    def __init__(self, log_dir: str = "model_io_logs"):
        self.log_dir = Path(log_dir)
        self.log_dir.mkdir(exist_ok=True)
        self.session_id = str(uuid.uuid4())
    
    def log_call(self, 
                 messages: list,
                 response: Any,
                 model: str,
                 usage: Dict):
        
        entry = {
            "timestamp": datetime.utcnow().isoformat(),
            "session_id": self.session_id,
            "model": model,
            "usage": usage,
            "messages_count": len(messages),
            "messages_preview": [
                {
                    "role": m.get("role"),
                    "content_preview": m.get("content", "")[:200]
                }
                for m in messages
            ],
            "response_preview": str(response)[:500]
        }
        
        # 追加写入 JSONL 文件
        with open(
            self.log_dir / f"{self.session_id}.jsonl", "a"
        ) as f:
            f.write(json.dumps(entry, ensure_ascii=False) + "\n")
```

### 集成到 Agent Loop

```typescript
class AgentWithModelLogging {
  private ioLogger: ModelIOLogger;
  
  async execute(task: string): Promise<string> {
    let context = this.buildContext(task);
    
    while (!this.isComplete(context)) {
      // 记录输入
      const inputSnapshot = this.ioLogger.snapshotInput({
        messages: context.messages,
        tools: context.tools,
        model: this.model,
      });
      
      // 模型调用
      const response = await this.model.call(context);
      
      // 记录输出
      this.ioLogger.log({
        ...inputSnapshot,
        response: response,
        duration_ms: response.duration,
        tokens_in: response.usage.promptTokens,
        tokens_out: response.usage.completionTokens,
        timestamp: Date.now(),
      });
      
      // 处理响应
      context = this.updateContext(context, response);
    }
    
    return context.result;
  }
}
```

## 常见误区

- **只记输出不记输入**：只知道模型"说了什么"但不知道"看到了什么"，没有任何调试价值
- **把所有 IO 无差别全量记录**：数据量很快失控，而且 99% 的完整 IO 记录永远不会被查看。分级记录才是可持续的做法
- **记录前不做脱敏**：模型 IO 中经常包含密钥和用户隐私。记录文件本身就是一级安全风险
- **同步记录阻塞主流程**：如果存储系统挂了，记录失败会导致整个智能体不可用

## 追问方向

- 一个 session 有 20 次模型调用，每次 messages 数组 50KB，你如何存储和查询？
- 模型 IO 记录中发现了用户的信用卡号，你怎么处理已有的记录和防止再次发生？
- 你如何保证模型 IO 记录系统本身不影响智能体的正常执行？
- 团队需要根据模型 IO 数据做成本分析，但你又不能把所有数据都存下来，怎么取舍？

## 学习建议

从最小的方案开始：在你的 agent loop 中每次模型调用后追加一行 JSON 到文件。先跑几天看看数据量，然后再决定要不要上正式的记录系统。关键是在"什么都不记"和"什么都记"之间找到适合自己的平衡点。
