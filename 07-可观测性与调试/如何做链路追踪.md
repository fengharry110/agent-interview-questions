# 如何做链路追踪？

## 面试问题

如何做链路追踪？

## 考察重点

- 是否理解分布式追踪（Distributed Tracing）的基本概念：Trace、Span、Context Propagation
- 是否能说清楚智能体系统中的链路追踪和传统微服务追踪的区别
- 是否有实际的追踪工具使用经验（OpenTelemetry、Jaeger、Zipkin 等）
- 是否考虑过异步模型调用和流式响应的追踪问题

## 核心结论

链路追踪在智能体系统的核心价值是**把一次用户请求的完整执行路径串联起来**——从用户输入、意图识别、任务规划、多次模型调用、多次工具执行到最终响应。实现方式可以基于 OpenTelemetry 标准：每个用户请求创建一个 Trace，每步决策/工具调用创建一个 Span，通过 Context Propagation 在异步边界传递追踪上下文。

```python
Trace: 一次完整的用户请求处理

│   ├── Span: 调用 LLM 分析检索结果 (LLM call)
│   └── Span: 工具调用: search_api (HTTP call)
├── Span: 执行步骤 2
│   ├── Span: 调用 LLM 决策 (LLM call)
│   ├── Span: 工具调用: write_file (File I/O)
│   └── Span: 工具调用: run_shell (Process)
└── Span: 生成最终回复
    └── Span: 调用 LLM 组织回复 (LLM call)
```

### 基于 OpenTelemetry 的实现

from opentelemetry import trace
from opentelemetry.trace import Status, StatusCode

class AgentTracer:

    基于 OpenTelemetry 的智能体链路追踪
    """
    
    def __init__(self, service_name: str = "agent-service"):
        self.tracer = trace.get_tracer(service_name)
    
    def trace_execution(self, session_id: str, user_goal: str):

        创建一次执行的主 Trace
        """
        with self.tracer.start_as_current_span(

                "session_id": session_id,

                tracer=self.tracer,

                "llm.model": model,
                "llm.max_tokens": 4096,
                "llm.temperature": 0

### 关键实现细节

#### 1. 上下文传播

智能体系统中，一个 Trace 需要跨多次模型调用和工具调用传递：

```python
class AgentContextPropagator:
    """
    在异步调用边界传递追踪上下文
    """
    
    def __init__(self):
        self._current_context = None
    
    async def with_trace(self, func, context):
        """
        在异步调用中保持追踪上下文
        """
        token = trace.set_span_in_context(
            context.span, context.otel_context
        )
        try:
            return await func()
        finally:
            trace.reset_context(token)
    
    def inject_to_llm_request(self, request: Dict) -> Dict:
        """
        将追踪信息注入 LLM 请求（用于调试）
        """
        request["_trace_id"] = self._current_context.trace_id
        request["_span_id"] = self._current_context.span_id
        return request
```

模型流式输出时，Span 的结束时机不是"收到完整响应"而是"流结束"：

#### 3. 工具调用的子 Span

工具调用可能触发 HTTP 请求、数据库查询、文件操作等子 Span：

```python
def trace_tool_with_subspans(tracer, tool_call: ToolCall):
    """
    追踪工具调用及其内部操作
    """
    with tracer.start_as_current_span(
        f"tool.{tool_call.name}",
        attributes={
            "tool.name": tool_call.name,
            "tool.args": str(tool_call.args)
        }
    ) as span:
        # 如果是 HTTP 调用，创建子 Span
        if tool_call.name == "web_search":
            with tracer.start_as_current_span(
                "http.request",
                attributes={
                    "http.method": "GET",
                    "http.url": tool_call.args["url"]
                }
            ) as http_span:
                result = execute_http(tool_call.args)
                http_span.set_attribute(
                    "http.status_code", result.status_code
                )
        else:
            result = execute_tool(tool_call)
        
        span.set_attribute("tool.status", result.status)
        span.set_attribute("tool.duration_ms", result.duration_ms)
        
        return result
```

### 追踪数据的可视化

```python
class TraceVisualizer:
    """
    执行轨迹的可视化
    """
    
    @staticmethod
    def to_timeline(trace: Trace) -> Timeline:
        """
        将 Trace 转为时间轴视图
        """
        events = []
        for span in trace.spans:
            events.append({
                "name": span.name,
                "start": span.start_time,
                "end": span.end_time,
                "duration_ms": span.duration_ms,
                "depth": span.depth,
                "status": "success" if span.status.is_ok else "error",
                "attributes": span.attributes
            })
        
        return Timeline(events=events)
    
    @staticmethod
    def to_waterfall(trace: Trace) -> WaterfallChart:
        """
        生成瀑布图数据
        每一行是一个 Span，宽度表示耗时
        """
        rows = []
        for span in sorted(
            trace.spans, key=lambda s: s.start_time
        ):
            rows.append({
                "name": "  " * span.depth + span.name,
                "start_offset_ms": span.start_time - trace.start_time,
                "duration_ms": span.duration_ms,
                "tool": span.attributes.get("tool.name"),
                "model": span.attributes.get("llm.model")
            })
        
        return WaterfallChart(rows=rows)
```

## 工程实践

### 最小链路追踪实现

```python
class SimpleTracer:
    """
    不需要外部依赖的简单追踪实现
    适合开发阶段使用
    """
    
    def __init__(self):
        self.traces: Dict[str, List[Span]] = {}
    
    def start_trace(self, session_id: str, name: str):
        trace_id = session_id
        self.traces[trace_id] = []
        return TraceContext(
            trace_id=trace_id,
            span_id=name
        )
    
    def start_span(self, ctx: TraceContext, name: str):
        span = Span(
            trace_id=ctx.trace_id,
            span_id=f"{ctx.span_id}/{name}",
            parent_span_id=ctx.span_id,
            name=name,
            start_time=time.time()
        )
        self.traces[ctx.trace_id].append(span)
        return TraceContext(
            trace_id=ctx.trace_id,
            span_id=span.span_id
        )
    
    def end_span(self, ctx: TraceContext, status: str = "ok"):
        for span in self.traces[ctx.trace_id]:
            if span.span_id == ctx.span_id:
                span.end_time = time.time()
                span.duration_ms = (
                    span.end_time - span.start_time
                ) * 1000
                span.status = status
                break
    
    def export(self, trace_id: str) -> Dict:
        """导出为 JSON"""
        return {
            "trace_id": trace_id,
            "spans": [
                {
                    "name": s.name,
                    "start": s.start_time,
                    "duration_ms": s.duration_ms,
                    "parent": s.parent_span_id,
                    "status": s.status
                }
                for s in self.traces[trace_id]
            ]
        }
```

### OpenTelemetry 集成示例

```python
from opentelemetry import trace
from opentelemetry.exporter.otlp.proto.grpc.trace_exporter import OTLPSpanExporter
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import BatchSpanProcessor

def setup_otel_tracing():
    """
    配置 OpenTelemetry 导出到 Jaeger/Zipkin
    """
    provider = TracerProvider()
    processor = BatchSpanProcessor(
        OTLPSpanExporter(
            endpoint="http://localhost:4317"
        )
    )
    provider.add_span_processor(processor)
    trace.set_tracer_provider(provider)
```

## 常见误区

- **把日志和追踪混为一谈**：日志是平铺的记录，追踪是带父子关系的树状结构。追踪比日志更适合理解"为什么 A 导致了 B"
- **只追踪模型调用，不追踪工具执行**：工具执行（尤其 HTTP 调用和文件操作）往往比模型推理还慢，不追踪就找不到瓶颈
- **没有在异步边界传播上下文**：在 asyncio 任务或线程池中丢失了追踪上下文，父子关系断裂，追踪变成了孤立的片段
- **Span 粒度过粗或过细**：一个 Span 覆盖整个 agent loop（太粗），或每次变量赋值都创建一个 Span（太细）。合理的粒度是"每次模型/工具调用一个 Span"
