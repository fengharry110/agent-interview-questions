# RAG 和长期记忆有什么区别？

## 面试问题

RAG 和 Agent 的长期记忆有什么区别？

## 考察重点

- 是否能区分外部知识库和用户/任务记忆
- 是否知道记忆需要写入、更新和遗忘机制
- 是否能说明隐私和准确性风险

## 参考回答

RAG 通常面向外部知识库，比如文档、网页、手册、代码说明。长期记忆更偏向 Agent 在和用户交互过程中积累的信息，比如用户偏好、历史任务、项目背景和常用约束。

RAG 的重点是检索已有知识，长期记忆还涉及什么时候写入、如何更新、如何避免错误记忆、如何让用户查看和删除记忆。

二者技术上都可能使用向量检索，但产品语义和安全要求不同。长期记忆更接近用户数据，因此更需要权限、隐私和可控性设计。

## 追问方向

- 什么信息不应该写入长期记忆？
- 记忆写错了怎么办？
- 记忆是否应该自动更新？

## 学习建议

回答时强调：RAG 是知识检索能力，memory 是状态和偏好管理能力。
