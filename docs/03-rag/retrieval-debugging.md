# 召回结果不准时如何排查？

## 面试问题

如果 RAG 召回结果不准，你会如何排查？

## 考察重点

- 是否能系统定位问题
- 是否知道 query、chunk、embedding、rerank 都可能出错
- 是否有评测意识

## 参考回答

我会先看用户问题对应的理想文档是否真的在知识库里。如果不在，那是数据覆盖问题。如果在，再看切分后相关信息是否被拆散或丢失。

然后检查 embedding 模型是否适合当前语言和领域，query 是否需要改写，metadata 过滤是否过严或过松。如果初召回结果大体相关但排序不好，可以考虑 reranker。

最后要建立一组固定测试问题，记录每个问题应该召回哪些文档，用指标和案例一起判断改动是否有效。

## 追问方向

- 如何判断是召回问题还是生成问题？
- 什么时候需要 reranker？
- 中文知识库 embedding 怎么选？

## 学习建议

排查 RAG 不要只看最终回答，要打印 top-k 检索结果和分数。
