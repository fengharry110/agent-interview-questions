# RAG 在 Agent 系统中起什么作用？

## 面试问题

RAG 在 Agent 系统中通常起什么作用？

## 考察重点

- 是否理解 RAG 是知识获取能力
- 是否知道 Agent 可以把检索当作工具
- 是否能说明 RAG 的局限

## 参考回答

RAG 可以让 Agent 获取外部知识，比如企业文档、API 文档、历史案例、用户手册或代码库说明。Agent 在需要知识时调用检索工具，把相关内容放入上下文，再进行推理和回答。

在 Agent 系统里，RAG 通常不是最终目的，而是一个能力模块。Agent 可能先检索资料，再调用业务 API，最后生成报告。

RAG 的效果取决于检索质量。如果召回内容不相关、过期或缺失，Agent 后续决策也可能出错。因此需要评估 chunking、embedding、reranking 和引用质量。

## 追问方向

- Agent 是否应该每一步都检索？
- 检索内容很多时如何控制上下文？
- 如何让 Agent 判断是否需要检索？

## 学习建议

把 RAG 看成 Agent 的一个工具，而不是把所有 Agent 问题都归结为 RAG 问题。
