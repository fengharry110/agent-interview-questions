# 什么是 Tool Calling？

## 面试问题

什么是 Tool Calling？为什么它对 Agent 很重要？

## 考察重点

- 是否理解模型通过结构化参数调用外部能力
- 是否知道 Tool Calling 和普通文本生成的区别
- 是否能解释工具结果如何回到模型上下文

## 参考回答

Tool Calling 是让模型根据任务需要选择某个工具，并生成结构化参数，由程序真正执行这个工具。工具可以是搜索、查数据库、读文件、发请求、运行代码等。

它重要的原因是模型本身只能生成文本，不能直接访问外部世界。通过 Tool Calling，模型可以把“我需要做什么”转换成真实系统里的函数调用。工具执行结果再作为 observation 返回给模型，模型继续推理或给出最终答案。

在 Agent 系统里，Tool Calling 是连接模型和外部环境的桥梁。

## 追问方向

- Tool Calling 和插件有什么区别？
- 工具参数为什么要结构化？
- 模型选错工具怎么办？

## 学习建议

学习重点是 schema、参数校验、错误处理和权限控制，不要只停留在“让模型调用函数”。
