# 如何设计一个好的工具 schema？

## 面试问题

你会如何设计一个适合 Agent 使用的工具 schema？

## 考察重点

- 是否知道工具名、描述、参数都影响模型选择
- 是否理解参数越清晰越容易稳定调用
- 是否重视输入校验和权限边界

## 参考回答

好的工具 schema 应该清楚表达工具能做什么、什么时候使用、需要哪些参数、返回什么结果。工具名应该具体，避免模糊。工具描述要告诉模型适用场景和限制。参数应该尽量少，但类型明确，必要字段和可选字段要清楚区分。

比如文件读取工具需要 path 参数，但不应该允许任意路径，应该限制在工作区内。命令执行工具更要谨慎，可以使用 allowlist 或人工确认。

此外，工具返回值也应该结构化，方便模型理解成功、失败、错误原因和下一步可用信息。

## 追问方向

- 参数过多会有什么问题？
- 如何让模型少调用错工具？
- 工具返回自然语言还是 JSON 更好？

## 学习建议

可以用 Pydantic 给每个工具定义输入输出模型，这是 Agent 工程里非常实用的习惯。
