# Agent 调用工具失败时应该怎么办？

## 面试问题

Agent 调用工具失败时，你会如何处理？

## 考察重点

- 是否有错误分类意识
- 是否知道哪些错误可以重试，哪些必须停止
- 是否能设计可观测的失败处理流程

## 参考回答

工具失败后，首先要区分错误类型。参数错误通常需要让模型修正参数再试。网络超时可以有限重试。权限错误不能盲目重试，应该提示需要授权。业务规则错误则需要把原因返回给 Agent，让它选择替代路径。

系统要给 Agent 返回清晰的错误结构，比如 error_type、message、retryable、suggestion。不要只返回一大段异常堆栈。

同时需要设置最大重试次数和最大步骤数，避免 Agent 卡在失败循环里。所有失败都应该记录日志，方便后续评测和调试。

## 追问方向

- 如何避免 Agent 对同一个错误反复重试？
- 工具失败信息应该暴露多少给模型？
- 哪些失败应该交给人工处理？

## 学习建议

可以把工具错误分成 validation_error、permission_denied、timeout、not_found、business_error 等类型。
