# 如何设计安全的文件和命令执行工具？

## 面试问题

如果 Agent 需要读写文件或执行命令，你会如何保证安全？

## 考察重点

- 是否理解真实工具有破坏性
- 是否知道路径限制、命令 allowlist、人工确认
- 是否能区分读操作和写操作风险

## 参考回答

文件工具首先应该限制工作目录，防止 Agent 读取或修改系统敏感路径。读文件和写文件要分成不同工具，写文件前最好生成 diff 或 patch，让用户确认。删除、移动、覆盖这类操作风险更高，需要更严格审批。

命令执行工具应该限制可执行命令范围，避免任意 shell。对于安装依赖、删除文件、上传数据、访问外部网络等操作，需要显式确认。命令输出也要截断和过滤，避免把敏感信息带回模型上下文。

安全设计的核心是最小权限、可审计、危险动作需要确认。

## 追问方向

- 为什么不建议直接给 Agent 一个 unrestricted shell？
- 如何防止路径穿越？
- 如何设计命令执行日志？

## 学习建议

可以参考 Codex 类工具的权限模型：读操作默认允许，写操作和危险命令根据范围请求授权。
