# Agent 项目如何写进简历？

## 面试问题

如果你做过 Agent 项目，你会如何把它写进简历？

## 考察重点

- 是否能把项目表达成工程成果
- 是否知道简历要写具体能力和结果
- 是否能突出 Agent 核心技术点

## 参考回答

Agent 项目写进简历时，不要只写“调用大模型实现智能助手”。更好的写法是说明系统解决了什么问题、用了哪些 Agent 能力、如何保证可靠性、有什么结果。

可以突出 tool calling、RAG、任务规划、权限控制、评测、日志、部署等点。例如：“实现基于 tool calling 的代码分析 Agent，支持仓库检索、文件读取、测试执行和补丁生成；设计权限确认机制和 30 条任务评测集，任务成功率达到 X%。”

如果没有真实业务数据，也可以写清楚测试集规模、功能覆盖和 demo 链接。

## 追问方向

- 项目没有上线，怎么体现价值？
- 如何避免简历看起来像套壳？
- 面试时如何讲 Agent 项目架构？

## 学习建议

准备一套固定项目故事：背景、问题、方案、难点、结果、复盘。
