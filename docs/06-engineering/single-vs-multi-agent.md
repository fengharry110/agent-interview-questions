# 单 Agent 和多 Agent 如何选择？

## 面试问题

什么时候使用单 Agent，什么时候使用多 Agent？

## 考察重点

- 是否有克制使用多 Agent 的判断力
- 是否知道多 Agent 会增加协调成本
- 是否能说出适合拆分的场景

## 参考回答

大多数场景应该先从单 Agent 开始，因为单 Agent 更简单、更容易调试、更容易评测。多 Agent 不一定更智能，反而可能带来通信成本、责任不清、循环讨论和结果不稳定。

当任务天然包含多个角色，且角色之间的目标、工具、上下文差异很大时，可以考虑多 Agent。比如一个代码系统里，Planner 负责拆任务，Coder 负责实现，Reviewer 负责检查。

使用多 Agent 时，要明确每个 Agent 的职责、输入输出、停止条件和冲突解决方式。

## 追问方向

- 多 Agent 如何共享上下文？
- 如何避免多个 Agent 互相扯皮？
- Reviewer Agent 是否真的可靠？

## 学习建议

面试里不要把多 Agent 说成万能方案。先说单 Agent，再说明什么时候值得拆。
