# 如何优化 Agent 的成本和延迟？

## 面试问题

Agent 系统通常成本和延迟较高，你会如何优化？

## 考察重点

- 是否理解多步调用会放大成本
- 是否能从模型、工具、上下文、缓存多个方向优化
- 是否知道质量和成本要权衡

## 参考回答

Agent 成本高的原因通常是多轮模型调用、长上下文、重复检索和工具调用。优化可以从几个方向入手。

第一，任务分级。简单任务用小模型或规则，复杂任务再用强模型。第二，减少不必要步骤，让 Agent 在需要时才调用工具。第三，控制上下文，只保留和当前决策相关的信息。第四，对稳定结果做缓存，比如检索结果、工具查询结果和中间分析。

延迟方面，可以并行执行独立工具，减少网络请求，设置超时和快速失败策略。优化时不能只追求便宜，还要看任务成功率是否下降。

## 追问方向

- 什么时候可以用小模型？
- 如何判断 Agent 步骤过多？
- 缓存会带来什么风险？

## 学习建议

回答时可以按“模型选择、上下文压缩、工具调用、缓存、并行”五个方向展开。
