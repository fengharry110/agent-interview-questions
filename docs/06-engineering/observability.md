# Agent 系统如何做日志和可观测性？

## 面试问题

Agent 系统如何设计日志和可观测性？

## 考察重点

- 是否知道 Agent 调试依赖完整轨迹
- 是否能设计 trace 字段
- 是否关注成本、延迟和安全

## 参考回答

Agent 系统应该记录完整执行轨迹，包括用户任务、模型输入输出、计划、工具调用、工具参数、工具结果、错误、重试、最终回答、token 使用量、耗时和成本。

日志要支持按任务、用户、工具、错误类型检索。对于敏感字段要脱敏。对于线上系统，还需要指标监控，比如任务成功率、平均延迟、工具失败率、人工确认率和成本趋势。

可观测性不是为了好看，而是为了定位失败、优化 prompt、发现高成本任务和支撑评测。

## 追问方向

- trace 和普通日志有什么区别？
- 如何避免日志泄露隐私？
- Agent 成本异常升高怎么排查？

## 学习建议

做作品集时，把 Agent 每一步执行过程展示出来，会比只展示最终回答更专业。
