# 什么是 Agent？它和普通 Chatbot 有什么区别？

## 面试问题

什么是 LLM Agent？它和普通 Chatbot 有什么区别？

## 考察重点

- 是否理解 Agent 的核心是行动能力，而不仅是对话能力
- 是否知道 Agent 可以调用工具、观察结果、继续决策
- 是否能说出 Agent 的适用场景和额外风险

## 参考回答

普通 Chatbot 主要是根据用户输入生成回复，它通常不主动改变外部环境，也不会执行复杂任务。LLM Agent 则是在大语言模型之上增加了目标、工具、状态和执行循环。

一个典型 Agent 会先理解任务目标，然后决定下一步动作，比如搜索资料、读取文件、调用 API、运行命令或修改数据。工具执行后，Agent 会观察结果，再决定是否继续执行下一步，直到任务完成或达到停止条件。

所以，Chatbot 更像是问答系统，而 Agent 更像是能完成任务的执行系统。Agent 的能力更强，但风险也更高，因为它可能调用真实工具，影响文件、数据库、业务流程或外部服务。

## 追问方向

- Agent 一定需要多轮规划吗？
- 什么情况下 Chatbot 就够了，不需要 Agent？
- Agent 的工具调用失败后应该怎么处理？

## 学习建议

可以用一个“订机票助手”或“代码修复助手”来解释 Agent。重点说清楚：它不是只回答，而是可以一步步完成任务。
