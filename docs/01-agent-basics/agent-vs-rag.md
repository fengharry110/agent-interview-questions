# Agent 和 RAG 应用有什么区别？

## 面试问题

Agent 和 RAG 应用有什么区别？它们之间是什么关系？

## 考察重点

- 是否能区分检索增强和自主执行
- 是否理解 RAG 可以作为 Agent 的一个工具
- 是否知道二者可以组合

## 参考回答

RAG 的核心是检索增强生成。系统先从知识库中检索相关内容，再把这些内容提供给模型生成答案。它主要解决模型不知道私有知识、知识过期、回答缺少依据的问题。

Agent 的核心是任务执行。Agent 可以根据目标选择不同工具，并根据工具结果继续下一步。RAG 可以是 Agent 的一个工具，比如 Agent 在回答问题前调用知识库检索工具，也可以在执行任务过程中多次查询资料。

简单说，RAG 解决“回答时参考什么资料”，Agent 解决“为了完成任务下一步做什么”。一个系统可以只有 RAG，没有 Agent；也可以是 Agent 使用 RAG。

## 追问方向

- RAG 应用什么时候会变成 Agent？
- 如果 RAG 检索错了，Agent 会发生什么问题？
- Agent 是否一定要有 RAG？

## 学习建议

面试时不要把 RAG 和 Agent 混成一个概念。可以画成：Agent 是外层决策者，RAG 是其中一个可调用工具。
