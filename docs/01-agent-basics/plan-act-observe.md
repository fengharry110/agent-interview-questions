# 什么是 Plan-Act-Observe 循环？

## 面试问题

请解释 Agent 中常见的 Plan-Act-Observe 循环。

## 考察重点

- 是否理解 Agent 的执行闭环
- 是否知道每一步的输入输出
- 是否能说明停止条件和失败处理

## 参考回答

Plan-Act-Observe 是 Agent 的一种基本执行模式。

Plan 阶段，Agent 理解用户目标和当前状态，决定下一步应该做什么。Act 阶段，Agent 调用某个工具或执行某个动作。Observe 阶段，系统把工具执行结果返回给 Agent，Agent 根据新的信息判断任务是否完成，或者继续下一轮。

这个循环通常需要边界条件，比如最大步骤数、最大花费、最大重试次数、工具调用权限和错误处理策略。没有这些限制，Agent 可能会陷入无意义循环，或者执行过多不必要的操作。

## 追问方向

- 如何防止 Agent 无限循环？
- Plan 是否应该显式保存？
- 每一步观察结果应该如何进入上下文？

## 学习建议

可以自己实现一个最小 Agent：输入目标，选择工具，执行工具，记录 observation，再决定下一步。
