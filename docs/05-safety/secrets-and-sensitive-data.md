# 如何保护密钥和敏感数据？

## 面试问题

Agent 系统中如何保护 API Key、密钥和敏感数据？

## 考察重点

- 是否知道模型上下文不是安全存储
- 是否理解最小暴露原则
- 是否关注日志和 trace 脱敏

## 参考回答

密钥不应该直接进入模型上下文。Agent 需要调用外部服务时，应该由后端工具持有密钥，模型只请求执行某个动作，而不是看到密钥本身。

敏感数据要按最小必要原则传给模型。日志、trace、错误信息里也要做脱敏，避免把 token、身份证、邮箱、客户数据等写入可见日志。

如果 Agent 可以访问文件或数据库，还需要权限控制和数据边界，防止它读取与任务无关的敏感内容。

## 追问方向

- 工具报错里包含密钥怎么办？
- 如何防止 Agent 把敏感信息发给外部 API？
- trace 是否可以长期保存？

## 学习建议

记住一句话：模型可以请求能力，但不应该直接拥有所有秘密。
