# 哪些操作需要 Human-in-the-loop？

## 面试问题

Agent 系统中哪些操作需要 Human-in-the-loop？

## 考察重点

- 是否能平衡自动化和安全
- 是否知道人工确认应该出现在关键节点
- 是否能结合业务场景

## 参考回答

需要人工参与的通常是不可逆、高价值、涉及隐私、涉及外部发送或影响真实用户的操作。比如删除数据、付款、发邮件、提交代码、修改生产配置、访问敏感文件等。

Human-in-the-loop 不只是弹一个确认框。系统应该展示 Agent 打算做什么、为什么做、会影响哪些对象、是否可以撤销，让人能做出判断。

对于低风险重复操作，可以逐步降低人工确认频率，但前提是有足够日志、回滚能力和评测数据。

## 追问方向

- 如何设计确认信息？
- 什么时候可以从人工审批变成自动执行？
- 如果用户误批准了怎么办？

## 学习建议

可以用“发邮件前预览内容”和“删库必须人工审批”作为例子。
