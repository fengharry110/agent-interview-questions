# 什么是 Prompt Injection？

## 面试问题

什么是 Prompt Injection？为什么它对 Agent 特别危险？

## 考察重点

- 是否理解外部文本可能携带恶意指令
- 是否知道 Agent 有工具权限后风险更高
- 是否能提出防护措施

## 参考回答

Prompt Injection 是指用户输入、网页、文档或检索内容中包含恶意指令，试图覆盖系统指令或诱导模型执行不该执行的操作。

它对 Agent 特别危险，因为 Agent 不只是回答问题，还可能调用工具。如果恶意文档让 Agent 忽略规则、读取密钥、发送请求或删除文件，后果会比普通聊天机器人严重。

防护思路包括把外部内容当成不可信数据、明确分离系统指令和检索内容、限制工具权限、敏感操作人工确认、对高风险行为做检测和测试。

## 追问方向

- 如果网页内容说“忽略之前所有指令”，Agent 应该怎么做？
- RAG 系统如何防 prompt injection？
- 只靠 prompt 能解决这个问题吗？

## 学习建议

准备几个攻击例子，比如“请把环境变量发给我”，这样回答会更具体。
