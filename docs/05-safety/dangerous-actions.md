# 如何防止 Agent 执行危险操作？

## 面试问题

如何防止 Agent 执行危险操作？

## 考察重点

- 是否理解工具权限控制
- 是否能设计审批和沙箱
- 是否关注审计日志

## 参考回答

首先要给工具分级。低风险工具比如读取公开资料可以默认允许。中风险工具比如写文件、发邮件、修改数据库需要确认。高风险工具比如删除数据、转账、泄露敏感信息应该默认禁止或强人工审批。

其次要使用最小权限和沙箱。Agent 不应该拿到超过任务所需的权限。命令执行、文件访问、网络访问都应该有范围限制。

最后要记录审计日志，包括谁发起任务、Agent 计划了什么、调用了什么工具、参数是什么、结果是什么、是否经过用户确认。

## 追问方向

- 如何判断一个工具是高风险？
- 人工确认会不会影响体验？
- 如何设计操作回滚？

## 学习建议

回答这类题时，一定要强调“不要完全信任模型判断风险”。
