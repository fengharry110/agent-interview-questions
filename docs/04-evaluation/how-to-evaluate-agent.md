# 如何评估一个 Agent 是否好用？

## 面试问题

你会如何评估一个 Agent 是否真正好用？

## 考察重点

- 是否知道 Agent 不能只靠主观体验评估
- 是否能设计任务集和指标
- 是否关注失败原因

## 参考回答

评估 Agent 首先要有代表性任务集，覆盖真实用户会做的事情。每个任务要定义成功标准，比如是否完成目标、是否调用正确工具、输出是否满足格式、是否遵守安全规则。

除了最终答案，还要评估执行过程。Agent 可能答案看起来对，但调用了错误工具或绕过了权限。常见指标包括任务成功率、工具调用准确率、平均步骤数、延迟、成本、安全违规率和人工介入率。

更重要的是分析失败类型，例如规划错误、检索错误、工具参数错误、上下文丢失、权限处理错误等。

## 追问方向

- 如何评估开放式回答质量？
- 如何判断一次工具调用是否正确？
- Agent eval 和普通单元测试有什么区别？

## 学习建议

为自己的 Agent 项目准备 20 到 50 个固定任务，比只做 demo 更有说服力。
