# 如何分析一次失败的 Agent 执行轨迹？

## 面试问题

如果一次 Agent 执行失败了，你会如何分析它的 trace？

## 考察重点

- 是否会从过程定位问题
- 是否能区分模型、工具、检索、权限和业务问题
- 是否有日志设计意识

## 参考回答

我会先看最终失败表现，再回到 trace 逐步检查。第一步看 Agent 是否正确理解目标。第二步看计划是否合理。第三步看工具选择是否正确。第四步看工具参数是否符合 schema。第五步看 observation 是否被正确理解。

如果使用了 RAG，还要检查检索结果是否相关。如果涉及权限，要看是否正确触发确认。如果模型在后续步骤忽略了前面的 observation，可能是上下文管理或提示词问题。

分析完成后，要给失败打标签，并决定是改 prompt、改工具 schema、改检索、改权限策略，还是补测试用例。

## 追问方向

- trace 中必须记录哪些字段？
- 如何保护 trace 里的敏感信息？
- 如何把失败案例转成回归测试？

## 学习建议

Agent 调试不要只看最后一句回答。真正有价值的信息在每一步 action 和 observation 里。
