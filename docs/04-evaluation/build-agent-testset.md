# 如何构建 Agent 测试集？

## 面试问题

你会如何为 Agent 构建测试集？

## 考察重点

- 是否知道测试集需要覆盖真实任务
- 是否能设计 expected outcome
- 是否关注边界情况和安全案例

## 参考回答

构建 Agent 测试集时，我会先收集真实用户任务或业务流程，把它们整理成标准输入、初始环境、可用工具和预期结果。每个测试用例应该说明成功条件，而不一定只要求固定文本答案。

测试集要覆盖简单任务、复杂多步任务、工具失败、权限不足、上下文不足、恶意输入等情况。对于 Agent 来说，过程同样重要，所以最好保存完整 trace，包括每一步工具调用和 observation。

测试集还应该持续迭代。线上发现的新失败案例，要沉淀回回归测试。

## 追问方向

- 如何处理没有唯一答案的问题？
- 如何构造 prompt injection 测试？
- 测试集会不会被 prompt 过拟合？

## 学习建议

可以用表格记录 case_id、用户任务、可用工具、预期行为、评分规则和失败标签。
