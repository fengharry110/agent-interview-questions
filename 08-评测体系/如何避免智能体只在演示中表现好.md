# 如何避免智能体只在演示中表现好？

## 面试问题

如何避免智能体只在演示中表现好？

## 考察重点

- 是否理解"演示 = 精心挑选的样本"和"线上 = 长尾分布"的本质区别
- 是否了解常见的"演示陷阱"（过拟合测试集、数据泄露）
- 是否有防止智能体系统过拟合的实践经验
- 是否建立了保障线上表现的机制

## 核心结论

智能体只在演示中表现好，本质上是因为**测试集和线上数据分布不一致**。演示使用的测试用例通常是精心挑选的、覆盖主流场景的，而线上会遇到各种没见过的边缘情况。避免方法是三管齐下：**测试集持续补充线上真实数据**、**建立线上线下指标对比机制**、**在 demo 之外做盲测**。

### 线上 vs 离线指标对比

```python
class OnlineOfflineComparison:
    """
    线上线下指标对比
    """
    
    def __init__(self):
        self.offline_evaluator = OfflineEvaluator()
        self.online_monitor = OnlineMonitor()
    
    def detect_drift(self) -> DriftReport:
        """
        检测线上线下表现差异
        """
        offline_metrics = self.offline_evaluator.last_evaluation()
        online_metrics = self.online_monitor.last_week_metrics()
        
        drifts = []
        for metric in self._common_metrics():
            offline_val = offline_metrics[metric]
            online_val = online_metrics[metric]
            
            if abs(offline_val - online_val) > 0.1:
                drifts.append(MetricDrift(
                    metric=metric,
                    offline=offline_val,
                    online=online_val,
                    gap=online_val - offline_val,
                    severity="high" if abs(online_val - offline_val) > 0.2 else "medium"
                ))
        
        return DriftReport(
            has_drift=len(drifts) > 0,
            drifts=drifts,
            recommendation=self._recommendation(drifts)
        )
```

```python
class BlindTesting:
    """
    盲测方法论
    """
    
    def __init__(self):
        self.testers = []  # 不了解系统的人
    
    def run_blind_test(self, test_cases: List[TestCase]) -> BlindTestReport:
        """
        盲测：测试者不知道系统的版本信息
        """
        results = []
        for case in test_cases:
            response = self.agent.run(case.input)
            
            # 测试者不知道这是哪个版本
            results.append({
                "case": case,
                "response": response,
                "tester_rating": None,  # 由测试者评分
                "expected_rating": case.expected_rating
            })
        
        return BlindTestReport(
            total=len(results),
            results=results,
            average_deviation=self._calculate_deviation(results)
        )
    
    def blind_comparison(
        self,
        version_a: str,
        version_b: str,
        test_cases: List[TestCase]
    ) -> ComparisonReport:
        """
        双盲对比测试
        """
        results_a = []
        results_b = []
        
        for case in test_cases:
            # 两个版本分别运行
            resp_a = self.get_version(version_a).run(case.input)
            resp_b = self.get_version(version_b).run(case.input)
            
            # 打乱顺序让测试者盲评
            pair = [
                {"version": "A", "response": resp_a},
                {"version": "B", "response": resp_b}
            ]
            random.shuffle(pair)
            
            results_a.append(pair[0])
            results_b.append(pair[1])
        
        return ComparisonReport(
            version_a=version_a,
            version_b=version_b,
            preferred_count=self._count_preferences(results_a, results_b)
        )
```

### 测试集新鲜度管理

```python
class TestSetFreshness:
    """
    测试集新鲜度管理
    """
    
    def __init__(self):
        self.test_set = TestSet()
        self.production_logs = ProductionLogStore()
    
    def refresh_test_set(self):
        """
        每周用线上数据更新测试集
        """
        # 1. 收集本周的线上数据
        weekly_logs = self.production_logs.get_last_n_days(7)
        
        # 2. 采样（按类型分层）
        sampled = self._stratified_sample(weekly_logs, n=100)
        
        # 3. 标注
        new_cases = []
        for log in sampled:
            case = TestCase(
                input=log.user_request,
                expected_output=log.expected_output,
                source="production",
                collected_at=log.timestamp
            )
            new_cases.append(case)
        
        # 4. 替换最旧的 20% 用例
        self.test_set.replace_oldest(new_cases, ratio=0.2)
    
    def _stratified_sample(self, logs, n: int) -> List:
        """分层采样保证覆盖度"""
        by_intent = defaultdict(list)
        for log in logs:
            by_intent[log.intent_type].append(log)
        
        sampled = []
        per_intent = n // len(by_intent)
        for intent, intent_logs in by_intent.items():
            sampled.extend(random.sample(
                intent_logs,
                min(per_intent, len(intent_logs))
            ))
        return sampled
```

## 常见误区

- **以为测试集分数高就等于产品好**：测试集只能反映"在已知场景上的表现"，无法覆盖未知的长尾场景
- **演示成功就宣布发布**：演示的 3-5 个用例不能代表整体质量，至少需要 50-100 个盲测用例
- **只改 prompt 不加测试集**：没有测试集的约束，prompt 修改只是凭感觉，改好改坏都不知道
- **测试集从不更新**：测试集几个月不更新，系统越来越好只说明过拟合越来越严重

## 学习建议

做一次"诚实演示"：打开线上真实请求的日志，闭着眼睛随机选 10 条，不用挑选直接跑。能跑通几条？如果不足 7 条，说明你之前的所有演示都是"精心挑选"的。这就是你改进的起点。每周做一次这种"诚实演示"，跟踪进展。
