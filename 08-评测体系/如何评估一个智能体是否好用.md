# 如何评估一个智能体是否好用？

## 面试问题

如何评估一个智能体是否好用？

## 考察重点

- 是否理解"好用"是主观感受，需要转化为可量化的指标
- 是否能区分"模型能力评估"和"系统质量评估"是两件不同的事
- 是否有从用户视角和系统视角分别定义评估标准的经验
- 是否了解没有评测体系就无法持续改进智能体

## 核心结论

评估一个智能体是否好用不能只靠"我觉得它答得不错"。需要从三个维度分别评估：**任务完成质量**（它把事情做对了吗）、**用户体验**（用户觉得好用吗）、**系统效率**（它花多少成本/时间把事情做完）。每个维度需要不同的指标和评估方法。

### 评估流程

```
测试集（100-500 条样本）
    │
    ▼
运行智能体系统 → 收集执行结果
    │
    ▼
自动评估（LLM-as-Judge + 规则检查）
    │
    ▼
人工抽检（10-20% 的样本）
    │
    ▼
生成评估报告
    ├── 总体评分
    ├── 分维度评分
    ├── 失败案例分析
    └── 改进建议
```

### LLM-as-Judge 评估

```python
class LLMAsJudge:
    """
    用 LLM 评估智能体的回答质量
    """
    
    JUDGE_PROMPT = """
    你是一个智能体评测专家。请评估以下智能体的回答质量。

    用户请求：{user_request}
    
    智能体执行过程：
    {execution_trace}
    
    智能体最终回答：
    {final_response}
    
    请从以下维度评分（1-5分）：
    1. 正确性：回答是否正确
    2. 完整性：是否覆盖了用户的所有需求
    3. 效率：工具调用是否合理、步骤是否简洁
    4. 安全性：是否有任何不安全的行为
    
    请输出 JSON 格式评分：
    {{
        "correctness": <分数>,
        "completeness": <分数>,
        "efficiency": <分数>,
        "safety": <分数>,
        "overall": <平均分>,
        "explanation": "<评分理由>"
    }}
    """
    
    def evaluate(self, test_case: TestCase) -> JudgeResult:
        response = self.judge_llm.complete(
            self.JUDGE_PROMPT.format(
                user_request=test_case.input,
                execution_trace=test_case.trace,
                final_response=test_case.response
            )
        )
        return JudgeResult(**json.loads(response))
```

## 工程实践

### 评估框架实现

```python
class AgentEvaluator:
    """
    智能体评估框架
    """
    
    def __init__(self):
        self.test_suite = TestSuite()
        self.judge = LLMAsJudge()
        self.metrics = MetricsCollector()
    
    def evaluate(self, agent, test_suite: TestSuite) -> Report:
        results = []
        
        for case in test_suite.cases:
            # 运行智能体
            start_time = time.time()
            response = agent.run(case.input)
            duration = time.time() - start_time
            
            # 收集指标
            metrics = self.metrics.collect(
                case=case,
                response=response,
                duration=duration
            )
            
            # LLM 评分
            judge_result = self.judge.evaluate(case, response)
            
            results.append({
                "case_id": case.id,
                "metrics": metrics,
                "judge_score": judge_result,
                "passed": judge_result.overall >= 3.5
            })
        
        return Report(
            total=len(results),
            pass_rate=sum(1 for r in results if r["passed"]) / len(results),
            avg_scores=self._aggregate_scores(results),
            failures=[r for r in results if not r["passed"]]
        )
```

## 常见误区

- **只看任务完成率，不看完成质量**：智能体可能"完成了"任务但结果质量很差，用户不满意
- **用单一指标衡量整体效果**：没有一个指标能完整反映智能体的表现，需要多维度综合评估
- **只在理想环境中评估**：测试集的数据和线上真实用户数据分布不同，离线评估结果不能代表线上表现
- **评测本身不评测**：LLM-as-Judge 也有偏差，需要定期用人工评分标定自动评分的准确性

## 追问方向

- 当自动评分和用户反馈不一致时，你以哪个为准？
- 你的评测成本占整体开发成本的比例是多少？
- 如何让评测覆盖长尾场景（罕见的用户请求）？
- 你多久做一次全面评估？每次评估的完整流程跑完需要多久？

## 学习建议

不要先搭建复杂的评测框架。从 10 个测试用例 + 人工评分开始，跑几次后你自然会知道需要什么样的自动化工具。评测体系的建设应该和智能体系统的建设同步进行，而不是事后补的。
