# 如何做回归评测？

## 面试问题

如何做回归评测？

## 考察重点

- 是否理解回归评测的目的是"检测改动是否破坏了已有功能"
- 是否能说清楚回归测试集的管理和维护方法
- 是否了解回归测试自动化运行的机制
- 是否有过因为回归测试及时发现了意外退化的经验

## 核心结论

回归评测的核心是：**每次改动（prompt 修改、工具变更、模型升级）都跑一遍固定的测试集，确保之前修好的问题没有回退，之前好用的功能没有被破坏**。关键在于"固定的测试集"——它必须包含曾经出过问题的案例，并且每次跑的结果都要和上一次对比。

## 参考回答

### 回归评测流程

```python
class RegressionTestPipeline:
    """
    回归评测流水线
    """
    
    def __init__(self):
        self.test_suite = self._load_suite()
        self.baseline_results = self._load_baseline()
    
    def run_regression(self, change_description: str) -> RegressionReport:
        """
        运行回归测试
        """
        current_results = {}
        regressions = []
        improvements = []
        
        for case in self.test_suite.cases:
            # 运行
            response = self.agent.run(case.input)
            score = self.evaluator.evaluate(case, response)
            current_results[case.id] = score
            
            # 对比基线
            if case.id in self.baseline_results:
                baseline = self.baseline_results[case.id]
                diff = score.overall - baseline.overall
                
                if diff < -0.3:
                    regressions.append(Regression(
                        case_id=case.id,
                        previous_score=baseline.overall,
                        current_score=score.overall,
                        diff=diff,
                        severity="high" if diff < -0.5 else "medium"
                    ))
                elif diff > 0.3:
                    improvements.append(Improvement(
                        case_id=case.id,
                        previous_score=baseline.overall,
                        current_score=score.overall,
                        diff=diff
                    ))
        
        # 保存当前结果为新的基线
        self._save_baseline(current_results)
        
        return RegressionReport(
            change_description=change_description,
            total_cases=len(self.test_suite.cases),
            regressions=regressions,
            improvements=improvements,
            overall_change=self._calculate_overall_change(
                current_results
            ),
            passed=len(regressions) == 0
        )
```

### 回归测试集

```python
class RegressionTestSuite:
    """
    回归测试集管理
    """
    
    def __init__(self):
        self.cases: List[TestCase] = []
        self.baseline: Dict[str, float] = {}
    
    def add_from_failure(self, case: TestCase):
        """
        从线上失败案例添加回归测试
        核心来源：之前出过问题、已经修好的案例
        """
        case.tags.append('regression')
        case.tags.append('from_failure')
        self.cases.append(case)
    
    def add_critical_path(self, case: TestCase):
        """
        添加核心路径测试
        核心来源：最重要的用户场景
        """
        case.tags.append('regression')
        case.tags.append('critical_path')
        self.cases.insert(0, case)  # 核心路径放前面
    
    def size_by_priority(self) -> Dict:
        """按优先级统计"""
        return {
            "P0 (critical)": sum(1 for c in self.cases if 'critical_path' in c.tags),
            "P1 (failure)": sum(1 for c in self.cases if 'from_failure' in c.tags),
            "P2 (normal)": sum(1 for c in self.cases if 'normal' in c.tags),
        }
```

### 回归结果分析

```python
class RegressionAnalyzer:
    """
    回归结果分析
    """
    
    def analyze(self, report: RegressionReport) -> Analysis:
        """分析回归结果"""
        
        # 1. 整体评估
        if not report.passed:
            critical_regressions = [
                r for r in report.regressions
                if r.severity == 'high'
            ]
            
            if critical_regressions:
                return Analysis(
                    verdict="BLOCKED",
                    message=f"发现 {len(critical_regressions)} 个严重回归，禁止发布"
                )
            else:
                return Analysis(
                    verdict="REVIEW_NEEDED",
                    message=f"发现 {len(report.regressions)} 个回归需确认"
                )
        
        return Analysis(
            verdict="PASSED",
            message="所有回归测试通过"
        )
    
    def generate_diff_report(
        self, 
        old_baseline: Dict,
        new_baseline: Dict
    ) -> DiffReport:
        """生成两次基线的差异报告"""
        diffs = []
        for case_id in old_baseline:
            old_score = old_baseline[case_id]
            new_score = new_baseline.get(case_id)
            
            if new_score and abs(new_score - old_score) > 0.1:
                diffs.append({
                    "case_id": case_id,
                    "old": old_score,
                    "new": new_score,
                    "change": "improved" if new_score > old_score else "regressed"
                })
        
        return DiffReport(diffs=diffs, significant_changes=len(diffs))
```

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  regression:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Run regression tests
        run: python -m agent.eval.regression
        
      - name: Check results
        run: |
          if [ -f regression_report.json ]; then
            python -c "
            import json
            report = json.load(open('regression_report.json'))
            if not report['passed']:
              print('REGRESSION FAILED')
              for r in report['regressions']:
                print(f'  - {r[\"case_id\"]}: {r[\"previous_score\"]} -> {r[\"current_score\"]}')
              exit(1)
            "
          fi

### 基线管理

```python
class BaselineManager:
    """
    回归基线管理
    """
    
    def __init__(self, storage_path: str = "baselines/"):
        self.storage_path = Path(storage_path)
        self.storage_path.mkdir(exist_ok=True)
    
    def save_baseline(self, name: str, results: Dict):
        """保存基线"""
        baseline = {
            "name": name,
            "timestamp": datetime.now().isoformat(),
            "results": results,
            "summary": {
                "total": len(results),
                "avg_score": mean(results.values())
            }
        }
        
        path = self.storage_path / f"{name}.json"
        json.dump(baseline, open(path, 'w'))
    
    def compare_baselines(self, old: str, new: str) -> Dict:
        """对比两个基线"""
        old_data = json.load(open(self.storage_path / f"{old}.json"))
        new_data = json.load(open(self.storage_path / f"{new}.json"))
        
        regression = []
        for case_id, new_score in new_data['results'].items():
            old_score = old_data['results'].get(case_id)
            if old_score and new_score < old_score:
                regression.append({
                    "case_id": case_id,
                    "old": old_score,
                    "new": new_score,
                    "drop": old_score - new_score
                })
        
        return {
            "old_baseline": old,
            "new_baseline": new,
            "total_cases": len(new_data['results']),
            "regressions": len(regression),
            "details": sorted(regression, key=lambda x: x['drop'], reverse=True)[:20]
        }
```

## 常见误区

- **回归测试集不更新**：线上出现了新问题，修好了但不加入回归测试集，下次还会再犯
- **只看回归不看进步**：回归测试只报告分数下降的用例，也要关注分数提升的用例——可能你的改动对某些场景特别好
- **基线覆写太随意**：每次跑完回归都自动覆盖基线，会导致逐渐"漂移"——问题慢慢被当成正常。建议每次覆写前人工确认
- **回归测试耗时太长**：测试集太大导致每次改动都要等很久才能部署。需要分层（快速回归 + 深度回归）

## 追问方向

- 你的回归测试集有多少用例？跑一次全量回归需要多久？
- 回归测试中发现一个用例分数下降了，你怎么确认是"真回归"还是"假阳性"？
- 当模型升级后很多用例分数都变了（不是单纯的变好变坏，而是模型风格变了），你怎么更新基线？
- 如果回归测试没过但紧急修复要上线，你怎么做风险决策？

## 学习建议

创建一个"回归测试清单"，包含你最核心的 10 个用户场景。在每次修改 prompt 或工具之前跑一遍，记下结果。改完再跑一遍，对比差异。坚持几周后你会自然发现需要更多的测试用例来覆盖更多场景。不要一开始就搭复杂的回归框架，10 个关键用例 + 手动对比就能抓住 80% 的问题。
