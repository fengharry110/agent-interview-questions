# 如何评估任务完成率？

## 面试问题

如何评估任务完成率？

## 考察重点

- 是否理解任务完成率不是非黑即白的二元指标，"部分完成"也是常见状态
- 是否能说清楚"任务完成"的判断标准和判断方法
- 是否了解自动评估和人工评估各自的局限性
- 是否有通过任务完成率驱动系统改进的经验

## 核心结论

任务完成率不能简单地用"智能体说完成了就算完成"。评估需要分三层：**任务是否结束**（有没有给出最终回复）、**结果是否有效**（回复是否对用户有帮助）、**目标是否实现**（用户的问题是否真正被解决）。最后一层最难自动化，通常需要用户反馈或人工评审。

### 完成率的三个层次

```python
class CompletionLevel(Enum):
    COMPLETED = "completed"
    """任务完成：用户目标已达成"""
    
    PARTIALLY_COMPLETED = "partially_completed"
    """部分完成：完成了部分目标，部分未达成"""
    
    ATTEMPTED = "attempted"
    """尝试过：智能体尝试了但没有完成"""
    
    FAILED = "failed"
    """失败：智能体未能完成或直接拒绝"""
    
    ERROR = "error"
    """系统错误：执行过程中出现异常中断"""


class CompletionEvaluator:
    """
    任务完成率评估器
    """
    
    def evaluate(self, trace: Trace) -> CompletionResult:
        """
        评估一次执行的任务完成情况
        """
        # 1. 检查是否正常结束
        if not self._has_proper_ending(trace):
            return CompletionResult(
                level=CompletionLevel.ERROR,
                reason="执行非正常终止",
                detail=trace.error_info
            )
        
        # 2. 检查智能体是否给出了实质性回复
        if self._is_refusal(trace):
            return CompletionResult(
                level=CompletionLevel.FAILED,
                reason="智能体拒绝了请求"
            )
        
        # 3. 检查目标达成程度
        goal_achievement = self._measure_goal_achievement(
            trace.user_goal, trace.final_response
        )
        
        if goal_achievement >= 0.9:
            return CompletionResult(
                level=CompletionLevel.COMPLETED,
                score=goal_achievement
            )
        elif goal_achievement >= 0.5:
            return CompletionResult(
                level=CompletionLevel.PARTIALLY_COMPLETED,
                score=goal_achievement,
                reason="部分目标未达成"
            )
        else:
            return CompletionResult(
                level=CompletionLevel.ATTEMPTED,
                score=goal_achievement,
                reason="尝试过但未达成主要目标"
            )
    
    def _measure_goal_achievement(
        self, goal: str, response: str
    ) -> float:
        """
        衡量目标达成程度（0.0 - 1.0）
        """
        # 方法：用 LLM 评估
        prompt = f"""
        用户目标：{goal}
        智能体回复：{response}
        
        请评估目标达成程度（0.0-1.0）：
        - 1.0 = 完全达成
        - 0.7 = 主要目标达成，细节有遗漏
        - 0.5 = 部分达成
        - 0.3 = 尝试了但没达到目标
        - 0.0 = 完全没达成
        
        只输出数字。
        """
        
        score = float(self.judge_llm.complete(prompt))
        return max(0.0, min(1.0, score))
```

### 判断标准

```python
class CompletionCriteria:
    """
    任务完成的判断标准
    """
    
    COMPLETION_SIGNALS = {
        "positive": [
            "明确的完成声明（已完成、已处理、已生成）",
            "提供了具体的输出结果",
            "用户确认满意",
            "执行了所有必要的工具调用"
        ],
        "negative": [
            "智能体说'我做不到'或'无法完成'",
            "循环后超时终止",
            "最终回复含糊不具体",
            "关键工具没有调用或调用失败"
        ],
        "ambiguous": [
            "完成了部分但没提剩余部分",
            "回复了但没给出具体结果",
            "需要进一步澄清"
        ]
    }
    
    @staticmethod
    def auto_detect_completion(trace: Trace) -> bool:
        """自动检测是否完成"""
        final_step = trace.steps[-1]
        
        # 智能体明确确认完成
        completion_phrases = [
            "已完成", "已处理", "已生成", "已修改",
            "完成", "done", "completed", "finished"
        ]
        
        if any(
            phrase in final_step.model_output
            for phrase in completion_phrases
        ):
            return True
        
        # 最后一步是工具调用（没有给出最终回复）→ 可能被截断
        if final_step.type == 'tool_call':
            return False
        
        return False
```

### 统计方法

```python
class CompletionStatistics:
    """
    完成率的统计方法
    """
    
    @staticmethod
    def calculate(traces: List[Trace]) -> Dict:
        """计算完成率统计"""
        total = len(traces)
        
        levels = Counter(
            self._classify(trace) for trace in traces
        )
        
        return {
            "total": total,
            "completion_rate": levels['completed'] / total,
            "partial_rate": levels['partially_completed'] / total,
            "failure_rate": levels['failed'] / total,
            "error_rate": levels['error'] / total,
            "distribution": dict(levels)
        }
    
    @staticmethod
    def trend(daily_results: List[Dict], days: int = 30) -> Trend:
        """完成率趋势"""
        return Trend(
            daily_rates=[r['completion_rate'] for r in daily_results],
            trend_direction=self._calculate_trend(daily_results),
            improvement=(
                daily_results[-1]['completion_rate'] -
                daily_results[0]['completion_rate']
            )
        )
```

## 常见误区

- **智能体说完成了就算完成**：模型可能给出"已完成"的确认但实际上没有执行必要的操作。需要验证工具调用日志
- **把完成率当唯一指标**：完成率很高但工具调用准确率很低，说明智能体在"糊弄"用户
- **忽略部分完成的情况**：用户请求了 3 件事，智能体只做了 2 件。全完成/没完成的二元分类信息量不够
- **不区分"不能完成"和"不应该完成"**：安全拒绝（如"我不能帮你删除系统文件"）是正确行为，不应计入失败

## 追问方向

- 当用户的请求本身就不明确时，任务完成的定义应该是什么？
- 如果智能体完成了任务但花了 50 步，这算完成吗？完成率是否需要和效率指标联动？
- 你如何用完成率来指导产品的改进决策？
- 完成率的波动多大范围算正常？什么情况下需要立即响应？

## 学习建议

先花一周时间手工标注你的智能体执行的 50 条结果，分别判断"完成了"、"部分完成"和"没完成"。你会发现有很多边界情况需要定规则（比如完成了 80% 算不算完成）。建立了规则后再写自动化评估脚本，但永远保留人工抽检的环节。
