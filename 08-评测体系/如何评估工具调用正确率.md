# 如何评估工具调用正确率？

## 面试问题

如何评估工具调用正确率？

## 考察重点

- 是否理解工具调用正确率不是一个单一指标，而是"选对工具"和"传对参数"两个维度的组合
- 是否能说清楚不同类型的工具错误（选错工具、参数错误、顺序错误）各有不同的评估方法
- 是否有自动评估工具调用的实践经验
- 是否了解工具调用评估的挑战（期望的工具可能不唯一）

## 核心结论

评估工具调用正确率需要拆解为两个子问题：**工具选择是否正确**（选没选对工具）和**参数传递是否正确**（参数格式和值是否正确）。前者适合用分类指标（精确率/召回率），后者适合用结构化比对（JSON Schema 校验）。评估时还需要考虑"多种工具组合都能完成任务"的情况，不能死板地要求唯一正确的工具。

### 评估框架

```python
class ToolCallEvaluator:
    """
    工具调用正确率评估
    """
    
    def evaluate(self, trace: Trace) -> ToolCallScore:
        """
        评估一次执行中所有工具调用的正确性
        """
        scores = []
        for step in trace.steps:
            if step.type == 'tool_call':
                score = self._evaluate_single_call(
                    step, trace.expected_tools
                )
                scores.append(score)
        
        return ToolCallScore(
            total_calls=len(scores),
            correct_tool_selection=sum(
                1 for s in scores if s.tool_correct
            ),
            correct_parameters=sum(
                1 for s in scores if s.params_correct
            ),
            tool_accuracy=sum(
                1 for s in scores if s.tool_correct
            ) / len(scores) if scores else 1.0,
            param_accuracy=sum(
                1 for s in scores if s.params_correct
            ) / len(scores) if scores else 1.0
        )
    
    def _evaluate_single_call(
        self,
        step: TraceStep,
        expected_tools: List[ExpectedToolCall]
    ) -> SingleCallScore:
        """
        评估单次工具调用
        """
        # 1. 工具选择是否正确
        expected_tool = self._find_matching_expectation(
            step.tool_name, expected_tools
        )
        tool_correct = expected_tool is not None
        
        # 2. 参数是否正确
        params_correct = False
        if tool_correct and expected_tool:
            params_correct = self._validate_parameters(
                step.tool_args,
                expected_tool.expected_args
            )
        
        return SingleCallScore(
            step_number=step.number,
            tool_name=step.tool_name,
            tool_correct=tool_correct,
            params_correct=params_correct,
            expected_tool=expected_tool.name if expected_tool else None
        )
```

### 自动校验参数

```python
class ParameterValidator:
    """
    工具参数校验
    """
    
    def validate(self, actual: Dict, schema: Dict) -> ParamValidation:
        """
        校验参数是否符合 JSON Schema 定义
        """
        errors = []
        
        # 1. 必填参数检查
        for required in schema.get('required', []):
            if required not in actual:
                errors.append({
                    "type": "missing_required",
                    "param": required,
                    "severity": "error"
                })
        
        # 2. 参数类型检查
        for param_name, param_value in actual.items():
            expected_type = (
                schema.get('properties', {})
                .get(param_name, {})
                .get('type')
            )
            
            if expected_type and not self._type_check(
                param_value, expected_type
            ):
                errors.append({
                    "type": "type_mismatch",
                    "param": param_name,
                    "expected": expected_type,
                    "actual": type(param_value).__name__,
                    "severity": "warning"
                })
        
        # 3. 枚举值检查
        for param_name, param_value in actual.items():
            enum_values = (
                schema.get('properties', {})
                .get(param_name, {})
                .get('enum')
            )
            if enum_values and param_value not in enum_values:
                errors.append({
                    "type": "invalid_enum",
                    "param": param_name,
                    "expected": enum_values,
                    "actual": param_value,
                    "severity": "error"
                })
        
        return ParamValidation(
            valid=len(errors) == 0,
            errors=errors,
            error_count=len(errors),
            severity="error" if any(
                e['severity'] == 'error' for e in errors
            ) else "warning"
        )
```

### 评估报告

```python
class ToolCallReport:
    """
    工具调用评估报告
    """
    
    def generate(self, results: List[ToolCallScore]) -> Report:
        return Report(
            summary={
                "tool_accuracy": (
                    sum(r.tool_correct for r in results) / len(results)
                ),
                "param_accuracy": (
                    sum(r.params_correct for r in results) / len(results)
                ),
                "total_calls": len(results),
            },
            by_tool=self._group_by_tool(results),
            failures=[
                r for r in results
                if not r.tool_correct or not r.params_correct
            ],
            top_failed_tools=self._top_failures(results, 5)
        )
    
    def _group_by_tool(self, results):
        """按工具名聚合"""
        groups = {}
        for r in results:
            tool = r.tool_name
            if tool not in groups:
                groups[tool] = []
            groups[tool].append(r)
        
        return {
            tool: {
                "total": len(scores),
                "accuracy": sum(s.tool_correct for s in scores) / len(scores)
            }
            for tool, scores in groups.items()
        }
```

### 集成到 CI

```python
class ToolCallCI:
    """
    工具调用正确率 CI 检查
    """
    
    def check(self, test_suite: TestSuite) -> bool:
        """CI 检查：工具调用正确率是否超过阈值"""
        evaluator = ToolCallEvaluator()
        results = []
        
        for case in test_suite.cases:
            agent_output = self.agent.run(case.input)
            score = evaluator.evaluate(agent_output)
            results.append(score)
        
        overall_accuracy = (
            sum(r.tool_accuracy for r in results) / len(results)
        )
        
        if overall_accuracy < self.threshold:
            raise CIError(
                f"Tool accuracy {overall_accuracy:.1%} "
                f"below threshold {self.threshold:.1%}"
            )
        
        return True
```

## 常见误区

- **追求 100% 工具选择准确率**：有些场景下多个工具都能完成任务，强迫选择"正确"的工具反而会降低灵活性。接受合理的等价替换
- **把参数校验当字符串匹配**："path=/data/file.txt" 和 "path": "/data/file.txt" 功能等价，不应该判错
- **忽略工具调用顺序的重要性**：即使每个工具都选对了，顺序错了也会导致失败（先删文件再读取）
- **只评估单次调用不看整体效率**：智能体可能调用了 5 次工具才完成本应 2 次就能完成的任务，这是"正确但低效"

## 追问方向

- 如何处理"智能体用了我没预料到的新方法完成任务，但确实有效"这种情况的评分？
- 你的工具调用正确率评估的 Inter-Rater Reliability（评分者间信度）如何？
- 当工具参数错误时，你如何区分"模型理解错了"和"工具定义不清晰"？
- 工具调用正确率的提高是否一定意味着整体体验的改善？

## 学习建议

从最直接的方法开始：对每个测试用例，人工标注"期望的工具调用序列"，然后和实际的工具调用序列做比对。先手工跑 20 条用例，感受一下评估的难度和微妙之处，再决定要不要写自动化评估脚本。
