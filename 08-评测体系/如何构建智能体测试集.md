# 如何构建智能体测试集？

## 面试问题

如何构建智能体测试集？

## 考察重点

- 是否理解测试集需要覆盖"正常场景"和"异常场景"
- 是否了解从真实用户数据中构建测试集的方法
- 是否能说清楚测试集的维护和管理策略
- 是否有构建过可用的智能体测试集的经验

## 核心结论

智能体测试集不是一次性建好的，而是持续积累的。构建策略是"三源汇流"：**线上真实数据**（用户反馈 + 执行轨迹）、**人工构造**（覆盖边界情况）、**自动生成**（用 LLM 变体）。每个来源解决不同的问题：线上数据保证覆盖率，人工构造保证质量，自动生成保证数量。

### 构建步骤

```python
class TestSetConstruction:
    """
    测试集构建流程
    """
    
    def __init__(self):
        self.cases: List[TestCase] = []
    
    def build_from_production(self, logs: List[SessionLog]):
        """从线上日志提取测试用例"""
        for log in logs:
            # 1. 提取用户请求
            case = TestCase(
                input=log.user_goal,
                expected_output=self._extract_expected(log),
                tags=["production", log.intent_type]
            )
            
            # 2. 如果有用户反馈，用它做期望结果
            if log.user_feedback:
                case.expected_output = log.user_feedback.expected
            
            self.cases.append(case)
    
    def build_manual(self, scenarios: List[Scenario]):
        """人工构造测试场景"""
        for scenario in scenarios:
            case = TestCase(
                input=scenario.user_input,
                expected_output=scenario.expected_output,
                tags=["manual", scenario.category],
                difficulty=scenario.difficulty
            )
            
            # 加边界情况
            if scenario.is_edge_case:
                case.tags.append("edge_case")
            
            self.cases.append(case)
    
    def build_with_llm(self, seed_cases: List[TestCase],
                       count: int) -> List[TestCase]:
        """用 LLM 基于种子用例生成变体"""
        prompt = """
        基于以下种子测试用例，生成 {count} 个变体。
        保持相同的意图但改变具体的表述方式。

        种子用例：
        {seed}

        要求：
        - 覆盖不同的表达方式
        - 覆盖不同的参数值
        - 覆盖不同的难度级别
        """
        
        generated = self.llm.generate(prompt)
        return self._validate_and_filter(generated)
```

## 深度展开

### 测试集的生命周期

```python
class TestSetLifecycle:
    """
    测试用例从创建到退役
    """
    
    def __init__(self):
        self.cases = []
    
    def add_case(self, case: TestCase):
        """添加新用例"""
        case.id = self._generate_id()
        case.created_at = datetime.now()
        case.state = "active"
        self.cases.append(case)
    
    def deprecate_case(self, case_id: str, reason: str):
        """弃用不再适用的用例"""
        case = self._find_case(case_id)
        case.state = "deprecated"
        case.deprecated_reason = reason
        case.deprecated_at = datetime.now()
    
    def review_cases(self):
        """定期审查测试集"""
        report = {
            "total_active": sum(1 for c in self.cases if c.state == "active"),
            "total_deprecated": sum(1 for c in self.cases if c.state == "deprecated"),
            "by_category": self._count_by_category(),
            "by_difficulty": self._count_by_difficulty(),
            "last_added": self._last_added_date(),
            "coverage_gaps": self._find_coverage_gaps()
        }
        return report
```

| 开发期 | 100-300 条 | 覆盖主流场景 |
| 上线前 | 300-1000 条 | 覆盖边界和异常 |
| 持续迭代 | 持续增长 | 每个线上失败都沉淀为用例 |

### 测试集管理工具

```python
class TestSetManager:
    """
    测试集管理器
    """
    
    def __init__(self, storage_path: str = "test_suite.json"):
        self.storage_path = storage_path
        self.cases = self._load()
    
    def add_batch(self, new_cases: List[TestCase]):
        """批量添加并去重"""
        existing_inputs = {c.input for c in self.cases}
        
        added = 0
        skipped = 0
        for case in new_cases:
            if case.input not in existing_inputs:
                self.cases.append(case)
                existing_inputs.add(case.input)
                added += 1
            else:
                skipped += 1
        
        self._save()
        return {"added": added, "skipped": skipped}
    
    def search_cases(self, query: str) -> List[TestCase]:
        """搜索测试用例"""
        results = []
        for case in self.cases:
            if (query.lower() in case.input.lower() or
                query.lower() in ' '.join(case.tags).lower()):
                results.append(case)
        return results
    
    def export_suite(self, format: str = "json") -> str:
        """导出测试集"""
        if format == "json":
            return json.dumps(
                [c.to_dict() for c in self.cases],
                ensure_ascii=False,
                indent=2
            )
        elif format == "csv":
            # 导出为 CSV 用于人工评审
            pass
```

## 常见误区

- **测试集只增不减**：用例会过期（模型升级、功能变更），需要定期清理
- **只覆盖成功路径**：失败路径（工具返回错误、超时等）比成功路径更容易暴露问题
- **测试集和真实用户分布不一致**：人工构造的用例可能过于理想化，需要和线上数据混合
- **测试集不维护**：没有定期审查和更新，用例慢慢落后于系统演进

## 追问方向

- 你的测试集和线上用户请求的分布差异有多大？多久检查一次？
- 如何评估测试集本身的质量？有没有指标来衡量"测试集好不好"？
- 当新功能上线时，测试集需要多少时间才能跟上？
- 测试集的维护成本占你多少开发时间？

## 学习建议

从今天开始记录你遇到的每一个"智能体做错了"的案例，每周花 30 分钟把它们整理成测试用例。坚持一个月后你就有了 20-30 个真实用例。然后基于这些种子用例，用 LLM 生成变体来扩展覆盖。测试集的建设不是项目，是习惯。
