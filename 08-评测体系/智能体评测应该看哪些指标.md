# 智能体评测应该看哪些指标？

## 面试问题

智能体评测应该看哪些指标？

## 考察重点

- 是否能系统地列出智能体评测的关键指标体系
- 是否理解不同角色（用户、开发者、业务方）关心的指标不同
- 是否能区分"模型指标"和"系统指标"
- 是否有通过指标驱动智能体迭代的实际经验

## 核心结论

智能体评测指标分为三个层次：**任务层**（任务完成率、正确率）、**过程层**（工具调用准确率、步骤效率）、**资源层**（成本、延迟）。每个层次服务于不同的决策场景。关键原则是：先定义"什么样的智能体算好"，再选择相应的指标，而不是先拉一堆指标再去解释。

### 指标的计算方式

```python
class MetricsCalculator:
    """
    各指标的计算方法
    """
    
    def task_completion_rate(self, results: List[TaskResult]) -> float:
        """任务完成率 = 明确完成的任务 / 总数"""
        completed = sum(
            1 for r in results
            if r.status == 'completed'
        )
        return completed / len(results)
    
    def tool_selection_accuracy(self, traces: List[Trace]) -> float:
        """工具选择准确率"""
        correct = 0
        total = 0
        for trace in traces:
            for step in trace.steps:
                if step.type == 'tool_call':
                    total += 1
                    if step.tool_name == step.expected_tool:
                        correct += 1
        return correct / total if total > 0 else 1.0
    
    def step_efficiency(self, trace: Trace) -> float:
        """步骤效率 = 最优步骤 / 实际步骤"""
        optimal = trace.estimated_optimal_steps
        actual = len(trace.steps)
        return optimal / actual if actual > 0 else 0
    
    def cost_per_task(self, total_cost: float, 
                      task_count: int) -> float:
        return total_cost / task_count if task_count > 0 else 0
```

## 深度展开

### 不同角色的关注点

| 角色 | 最关心的指标 | 关注原因 |
|------|------------|---------|
| **用户** | 任务完成率、满意度 | 智能体能不能解决我的问题 |
| **开发者** | 工具选择准确率、步骤效率 | 系统设计是否合理 |
| **业务方** | 成本/任务、人工介入率 | 投入产出比 |
| **管理者** | 整体趋势、回归率 | 系统是否在持续改进 |

### 北极星指标

选择合适的"北极星指标"（North Star Metric）很重要——用一个指标统领所有改进方向：

```python
class NorthStarMetric:
    """
    不同场景的北极星指标选择
    """
    
    SCENARIOS = {
        "customer_service": {
            "north_star": "customer_satisfaction",
            "reasoning": "客服智能体的最终目标是让用户满意"
        },
        "code_assistant": {
            "north_star": "task_completion_rate",
            "reasoning": "代码助手的价值在于完成任务"
        },
        "data_analysis": {
            "north_star": "answer_correctness", 
            "reasoning": "数据分析以正确性为第一优先级"
        },
        "internal_tool": {
            "north_star": "human_intervention_rate",
            "reasoning": "内部工具的价值在于减少人工操作"
        }
    }
    
    @staticmethod
    def select(scenario: str) -> str:
        return NorthStarMetric.SCENARIOS.get(
            scenario, {}
        ).get("north_star", "task_completion_rate")
```

```python
        {
            "between": ["安全性", "用户体验"],
            "relationship": "有时冲突",
            "explanation": "过于严格的安全检查会让用户觉得体验笨拙"
        }
    ]
    
    @staticmethod
    def recommend_balance(
        metrics: Dict[str, float],
        budget: Budget
    ) -> List[str]:
        """根据当前指标和预算推荐优化方向"""
        suggestions = []
        
        if metrics['cost_per_task'] > budget.max_cost:
            suggestions.append("降低模型规格或增加上下文压缩")
        elif metrics['task_completion_rate'] < 0.8:
            suggestions.append("考虑在关键路径上使用更强的模型")
        
        return suggestions
```

## 常见误区

- **指标越多越好**：指标太多会分散注意力。建议只关注 3-5 个核心指标，其他作为辅助参考
- **只看正面指标不看负面**：只报告完成率不报告失败原因，等于只报喜不报忧
- **指标和业务目标脱节**：技术指标很漂亮（工具准确率 99%），但用户不满意，说明指标选错了
- **不对指标做长期追踪**：单次评测的指标波动可能很大，需要看趋势而非绝对值

## 追问方向

- 你的指标体系中，哪些是 Leading Indicator（先行指标），哪些是 Lagging Indicator（滞后指标）？
- 如果工具准确率从 95% 降到 90%，但任务完成率反而上升了，你怎么解释这个现象？
- 你如何确保指标的可信度（没有被"刷分"的可能）？
- 指标的基准线是如何设定的？为什么选这个值？

## 学习建议

不要一次性实现所有指标。先找到对你业务最关键的 3 个指标，建立数据采集和看板，跑一个月看趋势。然后逐步增加辅助指标。关键是指标要能驱动决策——如果某个指标变了但你不知道该怎么办，那这个指标就没用。
