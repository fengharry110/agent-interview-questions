# 如何设计自动评分和人工评分结合的评测体系？

## 面试问题

如何设计自动评分和人工评分结合的评测体系？

## 考察重点

- 是否理解自动评分和人工评分各自的优劣势，以及如何互补
- 是否能说清楚"自动筛 + 人工审"的流水线设计
- 是否了解评分校准和评分者间信度的概念
- 是否有设计混合评测体系的实际经验

## 核心结论

自动评分和人工评分不是二选一，而是流水线关系：**自动评分跑全量，筛出可疑结果 → 人工评分只审可疑结果，节省精力 → 人工评分结果反哺自动评分模型**。自动评分负责效率（一天评 1000 条），人工评分负责准确性（一天评 50 条但每条更准）。两者的结合点在于"自动评分的不确定区域"——自动评分很有把握的不用审，很没把握的也不用审（直接返工），只有"不确定"的才送人工。

### 混合评测流水线

```python
class HybridEvaluationPipeline:
    """
    自动 + 人工混合评测流水线
    """
    
    def __init__(self):
        self.auto_evaluator = AutoEvaluator()
        self.human_review_platform = HumanReviewPlatform()
        self.calibration = CalibrationManager()
    
    async def evaluate(self, results: List[AgentResult]) -> EvalReport:
        """
        混合评测主流程
        """
        auto_scores = []
        human_review_queue = []
        
        for result in results:
            # 1. 自动评分
            auto_score = await self.auto_evaluator.score(
                result.case, result.response
            )
            
            # 2. 根据自动评分的置信度决定是否送人工
            decision = self._review_decision(auto_score)
            
            if decision == 'auto_approve':
                auto_scores.append({
                    **result.to_dict(),
                    'final_score': auto_score.score,
                    'method': 'auto'
                })
            
            elif decision == 'human_review':
                human_review_queue.append({
                    'result': result,
                    'auto_score': auto_score,
                    'reason': decision.reason
                })
            
            elif decision == 'auto_reject':
                # 自动评分极低，直接标记为失败
                auto_scores.append({
                    **result.to_dict(),
                    'final_score': 0,
                    'method': 'auto_reject'
                })
        
        # 3. 批量送人工审核
        human_scores = await self.human_review_platform.batch_review(
            human_review_queue
        )
        
        # 4. 合并结果
        all_scores = auto_scores + human_scores
        
        # 5. 校准自动评分模型
        self.calibration.update(human_scores)
        
        return EvalReport(
            total=len(all_scores),
            auto_count=len(auto_scores),
            human_count=len(human_scores),
            scores=all_scores,
            calibration_status=self.calibration.status()
        )
    
    def _review_decision(self, auto_score) -> ReviewDecision:
        """
        判断是否需要人工审核
        """
        confidence = auto_score.confidence
        score = auto_score.score
        
        # 高置信度+高分→自动通过
        if confidence > 0.9 and score > 4.0:
            return ReviewDecision('auto_approve', 'high_confidence_high_score')
        
        # 高置信度+低分→自动拒绝
        if confidence > 0.9 and score < 2.0:
            return ReviewDecision('auto_reject', 'high_confidence_low_score')
        
        # 低置信度→送人工
        if confidence < 0.7:
            return ReviewDecision('human_review', 'low_confidence')
        
        # 中等分数→送人工确认边界
        if 2.5 <= score <= 3.5:
            return ReviewDecision('human_review', 'borderline_score')
        
        # 其他→自动通过
        return ReviewDecision('auto_approve', 'acceptable_range')
```

### 评分校准

```python
class CalibrationManager:
    """
    自动评分的持续校准
    """
    
    def __init__(self):
        self.human_scores: List[Dict] = []
        self.auto_scores: List[Dict] = []
    
    def update(self, new_human_scores: List[Dict]):
        """
        用新的人工评分校准自动评分模型
        """
        self.human_scores.extend(new_human_scores)
        
        # 分析偏差
        biases = self._calculate_bias()
        
        # 如果偏差超过阈值，自动调整评分权重
        if abs(biases['avg_bias']) > 0.3:
            self._adjust_auto_evaluator(biases)
        
        return CalibrationStatus(
            sample_size=len(self.human_scores),
            avg_bias=biases['avg_bias'],
            adjusted=True if abs(biases['avg_bias']) > 0.3 else False
        )
    
    def _calculate_bias(self) -> Dict:
        """计算自动评分和人工评分的偏差"""
        differences = []
        for h_score in self.human_scores:
            auto = self._find_auto_score(h_score['result_id'])
            if auto:
                differences.append({
                    'human': h_score['score'],
                    'auto': auto['score'],
                    'diff': auto['score'] - h_score['score']
                })
        
        if not differences:
            return {'avg_bias': 0, 'std_bias': 0}
        
        return {
            'avg_bias': mean(d['diff'] for d in differences),
            'std_bias': stdev(d['diff'] for d in differences),
            'over_confidence_rate': sum(
                1 for d in differences if abs(d['diff']) > 1
            ) / len(differences)
        }
```

### 人工评分的质量控制

```python
class HumanReviewQuality:
    """
    人工评分的质量控制
    """
    
    def __init__(self):
        self.gold_standard_cases = self._load_gold_standard()
    
    def validate_reviewer(self, reviewer_id: str) -> ReviewerQuality:
        """
        评估评分者的质量
        """
        reviewer_scores = self._get_reviewer_scores(reviewer_id)
        
        # 对比黄金标准
        deviations = []
        for case in self.gold_standard_cases:
            if case.id in reviewer_scores:
                deviation = abs(
                    reviewer_scores[case.id] - case.standard_score
                )
                deviations.append(deviation)
        
        avg_deviation = mean(deviations) if deviations else float('inf')
        
        return ReviewerQuality(
            reviewer_id=reviewer_id,
            avg_deviation=avg_deviation,
            consistency=self._measure_consistency(reviewer_id),
            is_qualified=avg_deviation < 1.0
        )
    
    def _load_gold_standard(self) -> List[GoldCase]:
        """
        黄金标准测试集
        由资深专家标注的标准答案和评分
        """
        return [
            GoldCase(id=f"GS-{i}", standard_score=score)
            for i, score in enumerate([5, 4, 3, 2, 1])
        ]
```

### 混合评测的成本分析

```python
class HybridEvalCost:
    """
    混合评测的成本分析
    """
    
    @staticmethod
    def cost_analysis(
        total_cases: int,
        auto_cost_per_case: float,
        human_cost_per_case: float,
        human_review_rate: float
    ) -> CostBreakdown:
        """分析混合评测的成本"""
        auto_count = total_cases
        human_count = int(total_cases * human_review_rate)
        
        total_cost = (
            auto_count * auto_cost_per_case +
            human_count * human_cost_per_case
        )
        
        pure_auto_cost = total_cases * auto_cost_per_case
        pure_human_cost = total_cases * human_cost_per_case
        
        return CostBreakdown(
            total_cases=total_cases,
            auto_count=auto_count,
            human_count=human_count,
            total_cost=total_cost,
            cost_per_case=total_cost / total_cases,
            savings_vs_pure_human=pure_human_cost - total_cost,
            premium_vs_pure_auto=total_cost - pure_auto_cost
        )
```

### 评分者一致性

```python
class InterRaterReliability:
    """
    评分者间信度（Inter-Rater Reliability）
    """
    
    @staticmethod
    def kappa_score(
        rater_a: List[float],
        rater_b: List[float]
    ) -> float:
        """
        计算 Cohen's Kappa
        衡量两个评分者的一致性
        """
        from sklearn.metrics import cohen_kappa_score
        
        # 将连续分数离散化
        def discretize(scores):
            return [
                1 if s <= 2 else
                2 if s <= 3 else
                3 if s <= 4 else
                4
                for s in scores
            ]
        
        return cohen_kappa_score(
            discretize(rater_a),
            discretize(rater_b)
        )
```

## 常见误区

- **人工评分不需要质量控制**：人的评分也不一致，同一个评分者早上和下午打分可能不同。需要定期用黄金标准测试集校准
- **自动评分一劳永逸**：模型在变、用户请求在变，自动评分需要持续用人工评分校准
- **所有结果都送人工**：人工审核是瓶颈，应该只审"自动评分不确定"的边界情况。全量送人工就失去了自动化的意义
- **自动评分和人工评分标准不一致**：同一个答案自动评分打 4 分，人工打 2 分，说明评分标准定义不清楚。需要统一评分规范

## 追问方向

- 你的混合评测体系中，人工审核率控制在多少比较合理？
- 如何衡量人工评分本身的质量？谁又来审核审核者？
- 当自动评分和人工评分持续出现系统性偏差，你的处理流程是什么？
- 如果预算只够覆盖 20% 的人工抽检，你优先审哪些样本？

## 学习建议

先全人工评 100 条，建立标准。然后写一个简单的自动评分器（或者用 LLM-as-Judge），和人工评分对比。找出自动评分总是判错的那几类问题，针对性改进。当自动评分和人工评分的偏差 < 0.5 分时，可以开始全量自动 + 抽检人工的模式。不要一开始就追求完美的自动化，逐步演进。
